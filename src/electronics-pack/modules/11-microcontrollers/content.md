# Module 11: Microcontrollers

> **Tier**: 4 | **H&H Reference**: Ch.14-15 | **Safety Mode**: Gate

## Overview

A microcontroller (MCU) puts a CPU, RAM, flash memory, and dozens of peripherals on a single chip smaller than a postage stamp. Arduino, ESP32, STM32, PIC -- they all follow the same pattern: read inputs, make decisions, drive outputs. MCUs are in everything from toasters to satellites, from car engine controllers to medical devices. Unlike a general-purpose computer, an MCU is designed for one dedicated task and runs its program in an infinite loop from the moment power is applied. This module teaches how microcontrollers work from the pin level up, drawing from H&H Ch.14-15 throughout. You will learn to configure GPIO pins, communicate over serial protocols, handle interrupts, generate PWM signals, and read analog sensors -- the fundamental skills of embedded programming.

## Topics

### Topic 1: MCU Architecture

A microcontroller integrates a CPU core, program memory (flash), data memory (SRAM), and a peripheral bus onto one chip. The CPU fetches instructions from flash, executes them, and reads/writes data in SRAM. The peripheral bus connects the CPU to I/O blocks: GPIO, UART, SPI, I2C, ADC, timers, and more. Two main architectures exist: Harvard (separate instruction and data buses, used by AVR and PIC) and von Neumann (shared bus, used by ARM Cortex-M). The clock system uses an oscillator (internal RC or external crystal) to generate the system clock, typically 8-240 MHz. At reset, the MCU runs a power-on sequence: initialize the stack pointer, clear RAM, and jump to the entry point of the user program. Peripherals appear as addresses in the memory map -- writing to a specific address configures a timer or sets a GPIO pin. This is called memory-mapped I/O, and it is how software controls hardware. -- H&H Ch.14-15

### Topic 2: GPIO -- General Purpose I/O

GPIO pins are the most fundamental peripheral: each pin can be configured as a digital input or output. As an output, the pin drives high (sources current to Vcc) or low (sinks current to ground), controlling LEDs, relays, and other devices. As an input, the pin reads the logic level applied by an external circuit. A critical subtlety: unconnected input pins "float" at an undefined voltage, reading random values. Pull-up resistors (connecting the pin to Vcc through a resistor) or pull-down resistors (connecting to ground) establish a known default state. Most MCUs have internal pull-up/pull-down resistors that can be enabled in software. Open-drain outputs can only pull low; they need an external pull-up to go high, which allows multiple devices to share a wire (as in I2C). GPIO registers control everything: the direction register (input/output), the data register (current state), and the pull-enable register. -- H&H Ch.14-15

### Topic 3: UART Serial Communication

UART (Universal Asynchronous Receiver/Transmitter) sends data one bit at a time without a shared clock. The sender and receiver must agree on the baud rate (bits per second) beforehand. Each byte is wrapped in a frame: one start bit (always 0), eight data bits sent LSB first, and one stop bit (always 1). Common baud rates include 9600 (reliable over long wires) and 115200 (fast for debug output). The start bit signals the receiver to begin sampling; it then reads 8 data bits at the agreed rate and checks the stop bit. If the baud rates differ by more than about 3%, communication fails silently with garbled data. RS-232 uses +/-12V signaling for noise immunity over long cables; TTL-level UART uses 0V/3.3V (or 5V) for direct MCU connection. UART is used for debug consoles, GPS modules, Bluetooth modules, and any point-to-point serial link. -- H&H Ch.14-15

### Topic 4: SPI -- Serial Peripheral Interface

SPI is a synchronous, full-duplex, master-slave protocol using four wires: SCLK (clock, generated by master), MOSI (master out, slave in), MISO (master in, slave out), and CS (chip select, active low). The master drives SCLK and selects a slave by pulling its CS line low. On each clock edge, one bit shifts out on MOSI and one bit shifts in on MISO simultaneously -- this is full-duplex. Clock polarity (CPOL) and clock phase (CPHA) define four modes (0-3) controlling when data is sampled relative to the clock edge. SPI is fast (MHz range, up to 100+ MHz on some devices) because the shared clock eliminates baud rate agreement. It is used for displays, SD cards, flash memory, and high-speed ADCs/DACs. The drawback: each slave needs its own CS line, so connecting many devices requires many pins. -- H&H Ch.14-15

### Topic 5: I2C -- Inter-Integrated Circuit

I2C uses just two wires (SDA for data, SCL for clock) with pull-up resistors to support multiple masters and multiple slaves on the same bus. Each slave has a unique 7-bit address (up to 127 devices, though practical limits are lower). A transaction begins with a start condition (SDA goes low while SCL is high), followed by the 7-bit slave address, a read/write bit, and an ACK/NACK from the slave. Data bytes follow, each acknowledged by the receiver. A stop condition (SDA goes high while SCL is high) ends the transaction. Standard mode runs at 100 kHz, fast mode at 400 kHz, and fast-plus at 1 MHz. I2C is ideal for low-speed sensor networks: temperature sensors, EEPROMs, real-time clocks, and I/O expanders. Most sensors use a register-pointer model: write the register address, then read the data. -- H&H Ch.14-15

### Topic 6: Interrupts

Interrupts are hardware-triggered function calls that let the MCU respond to events without constantly polling. When an interrupt fires, the CPU saves its current context (program counter, registers), jumps to the interrupt service routine (ISR), executes it, then restores context and resumes the interrupted code. Sources include: pin-change interrupts (GPIO pin changes state), timer interrupts (timer overflows or reaches a compare value), UART receive interrupts (byte arrived), and external interrupts (button press, sensor alert). Each interrupt has a priority level; higher-priority interrupts can preempt lower-priority ISRs (nesting). Interrupt latency -- the time from event to ISR entry -- is typically a few clock cycles on ARM Cortex-M cores. The cardinal rule: keep ISRs short. Set a flag, copy data to a buffer, and return. Do the heavy processing in the main loop. Long ISRs block other interrupts and cause missed events. -- H&H Ch.14-15

### Topic 7: Timers and Counters

Hardware timers are counters driven by the system clock through a configurable prescaler. The prescaler divides the clock: a 16 MHz clock with prescaler 64 produces 250 kHz timer ticks. The counter increments on each tick and can count up or down. When the counter reaches its period (reload value), it overflows, resets to zero, and optionally triggers an interrupt. A compare match interrupt fires when the counter equals a programmable compare value -- this is the basis of PWM generation. Timers serve many purposes: generating precise delays, measuring pulse widths (input capture), counting external events (counter mode), generating PWM waveforms (compare output mode), and creating periodic interrupts for real-time scheduling. Most MCUs have multiple independent timer blocks with varying bit widths (8-bit, 16-bit, or 32-bit), each configurable with its own prescaler and mode. -- H&H Ch.14-15

### Topic 8: PWM -- Pulse Width Modulation

PWM is a digital technique to simulate analog output. A timer generates a square wave with a fixed frequency and a variable duty cycle (the fraction of each period the signal is high). Duty cycle = on_time / period. At 50% duty cycle, the average voltage is half the supply. PWM is used for LED dimming (brightness proportional to duty cycle), motor speed control (torque proportional to average voltage), servo positioning (pulse width encodes angle, typically 1-2 ms in a 20 ms period), and as a DAC substitute for slow-changing signals. Resolution depends on the timer: an 8-bit timer gives 256 levels (0.39% steps), a 16-bit timer gives 65536 levels. Higher PWM frequency reduces audible whine from motors and visible flicker from LEDs but also reduces resolution for a given timer clock. The relationship is: max_resolution = timer_clock / PWM_frequency. A 16 MHz timer at 1 kHz PWM gives 16000 levels. -- H&H Ch.14-15

### Topic 9: ADC on MCU

Most microcontrollers include a built-in SAR (successive approximation register) ADC. The ADC converts an analog voltage on a pin to a digital code proportional to V_in / V_ref. Typical specs: 10-12 bits resolution, 100 kSps to 1 MSps conversion rate, multiplexed input channels (one ADC shared across multiple pins via an analog mux). The reference voltage selection matters enormously: internal references (1.1V, 2.5V) are convenient but less accurate; external references are more precise but require a dedicated pin and component. The sample-and-hold circuit captures the input voltage at the start of conversion and holds it constant while the SAR logic performs binary search across code values. Conversion time depends on resolution and clock speed. For noisy signals, averaging multiple readings (oversampling) reduces noise by sqrt(N). DMA (direct memory access) can transfer ADC results to memory without CPU intervention for continuous high-speed acquisition. -- H&H Ch.14-15

### Topic 10: Common MCU Platforms

Arduino (ATmega328P) is an 8-bit AVR running at 16 MHz with 32KB flash, 2KB SRAM, and a beginner-friendly ecosystem. The Arduino IDE and vast library collection make it the standard entry point for embedded programming. ESP32 is a dual-core 240 MHz processor with built-in WiFi and Bluetooth Low Energy, making it the go-to platform for IoT projects. It has a 12-bit ADC, multiple SPI/I2C/UART peripherals, and costs under $5. STM32 (ARM Cortex-M series) is the professional choice, ranging from the low-power STM32L0 (32 MHz, 8KB RAM) to the high-performance STM32H7 (480 MHz, 1MB RAM, hardware floating point). The Raspberry Pi Pico (RP2040) features dual ARM Cortex-M0+ cores at 133 MHz with a unique PIO (Programmable I/O) subsystem that can implement custom serial protocols in hardware. Choosing the right MCU depends on: clock speed (processing power), peripheral set (what you need to connect), power consumption (battery life), memory (program size), and ecosystem (tools, libraries, community support). -- H&H Ch.14-15

### Level 3: Mathematical

> Baud rate: bits/second = f_clk / (prescaler * divisor). Timer period: T = (reload + 1) * (prescaler / f_clk). PWM duty cycle: D = compare_value / period_value. SPI clock: f_SPI = f_clk / (2 * (SPBRG + 1)). -- H&H Ch.14-15
