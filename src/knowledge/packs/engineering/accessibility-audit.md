# ENGR-101: Accessibility Audit

**Pack:** Engineering & Design — Foundational Knowledge Pack
**Audit Date:** 2026-02-20
**Overall Status:** Partially Accessible

## Feature Status

| Feature | Status | Notes |
|---------|--------|-------|
| Screen Reader Compatible | Partial | Design process instructions are clear; design sketches and blueprints need detailed descriptions |
| Large Text Support | Implemented | Written instructions scale well for larger fonts |
| High Contrast Mode | Implemented | High-contrast design tool interface available |
| Keyboard Navigation | Implemented | Design tools support keyboard input; CAD alternatives available |
| Alt Text for Images | Partial | Design specifications and material diagrams included; assembly drawings need enhancement |
| Media Captions | Not Started | No video demonstrations; captions planned for prototyping videos |

## Content-Specific Considerations

- **Design Sketches:** Engineering drawings and blueprints are visual-heavy. Text-based design specifications provided; learners with visual impairments can participate through tactile model building and collaborative design discussions.
- **Prototyping & Testing:** Hands-on building is inherently tactile and accessible. Clear verbal instructions enable full participation; safety concerns addressed with extra detail.
- **Materials Knowledge:** Material properties (strength, flexibility, weight) described; learners can explore through touch and manipulation.
- **Engineering Ethics:** Discussion-based ethical considerations are naturally accessible; case studies include written descriptions of design scenarios.

## Remediation Path

### Priority 1 (Critical)
1. Create detailed verbal descriptions of design specification drawings
2. Describe material properties tactilely (texture, weight, flexibility) for hands-on exploration
3. Enhance assembly instruction clarity for learners building from descriptions
4. Provide text-based alternatives for design constraint diagrams
5. Ensure testing procedure instructions are clear and sequential

### Priority 2 (Important)
1. Add captions to any prototyping demonstration videos
2. Create accessible versions of design iteration documentation
3. Develop keyboard shortcuts for CAD and design software
4. Build peer collaboration guides for mixed-ability design teams
5. Create accessible assessment rubrics for design projects

### Priority 3 (Enhancement)
1. Develop tactile design models for reference and learning
2. Create audio descriptions of prototype testing procedures
3. Build customizable design templates for different learning preferences
4. Create accessible failure analysis and troubleshooting guides

## WCAG 2.1 Compliance Notes

- **Target:** WCAG 2.1 Level AA
- **Current Status:** Level A for process instructions; Level AA for design tools
- **Major Gaps:**
  - Design drawing descriptions need expansion
  - CAD tool accessibility features incomplete
  - Material property descriptions could be more sensory
  - Video demonstrations not yet captioned
- **Strengths:**
  - Hands-on nature allows tactile learning
  - Clear sequential design process
  - Good contrast in design interfaces
  - Text-based specifications available
