# CODE-101: Computer Science & Coding

Computer science and coding as tools for thinking, problem-solving, and creating.

## What Is This Pack?

CODE-101 teaches coding and computer science not as isolated technical skills, but as ways to solve problems, express ideas precisely, and build complex things from simple parts. From unplugged algorithms through complexity analysis, this pack grows with learners across all ages and experience levels.

**Core Philosophy:** Code is how you tell computers what to do. Computer science is how you solve problems precisely and efficiently. This pack teaches both as tools for thinking.

## Quick Start

**New to coding?** Start with Module 1 (Computational Thinking) — no computer required.

**Ready to write code?** Module 2 (Programming Fundamentals) covers visual and text-based environments.

**Want to build things?** Module 3 (Building Projects) teaches project planning, testing, and debugging.

**Interested in depth?** Module 4 (Algorithms & Efficiency) explores why some solutions are better than others.

**Thinking about impact?** Module 5 (Computing & Society) covers ethics, AI, security, and digital citizenship.

## Key Features

### 5 Core Modules

1. **Computational Thinking & Algorithms** — Breaking problems into steps, recognizing patterns, designing algorithms
2. **Programming Fundamentals** — Variables, control flow, input/output in visual and text-based environments
3. **Building Projects & Problem-Solving** — Planning, iterating, testing, debugging, organizing code
4. **Algorithms & Efficiency** — Sorting, searching, complexity analysis, optimization
5. **Computing & Society** — Ethics, AI, security, digital citizenship, open source

### Progression Pathways

- **Visual to Text:** Foundation through visual environments (Scratch, Blockly), then text-based (Python, JavaScript)
- **Algorithm Focus:** For students interested in computer science depth
- **Applied & Practical:** Building useful projects and tools
- **Ethics-Aware:** Integrating ethical considerations throughout

## What Learners Can Do

### By End of Foundation Level
- Understand sequences and basic algorithms
- Create simple programs with visual tools
- Find and fix errors in code
- Recognize patterns and rules

### By End of Elementary Level
- Write programs with variables and loops
- Break problems into smaller parts
- Test their code to find bugs
- Create interactive games and animations

### By End of Middle School Level
- Write clear, organized code
- Use data structures appropriately
- Build complete projects with iteration
- Understand basic algorithm efficiency

### By End of High School Level
- Design algorithms and analyze efficiency (Big-O)
- Write modular, maintainable code
- Debug subtle problems systematically
- Build tools and visualizations with real data

## Assessment

### Formative (During Learning)
- Can they explain what code does?
- Do they test their code?
- How do they respond to errors?
- Do they see patterns and connections?
- Are they curious and asking questions?

### Summative (Evidence of Mastery)
- **Portfolio:** Working programs, documented code, project write-ups
- **Projects:** Complete game, useful tool, data analysis, algorithmic problem
- **Reflection:** Understanding what went wrong and how to fix it

## Parent/Mentor Guidance

### If You Don't Know How to Code...

Great news — you can still help! Programming is about clear thinking. Ask your child to:
- **Explain** what their code does
- **Trace through** the logic step-by-step
- **Predict** what will happen
- **Test** their guesses

You don't need to understand syntax.

### Key Questions to Ask

- "What are you trying to make the computer do?"
- "How would you explain this in steps to someone else?"
- "What do you think will happen if you...?"
- "How could you test if that works?"
- "What error is it giving you? What might that mean?"
