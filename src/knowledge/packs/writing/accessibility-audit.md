# WRIT-101: Accessibility Audit

**Status**: Alpha (2026-02-20) | **Target**: WCAG 2.1 AA

## Summary

WRIT-101 emphasizes close reading, narrative craft, and writing development through peer workshops. Key accessibility challenge: peer review and writing tools must support diverse input modalities; texts should be available in audio format; discussion platforms must be keyboard-navigable.

## Key Issues & Remediation

### 1. Text Availability
**Priority: HIGH** | **WCAG**: 1.2 (Alternatives for Time-based Media)
- Provide all assigned texts in multiple formats (print, digital, audio)
- Use accessible e-readers or proper PDF/HTML markup
- Ensure audio versions are professionally narrated
- Provide transcripts for audio-only access

### 2. Peer Review Tools
**Priority: HIGH** | **WCAG**: 2.1.1 (Keyboard), 4.1.2 (Name, Role, Value)
- Ensure annotation and feedback tools are keyboard-accessible
- Allow multiple feedback modalities (text, audio, video)
- Make comment threads navigable with screen readers
- Support voice-to-text input for commenting

### 3. Writing Tools
**Priority: MEDIUM** | **WCAG**: 2.1.1 (Keyboard), 1.1.1 (Non-text Content)
- Ensure word processors used are accessible (Google Docs, Office)
- Provide grammar/spell-check with screen reader support
- Allow multiple response formats (written, recorded, visual)
- Support text-to-speech for reviewing work

### 4. Literary Texts & Examples
**Priority: MEDIUM** | **WCAG**: 1.1.1 (Non-text Content)
- Ensure examples with visual illustrations have alt text
- Provide plain-text versions of formatted poetry
- Include audio excerpts of dramatic works
- Describe visual formatting (typeface changes, spacing) in text
