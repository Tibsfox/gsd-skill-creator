## PF-02 — hybrid query latency

```sql
WITH spatial_candidates AS MATERIALIZED (
  SELECT ... FROM atlas.symbols
  WHERE position IS NOT NULL
    AND ST_DWithin(position, ST_SetSRID(ST_MakePoint($2, $3), 0), $4)
)
SELECT ..., embedding <=> $1::vector AS semantic_distance
FROM spatial_candidates
ORDER BY embedding <=> $1::vector
LIMIT $5;
```

|---|---|---|---|
| p50 | TBD | — | pending |

Methodology placeholder:
- Same fixture as PF-01, with a 384-dim query vector sampled from the existing IVFFlat-indexed `atlas.symbols.embedding` distribution.
- `EXPLAIN ANALYZE` is captured per trial and verified by `assertGistLeadingFilter` from `src/atlas/spatial/hybrid-query.ts`.
- Without the MATERIALIZED hint, prior empirical results (Part A research mission §2.4.6) suggest the planner may invert the cost model and PF-02 latency degrades 10–100×; the hint is therefore the load-bearing structural choice the benchmark verifies.
