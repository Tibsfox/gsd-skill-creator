## Migration status

**Applied 2026-05-10.** PostGIS 3.6.3 is installed at the cluster level; migration `004-postgis.sql` ran successfully under the maple role; `atlas.symbols` carries `position geometry(Point, 0)` and `bbox geometry(Polygon, 0)` with both GiST indexes live (`idx_atlas_symbols_position_gist`, `idx_atlas_symbols_bbox_gist`).

**Position backfill applied 2026-05-10.** All 97,705 rows now have non-null `position` populated by a deterministic hash-based synthetic layout (in atlas logical coords, [0..999]² uniform). The hash function is `abs(hashtextextended(id, seed)) % 1000` — same id always maps to same coordinates, so the layout survives reindex.

```sql
UPDATE atlas.symbols
SET position = ST_SetSRID(ST_MakePoint(
  (abs(hashtextextended(id, 0)) % 1000)::float,
  (abs(hashtextextended(id, 1)) % 1000)::float
), 0)
WHERE position IS NULL;
-- UPDATE 97705
```

The synthetic layout is suitable for substrate-validation and demo work; future symbol-graph layout passes will replace it with semantically-meaningful coordinates.

**Live PMTiles archive built 2026-05-10.** `buildSymbolTilePyramid` was run against the backfilled atlas.symbols and produced a 9.19 MB / 5,461-tile / z=0..6 pyramid in ~2 s. The pmtiles@4 reader round-trips both the small-archive (root-only) and large-archive (leaf-directory) paths; tile-fetch IPC returns valid MVT bytes end-to-end.
