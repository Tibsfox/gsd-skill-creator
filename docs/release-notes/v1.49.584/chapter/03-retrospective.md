## New lessons emitted (#10156–#10167)

- **Lesson #10156: Combined three-phase milestone (polish + citation + forward-build) is operationally productive when phases share substrate.** v1.49.584 demonstrates that doing polish + citation triage + forward-cadence build in the same milestone is more efficient than three separate milestones — the citation findings inform polish AND inform the new build, the polish makes the new build's template clean, and a single ship pipeline (manifests + release-notes + tag + push + RH refresh) covers all three workstreams. ~50% scope multiplier on the standard forward-cadence milestone for ~30% additional ship-pipeline overhead.
- **Lesson #10157: V-flag citation triage frequently surfaces fact corrections, not just page references.** 8 of 22 carried V-flags (~36%) revealed substantive factual errors when researchers checked NTRS / NASA history / AGU Wiley / Mudgway PDF. Future V-flag triage sprints should expect substantive corrections in a similar fraction. Plan agent budget accordingly: a citation-triage agent that finds a fact error needs to do additional research to identify the correct citation (which the original error obscured), so per-V-flag agent budget should account for both citation-lookup AND citation-correction work.
- **Lesson #10158: Mid-flight fact corrections should be applied by main context, not queued to finishing agents.** When Phase B citation-triage cluster agents surface corrections that affect another agent's in-flight work (Phase A polish agent), queueing via SendMessage risks delivery-after-completion. Apply the correction inline. Future workflows: cluster agents return findings to main context; main context decides to apply or queue based on agent state.
- **Lesson #10159: Cross-track scorer specs should be cross-checked for inconsistencies.** MUS scorer's Pedagogical Takeaway regex `^#{1,6}\s+(?:\d+\.\s+)?pedagogical takeaway\s*$/im` (numeric prefix optional) vs ELC scorer's `^#{1,6}\s+pedagogical takeaway\s*$/im` (no numeric prefix) caused unnecessary scorer iteration when ELC build agent used `## 7. Pedagogical Takeaway` form. A meta-scorer would catch cross-track inconsistencies; for now, both tracks should standardize on bare-heading form (no numeric prefix).
- **Lesson #10160: Concept-registry .ts files for new MUS subject-specs must be authored alongside subject-spec.json.** MUS scorer's criterion 8 (concept-registry entry) walks `.college/departments/music/concepts/` looking for the `concept_id`. v1.66 build agent did not author the .ts; main context authored it post-completion. Future Phase C build agent prompt template should include concept-registry .ts authoring as a Phase C deliverable for MUS, not discovered at scorer stage.
- **Lesson #10161: GRACEFUL-ATTRITION is the operational complement of PERSISTENT-CONSTELLATION-LISTENER.** PCL describes the *sustained* operational mode where individual platforms maintain identity across decades; GA describes the *attritional* operational mode where individual platforms fail and the constellation absorbs the loss. Both modes are required for multi-platform multi-decade missions; design-discipline pairs them naturally. Future degrees that frame multi-platform multi-decade systems should explicitly braid both threads.
- **Lesson #10162: MUS Pass-1 closure unblocks Pass-2 backfill or Pass-3 frontier work.** Future MUS subject-picks beyond v1.66 should default to Pass-2 backfill (advancing Domain target counts beyond Pass-1 minimums) or Pass-3 frontier (entirely new domain primitives) rather than continuing Pass-1 below-target closures. Pass-1 complete is the engine-state delta worth tracking.
- **Lesson #10163: Era-anchoring `flight-hardware-mapping.csv` benign warning is a design-discipline early signal.** When ELC scorer reports "no year for degree in mapping" in a benign-warning state, that's a signal the CSV is drifting behind ship-cadence. Phase D back-fill discipline should include flight-hardware-mapping.csv update as a standard step, not on-discovery. v1.66 back-filled v1.65 + added v1.66 in same Phase D pass.
- **Lesson #10164: Inner-heliosphere vantage opens a new mission-design category.** Pioneer 9's INNER-vantage position (~0.75 AU perihelion) distinguishes it from Pioneer 6/7/8 near-1.0-AU vantages. Future inner-heliosphere missions form a natural category for which Pioneer 9 is the corpus opener at v1.66. Watchlist: Helios 1+2, Mariner 10, Parker Solar Probe.
- **Lesson #10165: Form-as-multiplicity-coordination is a substrate-independent organizational principle.** Adams's three-orchestral-choir palindromic structure, Olympic Marmot colony alarm-chorus coordination, and Pioneer 6/7/8/9 four-platform constellation deployment all instantiate the same primitive. Future degrees should look for this primitive shape in cross-track triads where the Form domain is in play.
- **Lesson #10166: Inner-heliosphere TID environment is engineering-distinct from outer-heliosphere or near-1-AU TID environments.** Pioneer 9's ~0.75 AU perihelion exposed it to higher cumulative TID dose than Pioneer 6/7/8 over the same duration. Modern inner-heliosphere missions (Parker Solar Probe) confront this regime with rad-hard SiGe BiCMOS process families that descend directly from the 1960s-era discrete-Si bipolar lineage Pioneer 9 demonstrated.
- **Lesson #10167: Endemic species (single-region) pair structurally well with constellation-closing missions.** Olympic Marmot is ENDEMIC to Olympic Peninsula — not found anywhere else. Pioneer 9 closed the four-platform constellation — not extensible to additional Pioneer-class platforms. The pairing is structural: a closing artifact and an endemic organism share the property of being irreplaceable specific instances of their category.
