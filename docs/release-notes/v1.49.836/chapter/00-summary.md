# v1.49.836 — `publish.mjs` Destination-Side Hand-Author Preservation

**Released:** 2026-05-27

## What shipped

Adds destination-side hand-author preservation to `tools/release-history/publish.mjs`. Mirrors `chapter.mjs`'s C04 source-side preservation (from v1.49.585) at the publisher layer. A hand-authored `docs/release-notes/<v>/chapter/<file>.md` is no longer silently clobbered by the publish copy.

`publish.mjs` previously copied bytes from `.planning/roadmap/<v>/<file>.md` (auto-generated by `chapter.mjs`) to `docs/release-notes/<v>/chapter/<file>.md` (GitHub mirror) and `www/tibsfox/com/Research/release-story/<v>/<file>.md` (tibsfox.com staging) without checking if the destination already had different content. The v834/v835 ships hit this on every chapter file (`00-summary.md`, `03-retrospective.md`, `99-context.md`), and the handoff documented `git checkout HEAD -- <chapters>` as the recurring mechanical workaround.

v836 closes the workaround deterministically:

- **NEW** `tools/release-history/opener-match.mjs` — extracts `normalizeOpener` + `openerMatches` from `chapter.mjs` into a side-effect-free module. Re-exported from `chapter.mjs` for backward compatibility.
- **MODIFIED** `tools/release-history/publish.mjs` — adds `shouldPublishToDestination(sourceContent, destPath, forceOverwrite)` helper (exported) mirroring the C04 decision tree, but at the destination side. Wires the gate into both the per-target and toplevel publish loops. Adds `--force-overwrite` flag as the explicit bypass. Guards `main()` auto-run with an `isEntryPoint` check so the helper can be imported from tests without triggering the script body (which would attempt a PG connect).
- **MODIFIED** `tools/release-history/chapter.mjs` — re-exports `normalizeOpener` + `openerMatches` from the new module; removes the inline definitions.
- **NEW** `tests/release-history/publish-preserves-handauthored.test.ts` — 6 tests covering the 5-step decision tree + the `--force-overwrite` bypass.

## Why this ship

Closes the publish chapter-overwrite friction class via the **Two-layer closure for procedure-rooted drift** pattern (Lesson #10431):

- **Layer 1 (source-eliminator, already existed):** `chapter.mjs`'s `writeChapterIdempotent` preserves hand-authored content in `.planning/roadmap/<v>/<file>` (C04 from v1.49.585).
- **Layer 2 (detector/preservation, this ship):** `publish.mjs`'s new `shouldPublishToDestination` preserves hand-authored content at the GitHub-mirror + tibsfox.com staging destinations.

v585 closed source-side preservation 251 ships ago, but the destination-side counterpart was structurally missing. The operator workaround (`git checkout HEAD -- <chapters>`) absorbed it for 251 ships until v836 paid down the symmetric layer.

This is the **second instance** of the two-layer-closure pattern (#10431):
- v813 STATE.md `v805→v806` drift: source-eliminator `tools/state-md-set-shipped.mjs` + detector `tools/pre-tag-gate.sh` step 0.5.
- v836 publish chapter-overwrite drift: source-eliminator `chapter.mjs` C04 (v585) + detector `publish.mjs` `shouldPublishToDestination` (this ship).
