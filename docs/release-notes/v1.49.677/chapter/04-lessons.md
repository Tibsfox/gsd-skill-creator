### Lesson #10385 candidate — SUB-AGENT-PROMPT-FILENAME-COORDINATION-DRIFT

**Substrate observation:** At v677 W1+W2 two parallel sub-agents were dispatched concurrently — one for NASA 1.130 `index.html`, one for 13 artifact files. Each sub-agent received the MISSION-BRIEF.md + degree-sync.json as substrate inputs but neither received an explicit filename manifest. The two sub-agents independently chose mostly-divergent filenames for the same conceptual artifacts:

| Index.html referenced | Artifact sub-agent wrote |
|---|---|
| `audio/soyuz-launch-baikonur.dsp` | `audio/baikonur-launch-1986-03-13.dsp` |
| `audio/mir-dock-forward-port.dsp` | `audio/mir-soft-docking-1986-03-15.dsp` |
| `audio/inter-station-transit.dsp` | `audio/inter-station-transit-1986-05-05.dsp` |
| `circuits/argon-16b-flight-computer-flowchart.{cir,html,md}` | `circuits/lyappa-hand-rotating-arm-control.{cir,html,md}` |
| `shaders/lyappa-arm-rotation.frag` + `shaders/mir-orbital-geometry.frag` | only `shaders/inter-station-orbital-transfer.frag` (one shader, not two) |
| `sims/orbital-transfer-mechanics.py` | `sims/orbital-transfer-mir-to-salyut7.py` |
| `sims/two-station-phasing-walker.py` | `sims/cosmonaut-cumulative-career-days.py` |
| `sims/mir-modular-expansion-explorer.html` | `sims/mir-15-year-occupancy-explorer.html` |
| `story/soyuz-t15-mission-narrative.{html,tex}` | `story/soyuz-t-15-the-bridge-between-stations.{html,tex}` |
| `shaders/viewer.html` | `shaders/viewer.html` (only match) |

Of 14 href references in the index, only 1 (`shaders/viewer.html`) actually matched the artifact sub-agent's chosen filenames at first dispatch.

**Substrate-form proposed:** SUB-AGENT-PROMPT-FILENAME-COORDINATION-DRIFT. Pattern for parallel multi-artifact dispatches: when two or more sub-agents are concurrently authoring complementary deliverables (index page referencing artifacts; manifest referencing files; catalog referencing pages), the dispatch coordinator must include an explicit shared filename manifest in BOTH prompts. Otherwise each sub-agent's authorial choice diverges and a post-dispatch reconciliation pass is required.

**Operational mitigation:** the v677 inline fix-up cost ~5 minutes + ~3K tokens (sed-driven mass-rename via Bash + Edit for 1 stale shader reference). The pre-dispatch coordination cost (compute the filename manifest, include in both prompts) would have been ~1 minute + ~500 tokens. Net token-economy + operator-time saving from the discipline is meaningful at scale.

**Operational template proposed:**

```text
Filename manifest (use these exact paths for both sub-agents):
- artifacts/story/{slug}.html
- artifacts/story/{slug}.tex
- artifacts/shaders/{shader-name}.frag
- artifacts/shaders/viewer.html
- artifacts/audio/{event-name-1}.dsp
- artifacts/audio/{event-name-2}.dsp
- artifacts/audio/{event-name-3}.dsp
- artifacts/circuits/{circuit-name}.cir
- artifacts/circuits/{circuit-name}.html
- artifacts/circuits/{circuit-name}.md
- artifacts/sims/{sim-name-1}.py
- artifacts/sims/{sim-name-2}.py
- artifacts/sims/{sim-name-3}.html
```

The dispatch coordinator fills in the slugs once based on degree-sync.json substrate content + passes the exact manifest to both sub-agents. Each sub-agent then has a normative naming contract to honor.

**Soak observation #1.** Applied at v677 first instance. ESTABLISHED candidate at any future parallel-multi-artifact dispatch milestone.
