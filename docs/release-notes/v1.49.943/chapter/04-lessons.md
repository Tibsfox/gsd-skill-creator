## Promoted

- **Lesson #10464 (NEW, promoted) — Pin the GREEN/BREAKING Set boundaries of a defer-biased gate with a mutation-proven discriminating fixture.** A flip-readiness gate sorts each CI-leg conclusion through a `GREEN` Set (`{success}` — advances the consecutive-green streak), a `BREAKING` Set (`{failure, timed_out, cancelled, action_required}` — breaks it and stops the walk), and a transparent remainder (everything else — `neutral`/`skipped`/`stale`/`startup_failure`/`in_progress`/`null`/missing). The gate's safety property is **defer-bias**: a misclassification may only DEFER the flip, never advance it. That property lives entirely in the two Set boundaries, and each has a silent failure direction — a non-`success` conclusion drifting INTO `GREEN` advances the flip on a non-green run; a flaky-infra conclusion (`timed_out`/`cancelled`/`action_required`) missing FROM `BREAKING` lets a flaky lane advance the flip. Neither drift reds the gate's happy-path tests (N plain greens pass wherever the boundary sits), so the boundaries must be pinned explicitly with a **discriminating** fixture: `[green, X, green]` @ `n=3`, asserting `streak === 2`, `ready === false`, `broke === null` — it reds if `X` moved into `GREEN` (streak would reach 3 / ready) OR into `BREAKING` (`broke` set). A `[green, X, green, green]` @ `n=3` shape pushes the streak to `n` in the transparent case too (`ready` true wherever `X` sits), so the GREEN-boundary test must use exactly two greens — keeping `ready === false` the load-bearing assertion. Pin every named transparent state, not just one. Verify by mutation: widening a Set must red exactly the pinning test; `git checkout` restores green.
  - **Two-instance promotion.** Instance 1: **cargo** gate, v1.49.938 (`cargo-flip-readiness.test.mjs`) — `BREAKING`+`GREEN` boundaries pinned after the v938 adversarial review found a tracked `timed_out`/`cancelled` could fall through and advance the flip. Instance 2: **macOS** gate, v1.49.942 (`macos-flip-readiness.test.mjs`) — the mirror.
  - **This ship's application.** The `stale`/`startup_failure`/`in_progress` named-transparent states are now test-pinned in **both** suites (the discriminating two-green shape, mutation-proven in both directions), completing the macOS<->cargo Set-boundary symmetry. Gate sources unchanged.
