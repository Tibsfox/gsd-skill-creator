**Types to define in try-session-runner.ts:**

```typescript
/** Full try-session definition with steps (loaded from try-sessions/ directory) */
export interface TrySessionDefinition {
  id: string;
  title: string;
  description: string;
  estimatedMinutes: number;
  prerequisites: string[];    // Concept IDs that should be explored first
  steps: TryStep[];
}

/** A single step in a try-session */
export interface TryStep {
  instruction: string;        // What to do
  expectedOutcome: string;    // What should happen
  hint?: string;              // If stuck
  conceptsExplored: string[]; // Which concepts this step touches
}

/** Runtime state of a try-session */
export interface TrySessionState {
  sessionId: string;
  title: string;
  status: 'active' | 'completed' | 'abandoned';
  currentStep: number;
  totalSteps: number;
  stepsCompleted: boolean[];  // Track which steps are marked complete
  conceptsExplored: string[]; // Accumulated deduplicated concept IDs
  startedAt: Date;
  completedAt?: Date;
}
```

**TrySessionRunner class:**

```typescript
export class TrySessionRunner {
  private state: TrySessionState;
  private definition: TrySessionDefinition;

  /** Start a session from a definition */
  static start(definition: TrySessionDefinition): TrySessionRunner;

  /** Load and start a session from a department's try-sessions/ directory */
  static async loadSession(
    loader: CollegeLoader,
    departmentId: string,
    sessionId: string
  ): Promise<TrySessionRunner>;

  getCurrentStep(): TryStep;
  nextStep(): TryStep | null;
  previousStep(): TryStep | null;
  goToStep(index: number): TryStep;
  completeStep(): TryStep | null;  // Mark current as complete, advance
  getProgress(): { currentStep: number; totalSteps: number; percentComplete: number };
  getPrerequisites(): string[];
  getConceptsExplored(): string[];
  getState(): TrySessionState;
}
```

**loadSession() reads from filesystem:**
1. Construct path: `{basePath}/{departmentId}/try-sessions/{sessionId}.json`
2. Parse JSON into TrySessionDefinition
3. Return new TrySessionRunner initialized with the definition

**Concept tracking:** Each time a step is visited (via nextStep, goToStep, or completeStep), its `conceptsExplored` array is merged into the session's accumulated set (deduplicated).

**Test setup:** Tests create a TrySessionDefinition inline (no filesystem needed for most tests). Test 11 (loadSession) creates a temp directory with a JSON try-session file.
