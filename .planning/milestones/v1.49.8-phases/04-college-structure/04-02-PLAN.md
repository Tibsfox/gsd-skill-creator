  <action>
Implement DepartmentExplorer class. It wraps CollegeLoader and ConceptRegistry to provide path-based navigation.

**Constructor:** Takes a `CollegeLoader` and `ConceptRegistry` (dependency injection).

**Path format:** `{departmentId}/{wingId}/{conceptId}` -- each segment is optional:
- `mathematics` -> loads summary, returns entry point concept
- `mathematics/algebra` -> loads algebra wing, returns first concept
- `mathematics/algebra/ratios` -> loads specific concept from algebra wing

**explore() method:**
1. Parse the path into segments: `[departmentId, wingId?, conceptId?]`
2. If only departmentId: call `loader.loadSummary()`, find entryPoint concept in registry
3. If departmentId + wingId: call `loader.loadWing()`, return first concept
4. If all three: call `loader.loadWing()`, find specific concept by ID
5. Build `relatedPaths` from concept relationships -- for each relationship, construct the navigable path `{targetDomain}/{wingId}/{targetId}`
6. Extract `pedagogicalContext` from the DEPARTMENT.md content (loader provides this in summary)

**listExplorablePaths() method:**
1. Load department summary to get wing list
2. For each wing, load wing content to get concept list
3. Return flattened array of all `dept/wing/concept` paths

**Error handling:** Invalid paths throw `ExplorationError` with message explaining valid path structure.

**Test setup:** Tests create a mock ConceptRegistry with test concepts that have relationships, and a CollegeLoader pointing at a temp directory with a test department. Mock concepts should include at least two concepts with a cross-reference relationship to verify relatedPaths generation.

Write tests FIRST (RED), then implement (GREEN).

    - Test 2: resolve() returns empty matches array when no cross-references exist to the target department
    - Test 3: resolve() for non-existent source concept throws ConceptNotFoundError
    - Test 4: resolve() finds both analogy and cross-reference relationship types
    - Test 5: resolveAll('mathematics') returns all cross-references from math department to any other department

  </behavior>
  <action>
Implement CrossReferenceResolver class that links concepts across departments.

**Constructor:** Takes a `ConceptRegistry` (dependency injection).

**resolve() method:**
1. Look up the source concept in the registry
2. Get all relationships of type 'analogy' or 'cross-reference'
3. Filter for relationships where the target concept belongs to `toDepartment`
4. Build CrossReferenceResult with matches array

**resolveAll() method:**
1. Get all concepts in the source department via `registry.search('', domain)`
2. For each concept, collect cross-reference and analogy relationships
3. Group by target department
4. Return array of CrossReferenceResult objects

**findBridges() method:**
1. Get all concepts in department A
2. Get all concepts in department B
3. Find pairs where A has a relationship pointing to B AND/OR B has a relationship pointing to A
4. Return array of bridge objects: `{ conceptA: string, conceptB: string, relationships: string[] }`

**Test setup:** Tests create a ConceptRegistry populated with:
- `math-exponential-decay` with analogy relationship to `culinary-cooling-curve`
- `culinary-cooling-curve` with cross-reference relationship to `math-exponential-decay`
- `math-ratios` with analogy relationship to `culinary-bakers-percentages`
- A concept with NO cross-references (for empty matches test)

Write tests FIRST (RED), then implement (GREEN).
