<feature>
  <name>CalibrationEngine — universal feedback loop with bounded adjustment</name>
  <files>.college/calibration/engine.ts, .college/calibration/engine.test.ts</files>
  <behavior>
    Stage 1 — OBSERVE: process() captures feedback.observedResult as ObservedResult with timestamp.
    Stage 2 — COMPARE: compare() returns a ComparisonDelta with category (derived from parameter names in feedback), magnitude (numeric difference from expected), and direction ('over' | 'under' | 'miss').
    Stage 3 — ADJUST: model.computeAdjustment(delta) returns raw adjustments. boundAdjustment() clamps each value: if |rawAdj / currentParam| > 0.20, clamp to sign(rawAdj) * 0.20 * currentParam.
    Stage 4 — RECORD: deltaStore.save(record) is called with the bounded CalibrationDelta. process() returns the persisted record.

    Bounded adjustment invariants (CAL-06):
    - Test 1: adjustment of +50 on a parameter valued 100 → clamped to +20 (20% of 100)
    - Test 2: adjustment of -5 on a parameter valued 100 → passes through unchanged (5% < 20%)
    - Test 3: adjustment of +25 on parameter valued 100 → clamped to +20
    - Test 4: adjustment of -22 on parameter valued 100 → clamped to -20
    - Test 5: adjustment of 0 → passes through (no change)

    Full loop test (CAL-01):
    - Given a mock DomainCalibrationModel and mock DeltaStore
    - When process() is called with UserFeedback { domain: 'test', observedResult: 'overdone', expectedResult: 'medium', parameters: { temperature: 350 } }
    - Then: observe stage produces ObservedResult with 'overdone' and timestamp
    - Then: compare stage produces ComparisonDelta with category 'temperature', direction 'over'
    - Then: adjust stage calls model.computeAdjustment with the delta
    - Then: record stage calls deltaStore.save exactly once with a CalibrationDelta
    - Then: returned CalibrationDelta has confidence in [0,1], timestamp is a Date, domainModel matches feedback.domain

    Model pluggability (CAL-02 proof):
    - engine.registerModel(model: DomainCalibrationModel) adds model to internal map
    - process() with feedback.domain = 'cooking' → selects cooking model
    - process() with unknown domain → throws Error('No calibration model registered for domain: X')
  </behavior>
  <implementation>
    After tests pass:

    1. Define supporting types in engine.ts (not re-exported from types.ts):
       - `UserFeedback`: { domain: string; translationId: string; observedResult: string; expectedResult: string; parameters: Record<string, number> }
       - `ObservedResult`: { raw: string; timestamp: Date; parameters: Record<string, number> }
       - `ComparisonDelta`: { category: string; magnitude: number; direction: 'over' | 'under' | 'miss' }
       - `DomainCalibrationModel`: extends CalibrationModel (from types.ts) adding computeAdjustment(delta: ComparisonDelta): Record<string, number> and confidence(delta: ComparisonDelta): number

    2. Implement CalibrationEngine class:
       - Constructor accepts DeltaStore (use the interface shape from delta-store.ts stub, define minimal interface here for now)
       - Private models: Map<string, DomainCalibrationModel>
       - registerModel(model: DomainCalibrationModel): void
       - Private observe(feedback: UserFeedback): ObservedResult
       - Private compare(observed: ObservedResult, expectedResult: string): ComparisonDelta
         - direction: 'over' if observed matches 'over/too much/high/burnt/dry', 'under' if 'under/too little/low/raw/wet', else 'miss'
         - magnitude: 1.0 (symbolic — actual magnitude from domain model; normalized at compare stage)
         - category: first key of feedback.parameters (primary parameter being calibrated)
       - Private boundAdjustment(rawAdj: Record<string, number>, currentParams: Record<string, number>): Record<string, number>
         - For each key: if |rawAdj[k] / currentParams[k]| > 0.20 → clamp to Math.sign(rawAdj[k]) * 0.20 * currentParams[k]
         - If currentParams[k] is 0 or undefined → pass through unchanged (avoid division by zero)
       - async process(feedback: UserFeedback): Promise<CalibrationDelta>
         - Steps: observe → compare → adjust (boundAdjustment applied) → record (deltaStore.save) → return

    3. Export: CalibrationEngine, UserFeedback, ComparisonDelta, ObservedResult, DomainCalibrationModel
  </implementation>
</feature>
