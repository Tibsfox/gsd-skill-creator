    **SafetyWarden class (.college/safety/safety-warden.ts):**

    Constructor takes an optional `now` parameter (Date factory function) for testable time. Maintains:
    - `boundaries: Map<string, SafetyBoundary>` -- registered safety boundaries
    - `dangerZones: Map<string, { temperature: number; startTime: Date }>` -- active danger zone tracking

    Methods:
    - `registerBoundaries(boundaries: SafetyBoundary[])`: Register boundaries. For duplicate parameters, keep the stricter limit (higher for minimum temps, lower for maximum times).
    - `check(input: SafetyCheckInput, mode: SafetyMode): SafetyCheckResult`: Core enforcement method. Looks up boundary for input.parameter. If no boundary found, returns {safe: true}. If boundary found:
      - Compare proposedValue against limit. For temp boundaries (parameter contains 'temp'), proposedValue must be >= limit. For time boundaries (parameter contains 'time' or 'zone'), proposedValue must be <= limit.
      - If safe, return {safe: true}.
      - If unsafe AND boundary.type === 'absolute':
        - annotate: {safe: false, action: 'annotate', warning: `SAFETY VIOLATION: ${boundary.reason}`, proposedValue: input.proposedValue, boundary}
        - gate: {safe: false, action: 'gate', requiresAcknowledgment: true, warning: boundary.reason, boundary}
        - redirect: {safe: false, action: 'redirect', safeValue: boundary.limit as number, reason: boundary.reason, boundary}
      - If unsafe AND boundary.type === 'warning':
        - annotate: {safe: false, action: 'annotate', warning: boundary.reason, proposedValue: input.proposedValue}
        - gate: {safe: false, action: 'gate', requiresAcknowledgment: true, warning: boundary.reason}
        - redirect: {safe: false, action: 'redirect', safeValue: boundary.limit as number, reason: boundary.reason}
    - `checkCalibrationOutput(adjustments: Record<string, number>, currentParams: Record<string, number>, mode: SafetyMode): Map<string, SafetyCheckResult>`: Checks all adjustments against boundaries. For each parameter in adjustments, computes the resulting value (currentParams[param] + adjustments[param]) and runs check(). Returns a Map of parameter -> SafetyCheckResult for any violations.
    - `trackDangerZone(itemId: string, temperatureF: number)`: If temperature is between 40 and 140 (inclusive), starts tracking. If outside that range, removes any existing tracking for this item (food left the danger zone).
    - `getActiveDangerZones(): DangerZoneEntry[]`: Returns all tracked items. For each, calculates elapsedMinutes from startTime to now(). Sets warning: true if elapsedMinutes >= 120.
    - `clearDangerZone(itemId: string)`: Removes tracking for an item.

    The constructor takes `now?: () => Date` for testable time injection. Defaults to `() => new Date()`.
