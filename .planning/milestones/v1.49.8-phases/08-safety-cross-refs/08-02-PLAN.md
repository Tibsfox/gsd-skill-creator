<objective>
Build the AllergenManager that ensures every ingredient substitution flags allergen implications, and create the safety module barrel export.

Purpose: SAFE-02 requires allergen warnings to be non-negotiable -- ingredient substitution must ALWAYS flag allergen implications. The AllergenManager maintains a database of ingredient-to-allergen mappings for the Big 9 (US FDA) allergens and checks every substitution proposal against it. This is safety-critical: a user with a peanut allergy must be warned when almond flour is suggested as a "nut-free" substitute (it is not -- tree nuts are separate from peanuts but both are Big 9).

Output: AllergenManager class, tests, safety module barrel export.

    **Internal Database (pre-populated):**
    - milk: ['milk', 'butter', 'ghee', 'cream', 'cheese', 'yogurt', 'whey', 'casein', 'lactose']
    - eggs: ['egg', 'eggs', 'mayonnaise', 'meringue', 'custard']

    - peanuts: ['peanut', 'peanuts', 'peanut butter', 'peanut oil']

    - sesame: ['sesame', 'sesame oil', 'sesame seeds', 'tahini', 'hummus']

    **Methods:**
    - `flagAllergens(ingredient: string): AllergenFlag[]`: Look up ingredient (case-insensitive) in the database. Return all matching allergen flags. If ingredient matches multiple categories (e.g., soy sauce -> soybeans + wheat), return flags for all.
    - `checkSubstitution(original: string, replacement: string): SubstitutionCheck`: Flag allergens in both the original and replacement. Return a SubstitutionCheck with:
      ```typescript
      interface SubstitutionCheck {
        original: AllergenFlag[];
        replacement: AllergenFlag[];
        safe: boolean;  // true if replacement introduces no NEW allergens
        warnings: string[];  // human-readable warnings
      }
      ```
      A substitution is "safe" if the replacement does not contain any allergens that the original did not. Warnings are generated for: same allergen in replacement, new allergen introduced, and when replacement has any allergens at all.
    - `getSubstitutions(ingredient: string): Array<{ingredient: string; allergenFlags: AllergenFlag[]}>`: Return known safe substitutions for the ingredient. Each substitution includes its own allergen flags so the user is ALWAYS informed. Pre-populate common substitutions:
      - milk -> oat milk, coconut milk, almond milk (with tree-nut flag), rice milk
      - eggs -> flax egg, chia egg, applesauce, commercial egg replacer
      - wheat flour -> rice flour, oat flour, almond flour (with tree-nut flag), coconut flour
      - butter -> coconut oil, olive oil, vegan butter
    - `registerIngredient(name: string, allergens: string[])`: Add a custom ingredient to the database. This allows departments to extend the allergen database.

    Export the SubstitutionCheck interface from types.ts.

    The AllergenManager is STATELESS with respect to user profiles -- it does not accept calibration state, user history, or preferences. This is by design per SAFE-02: allergen flagging fires regardless of user history.

    Write tests FIRST (RED), then implement (GREEN).
