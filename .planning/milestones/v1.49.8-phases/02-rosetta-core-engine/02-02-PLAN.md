      Rule 3: If taskType='explain', prefer panel matching user expertise: novice→natural/python, intermediate→python/lisp, advanced→lisp/pascal, expert→cpp/fortran/algol
      Rule 4: If taskType='compare', select 2-3 panels that illuminate different aspects (a systems + a heritage)
      Rule 5: If taskType='explore', suggest a panel NOT in recentPanels that has an expression for the concept
      Rule 6: Complex Plane integration — concept.complexPlanePosition.angle near 0 (±π/4) biases toward systems panels; near π/2 (±π/4) biases toward heritage panels; overrides rules 2-5 when angle is definitive
    - selectPanels always includes a non-empty rationale string explaining the decision
    - selectPanels falls back gracefully: if preferred panel has no expression for the concept, try next in preference order; if none match, use 'natural' panel
    - PanelRouter.selectPanels does NOT require the concept itself — it takes conceptId and context; the router consults registered panels' capabilities (not concept data directly)
