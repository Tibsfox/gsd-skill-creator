    - UserFeedback interface exported: { translationId: string, rating: 'helpful' | 'too-simple' | 'too-complex' | 'wrong-panel', comment?: string }
    - RosettaCore constructor accepts { registry: ConceptRegistry, router: PanelRouter, renderer: ExpressionRenderer, panelInstances: Map<PanelId, PanelInterface> }
    - translate(conceptId, context) steps:
        1. Look up concept in registry; throw ConceptNotFoundError if not found
        2. Load dependencies via registry.getDependencies (they appear in dependenciesLoaded)
        3. Get availablePanelIds from registry.getAvailablePanels(conceptId)
        4. Pass concept.complexPlanePosition into context.conceptComplexPosition
        5. Call router.selectPanels(context, availablePanelIds) to get PanelSelection
        6. Render primary: renderer.renderCalibrated if concept.calibration exists, else renderer.render at 'active' depth
        7. Render secondary panels (if any) at 'summary' depth
        8. Return Translation with a generated id (crypto.randomUUID() or Date.now().toString())
    - processFeedback(translationId, feedback) steps:
        Build a CalibrationDelta from the feedback:
          observedResult = feedback.rating
          expectedResult = 'helpful'
          adjustment = { complexity: feedback.rating === 'too-complex' ? -0.1 : feedback.rating === 'too-simple' ? 0.1 : 0 }
          confidence = 0.5  (stub — Phase 3 will refine)
          domainModel = 'feedback-stub'
          timestamp = new Date()
        Return the delta. Do NOT persist it (Phase 3 handles persistence).
    - ConceptNotFoundError is re-exported from './concept-registry.js' for consumers
  </behavior>
