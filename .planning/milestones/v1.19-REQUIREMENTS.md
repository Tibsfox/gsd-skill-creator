# Requirements: GSD Skill Creator

**Defined:** 2026-02-14
**Core Value:** Skills, agents, and teams must match official Claude Code patterns so they work correctly when loaded by Claude Code

## v1.19 Requirements

Requirements for v1.19 Budget Display Overhaul. Each maps to roadmap phases.

### Budget Model

- [x] **BM-01**: CumulativeBudgetResult includes both `installedTotal` and `loadableTotal` fields
- [x] **BM-02**: LoadingProjection type captures which skills fit within budget and which are deferred
- [x] **BM-03**: Loading projection respects tier priorities (critical > standard > optional)
- [x] **BM-04**: Loading projection uses same selection logic as BudgetStage pipeline
- [x] **BM-05**: Skills exceeding single-skill limit are flagged in the projection
- [x] **BM-06**: Loading projection is profile-aware (different budgets per agent profile)

### CLI Status

- [x] **BC-01**: "Installed Skills" section lists all skills with size as proportion of total installed
- [x] **BC-02**: "Loading Projection" section shows what fits within budget for the active profile
- [x] **BC-03**: Progress bar represents loading budget usage (not installed total)
- [x] **BC-04**: Per-skill percentage uses total installed as denominator (not budget limit)
- [x] **BC-05**: Over-budget installed total shows count-based summary ("3 of 14 skills fit")
- [x] **BC-06**: No negative headroom — show deferred count and oversized skill warnings instead
- [x] **BC-07**: Skills sorted by size descending with mini progress bars relative to largest skill
- [x] **BC-08**: JSON output mode (`--json`) includes both installed and loading projection data
- [x] **BC-09**: Color coding: green (<60%), cyan (60-79%), yellow (80-99%), red (>=100%) for loading budget

### Dashboard Gauge

- [x] **BD-01**: Gauge bar shows loading projection (not installed total)
- [x] **BD-02**: Hover tooltip on deferred skills area shows what was excluded
- [x] **BD-03**: Over-budget state shows filled bar with red outline (not impossible >100% width)
- [x] **BD-04**: Threshold transitions remain at 80% warning and 95% critical

### Budget Configuration

- [x] **BF-01**: Cumulative budget configurable in integration config (not just env var)
- [x] **BF-02**: Per-profile cumulative budgets (executor may have different limit than planner)
- [x] **BF-03**: Budget history tracks installed total and loaded total separately
- [x] **BF-04**: Trend line shows both dimensions (installed growth vs loading efficiency)

### Technical Constraints

- [x] **BT-01**: Backward compatible — existing env var SLASH_COMMAND_TOOL_CHAR_BUDGET still works
- [x] **BT-02**: No new runtime dependencies
- [x] **BT-03**: All changes covered by TDD (RED-GREEN cycle)
- [x] **BT-04**: Budget history format migration handles old single-value snapshots gracefully

## Future Requirements

None deferred — v1.19 is tightly scoped to budget display.

## Out of Scope

| Feature | Reason |
|---------|--------|
| BudgetStage pipeline changes | Already works correctly; only display is broken |
| Token-based counting in status | Chars are the display currency; tokens are internal |
| New dashboard pages | Existing gauge and silicon panel are the surface |
| Skill loading behavior changes | This is display-only; loading logic stays the same |
