# Roadmap: v1.19 — Budget Display Overhaul

**Goal:** Fix the budget display across CLI and dashboard by separating the installed skill inventory from loading projection, fixing percentages, and making the budget configurable.

## Phases

### Phase 149 — Budget Inventory Model
**Status:** complete (2026-02-14)
**Goal:** Users can inspect a budget model that clearly separates what is installed on disk from what will actually load into context, with tier-aware projection and profile awareness.
**Plans:** 2 plans
Plans:
- [x] 149-01-PLAN.md — LoadingProjection type, tier-based simulation logic, single-skill flagging, profile awareness
- [x] 149-02-PLAN.md — CumulativeBudgetResult with installedTotal/loadableTotal, formatBudgetDisplay update

**Requirements:** BM-01, BM-02, BM-03, BM-04, BM-05, BM-06, BT-02, BT-03

**Success Criteria:**
1. CumulativeBudgetResult exposes both installedTotal and loadableTotal as separate numeric fields
2. LoadingProjection lists which skills are loaded and which are deferred, sorted by tier priority (critical > standard > optional)
3. Skills exceeding the single-skill character limit appear as flagged in the projection output
4. Running the projection with different agent profiles produces different loaded/deferred sets
5. No new runtime dependencies are introduced (dev dependencies are fine)

### Phase 150 — CLI Status Display Redesign
**Status:** complete (2026-02-13)
**Goal:** Users see a two-section CLI status layout that shows their full installed inventory with proportional sizing and a loading projection with loaded/deferred breakdown.
**Depends on:** Phase 149
**Plans:** 3 plans
Plans:
- [x] 150-01-PLAN.md — Installed skills section with proportional percentages and size-relative mini bars
- [x] 150-02-PLAN.md — Loading projection section with loaded/deferred breakdown, count summary, oversized warnings, color coding
- [x] 150-03-PLAN.md — JSON output mode with both installed and projection data

**Requirements:** BC-01, BC-02, BC-03, BC-04, BC-05, BC-06, BC-07, BC-08, BC-09

**Success Criteria:**
1. Running `status` shows an "Installed Skills" section listing every skill with its size as a percentage of total installed (not budget limit)
2. Running `status` shows a "Loading Projection" section with a progress bar representing loading budget usage and a loaded/deferred skill count
3. Over-budget scenarios display "3 of 14 skills fit" style summaries with no negative headroom values
4. Running `status --json` returns structured data containing both installed inventory and loading projection
5. Progress bar and budget text use color coding (green/cyan/yellow/red) based on loading budget percentage thresholds

### Phase 151 — Dashboard Gauge & Budget Configuration
**Status:** complete (2026-02-14)
**Goal:** Users see accurate loading projection in the dashboard gauge and can configure cumulative budgets per profile in integration config, with historical tracking of both dimensions.
**Depends on:** Phase 149
**Plans:** 2 plans
Plans:
- [x] 151-01-PLAN.md — Dashboard gauge showing loading projection, deferred tooltip, over-budget red outline, threshold transitions
- [x] 151-02-PLAN.md — Integration config budget settings, per-profile cumulative budgets, env var backward compat, history format migration, dual-dimension trend line

**Requirements:** BD-01, BD-02, BD-03, BD-04, BF-01, BF-02, BF-03, BF-04, BT-01, BT-04

**Success Criteria:**
1. Dashboard budget gauge bar represents loading projection percentage (not installed total) with correct threshold transitions at 80% and 95%
2. Hovering over the deferred area of the gauge shows a tooltip listing excluded skills
3. Over-budget state renders a filled bar with red outline (no >100% width overflow)
4. Integration config accepts a cumulative budget field with per-profile limits (executor, planner, etc.)
5. Existing SLASH_COMMAND_TOOL_CHAR_BUDGET env var continues to work as a fallback when no config is set
