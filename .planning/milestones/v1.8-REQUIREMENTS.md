# Requirements: GSD Skill Creator v1.8

**Defined:** 2026-02-08
**Core Value:** Skills, agents, and teams must match official Claude Code patterns so they work correctly when loaded by Claude Code.

## v1.8 Requirements

Requirements for v1.8 milestone. Each maps to roadmap phases.

### Capability-Aware Planning

- [x] **CAP-01**: CAPABILITIES.md is auto-generated from available skills, agents, and teams (global + project-local) with machine-readable structured format ✓ Phase 54
- [x] **CAP-02**: ROADMAP.md phases declare capability verbs (use/create/after/adapt) referencing skills, agents, and teams ✓ Phase 55
- [x] **CAP-03**: Plan frontmatter includes capabilities field with skill/agent/team references inherited from phase declarations ✓ Phase 55
- [x] **CAP-04**: Executor context receives auto-injected skills based on plan capabilities field ✓ Phase 56
- [x] **CAP-05**: Phase `create` verb generates plan tasks that scaffold new skills/agents in project-local `.claude/` ✓ Phase 56
- [x] **CAP-06**: Phase `after` verb automatically invokes referenced capabilities after phase completion and verification ✓ Phase 60
- [x] **CAP-07**: Capability discovery runs automatically during new-project and new-milestone workflows to populate CAPABILITIES.md ✓ Phase 54

### Token Efficiency

- [x] **TOK-01**: Skills are ordered by cacheTier metadata (static/session/dynamic) with deterministic tiebreaking for prompt cache optimization ✓ Phase 57
- [x] **TOK-02**: Research artifacts can be compressed to distilled skill files (10-20x reduction) via compress-research CLI with staleness detection ✓ Phase 58
- [x] **TOK-03**: Skills declare modelGuidance metadata for whole-skill model targeting (opus/sonnet/haiku) with graceful skip on mismatch ✓ Phase 59
- [x] **TOK-04**: GSD agents have per-agent token budget profiles with priority tiers (critical/standard/optional) and 50/80/100% warning thresholds ✓ Phase 53
- [x] **TOK-05**: Read-only collector agents can be auto-generated for context-efficient information gathering with compressed summary output ✓ Phase 60
- [x] **TOK-06**: Parallelization advisor analyzes plan dependencies and recommends wave-based parallel vs sequential execution ✓ Phase 61
- [x] **TOK-07**: Context budget estimation CLI (budget-estimate) reports skill loading cost as token estimates with threshold warnings ✓ Phase 53

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Compaction-aware skills | Claude Code's compaction is opaque internal mechanism; skill metadata cannot influence it. Add compaction resilience guidelines to docs instead. |
| Per-instruction model coverage | 30-50% metadata overhead with unclear ROI; whole-skill targeting sufficient for v1.8 |
| Full token usage reporting | GSD inside Claude Code has no access to token usage metrics from extensions |
| Direct cache_control API calls | Skill-creator creates artifacts, not API requests; cacheTier provides ordering hints |
| Agent Teams native integration | High effort, needs Claude Code Agent Teams maturity (deferred to v2.0) |
| Cross-session memory beyond STATE.md | Deferred to v2.0 |
| MCP/A2A protocol support | Deferred to v2.0 |
| Skill registry/marketplace | Deferred to v2.0 |
