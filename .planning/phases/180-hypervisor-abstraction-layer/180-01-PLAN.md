  <action>
Create four library files that implement a hypervisor abstraction pattern. These files have NO shebang -- they are sourced, not executed directly (same pattern as discovery-common.sh).

**hypervisor-common.sh** -- Shared interface and dispatcher:
- Source discovery-common.sh for YAML helpers, has_command, warn
- Define the operation interface: every backend MUST implement these functions with the prefix `hv_{backend}_`:
  - `create` (name, cpus, ram_mb, disk_gb, iso_path) -- create a new VM
  - `start` (name) -- start a stopped VM
  - `stop` (name) -- graceful shutdown (try ACPI first, force after 30s timeout)
  - `snapshot` (name, snap_name) -- create a named snapshot
  - `destroy` (name) -- remove VM and its storage completely
  - `status` (name) -- print state: running/stopped/paused/not-found
  - `list` () -- print all VMs managed by this backend, one per line
- `hv_detect_backend()` -- reads `infra/inventory/hardware-capabilities.yaml` (or path from $CAPABILITIES_YAML), checks hypervisor section. Priority order: kvm > vmware > virtualbox (prefer open standard). Returns backend name string or "none".
- `hv_load_backend(backend)` -- sources the matching hv-{backend}.sh file from the same lib/ directory. Exits 1 with clear message if backend file not found.
- `hv_dispatch(operation, ...)` -- calls `hv_{BACKEND}_{operation} "$@"` where BACKEND is the loaded backend name. Validates operation is in the known set before dispatching.
- `hv_require_backend()` -- checks that a backend has been loaded, exits 1 with "No hypervisor backend loaded. Run hv_load_backend first." if not.

**hv-kvm.sh** -- KVM/libvirt backend:
- All functions prefixed `hv_kvm_`
- `hv_kvm_create`: use `virt-install --name $name --vcpus $cpus --memory $ram_mb --disk size=$disk_gb --cdrom $iso_path --os-variant centos-stream9 --network bridge=virbr0 --graphics none --noautoconsole --wait 0`. If VM already exists (virsh dominfo returns 0), warn and return 0 (idempotent).
- `hv_kvm_start`: `virsh start $name`. If already running (domstate == "running"), return 0.
- `hv_kvm_stop`: `virsh shutdown $name`, then poll `virsh domstate` for up to 30 seconds. If still running after 30s, `virsh destroy $name` (force off). If already shut off, return 0.
- `hv_kvm_snapshot`: `virsh snapshot-create-as $name --name $snap_name`. If snapshot exists, warn and return 0.
- `hv_kvm_destroy`: `virsh destroy $name 2>/dev/null; virsh undefine $name --remove-all-storage`. If VM doesn't exist, return 0.
- `hv_kvm_status`: parse `virsh domstate $name` output. Map to running/stopped/paused/not-found.
- `hv_kvm_list`: `virsh list --all --name` filtered to non-empty lines.
- Guard all commands with `has_command virsh || { warn "virsh not found"; return 1; }`

**hv-vmware.sh** -- VMware Workstation backend:
- All functions prefixed `hv_vmware_`
- Uses `vmrun` CLI (VMware Workstation/Fusion CLI tool)
- VM paths stored under `$HOME/vmware/{name}/{name}.vmx` convention
- `hv_vmware_create`: Generate a minimal .vmx file with numvcpus, memsize, disk size, ide0:0.fileName for ISO. Create the virtual disk with `vmware-vdiskmanager -c -s ${disk_gb}GB -a lsilogic -t 0 $disk_path` (or `qemu-img create -f vmdk` as fallback). If .vmx exists, warn and return 0.
- `hv_vmware_start`: `vmrun start $vmx_path nogui`. If already running (vmrun list contains path), return 0.
- `hv_vmware_stop`: `vmrun stop $vmx_path soft`. If not running, return 0. Fallback to `vmrun stop $vmx_path hard` after 30s.
- `hv_vmware_snapshot`: `vmrun snapshot $vmx_path $snap_name`.
- `hv_vmware_destroy`: `vmrun stop $vmx_path hard 2>/dev/null; rm -rf $vm_dir`. If dir doesn't exist, return 0.
- `hv_vmware_status`: check `vmrun list` for vmx path. Map to running/stopped/not-found.
- `hv_vmware_list`: parse `vmrun list` output, extract VM names from paths.
- Guard: `has_command vmrun || { warn "vmrun not found"; return 1; }`

**hv-vbox.sh** -- VirtualBox backend:
- All functions prefixed `hv_vbox_`
- Uses `VBoxManage` CLI
- `hv_vbox_create`: `VBoxManage createvm --name $name --ostype RedHat_64 --register`, then `modifyvm` for cpus, memory. `createhd --filename ... --size ${disk_gb_to_mb}`. `storagectl` + `storageattach` for disk and ISO. If VM exists (showvminfo returns 0), warn and return 0.
- `hv_vbox_start`: `VBoxManage startvm $name --type headless`. If already running, return 0.
- `hv_vbox_stop`: `VBoxManage controlvm $name acpipowerbutton`. Poll state for 30s, then `VBoxManage controlvm $name poweroff` if still running.
- `hv_vbox_snapshot`: `VBoxManage snapshot $name take $snap_name`.
- `hv_vbox_destroy`: `VBoxManage controlvm $name poweroff 2>/dev/null; VBoxManage unregistervm $name --delete`. If not found, return 0.
- `hv_vbox_status`: parse `VBoxManage showvminfo $name --machinereadable | grep VMState=`. Map to running/stopped/paused/not-found.
- `hv_vbox_list`: `VBoxManage list vms` parsed to extract names.
- Guard: `has_command VBoxManage || { warn "VBoxManage not found"; return 1; }`

**All files must follow established patterns:**
- `set -euo pipefail` at top

- Use `has_command` from discovery-common.sh
- Use `warn` for non-fatal messages to stderr
- All functions use `local` variables
- Idempotent: every operation checks current state before acting, returns 0 if already in target state

  <action>
Create the main CLI entry point that users and downstream automation call to manage VMs.

**infra/scripts/vm-ctl.sh:**
- Shebang: `#!/usr/bin/env bash`
- `set -euo pipefail`
- Source `lib/discovery-common.sh` and `lib/hypervisor-common.sh` from SCRIPT_DIR
- Parse arguments: `vm-ctl.sh [--backend kvm|vmware|vbox] <operation> [args...]`
  - `--backend` flag overrides auto-detection (useful for testing or multi-hypervisor hosts)
  - If no `--backend`, call `hv_detect_backend` to auto-select from hardware-capabilities.yaml
  - Default CAPABILITIES_YAML path: `${SCRIPT_DIR}/../inventory/hardware-capabilities.yaml` with fallback to `${SCRIPT_DIR}/../local/hardware-capabilities-local.yaml`
- Operations map directly to `hv_dispatch`:
  - `vm-ctl.sh create <name> --cpus N --ram-mb N --disk-gb N --iso PATH`
  - `vm-ctl.sh start <name>`
  - `vm-ctl.sh stop <name>`
  - `vm-ctl.sh snapshot <name> <snapshot-name>`
  - `vm-ctl.sh destroy <name>`
  - `vm-ctl.sh status <name>`
  - `vm-ctl.sh list`
  - `vm-ctl.sh detect` -- print which backend was auto-detected (diagnostic)
  - `vm-ctl.sh backends` -- list all available backends on this machine
- `--help` flag prints usage with all operations, examples, and option descriptions
- Before dispatching: call `hv_load_backend` with resolved backend, print `[vm-ctl] Using backend: $BACKEND` to stderr
- Exit codes: 0 = success, 1 = operation error, 2 = no backend available, 3 = invalid arguments
- Create operation argument parsing: `create` requires --cpus, --ram-mb, --disk-gb, --iso. Validate all are provided, cpus > 0, ram-mb > 0, disk-gb > 0, iso path exists (or warn if not for dry-run)
- `backends` subcommand: check each backend (kvm/vmware/vbox), print "available" or "not found" status for each
- Mark executable: `chmod +x infra/scripts/vm-ctl.sh`

**Follow established CLI patterns from existing scripts:**
- deploy-pxe.sh style argument parsing with while/case
- Status messages to stderr, data output to stdout
- Consistent formatting: `[vm-ctl] message` prefix for status messages
