Create a container runtime abstraction that provides Minecraft server deployment when VMs are unavailable.

**infra/scripts/lib/container-fallback.sh** -- Container runtime abstraction (no shebang, sourced):
- `set -euo pipefail`
- Source discovery-common.sh for has_command, warn, YAML helpers
- Detect runtime: `ct_detect_runtime()` -- returns "podman" if podman available, "docker" if docker available, "none" if neither. Podman preferred (rootless, daemonless).
- Store detected runtime in `_CT_RUNTIME` variable
- `ct_cmd()` -- wrapper that calls `$_CT_RUNTIME "$@"`. All subsequent functions use this.
- Container operations (all prefixed `ct_`):
  - `ct_create(name, image, port_mappings, env_vars, volume_mounts)`:
    - Build `$_CT_RUNTIME run -d --name $name --restart unless-stopped`
    - Add `-p host:container` for each port mapping (space-separated list of "host:container" pairs)
    - Add `-e KEY=VALUE` for each env var (space-separated list of "KEY=VALUE" pairs)
    - Add `-v host:container` for each volume mount (space-separated list of "host:container" pairs)
    - Append image name at end
    - If container already exists (inspect returns 0), warn and return 0 (idempotent)
  - `ct_create_minecraft(name, ram_mb, server_port, rcon_port, data_dir)`:
    - Convenience wrapper specifically for Minecraft server deployment
    - Uses `itzg/minecraft-server` image (most popular, well-maintained Minecraft Docker image)
    - Environment: `EULA=TRUE`, `TYPE=FABRIC`, `VERSION=1.21.4`, `MEMORY=${ram_mb}M`, `ENABLE_RCON=true`
    - Ports: `${server_port}:25565`, `${rcon_port}:25575`
    - Volume: `${data_dir}:/data`
    - Delegates to `ct_create` with assembled arguments
  - `ct_start(name)`: `ct_cmd start $name`. If already running, return 0.
  - `ct_stop(name)`: `ct_cmd stop $name`. If already stopped, return 0. Timeout 30s.
  - `ct_destroy(name)`: `ct_cmd stop $name 2>/dev/null; ct_cmd rm -f $name 2>/dev/null`. If not found, return 0.
  - `ct_status(name)`: inspect container, map to running/stopped/not-found. Output single word to stdout.
  - `ct_list()`: `ct_cmd ps -a --format '{{.Names}}'` (works for both podman and docker).
  - `ct_logs(name, lines)`: `ct_cmd logs --tail $lines $name`. Default lines=50.
  - `ct_exec(name, command...)`: `ct_cmd exec $name "$@"`. For running RCON commands inside the container.

**infra/scripts/container-ctl.sh** -- CLI entry point (with shebang):
- `#!/usr/bin/env bash` + `set -euo pipefail`
- Source `lib/discovery-common.sh` and `lib/container-fallback.sh` from SCRIPT_DIR
- Parse arguments: `container-ctl.sh [--runtime podman|docker] <operation> [args...]`
  - `--runtime` overrides auto-detection
  - If no `--runtime`, call `ct_detect_runtime` to auto-select
- Operations:
  - `container-ctl.sh minecraft <name> --ram-mb N [--port N] [--rcon-port N] [--data-dir PATH]`
    - Defaults: port=25565, rcon-port=25575, data-dir=$HOME/minecraft-data/$name
    - Calls `ct_create_minecraft` with resolved arguments
  - `container-ctl.sh create <name> --image IMAGE [--port HOST:CONTAINER...] [--env KEY=VALUE...] [--volume HOST:CONTAINER...]`
  - `container-ctl.sh start <name>`
  - `container-ctl.sh stop <name>`
  - `container-ctl.sh destroy <name>`
  - `container-ctl.sh status <name>`
  - `container-ctl.sh list`
  - `container-ctl.sh logs <name> [--lines N]`
  - `container-ctl.sh exec <name> -- <command...>`
  - `container-ctl.sh detect` -- print detected runtime
- `--help` prints usage with examples including the Minecraft shortcut
- Exit codes: 0 = success, 1 = operation error, 2 = no runtime available, 3 = invalid arguments
- Status messages to stderr with `[container-ctl]` prefix
- Mark executable: `chmod +x infra/scripts/container-ctl.sh`

```yaml
# Hypervisor Capability Matrix
# Documents supported operations and features per backend
# Consumed by: Phase 172 (VM provisioning), Phase 173 (server foundation)
# Updated by: Phase 180 (hypervisor abstraction layer)

version: 1.0.0

# VM backends -- managed by vm-ctl.sh
capability_matrix:
  kvm:
    type: hypervisor
    tool: virsh
    tool_package: libvirt-client  # dnf/apt package name varies by distro
    operations:
      create: true
      start: true
      stop: true
      snapshot: true
      destroy: true
      status: true
      list: true
      clone: true           # virsh vol-clone + virt-clone
      migrate: true          # virsh migrate (live migration)
      console: true          # virsh console
      resize_cpu: true       # virsh setvcpus
      resize_memory: true    # virsh setmem (with balloon driver)
    features:
      headless: true
      gpu_passthrough: true  # requires IOMMU
      nested_virtualization: true
      network_bridge: true
      storage_pool: true
      cloud_init: true
      pxe_boot: true
    resource_overhead:
      min_ram_mb: 256        # qemu process overhead
      cpu_overhead_percent: 2
    notes: "Primary recommended backend. Open standard, best Linux integration."

  vmware:
    type: hypervisor
    tool: vmrun
    tool_package: vmware-workstation  # manual install (no distro package)
    operations:
      create: true           # via .vmx generation + vmware-vdiskmanager
      start: true
      stop: true
      snapshot: true
      destroy: true
      status: true
      list: true
      clone: true            # vmrun clone
      migrate: false
      console: false         # GUI only
      resize_cpu: false      # requires VM shutdown + .vmx edit
      resize_memory: false   # requires VM shutdown + .vmx edit
    features:
      headless: true         # vmrun start nogui
      gpu_passthrough: false # not supported on Workstation
      nested_virtualization: true
      network_bridge: true
      storage_pool: false
      cloud_init: false      # manual only
      pxe_boot: true
    resource_overhead:
      min_ram_mb: 512
      cpu_overhead_percent: 3
    notes: "Proprietary. Good compatibility, but no live migration or GPU passthrough."

  virtualbox:
    type: hypervisor
    tool: VBoxManage
    tool_package: virtualbox
    operations:
      create: true
      start: true
      stop: true
      snapshot: true
      destroy: true
      status: true
      list: true
      clone: true            # VBoxManage clonevm
      migrate: false
      console: false         # GUI or vrde
      resize_cpu: false      # requires VM shutdown
      resize_memory: false   # requires VM shutdown
    features:
      headless: true         # startvm --type headless
      gpu_passthrough: false
      nested_virtualization: true  # VT-x passthrough
      network_bridge: true
      storage_pool: false
      cloud_init: false
      pxe_boot: true
    resource_overhead:
      min_ram_mb: 512
      cpu_overhead_percent: 5
    notes: "Cross-platform. Higher overhead than KVM. No GPU passthrough."

# Container backends -- managed by container-ctl.sh
# Used as fallback when no hypervisor is available
  podman:
    type: container
    tool: podman
    tool_package: podman
    operations:
      create: true
      start: true
      stop: true
      snapshot: false        # use podman commit for image snapshots
      destroy: true
      status: true
      list: true
      clone: false
      migrate: false
      console: true          # podman exec -it
      resize_cpu: true       # podman update --cpus
      resize_memory: true    # podman update --memory
    features:
      headless: true
      gpu_passthrough: false
      nested_virtualization: false
      network_bridge: true   # podman network
      storage_pool: false
      cloud_init: false
      pxe_boot: false
      rootless: true         # key differentiator from Docker
      daemonless: true       # no daemon required
      minecraft_image: "itzg/minecraft-server"
    resource_overhead:
      min_ram_mb: 64
      cpu_overhead_percent: 1
    notes: "Preferred container runtime. Rootless, daemonless, drop-in Docker replacement."

  docker:
    type: container
    tool: docker
    tool_package: docker-ce
    operations:
      create: true
      start: true
      stop: true
      snapshot: false
      destroy: true
      status: true
      list: true
      clone: false
      migrate: false
      console: true
      resize_cpu: true
      resize_memory: true
    features:
      headless: true
      gpu_passthrough: false
      nested_virtualization: false
      network_bridge: true
      storage_pool: false
      cloud_init: false
      pxe_boot: false
      rootless: false        # rootless mode exists but not default
      daemonless: false      # requires dockerd
      minecraft_image: "itzg/minecraft-server"
    resource_overhead:
      min_ram_mb: 128        # dockerd overhead
      cpu_overhead_percent: 1
    notes: "Fallback container runtime when Podman unavailable."

# Backend selection priority (used by vm-ctl.sh and container-ctl.sh)
selection_priority:
  hypervisors: [kvm, vmware, virtualbox]
  containers: [podman, docker]
  fallback_order: [kvm, vmware, virtualbox, podman, docker]

# Minimum requirements for Minecraft server deployment
minecraft_requirements:
  vm:
    min_ram_mb: 4096
    min_cpus: 2
    min_disk_gb: 20
    required_ports: [25565, 25575]
  container:
    min_ram_mb: 2048         # containers are more efficient
    min_cpus: 2
    min_disk_gb: 10
    required_ports: [25565, 25575]
```
