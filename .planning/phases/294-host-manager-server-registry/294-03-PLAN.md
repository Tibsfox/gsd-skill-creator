<objective>
Build the trace emission layer and Tauri IPC commands: a TraceEmitter that records structured trace events for every MCP message and emits them via Tauri's event system, plus IPC command handlers that expose the host manager to the frontend.

Purpose: Without tracing, MCP communication is invisible -- debugging requires log diving. Without IPC commands, the frontend has no way to connect/disconnect servers or invoke tools. This plan makes the host manager observable and controllable from the UI.

Output: `trace.rs` (trace recording and emission), `commands.rs` (Tauri IPC handlers), updated `mod.rs`, `lib.rs`, `state.rs`.
</objective>

  <action>
Create `src-tauri/src/mcp_host/trace.rs` implementing MCP message trace recording and emission.

**TraceEmitter struct:**
```rust
pub struct TraceEmitter {
    buffer: Vec<TraceEvent>,
    max_buffer: usize,
}
```

**Methods on TraceEmitter:**

1. `pub fn new(max_buffer: usize) -> Self` -- Creates with empty buffer. Default max_buffer = 1000.

2. `pub fn record_outgoing(&mut self, server_id: &str, method: &str, payload: Option<serde_json::Value>) -> String` -- Creates a TraceEvent with direction=Outgoing, timestamp=current unix ms (use `std::time::SystemTime::now().duration_since(UNIX_EPOCH)`), generates uuid for id. No latency_ms (outgoing). Pushes to buffer. If buffer exceeds max_buffer, remove oldest (VecDeque would be ideal -- use VecDeque instead of Vec). Returns the trace event id (for correlating with response).

3. `pub fn record_incoming(&mut self, server_id: &str, method: &str, request_timestamp: u64, payload: Option<serde_json::Value>, error: Option<String>) -> TraceEvent` -- Creates TraceEvent with direction=Incoming. Computes latency_ms from request_timestamp to now. Pushes to buffer. Returns the created TraceEvent.

4. `pub fn emit(&self, app_handle: &tauri::AppHandle, event: &TraceEvent)` -- Emits the TraceEvent as a Tauri event named "mcp-trace" using `app_handle.emit("mcp-trace", event)`. Serialize the event as the payload. If emit fails, log to stderr but do not propagate error.

5. `pub fn get_recent(&self, count: usize) -> Vec<&TraceEvent>` -- Returns the most recent `count` events from the buffer.

6. `pub fn get_by_server(&self, server_id: &str, count: usize) -> Vec<&TraceEvent>` -- Returns recent events filtered by server_id.

7. `pub fn clear(&mut self)` -- Clears the buffer.

8. `pub fn len(&self) -> usize` -- Returns buffer size.

**Use VecDeque<TraceEvent>** for the buffer to get efficient push_back + pop_front for ring-buffer behavior.

**Tests (in trace.rs):**
- `record_outgoing_creates_event` -- record outgoing, buffer len is 1, event has correct direction
- `record_incoming_computes_latency` -- record incoming with timestamp from 100ms ago, latency_ms > 0
- `buffer_caps_at_max` -- set max_buffer=2, record 3 events, buffer len is 2, oldest removed
- `get_by_server_filters` -- record events for 2 servers, filter returns only matching server

Create `src-tauri/src/mcp_host/commands.rs` with Tauri IPC command handlers.

**McpHostState (add to state.rs or define in commands.rs):**
The HostManager needs to be in Tauri managed state. Since HostManager contains async operations and needs mutable access, wrap in `tokio::sync::Mutex` (NOT std::sync::Mutex, which would block the async runtime):
```rust
pub struct McpHostState {
    pub manager: HostManager,
    pub trace: TraceEmitter,
}
```

**IPC Commands (each is a `#[tauri::command]`):**

1. `pub async fn mcp_connect(server_id: String, config: TransportConfig, state: tauri::State<'_, tokio::sync::Mutex<McpHostState>>) -> Result<String, String>` -- Locks state, calls manager.connect_server(). Returns status as serialized JSON string. On error returns the error message.

2. `pub async fn mcp_disconnect(server_id: String, state: tauri::State<'_, tokio::sync::Mutex<McpHostState>>) -> Result<(), String>` -- Locks state, calls manager.disconnect_server().

3. `pub async fn mcp_list_servers(state: tauri::State<'_, tokio::sync::Mutex<McpHostState>>) -> Result<Vec<ServerInfo>, String>` -- Locks state, returns manager.list_servers(). ServerInfo is already Serialize.

4. `pub async fn mcp_call_tool(tool_name: String, params: serde_json::Value, state: tauri::State<'_, tokio::sync::Mutex<McpHostState>>, app: tauri::AppHandle) -> Result<ToolCallResult, String>` -- Locks state. Records outgoing trace event. Calls manager.call_tool(). Records incoming trace event. Emits both trace events via app handle. Returns ToolCallResult.

5. `pub async fn mcp_get_trace(count: Option<usize>, server_id: Option<String>, state: tauri::State<'_, tokio::sync::Mutex<McpHostState>>) -> Result<Vec<TraceEvent>, String>` -- Locks state. If server_id provided, calls trace.get_by_server(). Otherwise calls trace.get_recent(). Returns cloned Vec of TraceEvents. Default count = 50.
