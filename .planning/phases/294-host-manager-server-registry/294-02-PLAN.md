<objective>
Build the tool routing and config persistence layers: a ToolRouter that maps tool names to servers and dispatches calls with timeout handling, and a ServerRegistry that persists server configurations and trust/health state to disk.

Purpose: Without routing, callers would need to know which server owns which tool. Without persistence, every app restart loses all server configs. These are the "brain" (routing) and "memory" (registry) of the host manager.

Output: `router.rs` (tool dispatch), `registry.rs` (config persistence), updated `manager.rs` (integration), updated `mod.rs`.
</objective>

    /// Maps tool name -> server id

**Methods on ToolRouter:**

1. `pub fn new() -> Self` -- empty index.

2. `pub fn rebuild_index(&mut self, connections: &HashMap<String, ServerConnection>)` -- Clears existing index. For each connected server with capabilities, maps each tool.name -> server.id. If duplicate tool names exist across servers, last-write-wins (log a warning to stderr).

3. `pub fn resolve(&self, tool_name: &str) -> Option<&str>` -- Returns the server_id that owns this tool, or None.

4. `pub async fn call_tool(&self, tool_name: &str, params: serde_json::Value, manager: &mut HostManager) -> Result<ToolCallResult, String>` -- Resolves tool name to server_id. Gets mutable connection from manager. Builds JSON-RPC `tools/call` request with params `{"name": tool_name, "arguments": params}`. Calls connection.send_request (which already has 30s timeout). Records latency using std::time::Instant. Returns ToolCallResult with success/error/latency_ms. If tool not found in index, return error "Unknown tool: {name}". If server not found or not connected, return error "Server unavailable: {server_id}".

5. `pub fn tool_count(&self) -> usize` -- Returns total number of indexed tools.

6. `pub fn tools_for_server(&self, server_id: &str) -> Vec<String>` -- Returns all tool names mapped to a specific server.
