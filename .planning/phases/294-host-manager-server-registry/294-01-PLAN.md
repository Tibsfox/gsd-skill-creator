Create `src-tauri/src/mcp_host/connection.rs` implementing the low-level connection to a single MCP server.

**Add to Cargo.toml dependencies:**
- `uuid = { version = "1", features = ["v4"] }` (for generating server IDs and message IDs)

**ConnectionStatus enum:**
```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
#[serde(rename_all = "lowercase")]
pub enum ConnectionStatus {
    Connecting,
    Connected,
    Disconnected,
    Failed,
}
```

**ServerConnection struct:**
```rust
pub struct ServerConnection {
    pub id: String,
    pub config: TransportConfig,
    pub status: ConnectionStatus,
    pub capabilities: Option<ServerCapability>,
    child: Option<tokio::process::Child>,
    stdin: Option<tokio::process::ChildStdin>,
    stdout_lines: Option<tokio::io::Lines<tokio::io::BufReader<tokio::process::ChildStdout>>>,
    restart_count: u32,
    last_restart: Option<std::time::Instant>,
}
```

**Methods on ServerConnection:**

1. `pub fn new(id: String, config: TransportConfig) -> Self` -- creates with Disconnected status.

2. `pub async fn spawn(&mut self) -> Result<(), String>` -- Only handles `TransportConfig::Stdio`. Uses `tokio::process::Command` to spawn the child process with the given command, args, and env. Sets up stdin (piped) and stdout (piped). Wraps stdout in BufReader + lines(). Sets status to Connecting. Then calls `self.handshake().await` to complete initialization. On success sets status to Connected. On failure sets status to Failed.

3. `async fn handshake(&mut self) -> Result<ServerCapability, String>` -- Implements MCP initialize handshake:
   - Build JSON-RPC request: `{"jsonrpc":"2.0","id":"init-1","method":"initialize","params":{"protocolVersion":"2025-03-26","capabilities":{},"clientInfo":{"name":"gsd-os","version":"0.1.0"}}}`
   - Write to stdin as single line + newline (JSON-RPC over stdio is newline-delimited)
   - Read response line from stdout
   - Parse response JSON. Extract `result.capabilities` and `result.serverInfo`
   - Send `initialized` notification: `{"jsonrpc":"2.0","method":"notifications/initialized"}`
   - Call `self.discover_capabilities().await` to populate full capability list
   - Return ServerCapability

4. `async fn discover_capabilities(&mut self) -> Result<(), String>` -- Sends `tools/list`, `resources/list`, `prompts/list` JSON-RPC requests sequentially (each with unique id). Parses responses into Vec<Tool>, Vec<Resource>, Vec<Prompt>. Stores in self.capabilities. If a list method returns error (server doesn't support it), use empty vec -- do not fail.

5. `pub async fn send_request(&mut self, method: &str, params: Option<serde_json::Value>) -> Result<serde_json::Value, String>` -- Generic JSON-RPC request sender. Generates unique id with uuid. Writes request line to stdin. Reads response line from stdout. Parses and returns result or error. Apply a 30-second timeout using `tokio::time::timeout`.

6. `pub async fn disconnect(&mut self) -> Result<(), String>` -- Sends `notifications/cancelled` or just kills the process. Try graceful: write shutdown notification, wait up to 2 seconds with `tokio::time::timeout`. Then kill child process. Set status to Disconnected. Clean up stdin/stdout handles.

7. `pub fn is_alive(&mut self) -> bool` -- Checks if child process is still running via `child.try_wait()`. Returns false if process has exited.

8. `pub fn backoff_duration(&self) -> std::time::Duration` -- Returns exponential backoff: `min(1 << restart_count, 30)` seconds. Capped at 30s.

9. `pub fn should_restart(&self) -> bool` -- Returns true if restart_count < 5 and status is Failed.

10. `pub fn record_restart(&mut self)` -- Increments restart_count, records last_restart timestamp.

**Tests (in connection.rs):**
- `connection_status_serializes_lowercase` -- verifies ConnectionStatus::Connected serializes to "connected"
- `backoff_duration_exponential` -- verifies backoff sequence 1, 2, 4, 8, 16, 30, 30
- `should_restart_within_limit` -- true for restart_count < 5, false at 5

Do NOT try to write integration tests that spawn real processes -- those will be in Phase 303.

**HostManager struct:**
```rust
pub struct HostManager {
    connections: HashMap<String, ServerConnection>,
}
```

**Methods on HostManager:**

1. `pub fn new() -> Self` -- empty connections map.

2. `pub async fn connect_server(&mut self, id: String, config: TransportConfig) -> Result<ConnectionStatus, String>` -- Creates a new ServerConnection, calls spawn(), stores in connections map. Returns the connection status. If id already exists and is Connected, return error "Server already connected".

3. `pub async fn disconnect_server(&mut self, id: &str) -> Result<(), String>` -- Looks up connection by id, calls disconnect(). Removes from map. Returns error if not found.

4. `pub fn get_status(&self, id: &str) -> Option<ConnectionStatus>` -- Returns current status of a server.

5. `pub fn list_servers(&self) -> Vec<ServerInfo>` -- Returns a Vec of ServerInfo structs (id, status, capabilities summary). Define ServerInfo:
```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(rename_all = "camelCase")]
pub struct ServerInfo {
    pub id: String,
    pub status: ConnectionStatus,
    pub server_name: Option<String>,
    pub tool_count: usize,
    pub resource_count: usize,
    pub prompt_count: usize,
}
```

6. `pub async fn check_health(&mut self)` -- Iterates all connections. For each: calls is_alive(). If not alive and was Connected, sets status to Failed. Collects failed server ids.

7. `pub async fn restart_failed(&mut self) -> Vec<String>` -- For each connection with status Failed: check should_restart(). If yes, check backoff_duration against time since last_restart. If enough time has passed, call record_restart(), then re-spawn. Returns list of server ids that were restarted.

8. `pub fn server_count(&self) -> usize` -- Returns number of connections.

9. `pub fn get_connection(&self, id: &str) -> Option<&ServerConnection>` -- Returns immutable reference to a connection.

10. `pub fn get_connection_mut(&mut self, id: &str) -> Option<&mut ServerConnection>` -- Returns mutable reference to a connection.
