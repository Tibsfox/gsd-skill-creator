**05-phase-d-test.md** -- Phase D: System Assembly, Integration, and Test chapter:
- Title: "Chapter 5: System Assembly, Integration, and Test -- Proving It Works"
- NASA SE cross-ref header: "SE Phase: Phase D | SP-6105 SS 5.2-5.3 | NPR 7123.1 SS 5.2 | Review Gate: SIR"
- Section 1: Service Deployment Order -- deploy services following the integration dependency order from the reference document:
  1. Keystone (identity -- everything depends on this)
  2. Glance (images -- Nova needs images to launch instances)
  3. Nova (compute -- needs Keystone auth and Glance images)
  4. Neutron (networking -- needs Keystone auth, Nova needs Neutron)
  5. Cinder (storage -- needs Keystone auth, Nova uses for block storage)
  6. Horizon (dashboard -- needs all of the above)
  7. Heat (orchestration -- needs all of the above)
  8. Swift (object storage -- needs Keystone auth, independent of others)
  Explain why this order matters (per SP-6105 SS 5.2 Product Integration)
- Section 2: Service-by-Service Verification -- for each service, document the individual verification checks:
  - Keystone: `openstack token issue`, service catalog listing, user creation
  - Nova: hypervisor list, flavor creation, keypair creation
  - Neutron: network creation, subnet creation, security group listing
  - Cinder: volume type listing, volume creation
  - Glance: image upload, image listing
  - Horizon: web dashboard accessibility
  - Heat: stack create from template
  - Swift: container creation, object upload
- Section 3: Integration Testing -- end-to-end scenarios that exercise cross-service interfaces:
  - Scenario 1: authenticate, create project, configure network, launch instance, attach storage, access via floating IP
  - Scenario 2: create Heat stack that provisions network + instance + storage
  - Each scenario maps to a requirement for verification traceability
- Section 4: Performance Baseline -- establish baseline metrics: instance launch time, API response times, storage I/O performance, network throughput between instances
- Section 5: Security Audit -- vulnerability assessment, RBAC policy verification, certificate validation (chain, expiry, SANs), network segmentation verification (management vs tenant traffic isolation)
- Section 6: TAID Verification Summary -- table mapping each major verification to its method (Test, Analysis, Inspection, Demonstration per SP-6105 SS 5.3), with specific test procedures and expected results
- Section 7: Phase Gate Criteria -- checklist for SIR (Integration Test Review): all services deployed and individually verified, integration tests pass end-to-end scenarios, performance baseline established, security audit complete with no critical findings
- Include SP-6105 SS 5.2, SS 5.3 and NPR 7123.1 cross-references throughout
