## Key Questions Answered

### Q1: Desktop integration — does v1.21 unify prior infrastructure?
**YES — emphatically.** v1.21 applies P-004 (Build Forward, Refine Backward) at the largest scale yet. Dashboard (v1.12), terminal (v1.15), budget display (v1.19), and design tokens (v1.20) are all unified into the desktop shell. DashboardHost renders existing generator HTML. Palette bridge connects user-style to existing theme system. Terminal emulator wraps the existing terminal concept in a proper PTY.

### Q2: P6 composition ceiling?
The generator pipeline (~25 modules) is unchanged. Desktop introduces its own 10-module composition. Two parallel composition pipelines now exist: the CLI-side generator and the desktop-side shell. They connect via IPC bridge (DashboardHost renders generator output). No ceiling hit; instead, a second composition track emerged.

### Q3: Collector interface unification?
Not addressed in v1.21. Desktop has its own data pipeline (IPC → bridge → host) separate from the collector pattern. Carry forward.

### Q4: Test ratio at unknown scale?
0.99x overall on 106 commits. Desktop TS alone: 1.18x. The Rust backend drags the ratio below 1.0x. This is the first time the overall ratio dips below 1.0x — a structural consequence of introducing a language (Rust) where unit testing requires Tauri runtime.

### Q5: Graceful degradation spread?
**YES.** The CRT engine falls back to CSS-only rendering when WebGL2 is unavailable. Boot sequence has skip detection. Calibration wizard has skip logic. The "graceful degradation" principle from the generator has spread to the desktop.
