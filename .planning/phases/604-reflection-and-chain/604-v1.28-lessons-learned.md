## Key Lessons from v1.28

### L1: Test Ratio Finally Crosses Target (1.15x)
Den's disciplined 1.26x pairing + knowledge's 288+ validator tests push combined ratio above 0.9x for the first time. The key: Den tested every agent module individually AND added integration tests. Future packs should follow Den's testing discipline.

### L2: Rapid Content Generation Strains Quality Control (9 Fix Commits)
Creating 35 knowledge packs in 2 days produced 9 YAML schema fixes. All are trivial (quoting values, expanding schemas) but the volume shows the pack template needs schema pre-validation. A Zod-based YAML linter for .skillmeta before commit would have caught all 9.

### L3: Den Codifies APT Operational Patterns
The Den agent framework (executor, verifier, sentinel, chronicler, planner, monitor, coordinator, dispatcher) maps directly to the APT team roles discovered during chain execution (flight-ops, watchdog, capcom, lab-director). This is the project building its own infrastructure from observed patterns — the strongest example of "cartography, not construction."

### L4: 12-Layer Den Pipeline Approaches Chain Depth Record
Den's ISA-to-chipset pipeline (12 layers) is the second-deepest vertical composition after v1.23's 17-layer AGC. Combined with knowledge's 8 layers and ~57 total modules, this is the broadest AND second-deepest release in chain history.

### L5: Scale Independence Continues to Hold
174 commits / +116K lines scores 4.15 vs chain average 4.236. Prior largest: v1.23 (146 commits) at 4.52, v1.22 (126 commits) at 3.88. Release size does not predict quality — discipline does.
