Create `skills/openstack/kolla-ansible/SKILL.md`. This is the deployment meta-skill that satisfies SKILL-02. Kolla-Ansible manages the entire OpenStack deployment lifecycle: bootstrap, deploy, reconfigure, and upgrade.

**Frontmatter:**
- `name: openstack-kolla-ansible`
- `description:` Mention: deployment engine, containerized OpenStack, bootstrap, deploy, reconfigure, upgrade, globals.yml, inventory, container management, rolling upgrades. This is the skill that manages the deployment lifecycle for all OpenStack services.
- Trigger intents: "kolla-ansible", "kolla", "deploy openstack", "bootstrap", "reconfigure", "upgrade openstack", "globals.yml", "inventory", "container", "docker"
- Trigger contexts: "deploying openstack", "upgrading openstack", "reconfiguring services", "managing openstack containers", "bootstrap environment"

**Body sections:**

1. **Introduction** - Kolla-Ansible's role as the production-grade containerized deployment engine, how it uses Ansible playbooks to manage Docker containers for each OpenStack service, relationship to Kolla (image building), supported operating systems (CentOS Stream 9)

2. **Bootstrap** (SKILL-02 specific) - Complete bootstrap procedure:
   - Prerequisites: Python 3, pip, virtual environment setup
   - Install kolla-ansible: `pip install kolla-ansible` (version pinning)
   - Generate configuration: `kolla-ansible install-deps`, copy globals.yml and passwords.yml
   - Generate passwords: `kolla-genpwd`
   - Configure globals.yml: `kolla_base_distro`, `kolla_install_type`, `openstack_release`, `kolla_internal_vip_address`, `network_interface`, `neutron_external_interface`, `enable_*` flags for each service
   - Configure inventory: all-in-one vs multinode
   - Bootstrap servers: `kolla-ansible -i all-in-one bootstrap-servers`
   - Pre-deployment checks: `kolla-ansible -i all-in-one prechecks`

3. **Deploy** (SKILL-02 specific) - Complete deployment procedure:
   - `kolla-ansible -i all-in-one deploy`
   - Post-deploy: `kolla-ansible -i all-in-one post-deploy`
   - Verify: source admin-openrc.sh, run `openstack service list`, verify each service
   - Init runonce script for demo resources: `./init-runonce` (creates networks, images, flavors)

4. **Reconfigure** (SKILL-02 specific) - How to change configuration after deployment:
   - Modify globals.yml or service-specific config overrides
   - Config override directory structure: `/etc/kolla/config/{service}/`
   - Run: `kolla-ansible -i all-in-one reconfigure`
   - When to reconfigure vs redeploy (service enable/disable requires deploy, config changes use reconfigure)
   - Verify changes took effect

5. **Upgrade** (SKILL-02 specific) - OpenStack version upgrade procedure:
   - Pre-upgrade: backup databases, snapshot VMs, document current state
   - Update kolla-ansible version, update container images
   - `kolla-ansible -i all-in-one pull` (pull new images)
   - `kolla-ansible -i all-in-one upgrade` (rolling upgrade)
   - Post-upgrade verification: all services healthy, API versions correct
   - Rollback procedure if upgrade fails

6. **Troubleshoot** - Common failure modes:
   - Bootstrap fails (Python dependency conflicts, Docker not installed, network interface not found)
   - Prechecks fail (port conflicts, missing packages, firewall rules, SELinux)
   - Deploy fails (image pull failure, container start failure, service dependency timeout)
   - Reconfigure fails (invalid config override, YAML syntax error, service won't restart)
   - Upgrade fails (database migration error, incompatible config, container image version mismatch)
   - Container keeps restarting (check `docker logs {container}`, configuration error, dependency not ready)
   - Password/credential issues (passwords.yml mismatch, `kolla-genpwd` clobbering existing passwords)

7. **Container Management** - Kolla-specific container operations: `docker ps`, `docker logs`, `docker exec`, `docker inspect`, restarting individual services, checking container health, log locations (`/var/log/kolla/`), config locations inside containers

8. **NASA SE Cross-References** - Maps to entire lifecycle: Phase C (bootstrap/deploy = "fabrication"), Phase D (prechecks/post-deploy = "integration & test"), Phase E (reconfigure/upgrade = "operations & sustainment")

**Token budget:** Target 7-8K tokens. This is the most important operational skill -- the deployment engine that manages everything. Use the full budget. The bootstrap/deploy/reconfigure/upgrade sections must be detailed enough that an operator can follow them.
