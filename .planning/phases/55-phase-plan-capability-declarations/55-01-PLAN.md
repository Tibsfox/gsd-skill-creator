  <action>
Create `src/capabilities/roadmap-capabilities.ts` implementing `parseCapabilityDeclarations(phaseLines: string[]): CapabilityRef[]`.

Algorithm (from research):
1. Iterate through lines looking for `**Capabilities**:` (regex: `/^\*\*Capabilities\*\*\s*:/`)
2. On match, check for inline content after the colon (single-line format)
3. If no inline content, enter multi-line mode collecting `- verb: type/name, type/name` lines
4. Multi-line block terminates when encountering: another `**` metadata line, a heading (`#`), or a numbered list (`1.`)
5. Parse each capability line with `parseCapabilityLine(line)`:
   - Extract verb (use|create|after|adapt) before the colon
   - Split remaining by comma
   - For each entry, split by `/` to get type and name
   - Filter: type must be `skill|agent|team`, name must be non-empty
6. Return flat array of CapabilityRef objects

Export only `parseCapabilityDeclarations` as the public API. Keep `parseCapabilityLine` as a private helper.

Use the CapabilityVerb, CapabilityType, CapabilityRef types from `./types.js`.
