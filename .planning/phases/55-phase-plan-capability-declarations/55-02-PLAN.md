  <action>
Create `src/capabilities/manifest-parser.ts` implementing `parseManifest(content: string): CapabilityManifest`.

Implementation:
1. Use `gray-matter` to extract YAML frontmatter (version, generatedAt, contentHash)
2. Parse markdown body by finding section headings (`## Skills`, `## Agents`, `## Teams`)
3. For each section, extract table rows (lines starting with `|` that are not header/separator rows)
4. Split each row by `|`, trim cells, and map to typed objects

**parseTableSection<T> helper** (generic):
- Takes body string, start heading, end heading (or undefined for last section), and a row parser function
- Finds the section between headings
- Filters lines: must start with `|`, must not contain `---` (separator), must not be the header row (detect by checking if first data cell matches known header like "Name")
- Splits cells by `|`, trims, passes to row parser

**Row parsers:**
- `parseSkillRow(cells)`: `{ name, scope, description, contentHash }` from 4 cells
- `parseAgentRow(cells)`: `{ name, scope, description, tools, model, contentHash }` from 6 cells. Convert `-` to undefined for tools/model.
- `parseTeamRow(cells)`: `{ name, scope, description, topology, memberCount, contentHash }` from 6 cells. Convert `-` to undefined for description/topology. Parse memberCount as number.

**Handle markdown escaping:** The renderer escapes `|` as `\|`. The parser should unescape `\|` back to `|` in cell values.

Handle "No skills/agents/teams discovered." gracefully (return empty array for that section).

Export only `parseManifest`.
  </action>
