## Key Lessons from v1.22

### L1: Domain Shift Breaks Score Continuity
The quality rubric (Completeness/Depth/Connections/Honesty) was calibrated on TypeScript engineering. When the project shifts to bash/YAML infrastructure, the rubric scores lower — not because quality dropped, but because the rubric's depth and honesty dimensions weight TypeScript-native qualities (type safety, composition, test ratios). **Recommendation:** Track per-language quality metrics alongside overall score. Acknowledge domain-shift positions in the trend analysis.

### L2: Infrastructure as Educational Content
v1.22 is really a **knowledge map** of infrastructure automation: how to provision VMs, deploy Minecraft, manage mods, emulate Amiga, build educational worlds. The scripts are the territory; the skills/agents/teams are the map. This aligns with the project's core vision: "gsd-skill-creator is cartography, not construction."

### L3: Chipset Assembly as Integration Architecture
The `.chipset/` schema + configuration YAML is the first formal schema for composing skills, agents, and teams into a cohesive system. JSON Schema draft-07 validation, trigger routing matrix, budget tracking, coordination topologies. This is the architectural bridge between the infra domain and the skill-creator domain.

### L4: Three Composition Tracks
The project now has three parallel composition paradigms:
1. **TypeScript CLI:** Generator pipeline (~25 modules, src/dashboard/)
2. **TypeScript Desktop:** Shell pipeline (~10 module groups, desktop/src/)
3. **Bash Infrastructure:** Script pipeline (57 scripts + 8 shared libraries, infra/)
Each has its own quality characteristics, test paradigm, and composition style.

### L5: Documentation as First-Class Deliverable
v1.22's 4 operational runbooks (day-1 deployment, day-2 operations, incident response, server update) are genuine operational engineering. These aren't afterthought docs — they're the primary value for anyone deploying the system. P7 (Documentation) hits its strongest point in the chain.
