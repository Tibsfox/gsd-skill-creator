  <action>
Create `src/cloud-ops/dashboard/types.ts` with shared types:
- `ServiceHealthEntry`: `{ name: OpenStackServiceName; status: ServiceStatus; lastCheck: string; message?: string }` — one per service
- `AlertEntry`: `{ severity: 'critical' | 'warning' | 'info'; source: CommunicationLoopName; message: string; timestamp: string }` — from health/cloud-ops loops
- `CrewStatus`: `{ name: string; profile: 'scout' | 'patrol' | 'squadron'; activeRoles: number; totalRoles: number }` — crew activation
- `BudgetStatus`: `{ used: number; ceiling: number; warning: boolean; blocked: boolean }` — token budget
- `LoopHealth`: `{ name: CommunicationLoopName; operational: boolean; lastMessage?: string }` — loop health
- `MissionTelemetry`: `{ crews: CrewStatus[]; budget: BudgetStatus; loops: LoopHealth[] }` — combined telemetry
- `CloudOpsPanelData`: `{ enabled: boolean | null; services: ServiceHealthEntry[]; alerts: AlertEntry[]; telemetry: MissionTelemetry }` — full panel data
- `DocEntry`: `{ id: string; title: string; type: 'chapter' | 'procedure' | 'runbook'; path: string; sePhaseRef?: string }` — navigation entry
- `DocContent`: `{ entry: DocEntry; body: string; crossRefs: Array<{ label: string; targetId: string }> }` — rendered content
- `DocConsoleData`: `{ enabled: boolean | null; entries: DocEntry[]; activeContent?: DocContent }` — full console data

Import types from `src/types/openstack.ts`. Use `export type` for all type exports. Follow the pattern from silicon-panel.ts types section.

Create `src/cloud-ops/dashboard/cloud-ops-panel.ts` with pure render functions:
- `renderServiceHealth(services: ServiceHealthEntry[]): string` — renders a table/grid of service status with color-coded diamond indicators per silicon-panel pattern. Maps ServiceStatus to colors: active=green, inactive=dim, error=red, maintenance=yellow, unknown=dim.
- `renderAlertSummary(alerts: AlertEntry[]): string` — renders alert list sorted by severity (critical first). Each alert shows severity badge, source loop name, message, and relative timestamp. Empty alerts = "No active alerts" message.
- `renderMissionTelemetry(telemetry: MissionTelemetry): string` — renders crew activation bars (activeRoles/totalRoles as percentage), budget gauge (used/ceiling with warning/blocked colors), and loop health indicators (9 dots, green=operational, red=down).
- `renderCloudOpsPanel(data: CloudOpsPanelData): string` — top-level renderer with progressive enhancement: null=empty string, false=disabled message, true=full panel combining service health + alerts + telemetry sections.

Use CSS class naming convention `cop-*` (cloud ops panel). Follow the exact progressive enhancement pattern from silicon-panel.ts. Pure functions only, no I/O.

Create test file with full coverage:
- Progressive enhancement: null returns empty, false returns disabled, true returns full panel
- Service health: all 5 status types render with correct CSS classes
- Alert summary: sorted by severity, empty state shown
- Mission telemetry: crew bars, budget gauge, loop indicators
- Edge cases: 0 services, 0 alerts, all loops down, budget at 95%+ (blocked state)
  </action>
  <verify>
Run `npx vitest run src/cloud-ops/dashboard/cloud-ops-panel.test.ts` -- all tests pass. Verify that `renderCloudOpsPanel` with null returns empty string, with false returns disabled message, and with sample data returns HTML containing all 8 service names and telemetry sections.
  </verify>
  <done>Cloud ops panel renders service health for all 8 OpenStack services with color-coded status, alert summaries from communication loops, and mission telemetry (crews, budget, loops). Progressive enhancement matches silicon-panel pattern.</done>
