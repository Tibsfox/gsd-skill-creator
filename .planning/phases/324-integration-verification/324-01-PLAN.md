**1. Header and Purpose**
- Title: "End-to-End Deployment Verification Procedure"
- Requirement traceability: VERIF-06
- NASA SE phase mapping: Phase D (System Assembly, Integration and Test, Launch) -- SIR gate
- References to pre-deploy-gates.yaml and post-deploy-gates.yaml

**2. Pre-Conditions**
- Hardware meeting minimum requirements (PRE-001 thresholds: 4 CPU cores, 16GB RAM, 100GB disk, VT-x/AMD-V)
- Network connectivity confirmed (PRE-002: management interface, default route, DNS, internet)
- Resources sufficient (PRE-003: 8GB free RAM, 50GB free disk, no port conflicts, Docker running)
- OS compatible (PRE-004: CentOS Stream 9/Ubuntu 22.04/Rocky 9, Python 3, Ansible, Docker, Git)

**3. Verification Steps (7 stages)**

Stage 1 -- Fresh System Baseline:
- Record system state (hostname, IP, disk, RAM, CPU, kernel)
- Verify no OpenStack services running
- Verify no Kolla containers present
- Record git commit hash for traceability

Stage 2 -- Pre-Deploy Gate Execution:
- Execute each of PRE-001 through PRE-004 from pre-deploy-gates.yaml
- Record pass/fail for each gate with actual values vs expected
- All 4 gates must pass (action: block) before proceeding
- Reference: NASA CDR gate equivalent

Stage 3 -- Deployment Crew Activation:
- Reference deployment-crew.yaml Squadron activation (12 roles)
- FLIGHT issues deployment directive via command loop
- PLAN produces deployment plan referencing kolla-ansible skill
- EXEC agents execute deployment using domain skills
- SCOUT surveys system state before and after each service deployment

Stage 4 -- Kolla-Ansible Deployment:
- Bootstrap: `kolla-ansible bootstrap-servers`
- Pre-checks: `kolla-ansible prechecks`
- Deploy: `kolla-ansible deploy`
- Post-deploy: `kolla-ansible post-deploy`
- Record each command's exit status and duration

Stage 5 -- Post-Deploy Gate Execution:
- Execute POST-001 through POST-009 from post-deploy-gates.yaml in dependency order
- Group 1: POST-001 (Keystone auth) -- must pass first
- Group 2: POST-002 through POST-007 in parallel -- service checks
- Group 3: POST-008, POST-009 -- integration and doc verification
- Record pass/fail/warn/skip for each gate
- All blocking gates (POST-001, 002, 003, 007) must pass

Stage 6 -- Operations Handoff:
- Reference crew-handoff.yaml procedure
- Deployment crew produces handoff package (system state, service status, known issues)
- Operations crew activates (8 roles per operations-crew.yaml)
- SURGEON begins health monitoring
- CAPCOM confirms handoff complete to human operator
- Verify operations crew has full system context from deployment

Stage 7 -- Documentation Verification:
- Run doc-verifier drift detection against running system
- Verify sysadmin guide chapter accuracy (service endpoints, config values)
- Verify ops manual procedures match running service versions
- Verify runbook preconditions match system state
- Verify quick reference card (ports, service names, log locations)
- Record any drift findings

**4. Expected Results Table**
- Table mapping each stage to pass criteria and actual results

**5. Git History Requirements**
- Every deployment decision committed with configuration change rationale
- Verification results committed as structured YAML
- Reference: INTEG-05

**6. Failure Handling**
- Pre-deploy gate failure: remediation steps, re-test procedure
- Deployment failure: rollback procedure (kolla-ansible destroy), root cause analysis
- Post-deploy gate failure: per-service remediation from gate configs
- Documentation drift: doc-sync loop trigger procedure

Do NOT include actual deployment commands to run -- this is a verification PROCEDURE document, not a deployment script. The script in Task 2 handles execution.

**Part A: Create `scripts/e2e-deployment-verification.sh`**

Create an executable bash verification script that automates the 7-stage deployment verification from the document. The script must:

1. Use `set -euo pipefail` for safety
2. Accept a `--dry-run` flag that prints what would be checked without executing destructive operations
3. Source the pre-deploy and post-deploy gate configs (parse YAML gate IDs and commands)
4. Implement each of the 7 verification stages as bash functions:

```bash
stage_1_baseline()       # Record system state, verify no existing OpenStack
stage_2_predeploy()      # Execute PRE-001 through PRE-004, record results
stage_3_crew_activation() # Verify crew configs exist and validate
stage_4_deployment()     # Check kolla-ansible availability, record version
stage_5_postdeploy()     # Execute POST-001 through POST-009 in dependency order
stage_6_handoff()        # Verify crew-handoff.yaml, check ops crew config
stage_7_doc_verify()     # Check doc artifacts exist, verify structure
```

5. Each stage function:
   - Prints a header with stage number and NASA SE phase reference
   - Runs the checks for that stage
   - Records results to the results YAML template
   - Returns 0 on pass, 1 on fail
   - On blocking failure, reports which stage failed and which remediation to follow

6. Output a summary at the end:
   - Total stages: 7
   - Passed / Failed / Skipped counts
   - Overall verdict: PASS or FAIL
   - Path to results file

7. The script MUST handle the case where OpenStack is not deployed (common during development) -- in that case, stages 4-5 should gracefully skip with status "skip" and a note that deployment is required.

8. Record the git commit hash at the start for traceability (INTEG-05 requirement).

9. Make the script executable (chmod +x).
