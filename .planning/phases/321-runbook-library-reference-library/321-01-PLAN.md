The 5 runbooks are:

1. **RB-KEYSTONE-001 -- Token Issuance Failure Troubleshooting**: Covers expired Fernet keys, clock skew between nodes, memcached session issues, database connectivity. Steps: check keystone-manage fernet_setup status, verify NTP sync, check memcached, verify DB connection. Rollback: restore previous Fernet key repository.

2. **RB-KEYSTONE-002 -- Service Catalog Endpoint Repair**: Covers missing endpoints, wrong URLs, incorrect region assignments. Steps: openstack endpoint list, compare against expected, recreate missing endpoints with openstack endpoint create. Rollback: restore from backup catalog dump.

3. **RB-KEYSTONE-003 -- RBAC Policy Troubleshooting**: Covers policy.yaml/policy.json misconfiguration, project scoping errors, role assignment gaps. Steps: check policy files, verify role assignments with openstack role assignment list, test with openstack token issue --os-project-name. Rollback: restore previous policy file from git.

4. **RB-KEYSTONE-004 -- TLS Certificate Renewal**: Covers certificate expiration, chain validation failures, intermediate CA issues. Steps: check cert expiry with openssl x509 -enddate, generate new certs, deploy via kolla-ansible certificates, restart affected services. Rollback: restore previous certificate set.

5. **RB-KEYSTONE-005 -- Fernet Key Rotation**: Covers scheduled Fernet key rotation procedure and emergency rotation after key compromise. Steps: keystone-manage fernet_rotate, verify key distribution across nodes, validate existing tokens still work. Rollback: restore previous key repository from backup.

The 6 runbooks are:

1. **RB-NOVA-001 -- Instance Launch Failure Diagnosis**: Covers the full diagnostic chain when `openstack server create` fails. Steps: check nova-scheduler logs for NoValidHost, verify placement service with `openstack resource provider list`, check Neutron port allocation, verify Glance image availability with `openstack image show`, check Keystone auth. Cross-references RB-KEYSTONE-001, RB-NEUTRON-001, RB-GLANCE-001.

2. **RB-NOVA-002 -- Scheduler and Placement Troubleshooting**: Covers host aggregate mismatches, availability zone problems, resource provider inventory sync. Steps: `openstack aggregate list`, `openstack resource provider inventory list`, check nova-scheduler.log for FilterScheduler output. Rollback: remove host from aggregate, resync placement.

3. **RB-NOVA-003 -- Hypervisor Connectivity Recovery**: Covers libvirt connection failures, nova-compute service down, resource reporting discrepancies. Steps: `openstack hypervisor list`, check libvirtd status, verify virt_type in nova.conf, `systemctl restart nova-compute`. Rollback: failover instances to other hypervisors if available.

4. **RB-NOVA-004 -- Live Migration Procedure**: Covers pre-migration checks, execution, and troubleshooting failed migrations. Steps: verify shared storage or block migration support, check libvirt TCP/TLS connectivity between hosts, `openstack server migrate --live`, monitor migration progress. Rollback: abort migration with `nova live-migration-abort`.

5. **RB-NOVA-005 -- Compute Resource Exhaustion Response**: Covers vCPU/RAM/disk exhaustion on hypervisors. Steps: `openstack hypervisor stats show`, identify overcommit ratios, check resource reservation settings, evaluate instance consolidation. Rollback: adjust resource allocation ratios temporarily.

6. **RB-NOVA-006 -- Compute Service Recovery After Host Failure**: Covers recovering instances after a compute node outage. Steps: mark host down, evacuate instances with `nova host-evacuate`, verify instance state on target hosts, clean up orphaned resources. Rollback: reverse evacuation if original host recovers.

Each runbook must include exact CLI commands, expected output, failure branches, SE Phase Reference (NPR 7123.1 SS 3.2 Process 9), and cross-references to related runbooks (especially Keystone, Neutron, Glance, and Cinder where compute depends on them).
