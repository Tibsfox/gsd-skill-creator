2. **RB-NEUTRON-002 -- DHCP Agent Failure Recovery**: Covers DHCP agent down, port binding failures, namespace issues. Steps: `openstack network agent list --agent-type dhcp`, check dnsmasq process, verify DHCP namespace exists, restart neutron-dhcp-agent, verify IP assignment. Rollback: reassign DHCP agent to network.

3. **RB-NEUTRON-003 -- Floating IP Troubleshooting**: Covers allocation exhaustion, association state mismatch, external network issues. Steps: `openstack floating ip list`, verify external network with `openstack network show`, check router gateway, verify NAT rules in router namespace. Rollback: disassociate and reassociate floating IP.

4. **RB-NEUTRON-004 -- Security Group Rule Debugging**: Covers rule conflicts, default deny behavior, conntrack issues. Steps: `openstack security group rule list`, verify iptables/nftables rules on compute host, check conntrack table, test with `openstack security group rule create`. Rollback: revert to previous security group rules.

5. **RB-NEUTRON-005 -- OVS/OVN Bridge Recovery**: Covers bridge status issues, flow table corruption, controller connectivity loss. Steps: `ovs-vsctl show`, `ovs-ofctl dump-flows br-int`, check OVN controller with `ovn-sbctl show`, restart openvswitch service. Rollback: rebuild OVS bridges from Neutron state.

6. **RB-NEUTRON-006 -- Tenant Network Isolation Verification**: Covers VXLAN/VLAN segmentation validation, tunnel endpoint verification. Steps: verify segmentation IDs with `openstack network show --column provider:segmentation_id`, check tunnel endpoints, validate isolation between tenant networks. Create 5 Cinder runbooks and 4 Glance runbooks following the exact format from skills/methodology/runbook-generator/SKILL.md.

**Cinder runbooks (5):**

1. **RB-CINDER-001 -- Volume Creation Failure Diagnosis**: Covers backend capacity exhaustion, driver errors, scheduling failures. Steps: `openstack volume service list`, `cinder-manage service list`, check LVM VG space with `vgs`, check Cinder scheduler logs. Rollback: clean up stuck volumes with `openstack volume set --state error`.

2. **RB-CINDER-002 -- Volume Attachment Troubleshooting**: Covers iSCSI target issues, multipath problems, Nova coordination failures. Steps: check volume attachment state, verify iSCSI targets with `iscsiadm -m session`, check Nova instance block device mapping, verify connector info. Cross-references RB-NOVA-001. Rollback: force-detach with `openstack volume set --detached`.

3. **RB-CINDER-003 -- LVM Backend Recovery**: Covers LVM thin pool exhaustion, metadata overflow, VG corruption. Steps: check `lvs`, `vgs`, thin pool usage with `lvs -a`, extend thin pool, verify Cinder volume mappings. Rollback: activate LV from backup metadata.

4. **RB-CINDER-004 -- Snapshot Management Troubleshooting**: Covers snapshot creation failures, orphaned snapshots, consistency issues. Steps: `openstack volume snapshot list`, check backend snapshot status, clean up orphaned snapshots. Rollback: delete failed snapshot, retry with quiesce.

5. **RB-CINDER-005 -- Volume Migration Between Backends**: Covers moving volumes between storage backends. Steps: pre-migration checks (volume detached, no snapshots), `openstack volume migrate`, monitor migration progress, verify data integrity. Rollback: abort migration, restore original volume location.

**Glance runbooks (4):**

1. **RB-GLANCE-001 -- Image Upload Failure Troubleshooting**: Covers format validation errors, backend storage full, hash mismatch. Steps: check `openstack image create` error, verify backend storage space, check Glance API logs, verify image format with `qemu-img info`. Rollback: delete partial upload, retry.

2. **RB-GLANCE-002 -- Image Format Conversion**: Covers converting between raw, qcow2, vmdk. Steps: `qemu-img convert`, verify conversion with `qemu-img info`, upload converted image, verify image properties. Rollback: retain original image, delete converted copy.

3. **RB-GLANCE-003 -- Image Metadata and Visibility Management**: Covers property inheritance issues, visibility settings (public/private/shared/community), image member management. Steps: `openstack image set`, `openstack image add project`. Rollback: restore previous metadata values.

4. **RB-GLANCE-004 -- Glance Backend Storage Recovery**: Covers backend filesystem issues, corrupted image files, storage quota exhaustion. Steps: verify backend with `openstack image list`, cross-check filesystem entries, repair orphaned files, rebuild cache. Rollback: restore images from backup.

All runbooks must include exact CLI commands, expected output, failure branches, SE Phase Reference, and cross-references.
