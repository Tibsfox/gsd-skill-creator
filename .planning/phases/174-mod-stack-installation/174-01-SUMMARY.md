## Decisions Made
- Modrinth API v2 used for download URL resolution with User-Agent header per API etiquette
- SHA-256 from local-values treated as authoritative over API-provided hash for reproducibility
- Exit code convention: 0=up-to-date, 1=error, 2=updates available (enables scripted cron checks)
- check-mod-updates.sh requires jq (not optional like deploy-mods.sh) since its core function is API parsing
- Mod manifest written at SERVER_DIR/mod-manifest.yaml (not mods/) to keep manifest separate from jar files
