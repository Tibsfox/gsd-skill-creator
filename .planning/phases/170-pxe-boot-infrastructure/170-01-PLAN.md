<objective>
Create dnsmasq PXE/TFTP/DHCP configuration templates and a CentOS Stream 9 boot media download/verification script, plus a TFTP root directory setup script.

Purpose: PXE boot requires three infrastructure pieces: (1) a DHCP server that tells booting clients where to find the bootloader, (2) a TFTP server that delivers the bootloader and kernel, and (3) the actual boot media files. This plan creates the configuration templates and acquisition scripts. The template/local-values pattern ensures zero secrets or site-specific data in version control.

Output:
- `infra/templates/dnsmasq/pxe-boot.conf.template` -- dnsmasq configuration adding DHCP/TFTP/PXE to existing DNS
- `infra/templates/dnsmasq/pxe-boot.local-values.example` -- example local values showing all template variables
- `infra/scripts/download-centos-boot-media.sh` -- downloads and verifies CentOS Stream 9 vmlinuz + initrd.img
- `infra/scripts/setup-tftp-root.sh` -- creates TFTP directory structure for BIOS and UEFI boot
</objective>

  <action>
    Create the dnsmasq PXE configuration using the template/local-values pattern. The template contains all configuration with `${VARIABLE}` placeholders; the local-values file provides site-specific values at deployment time.

    **IMPORTANT DESIGN CONSTRAINT:** This configuration EXTENDS an existing dnsmasq instance that already serves DNS on infra-01. The PXE config must NOT include any `server=`, `address=`, or `listen-address` directives that would conflict with the existing DNS setup. It adds ONLY DHCP, TFTP, and PXE-specific directives.

    Create `infra/templates/dnsmasq/pxe-boot.conf.template`:

    ```conf
    # PXE Boot Configuration for dnsmasq
    # Template: replace ${VAR} with values from local-values
    # Include this file via conf-dir or --conf-file alongside existing DNS config
    #
    # DOES NOT configure DNS — that is handled by the existing dnsmasq config.
    # This file adds DHCP + TFTP + PXE boot services only.

    # --- DHCP Configuration ---
    # Enable DHCP only on the provisioning interface/range
    # Scoped to PXE clients only — will NOT hand out leases to existing VMs
    dhcp-range=${DHCP_RANGE_START},${DHCP_RANGE_END},${DHCP_NETMASK},${DHCP_LEASE_TIME}

    # Gateway and DNS for PXE clients
    dhcp-option=option:router,${GATEWAY_IP}
    dhcp-option=option:dns-server,${DNS_SERVER_IP}

    # --- TFTP Configuration ---
    enable-tftp
    tftp-root=${TFTP_ROOT}

    # --- PXE Boot Configuration ---
    # Tag BIOS and UEFI clients differently for correct bootloader selection
    # Architecture detection via DHCP option 93 (client system architecture)
    dhcp-match=set:efi-x86_64,option:client-arch,7
    dhcp-match=set:efi-x86_64,option:client-arch,9
    dhcp-match=set:bios,option:client-arch,0

    # BIOS clients get pxelinux.0
    dhcp-boot=tag:bios,pxelinux.0

    # UEFI clients get the GRUB EFI bootloader
    dhcp-boot=tag:efi-x86_64,grubx64.efi

    # PXE prompt — brief pause before auto-booting
    pxe-prompt="GSD PXE Boot", ${PXE_PROMPT_TIMEOUT}

    # PXE service advertisement
    pxe-service=x86PC,"GSD Network Boot (BIOS)",pxelinux
    pxe-service=x86-64_EFI,"GSD Network Boot (UEFI)",grubx64.efi

    # --- DHCP Logging ---
    # Log DHCP transactions for debugging PXE issues
    log-dhcp

    # --- Safety ---
    # Only act as DHCP server on the provisioning network
    # Bind to the specific interface to prevent rogue DHCP on other networks
    interface=${PXE_INTERFACE}
    bind-interfaces
    ```

    Create `infra/templates/dnsmasq/pxe-boot.local-values.example`:

    ```yaml
    # PXE Boot Local Values
    # Copy to infra/local/pxe-boot.local-values and fill in site-specific values.
    # This file is an EXAMPLE — never commit actual values.

    # DHCP range for PXE provisioning (must not overlap existing static IPs)
    # Keep this range small — only needs enough IPs for VMs being provisioned
    dhcp_range_start: "192.168.122.200"
    dhcp_range_end: "192.168.122.250"
    dhcp_netmask: "255.255.255.0"
    dhcp_lease_time: "1h"

    # Network settings for PXE-booted clients
    gateway_ip: "192.168.122.1"
    dns_server_ip: "192.168.122.10"

    # TFTP root directory (where boot files are served from)
    tftp_root: "/var/lib/tftpboot"

    # Network interface for PXE/DHCP services
    pxe_interface: "virbr0"

    # PXE prompt timeout in seconds (0 = immediate boot)
    pxe_prompt_timeout: "5"

    # CentOS Stream 9 mirror (for boot media download)
    centos_mirror: "https://mirror.stream.centos.org/9-stream/BaseOS/x86_64/os"

    # Kickstart server URL (where kickstart files will be served via HTTP)
    kickstart_server_url: "http://192.168.122.10:8080/kickstart"
    ```

    **Key design decisions:**
    - `bind-interfaces` + specific `interface=` prevents rogue DHCP on unintended networks
    - DHCP range is deliberately small (200-250) — only for VMs being provisioned, not permanent leases
    - BIOS/UEFI detection uses DHCP option 93 (client-arch), the standard PXE approach
    - `dhcp-match` with `set:` tags enables conditional bootloader selection
    - No `server=` or `address=` directives — DNS is handled by existing config
    - Configuration is meant to be included via `conf-dir` or `--conf-file`

  <action>
    Create two scripts: one that downloads and verifies CentOS Stream 9 boot media, and one that sets up the TFTP root directory structure.

    **Create `infra/scripts/setup-tftp-root.sh`** (executable, bash):

    Sets up the standard TFTP directory layout for both BIOS and UEFI PXE boot:

    ```
    ${TFTP_ROOT}/
      pxelinux.0              # BIOS bootloader (from syslinux)
      ldlinux.c32             # Required by pxelinux
      libutil.c32             # Menu support
      menu.c32                # Simple menu
      vesamenu.c32            # Graphical menu (BIOS)
      pxelinux.cfg/
        default               # BIOS boot menu (created by Plan 02)
      grubx64.efi             # UEFI bootloader (from grub2-efi)
      grub.cfg                # UEFI boot menu (created by Plan 02)
      centos-stream-9/
        vmlinuz               # CentOS kernel
        initrd.img            # CentOS initramfs
      gsd/
        splash.png            # GSD boot splash (created by Plan 02)
    ```

    The script should:
    - Accept `--tftp-root` argument (defaults to `/var/lib/tftpboot`)
    - Create all directories with correct permissions (755, owned by root or tftp-serving user)
    - Copy BIOS bootloader files from syslinux package:
      - `pxelinux.0` from `/usr/share/syslinux/` or `/usr/lib/syslinux/modules/bios/`
      - `ldlinux.c32`, `libutil.c32`, `menu.c32`, `vesamenu.c32` from syslinux modules
    - Copy UEFI bootloader from grub2-efi package:
      - `grubx64.efi` from `/boot/efi/EFI/centos/` or `/usr/lib/grub/x86_64-efi/`
      - Or use `grub2-mknetdir` if available to generate the EFI network bootloader
    - Use `command -v` to check for required packages; print clear instructions if missing
    - Check if running as root (TFTP root setup typically needs root)
    - Idempotent: running twice produces same result without errors
    - Use `set -euo pipefail` for safety

    **Create `infra/scripts/download-centos-boot-media.sh`** (executable, bash):

    Downloads CentOS Stream 9 boot kernel and initramfs for PXE serving:

    1. Accept arguments:
       - `--mirror` (CentOS mirror URL, defaults to `https://mirror.stream.centos.org/9-stream/BaseOS/x86_64/os`)
       - `--tftp-root` (where to place files, defaults to `/var/lib/tftpboot`)
       - `--verify-only` (just check existing files, don't download)

    2. Download these files:
       - `${MIRROR}/images/pxeboot/vmlinuz` -> `${TFTP_ROOT}/centos-stream-9/vmlinuz`
       - `${MIRROR}/images/pxeboot/initrd.img` -> `${TFTP_ROOT}/centos-stream-9/initrd.img`

    3. Verification:
       - Download `${MIRROR}/images/pxeboot/CHECKSUM` (or compute SHA256 if no checksum file)
       - Verify downloaded files against checksums: `sha256sum -c`
       - If verification fails, print error and exit 1
       - If `--verify-only` and files don't exist, report what's missing

    4. Safety:
       - Use `set -euo pipefail`
       - Use `curl -fSL --retry 3` for downloads (fail on HTTP errors, follow redirects, retry)
       - Check available disk space before downloading (~100MB needed)
       - Create a `.download-verified` marker file after successful verification
       - Skip download if files exist and `.download-verified` marker is recent (within 7 days)
       - Print clear progress messages during download

    5. Output:
       - Print summary after completion: file paths, sizes, checksums, verification status
       - Exit 0 on success, 1 on verification failure, 2 on network/download error

    **Script design principles (both scripts):**
    - `set -euo pipefail` header
    - All paths parameterized via arguments (no hardcoded paths)
    - Graceful error messages pointing to what's wrong and how to fix it
    - Idempotent (safe to run multiple times)
    - No root required for download; root required for TFTP root setup (check and warn)
  </action>
