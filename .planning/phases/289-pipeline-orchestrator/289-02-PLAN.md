<feature>
  <name>Pipeline orchestrator (runPipeline)</name>
  <files>src/vtm/pipeline.ts, src/vtm/__tests__/pipeline.test.ts, src/vtm/index.ts</files>
  <behavior>
    **Function signature:**
    ```
    runPipeline(input: string | VisionDocument, config?: PipelineConfig): PipelineResult
    ```

    **Input handling:**
    - If `input` is a string: treated as raw vision document markdown, runs vision stage
    - If `input` is a VisionDocument object: skips parsing, runs from validation onward
    - When `config.speed` is 'mission-only' and input is VisionDocument: skips both vision parsing and research

    **Stage 1: Vision (stage = 'vision')**
    1. If input is string: call `parseVisionDocument(input)` -- if ParseResult.success is false, return PipelineError with stage='vision', recoverable=false
    2. If input is VisionDocument: use directly
    3. Unless `skipValidation`: call `validateVisionDocument(visionDoc)` -- collect diagnostics (don't fail on warnings)
    4. Unless `skipQualityCheck`: call `checkQuality(visionDoc)` -- collect diagnostics
    5. Call `classifyArchetype(visionDoc)` -- store for result
    6. Call `extractDependencies(input)` if input was string (use visionDoc.dependsOn otherwise)
    7. Produce VisionStageResult

    **Speed selection (between stage 1 and 2):**
    Call `selectPipelineSpeed(visionDoc, config)` to determine speed. If speed is 'mission-only', skip to stage 3. If speed is 'skip-research', skip stage 2.

    **Stage 2: Research (stage = 'research', skipped when speed != 'full')**
    1. Call `compileResearch(visionDoc)` -- produces ResearchReference
    2. Call `checkSourceQuality(research)` -- collect diagnostics
    3. Call `chunkKnowledge(research)` -- produce KnowledgeTiers
    4. Call `extractSafety(research)` -- produce SafetySection
    5. Produce ResearchStageResult
    6. On error: return PipelineError with stage='research', partialOutput includes vision, recoverable=true

    **Stage 3: Mission (stage = 'mission')**
    1. Call `assembleMissionPackage(visionDoc, research?)` -- produces MissionPackage
    2. Call `validateSelfContainment(missionPackage.componentSpecs)` -- collect diagnostics
    3. If `includeCache` (default true): call `generateCacheReport(missionPackage.waveExecutionPlan, missionPackage.componentSpecs)` -- produce CacheReport
    4. Call `validateBudget(budgetTasks)` where budgetTasks are derived from componentSpecs (map each to { id: spec.name, model: spec.modelAssignment, estimatedTokens: spec.estimatedTokens })
    5. Produce MissionStageResult
    6. On error: return PipelineError with stage='mission', partialOutput includes vision + research, recoverable=true

    **Result assembly (PIPE-04):**
    Build PipelineResult with:
    - `success: true`
    - `speed`: the speed used
    - `stages`: collected stage results
    - `fileManifest`: derive from MissionPackage -- list each component spec, milestone spec, wave plan, test plan, readme as entries with { name, type, size } where type is 'milestone-spec'|'component-spec'|'wave-plan'|'test-plan'|'readme' and size is human-readable estimated token count
    - `executionSummary`: from missionPackage.executionSummary, map to the PipelineResult shape (modelSplit = opus/sonnet/haiku from executionSummary)
    - `durationMs`: elapsed time from start to end (use Date.now() delta)

    **Error wrapping:**
    Wrap each stage in try/catch. On catch, produce PipelineError with:
    - `stage`: the current stage name
    - `error`: the caught error
    - `partialOutput`: all successfully completed stage results
    - `recoverable`: true if at least vision stage completed (vision data can be reused)

    Return PipelineResult with success=false and error field populated.

    **Barrel export:**
    Add `export * from './pipeline.js';` to src/vtm/index.ts

    **Test cases:**
    - Full pipeline with valid vision markdown produces success=true with all three stages populated
    - Full pipeline fileManifest includes milestone-spec, component-spec(s), wave-plan, test-plan, readme entries
    - Full pipeline executionSummary has modelSplit with opus/sonnet/haiku percentages
    - Skip-research pipeline skips research stage, stages.research is undefined
    - Mission-only pipeline with VisionDocument input skips parsing and research
    - Invalid markdown input returns success=false, error.stage='vision', error.recoverable=false
    - Pipeline with includeCache=false omits cacheReport from mission stage
    - Pipeline speed auto-detection delegates to selectPipelineSpeed
    - durationMs is a positive number
    - Error in mission stage returns partialOutput with vision (and research if full speed)
  </behavior>
