<feature>
  <name>Pipeline types and speed selector</name>
  <files>src/vtm/pipeline.ts, src/vtm/__tests__/pipeline.test.ts</files>
  <behavior>
    Define the following types and function:

    **Types (all exported):**

    1. `PipelineSpeed` -- re-export from research-utils.ts (already defined as 'full' | 'skip-research' | 'mission-only')

    2. `PipelineStage` -- literal union: 'vision' | 'research' | 'mission'

    3. `PipelineConfig` -- configuration for a pipeline run:
       - `speed?: PipelineSpeed` -- manual override (if omitted, auto-detected via speed selector)
       - `skipValidation?: boolean` -- skip vision document validation (default false)
       - `skipQualityCheck?: boolean` -- skip quality checker (default false)
       - `includeCache?: boolean` -- include cache optimization report in result (default true)

    4. `VisionStageResult` -- typed intermediate from vision stage:
       - `visionDoc: VisionDocument` -- parsed vision document
       - `diagnostics: VisionDiagnostic[]` -- validation diagnostics
       - `archetype: Archetype` -- classified archetype
       - `dependencies: string[]` -- extracted dependencies

    5. `ResearchStageResult` -- typed intermediate from research stage:
       - `research: ResearchReference` -- compiled research
       - `knowledgeTiers: KnowledgeTiers` -- chunked tiers
       - `safety: SafetySection` -- extracted safety
       - `sourceDiagnostics: SourceDiagnostic[]` -- source quality issues

    6. `MissionStageResult` -- typed intermediate from mission stage:
       - `missionPackage: MissionPackage` -- assembled mission
       - `selfContainmentDiagnostics: SelfContainmentDiagnostic[]` -- containment validation
       - `cacheReport?: CacheReport` -- optional cache optimization report
       - `budgetValidation: BudgetValidationResult` -- model budget validation

    7. `PipelineError` -- structured error for stage failures:
       - `stage: PipelineStage` -- which stage failed
       - `error: Error | string` -- the underlying error
       - `partialOutput: { vision?: VisionStageResult; research?: ResearchStageResult }` -- what was produced before failure
       - `recoverable: boolean` -- whether partial output is usable (true if at least vision stage completed)

    8. `PipelineResult` -- the complete pipeline output:
       - `success: boolean`
       - `speed: PipelineSpeed` -- speed used (auto-detected or overridden)
       - `stages: { vision: VisionStageResult; research?: ResearchStageResult; mission?: MissionStageResult }`
       - `error?: PipelineError` -- present when success is false
       - `fileManifest: Array<{ name: string; type: string; size: string }>` -- list of output files
       - `executionSummary: { totalTasks: number; parallelTracks: number; sequentialDepth: number; modelSplit: { opus: { count: number; percentage: number }; sonnet: { count: number; percentage: number }; haiku: { count: number; percentage: number } }; estimatedWallTime: string; totalTests: number; safetyCriticalTests: number }` -- from MissionPackage.executionSummary
       - `durationMs: number` -- total pipeline execution time

    **Function:**

    `selectPipelineSpeed(visionDoc: VisionDocument, config: PipelineConfig): PipelineSpeed`
    - If `config.speed` is set, return it directly (manual override)
    - Otherwise, call `detectResearchNecessity(visionDoc)` and return the recommended speed
    - This is a pure function with no side effects

    **Test cases:**
    - selectPipelineSpeed with explicit override returns override value -> 'mission-only'
    - selectPipelineSpeed without override delegates to detectResearchNecessity -> result matches recommendation
    - selectPipelineSpeed with safety-domain vision doc returns 'full' (via detectResearchNecessity)
    - selectPipelineSpeed with infrastructure vision doc returns 'skip-research'
    - PipelineError with vision-only partial output has recoverable: true verified structurally
    - PipelineResult type validates file manifest array structure
  </behavior>
