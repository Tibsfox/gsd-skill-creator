<objective>
Create the adaptive configuration generator that reads all discovery and assessment data to produce a complete local-values YAML for every downstream consumer in the project.

Purpose: This is the culmination of the hardware adaptation engine -- the single script a user runs after hardware discovery to get a complete, ready-to-use configuration file. It bridges the gap between "what hardware do I have" (Phases 169/178) and "what configuration should I use" (Phases 171-177, 182-183). The output drives kickstart templates, JVM flags, server.properties, mod configuration, UAE settings, and network/firewall setup.

Output: Adaptive config generator script, test suite, and documented example output.
</objective>

Create `infra/scripts/generate-local-values.sh` that reads hardware-capabilities.yaml + resource-budget.yaml + GPU/audio assessments and produces a complete `local-values.yaml` for all downstream consumers.

**Usage:** `generate-local-values.sh [--capabilities <path>] [--budget <path>] [--output <path>]`
- Default capabilities: `infra/inventory/hardware-capabilities.yaml` (or local variant)
- Default budget: `infra/local/resource-budget.yaml`
- Default output: `infra/local/local-values.yaml`
- If budget file doesn't exist, run `calculate-budget.sh` automatically to generate it
- If capabilities file doesn't exist, error with helpful message pointing to `discover-all.sh`

Source `infra/scripts/lib/discovery-common.sh` for YAML emission. Use `set -euo pipefail`.

**Process:**
1. Read hardware-capabilities.yaml (all sections via section-aware awk parsing)
2. Read resource-budget.yaml (host_reserved, vm_available, minecraft_vm, tier)
3. Run assess-gpu.sh on capabilities file, capture output
4. Run assess-audio.sh on capabilities file, capture output
5. Derive tier-adaptive configuration values
6. Emit complete local-values YAML

**Output YAML sections (all required by downstream phases):**


**Tier-adaptive values (the core adaptation logic):**

| Setting | Minimal (16GB) | Comfortable (32GB) | Generous (64GB+) |
|---------|---------------|-------------------|------------------|
| JVM heap_min_mb | 1024 | 2048 | 4096 |
| JVM heap_max_mb | `minecraft_vm.ram_gb * 1024 - 512` (leave OS overhead) | same formula | same formula |
| GC type | G1GC | G1GC | ZGC (if >= 8GB heap) |
| GC flags (G1GC) | `-XX:+UseG1GC -XX:MaxGCPauseMillis=50 -XX:G1HeapRegionSize=16M` | same + `-XX:G1NewSizePercent=30 -XX:G1MaxNewSizePercent=40` | same as comfortable |
| GC flags (ZGC) | n/a | n/a | `-XX:+UseZGC -XX:+ZGenerational` |
| Extra JVM flags | `-XX:+ParallelRefProcEnabled -XX:+DisableExplicitGC` | same + `-XX:+AlwaysPreTouch` | same as comfortable |
| view_distance | 10 | 16 | 24 |
| simulation_distance | 6 | 10 | 12 |
| max_players | 5 | 10 | 20 |
| entity_broadcast_range | 50 | 100 | 150 |
| syncmatica max size | 100000 | 500000 | 1000000 |

**Hypervisor preference logic:** KVM if available (open standard preference per PROJECT.md), then VMware, then VirtualBox. If none but podman/docker available, set preferred=container.

**Network interface:** Parse the network section from capabilities, find the first interface with type=ethernet and state=up. Fall back to "eth0" if none found.
