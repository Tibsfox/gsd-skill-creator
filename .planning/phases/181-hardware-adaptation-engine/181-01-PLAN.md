Create `infra/scripts/assess-gpu.sh` that reads a hardware-capabilities YAML (or hardware-values YAML) and produces a gpu_assessment YAML section to stdout. Source `infra/scripts/lib/discovery-common.sh` for YAML emission helpers.

**Input:** Path to capabilities YAML (first argument). Uses section-aware awk parsing (established pattern from calculate-budget.sh) to extract from the `gpu:` and `hypervisor:` sections: present, vendor, family, vram_gb, driver, iommu_available, and from hypervisor: kvm, nested_virtualization.

**Output YAML structure (to stdout):**
```yaml
gpu_assessment:
  compute_capable: true|false      # Can run CUDA/OpenCL workloads
  compute_level: none|basic|standard|advanced
  rendering_capable: true|false     # Can do hardware-accelerated rendering
  rendering_backend: none|mesa|nvidia|amdgpu
  passthrough_viable: true|false    # IOMMU + discrete GPU + KVM = can passthrough to VM
  passthrough_method: none|vfio     # Only vfio for now
  vram_tier: none|low|medium|high  # none=0, low=1-4GB, medium=4-8GB, high=8GB+
  driver_status: none|opensource|proprietary
  uae_display: software|opengl|vulkan  # Recommended UAE display backend
```

**Classification logic:**
- `compute_capable`: true if vendor is nvidia with proprietary driver, OR vendor is amd with amdgpu driver
- `compute_level`: none (no GPU), basic (integrated Intel), standard (AMD discrete), advanced (NVIDIA with CUDA)
- `rendering_capable`: true if any GPU present with a working driver
- `rendering_backend`: mesa for Intel/AMD open-source, nvidia for proprietary NVIDIA, amdgpu for AMD, none if no GPU
- `passthrough_viable`: true only if iommu_available=true AND gpu present AND vendor != intel AND kvm=true
- `vram_tier`: based on vram_gb value (handle null/empty as 0)
- `driver_status`: opensource for mesa/amdgpu/i915/nouveau, proprietary for nvidia, none if no driver
- `uae_display`: vulkan if rendering_capable and modern driver, opengl if rendering_capable, software if no GPU

Use `set -euo pipefail`. Exit 0 always (assessment of "no GPU" is valid). Print diagnostic info to stderr.

Create `infra/scripts/assess-audio.sh` that reads a hardware-capabilities YAML and produces an audio_assessment YAML section to stdout. Source `infra/scripts/lib/discovery-common.sh` for YAML emission helpers.

**Input:** Path to capabilities YAML (first argument). Uses section-aware awk parsing to extract from the `audio:` section: present, server, alsa card_count and cards, midi present and port_count, and capabilities (has_audio_output, has_audio_input, has_midi).

**Output YAML structure (to stdout):**
```yaml
audio_assessment:
  usable: true|false                # Has at least one audio output device
  server: none|alsa|pulseaudio|pipewire
  server_tier: none|basic|standard|advanced
  routing_method: none|direct_alsa|pulse|pipewire
  midi_available: true|false
  midi_port_count: 0
  uae_audio_backend: none|alsa|sdl|pulse|pipewire
  uae_midi_backend: none|alsa
  recommended_sample_rate: 44100|48000
  recommended_buffer_size: 256|512|1024
```

**Classification logic:**
- `usable`: true if audio present=true and has_audio_output=true
- `server_tier`: none (no audio), basic (ALSA only, no server), standard (PulseAudio), advanced (PipeWire)
- `routing_method`: pipewire if server=pipewire, pulse if server=pulseaudio, direct_alsa if server=none/alsa but cards present, none if no audio
- `uae_audio_backend`: pipewire if available (best latency), pulse if PulseAudio, alsa if ALSA only, none if no audio. UAE (FS-UAE) supports SDL audio which auto-detects, so sdl is the safe fallback when server is present.
- `uae_midi_backend`: alsa if midi present=true (ALSA MIDI is universal on Linux), none otherwise
- `recommended_sample_rate`: 48000 if PipeWire (native rate), 44100 if PulseAudio/ALSA (Amiga-compatible rate)
- `recommended_buffer_size`: 256 if PipeWire (low latency capable), 512 if PulseAudio, 1024 if ALSA only (conservative)

Use `set -euo pipefail`. Exit 0 always. Print diagnostics to stderr.
