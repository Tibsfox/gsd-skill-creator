  <action>
Create `infra/scripts/lib/fw-abstraction.sh` -- a sourced library that provides firewall management operations dispatched to the correct backend.

**Detection strategy (in priority order):**
1. If `FW_BACKEND` env var is set, use it directly (allows override)
2. If `infra/inventory/hardware-capabilities.yaml` exists, parse `firewall:` from the distro section
3. If neither, run `discover-distro.sh` inline and capture the firewall field
4. If all fail, attempt runtime detection: check for active firewalld, then ufw, then iptables

**Public functions:**

`fw_detect_backend()` -- Detect and set `_FW_BACKEND` variable. Print the detected backend to stderr. If no firewall tool is found, set to "none" and warn (not an error -- some systems have no firewall). Return 0 always.

`fw_open_port <port> <protocol>` -- Open a port (protocol is "tcp" or "udp", defaults to "tcp"):
- firewalld: `sudo firewall-cmd --permanent --add-port=<port>/<proto> && sudo firewall-cmd --reload`
- ufw: `sudo ufw allow <port>/<proto>`
- iptables: `sudo iptables -A INPUT -p <proto> --dport <port> -j ACCEPT` followed by persistence attempt via `iptables-save` if available
- none: warn that no firewall is active, port is already accessible, return 0
Must be idempotent: if port is already open, succeed silently without error.

`fw_close_port <port> <protocol>` -- Close a port:
- firewalld: `sudo firewall-cmd --permanent --remove-port=<port>/<proto> && sudo firewall-cmd --reload`
- ufw: `sudo ufw delete allow <port>/<proto>`
- iptables: `sudo iptables -D INPUT -p <proto> --dport <port> -j ACCEPT` (ignore error if rule doesn't exist)
- none: warn, return 0
Must be idempotent: closing an already-closed port succeeds silently.

`fw_open_service <service_name>` -- Open a well-known service by name (convenience wrapper):
- firewalld: `sudo firewall-cmd --permanent --add-service=<name> && sudo firewall-cmd --reload`
- ufw: `sudo ufw allow <name>`
- iptables: resolve service to port via /etc/services and call fw_open_port
- none: warn, return 0

`fw_list_ports()` -- List currently open ports:
- firewalld: `sudo firewall-cmd --list-ports --list-services`
- ufw: `sudo ufw status numbered`
- iptables: `sudo iptables -L INPUT -n --line-numbers | grep ACCEPT`
- none: print "no firewall active"
Output goes to stdout for parsing by callers.

`fw_status()` -- Print firewall backend and active/inactive status:
- firewalld: `systemctl is-active firewalld`
- ufw: `ufw status | head -1`
- iptables: `iptables -L | head -1` (presence of rules)
- none: "no firewall detected"

**Idempotency implementation:**
- Before opening a port, check if it's already open using fw_list_ports output or tool-specific check
- For firewalld: `firewall-cmd --query-port=<port>/<proto>` returns 0 if already open
- For ufw: grep the port from `ufw status`
- For iptables: grep the port from `iptables -L INPUT -n`

**Error handling:**
- Unknown backend: print clear error listing supported backends (firewalld, ufw, iptables, none), return 1
- Missing sudo: if commands fail with permission denied, suggest running with sudo
- Invalid port number: validate port is 1-65535 before dispatching
- Invalid protocol: validate protocol is "tcp" or "udp"

**Conventions:**
- Source discovery-common.sh for has_command, warn, safe_read
- Use SCRIPT_DIR detection pattern consistent with other lib files
- `set -euo pipefail`
- No shebang (sourced library)
- Header comment block documenting purpose, usage example, and supported backends
