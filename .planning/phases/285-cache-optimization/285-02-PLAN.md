- Validates at EVERY wave boundary -- cumulative timing: if Wave 0 takes 3min and Wave 1 takes 3min, Wave 2's caches from Wave 0 are at 6min -> violation

- affectedConsumers lists task IDs in the consumer wave that depend on the stale producer wave
- mitigation suggests splitting the slow wave or moving non-critical tasks to a later wave

Where TokenSavingsReport is:
```ts
interface TokenSavingsReport {
  skillCaching: { entries: number; tokensSaved: number };
  schemaReuse: { entries: number; tokensSaved: number };
  knowledgeTierOptimization: { entries: number; tokensSaved: number };
  totalTokensSaved: number;
}
```

Tests:
- Returns zero savings for all categories when inputs are empty arrays
- Skill caching savings: for each SharedLoadEntry where cacheable=true, savings = estimatedTokens * (tasks.length - 1) (cache once, reuse N-1 times)
- Schema reuse savings: for each SchemaReuseEntry, savings = encode(schema name).length * consumerTasks.length (each consumer avoids re-parsing the schema)
- Knowledge tier savings: sum of all KnowledgeTierEntry.savings values
- Reports per-category token counts as absolute numbers (not percentages, per CONTEXT.md decision)
- totalTokensSaved = sum of all three categories
