  <action>
Create `infra/scripts/lib/vm-common.sh` -- a sourced library (no shebang, like discovery-common.sh) providing:

1. **Backend auto-detection function** `vm_detect_backend()`:
   - Reads hypervisor section from hardware-capabilities.yaml (path: `infra/inventory/hardware-capabilities.yaml`, also check `../inventory/hardware-capabilities.yaml` relative to script)
   - Priority order: KVM first (open standard, per PROJECT.md "libvirt/KVM as primary hypervisor"), then VMware
   - For KVM: verify `virsh` command available AND `hypervisor.kvm: true` in capabilities
   - For VMware: verify `vmrun` command available AND `hypervisor.vmware: true` in capabilities
   - Sets `VM_BACKEND` variable to "kvm" or "vmware"
   - Returns exit 1 with descriptive error if no usable backend found
   - Allow `VM_BACKEND` environment variable override (explicit backend selection)

2. **Parameter validation function** `vm_validate_params()`:
   - Validates required parameters: VM_NAME, VM_RAM_MB, VM_VCPUS, VM_DISK_GB
   - VM_NAME must be alphanumeric + hyphens only (safe for both virsh and vmrun)
   - VM_RAM_MB must be positive integer >= 512
   - VM_VCPUS must be positive integer >= 1
   - VM_DISK_GB must be positive integer >= 10
   - Returns exit 1 with clear error listing all validation failures (not just first)

3. **VM state query function** `vm_exists()`:
   - Takes VM_NAME, delegates to backend-specific check
   - Returns 0 if VM exists, 1 if not (for idempotent operations)

4. **VM state function** `vm_is_running()`:
   - Takes VM_NAME, delegates to backend-specific check
   - Returns 0 if running, 1 if stopped/not-found

5. **Logging helpers** matching the project pattern:
   - `vm_info`, `vm_ok`, `vm_warn`, `vm_error`, `vm_die` -- same colored format as deploy-pxe.sh
   - `vm_step` for numbered progress steps

6. **Local-values loader** `vm_load_values()`:
   - Reads a VM provisioning local-values file using same YAML parsing as calculate-budget.sh
   - Populates VM_NAME, VM_RAM_MB, VM_VCPUS, VM_DISK_GB, VM_ISO_PATH, VM_NETWORK, VM_STORAGE_DIR from values file
   - Supports reading from resource-budget.yaml for minecraft_vm allocation values as defaults

Follow established patterns:
- `set -euo pipefail` at top
- All functions use `local` variables
- No side effects when sourced
- Comments explain what each function does

Create `infra/templates/vm/vm-provisioning.local-values.example` documenting all VM provisioning variables:
- vm_name, vm_ram_mb, vm_vcpus, vm_disk_gb
- vm_iso_path (CentOS ISO or PXE boot)
- vm_network (bridge name, e.g., virbr0 for KVM, vmnet8 for VMware)
- vm_storage_dir (where VM disks live)
- vm_os_variant (e.g., centos-stream9)
- kickstart_url (for PXE/kickstart provisioning)
- Comments explaining each variable and typical values per hypervisor

Create `infra/scripts/vm-backend-kvm.sh` implementing VM operations via libvirt/virsh:

1. Source `lib/vm-common.sh` for shared functions
2. Implement these functions (each idempotent -- safe to call repeatedly):

   **`kvm_create()`** -- Create a new KVM VM
   - Use `virt-install` with: --name, --ram, --vcpus, --disk (path, size, format=qcow2), --os-variant
   - Network: --network bridge=${VM_NETWORK:-virbr0}
   - For PXE boot: --pxe --boot network,hd
   - For ISO: --cdrom ${VM_ISO_PATH} --boot cdrom,hd
   - If kickstart URL provided: --extra-args "inst.ks=${KICKSTART_URL}"
   - Idempotent: if `virsh dominfo ${VM_NAME}` succeeds, log "VM already exists" and return 0
   - Create disk in ${VM_STORAGE_DIR:-/var/lib/libvirt/images}

   **`kvm_start()`** -- Start VM
   - `virsh start ${VM_NAME}`
   - Idempotent: if already running (`virsh domstate` returns "running"), return 0

   **`kvm_stop()`** -- Graceful shutdown with force fallback
   - `virsh shutdown ${VM_NAME}`, wait up to 60 seconds
   - If still running after timeout, `virsh destroy ${VM_NAME}` (force stop)
   - Idempotent: if already shut off, return 0

   **`kvm_snapshot()`** -- Create named snapshot
   - `virsh snapshot-create-as ${VM_NAME} --name ${SNAPSHOT_NAME} --description "..."`
   - Idempotent: if snapshot name exists, log and return 0

   **`kvm_clone()`** -- Clone from existing VM or snapshot
   - `virt-clone --original ${SOURCE_VM} --name ${VM_NAME} --auto-clone`
   - Source VM must be shut off (stop it if running)
   - Idempotent: if target VM exists, log and return 0

   **`kvm_destroy()`** -- Remove VM completely
   - Stop if running, `virsh undefine ${VM_NAME} --remove-all-storage --snapshots-metadata`
   - Idempotent: if VM does not exist, return 0

   **`kvm_vm_exists()`** -- Check if VM exists (return 0/1)
   **`kvm_vm_is_running()`** -- Check if VM is running (return 0/1)

3. Main dispatch: parse subcommand (create|start|stop|snapshot|clone|destroy) + args, call corresponding function
4. `--dry-run` support: log what would happen without executing

Create `infra/scripts/vm-backend-vmware.sh` implementing same operations via vmrun:

1. Source `lib/vm-common.sh` for shared functions
2. Same function signatures but VMware-specific:

   **`vmware_create()`** -- Create VMware VM
   - Generate .vmx file from parameters (template inline or heredoc)
   - Create virtual disk: `vmware-vdiskmanager -c -s ${VM_DISK_GB}GB -a lsilogic -t 1 disk.vmdk`
   - Set RAM, vCPUs, network adapter (nat or custom), guest OS type
   - For PXE boot: set bios.bootOrder = "ethernet0,hdd"
   - For ISO: set ide0:0.fileName to ISO path
   - Idempotent: if .vmx already exists, log and return 0

   **`vmware_start()`** -- `vmrun start ${VMX_PATH} nogui`
   - Idempotent: check `vmrun list` first

   **`vmware_stop()`** -- `vmrun stop ${VMX_PATH} soft`, fallback to `hard`
   - Idempotent: if not in `vmrun list`, return 0

   **`vmware_snapshot()`** -- `vmrun snapshot ${VMX_PATH} ${SNAPSHOT_NAME}`
   - Idempotent: check `vmrun listSnapshots` first

   **`vmware_clone()`** -- `vmrun clone ${SOURCE_VMX} ${DEST_VMX} full ${SNAPSHOT_NAME}`
   - Idempotent: if dest .vmx exists, log and return 0

   **`vmware_destroy()`** -- Stop + `vmrun deleteVM ${VMX_PATH}`
   - Idempotent: if .vmx does not exist, return 0

   **`vmware_vm_exists()`** / **`vmware_vm_is_running()`**

3. VMware-specific: VM_STORAGE_DIR defaults to ~/vmware/
4. .vmx generation: use heredoc with GSD header comment, parameterized values
5. Same `--dry-run` and subcommand dispatch pattern as KVM backend

Both scripts follow established patterns:
- `#!/usr/bin/env bash` + `set -euo pipefail`
- Colored logging via vm-common.sh helpers
- Usage/help functions
- Clear error messages on missing prerequisites (e.g., "virsh not found -- install libvirt-daemon-system")
