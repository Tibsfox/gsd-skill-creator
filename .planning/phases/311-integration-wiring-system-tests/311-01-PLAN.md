  <action>
Create `src/brainstorm/integration/session-bus.ts` with a `SessionBus` class that implements the 4-communication-loop architecture from the vision doc:

**Loop architecture:**
- `session/` — Facilitator ↔ all technique agents (phase transitions, technique activation/deactivation)
- `capture/` — All agents → Scribe (ideas, questions, clusters, evaluations)
- `user/` — CAPCOM ↔ human user (problem statement, preferences, feedback)
- `energy/` — Energy signals → Facilitator (fatigue signals, momentum indicators)

**SessionBusConfig interface:**
```typescript
export interface SessionBusConfig {
  brainstormDir: string;
  sessionId: string;
}
```

**SessionBus class — key methods:**

`constructor(config: SessionBusConfig)` — stores config, computes base path `{brainstormDir}/sessions/{sessionId}/bus/`

`async publish(loop: 'session' | 'capture' | 'user' | 'energy', message: BrainstormMessage): Promise<string>` — the core write method:
  - Calls `brainstormMessageFilename(message.from, message.to)` from `../shared/schemas.js` for collision-resistant filename
  - Validates message with `BrainstormMessageSchema.parse(message)` before writing
  - Writes to `{baseDir}/{loop}/{filename}` using `fs.writeFile` (JSON.stringify)
  - Returns the full file path written

`async poll(loop: 'session' | 'capture' | 'user' | 'energy', since?: number): Promise<BrainstormMessage[]>` — reads pending messages:
  - `fs.readdir` on the loop directory, sort by filename (monotonic counter ensures correct order)
  - Filter by `since` timestamp if provided (parse timestamp from filename prefix)
  - Read each `.msg` file, parse with `BrainstormMessageSchema.parse(JSON.parse(content))`
  - Return array of valid messages, silently skip corrupted files

`async drain(loop: 'session' | 'capture' | 'user' | 'energy'): Promise<BrainstormMessage[]>` — reads and deletes all messages in loop (for processed message cleanup)

`routeMessage(message: BrainstormMessage): 'session' | 'capture' | 'user' | 'energy'` — pure routing logic (synchronous):
  - `type === 'idea' | 'question' | 'cluster' | 'evaluation' | 'artifact_ready'` → `capture`
  - `type === 'energy_signal'` → `energy`
  - `type === 'user_input'` → `user`
  - `type === 'phase_transition' | 'technique_switch' | 'timer_event' | 'hat_color' | 'rule_violation' | 'agent_activated' | 'agent_deactivated'` → `session`

`async publishRouted(message: BrainstormMessage): Promise<string>` — calls `routeMessage()` then `publish()`, convenience wrapper

**Important constraints from STATE.md:**
- Use `brainstormMessageFilename()` from `../shared/schemas.js` — NOT timestamp-only filenames (confirmed collision-prone in Den bus)
- All filesystem functions take config as first-param style — no `process.cwd()` in method bodies
- Validate with Zod schemas on BOTH write (before writing) and read (after parsing JSON)
- `src/brainstorm/` has zero imports from `den/`, `vtm/`, `knowledge/`

Only imports allowed: `node:fs/promises`, `node:path`, `../shared/types.js`, `../shared/schemas.js`
