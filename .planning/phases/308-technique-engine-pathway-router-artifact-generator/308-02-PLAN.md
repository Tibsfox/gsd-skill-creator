**Brainwriting 6-3-5** (`collaborative/brainwriting-635.ts`) — HIGHEST CARE:
This is the most complex collaborative technique. The key mechanic is progressive building — each round BUILDS on the previous round.

Config parameters: `{ rounds: 6, ideas_per_round: 3, round_duration_ms: 300_000, simulated_participants: 4, generation_context: 'You are participant ${participant_number} of 4. Round ${round}. Previous participants wrote: [${previous_round_summaries}]. Generate 3 new ideas that EXPLICITLY BUILD ON what previous participants wrote. Each idea must reference or extend a previous idea (set parent_id to the idea you are building on). Ideas should be progressively more developed than Round 1.' }`

Internal state:
- `rounds: Map<number, Idea[]>` — round_number → ideas from that round
- `current_round: number` (starts at 1)
- `total_ideas: number`

`generateRound(round_number, previous_output?)`:
- Round 1: Generate `ideas_per_round * simulated_participants` ideas (12 total) with NO parent_id (first round, nothing to build on)
- Rounds 2-6: For each idea in this round, set `parent_id` to an idea from the PREVIOUS round (round_number - 1). Distribute: participant 1 builds on idea[0], participant 2 builds on idea[1], etc., cycling if needed
- Store round result in `rounds` map
- `metadata.round = round_number`

`isComplete()`: Returns true when `current_round >= rounds` (6 completed)

**Round Robin** (`collaborative/round-robin.ts`):
Config: `{ max_rounds: 10, ideas_per_turn: 1, round_timeout_ms: 120_000, simulated_agents: 4, generation_context: 'Round robin turn. Each participant contributes exactly 1 idea per turn. No agent dominates. Natural rhythm builds momentum. Your turn: add one idea building on the session so far.' }`
- `generateRound()`: Returns `simulated_agents` ideas (one per simulated turn), each with `source_agent: 'ideator'` and `tags: ['round-robin-turn-${round_number}']`
- `isComplete()`: Returns true when `current_round >= max_rounds`

**Brain-Netting** (`collaborative/brain-netting.ts`):
Config: `{ duration_ms: 1_200_000, prompt_interval_ms: 300_000, generation_context: 'Asynchronous contribution to a shared idea pool. Ideas can be revisited and built upon over longer timeframes. Add to the pool whenever inspiration strikes — no pressure to generate continuously.' }`
- `generateRound()`: Returns Idea[] with timestamps showing asynchronous spread. Uses `Date.now()` for timestamps. Ideas have no specific parent_id constraint.
- `isComplete()`: Returns true when `elapsed_ms >= duration_ms`

**Rolestorming** (`collaborative/rolestorming.ts`):

- `generateRound()`: Returns Idea[] with `perspective: personas[current_persona_index]` populated on EVERY idea. Advance `current_persona_index` after `duration_per_persona_ms` elapsed.
- `isComplete()`: Returns true when all personas have been covered (`personas_completed.length >= personas.length`)

Create the Artifact Generator — pure render functions that transform session data into polished documents.

**Design principle:** All rendering functions are PURE — they take SessionState (or subsets) and return strings. No filesystem writes. No side effects. The Scribe agent calls these and handles writing.

**`src/brainstorm/artifacts/templates/transcript.ts`**:
Export `renderTranscript(session: SessionState): string`

Produces this exact Markdown structure:
```markdown
# Brainstorm Session: {problem_statement}
**Date:** {ISO date} | **Duration:** {elapsed_ms formatted as Xh Xm} | **Pathway:** {active_pathway ?? 'Free-Form'}
**Techniques Used:** {techniques_used.join(', ')}

---

## Phase: Explore
*Entered: {timestamp}*

## Phase: Diverge
*Entered: {timestamp}*

### Technique: {technique_name} ({duration_label})
- {idea.content}
- {idea.content}
...

### Technique: {technique_name} — {scamper_lens | hat_color | round label}
- {idea.content} *(built on: {parent_idea.content if parent_id})*
...

## Phase: Organize
### Clusters
**{cluster.label}:** {idea_count} ideas
- {idea.content}
...

## Phase: Converge
### Top Ideas (by composite score)
| Rank | Idea | Feasibility | Impact | Alignment | Risk | Score |
|------|------|-------------|--------|-----------|------|-------|
...

## Phase: Act
### Action Plan
| Priority | Action | Source Idea | Owner | Deadline | Status |
...

---
## Session Statistics
- Total ideas: {n} | Questions: {n} | Clusters: {n}
- Techniques used: {n} | Duration: {formatted}
```

Use `session.metadata.phase_history` to determine which phases occurred and their timestamps. Use `session.ideas` grouped by `phase` and `source_technique`. For SCAMPER ideas, include `scamper_lens`. For Six Thinking Hats ideas, include `hat_color`. Format `elapsed_ms` as: `const formatDuration = (ms: number) => { const m = Math.floor(ms / 60000); const h = Math.floor(m / 60); return h > 0 ? \`${h}h ${m % 60}m\` : \`${m}m\`; }`

**`src/brainstorm/artifacts/templates/action-plan.ts`**:
Export `renderActionPlan(items: ActionItem[], ideas: Idea[]): string`

Produces standalone Markdown action plan. Each item MUST include the original source idea text (looked up from `ideas` array by `source_idea_ids[0]`):
```markdown
# Action Plan
*Generated: {date}*

## High Priority
| # | Action | Source Idea | Owner | Deadline | Status |
|---|--------|-------------|-------|----------|--------|
| 1 | {description} | "{source_idea_text}" | {owner ?? 'Unassigned'} | {deadline ?? 'TBD'} | {status} |

## Medium Priority
...

## Low Priority
...
```

Critical: `source_idea_ids` MUST be traced back to actual idea content. If an action item has no source_idea_ids, use `"No source idea linked"`. This prevents the "action plan omits source ideas" UX pitfall from PITFALLS.md.

**`src/brainstorm/artifacts/templates/json-export.ts`**:
Export `renderJsonExport(session: SessionState): string`

Returns `JSON.stringify(session, null, 2)` — complete SessionState as formatted JSON. The SessionState Zod schema handles all field serialization. Validate with `SessionStateSchema.parse(session)` before serializing to catch any runtime corruption.
