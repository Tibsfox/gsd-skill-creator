Define and export these interfaces:
```typescript
interface ITechniqueEngine {
  loadTechnique(id: TechniqueId): TechniqueInstance;
  getConfig(id: TechniqueId): TechniqueConfig;
  listByCategory(category: TechniqueCategory): TechniqueConfig[];
  listBySituation(situation: string): TechniqueConfig[];
}

interface TechniqueInstance {
  id: TechniqueId;
  config: TechniqueConfig;
  initialize(problem: string, session: SessionState): void;
  generateRound(round_number: number, previous_output?: Idea[] | Question[]): TechniqueOutput;
  getState(): Record<string, unknown>;
  isComplete(): boolean;
}

type TechniqueOutput = {
  ideas?: Idea[];
  questions?: Question[];
  prompts?: string[];
  visualization?: VisualizationData;
  metadata: { round: number; technique: TechniqueId; duration_ms: number };
};

// VisualizationData for future GSD-OS rendering (stored, not rendered in v1.32)
type VisualizationData = {
  type: 'tree' | 'grid' | 'sequence' | 'clusters';
  nodes: Array<{ id: string; label: string; parent_id?: string; position?: { x: number; y: number } }>;
};
```

Implement `TechniqueEngine` class:
- Internal registry: `Map<TechniqueId, () => TechniqueInstance>` — lazy factory, not eager instantiation
- `loadTechnique(id)`: Looks up factory in registry, calls it, returns instance. Throws `Error('Unknown technique: ${id}')` if not found
- `getConfig(id)`: Returns the technique's static TechniqueConfig (call loadTechnique, return config property)
- `listByCategory(category)`: Filters registry entries by category, returns TechniqueConfig[] sorted by name
- `listBySituation(situation)`: Searches situation string against each technique's description and name using simple keyword matching (no NLP needed — this is lightweight hint matching, not the Pathway Router's signal analysis)

The registry is populated by calling `register(id, factory)` in the constructor after importing all technique modules. Individual technique modules export a factory function `createFreewritingTechnique(): TechniqueInstance`, etc.

Import from `../shared/types.js` — use `.js` extension (ESM). Import `TECHNIQUE_DEFAULTS` from `../shared/constants.js`.

Do NOT import from `src/den/`, `src/vtm/`, `src/knowledge/`, or any other project module.

Create all 4 individual technique modules. Each exports a class implementing `TechniqueInstance` and a factory function.

**Technique-specific generation context (REQUIRED — prevents fidelity erosion per Pitfall 5):**
Each technique stores a `generation_context` string in its config.parameters that encodes the unique mechanic for that technique. This context is what distinguishes freewriting output from rapid ideation output.

**Freewriting** (`individual/freewriting.ts`):
- Config: `{ id: 'freewriting', category: 'individual', default_duration_ms: 600_000, parameters: { min_ideas_per_minute: 3, generation_context: 'Continuous stream of consciousness — no editing, no censoring, no stopping. Each idea flows associatively from the previous. Ideas can be rough, incomplete, tangential. DO NOT filter for quality.' } }`
- `isComplete()`: Returns true when `(elapsed_ms / 60_000) * min_ideas_per_minute` ideas have been generated (stream target met), OR `elapsed_ms >= default_duration_ms`
- `generateRound()`: Returns Idea[] with high diversity. Ideas are SHORT (1 sentence max). Each has `source_technique: 'freewriting'`, `tags: []`, `timestamp: Date.now()`, UUID id from `crypto.randomUUID()`. `parent_id` is set to previous idea's id to form an associative chain (simulates stream-of-consciousness).

**Mind Mapping** (`individual/mind-mapping.ts`):
- Config: `{ id: 'mind-mapping', category: 'individual', default_duration_ms: 900_000, parameters: { max_branch_depth: 3, branches_per_node: [3, 5], generation_context: 'Radial expansion from central topic. Each idea is a branch or sub-branch. Ideas at any depth. Connections between branches are themselves ideas. Non-linear — jump to wherever energy is highest.' } }`
- Internal state tracks `root_node_id` (set on initialize from problem string), `nodes: Map<string, {id, content, parent_id, depth}>`, `current_depth: number`
- `generateRound(round_number, previous_output?)`: Round 1 generates 3-5 top-level branches (parent_id = root). Subsequent rounds generate sub-branches off existing nodes at next depth level. Returns Idea[] with `parent_id` set correctly. Also sets `visualization: { type: 'tree', nodes: [...all_nodes] }`
- `isComplete()`: Returns true when `elapsed_ms >= default_duration_ms` OR `max_branch_depth` fully populated

**Rapid Ideation** (`individual/rapid-ideation.ts`):
- Config: `{ id: 'rapid-ideation', category: 'individual', default_duration_ms: 60_000, parameters: { target_count: 10, generation_context: 'SPEED OVER QUALITY. Maximum 10 words per idea. No elaboration. No explanation. Fire them out as fast as possible. Quantity is the ONLY metric.' } }`
- Internal state: `idea_count: number`
- `generateRound()`: Returns burst of Idea[] (5-10 ideas per call). Ideas MUST be short — enforce a 10-word max on `content` field (truncate at word boundary if necessary during generation).
- `isComplete()`: Returns true when `idea_count >= target_count`

**Question Brainstorming** (`individual/question-brainstorming.ts`):
- Config: `{ id: 'question-brainstorming', category: 'individual', default_duration_ms: 600_000, parameters: { min_questions: 15, generation_context: 'Generate QUESTIONS ONLY. Never answers. Build comprehensive understanding of the problem space. If an answer emerges, reframe it as a deeper question. Cover all W-categories: who, what, where, when, why, how.' } }`
- `generateRound()`: Returns Question[] (NOT Idea[]). Each question has `category` (who/what/where/when/why/how), `source_technique: 'question-brainstorming'`, `timestamp: Date.now()`, UUID id
- `isComplete()`: Returns true when `question_count >= min_questions`

**Common pattern for all techniques:**
- `initialize(problem, session)`: Store problem and session reference, reset internal counters, record `start_time = Date.now()`
- `getState()`: Return `{ idea_count | question_count, elapsed_ms, round_number }` (technique-specific fields)
- All IDs via `crypto.randomUUID()` (from `node:crypto`)
- Import types from `../../shared/types.js`

Do NOT use random() for idea content generation — generate deterministic placeholder content based on problem statement + technique context. The content fields use the generation_context to guide what the implementing LLM/agent will actually produce — these are configuration signals, not runtime generators.
