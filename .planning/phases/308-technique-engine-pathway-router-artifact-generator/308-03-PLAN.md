**Five Whys** (`analytical/five-whys.ts`):
Config:
```typescript
{
  id: 'five-whys',
  category: 'analytical',
  default_duration_ms: 600_000,
  parameters: {
    max_depth: 5,
    chains: 3,  // number of parallel causal chains
    generation_context: 'You are at depth ${depth} of a Five Whys causal chain. The previous Why was: "${previous_why_content}". Ask WHY that is true. Each question must CAUSALLY reference the previous question — it should be answerable as "because..." of the previous why. This is root cause analysis, not brainstorming. Depth 1 = symptoms. Depth 5 = root causes.',
  }
}
```

Internal state:
- `chains: Map<number, Question[]>` — chain_number → questions in that chain (chain is indexed 0, 1, 2)
- `current_chain: number` (starts at 0, cycles through `chains` count)
- `completed_chains: number` (chains that have reached depth 5)

`initialize(problem, session)`:
- Create initial "Why 0" seed question for each chain: `{ id: uuid, content: problem_statement, category: 'why', source_technique: 'five-whys', depth: 0, timestamp: Date.now() }`
- Store all chain seeds in `this.chains`

`generateRound(round_number)`:
- For the current active chain, find the deepest question in that chain
- Generate a new question that CAUSALLY follows from the previous: `depth = previous.depth + 1`, `parent_id = previous.id`
- Category: `'root-cause'` for depth 5, `'why'` for depths 1-4
- Advance chain when `max_depth` reached for current chain
- Returns Question[] (NOT Idea[])

`isComplete()`: Returns true when `completed_chains >= chains` (all chains have reached depth 5)

Both: Import from `../../shared/types.js` and `../../shared/constants.js`. Use `crypto.randomUUID()`. No imports from other project modules.
