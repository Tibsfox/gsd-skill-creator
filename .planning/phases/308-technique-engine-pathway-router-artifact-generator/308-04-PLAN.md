**Affinity Mapping** (`visual/affinity-mapping.ts`):
This technique is UNIQUE — it organizes existing ideas into clusters, not generating new ones.

Config:
```typescript
{
  id: 'affinity-mapping',
  category: 'visual',
  default_duration_ms: 900_000,
  parameters: {
    min_clusters: 2,
    max_clusters: 8,
    generation_context: 'Sort the provided ideas into natural groupings. Find themes. Label each cluster with a descriptive name. All ideas deserve placement — do not discard any. Generate between 2 and 8 clusters.',
  }
}
```

CRITICAL BEHAVIOR: This technique organizes EXISTING ideas from `previous_output`. It does NOT generate new ideas.

Internal state:
- `clusters: Cluster[]` — the resulting clusters
- `is_complete_flag: boolean`

`generateRound(round_number, previous_output?: Idea[] | Question[])`:
- `previous_output` is REQUIRED for this technique. If undefined or empty, return empty TechniqueOutput with a prompt: `['No ideas to cluster — run a generative technique first']`
- Takes the Idea[] from previous_output and groups them into clusters
- Cluster count: clamp between `min_clusters` and `max_clusters` (never < 2, never > 8)
- **Clustering algorithm** (using natural.TfIdf from the existing stack):
  - Import `natural` from `'natural'`
  - Create TfIdf instance, add each idea's content as a document
  - Group ideas with high TF-IDF similarity into the same cluster
  - Label each cluster using the most common significant terms
  - If TfIdf not available or throws, fall back to: split ideas evenly into `Math.min(max_clusters, Math.ceil(ideas.length / 5))` clusters
- Return `TechniqueOutput` with `visualization: { type: 'clusters', nodes: clusters.flatMap(c => c.idea_ids.map(id => ({ id, label: ideas_map.get(id)?.content ?? id, parent_id: c.id }))) }`
- Note: TechniqueOutput type has `ideas?: Idea[]` — for affinity mapping, return empty `ideas: []` and populate the `visualization` field. The Mapper agent will read clusters from session state, not from ideas.
- Store clusters in internal state; set `is_complete_flag = true` after first successful run (one pass is sufficient)

`isComplete()`: Returns `is_complete_flag` — true after first clustering pass completes

Both: Import from `../../shared/types.js`. Use `crypto.randomUUID()` for cluster IDs. No imports from den/, vtm/, knowledge/.

  <files>src/brainstorm/techniques/visual/lotus-blossom.ts</files>
  <action>
Create the Lotus Blossom technique module — forced elaboration to 64 ideas (8 themes × 8 ideas each).

Config:
```typescript
{
  id: 'lotus-blossom',
  category: 'visual',
  default_duration_ms: 900_000,
  parameters: {
    central_concept: '',  // set from problem statement in initialize()
    themes_count: 8,
    ideas_per_theme: 8,
    generation_context: 'Lotus Blossom grid expansion. Central concept: "${central_concept}". Theme: "${current_theme}". Generate idea ${idea_number_in_theme} of 8 for this theme. This is FORCED ELABORATION — you must generate all 8 ideas per theme, pushing past the obvious first responses into creative territory. Each idea explores a different angle of the theme.',
  }
}
```

Internal state:
```typescript
{
  central_concept: string;           // from problem statement
  themes: Map<number, {              // 8 themes (index 0-7)
    id: string;                      // UUID
    label: string;                   // theme name (first idea of the theme?)
    ideas: Idea[];                   // up to 8 ideas per theme
  }>;
  current_theme_index: number;       // 0-7
  total_ideas: number;
}
```

`initialize(problem, session)`:
- Store `central_concept = problem`
- Create 8 empty theme entries in `themes` map
- Set `current_theme_index = 0`

Lotus Blossom structure:
- The central node is the problem statement (not an idea itself)
- 8 theme nodes radiate from center — each theme is like a mini-freewriting topic
- 8 ideas radiate from each theme node — specific ideas for that theme
- Total: 64 ideas

`generateRound(round_number)`:
- Determine current theme: `themes.get(current_theme_index)`
- If current theme has < 8 ideas, generate 1-2 more ideas for it
  - Each new idea has `parent_id = theme.id` (links idea to its theme node)
  - `tags: ['lotus-blossom', `theme-${current_theme_index}`, `idea-${theme.ideas.length + 1}-of-8`]`
- If current theme has 8 ideas, advance `current_theme_index`
- If all 8 themes are full, isComplete returns true

First idea of each theme determines the theme's label (set `theme.label = first_idea.content.substring(0, 30)`)

Theme IDs are UUIDs — generated in `initialize()`. They serve as `parent_id` for all ideas in that theme.

`isComplete()`: Returns true ONLY when `total_ideas >= themes_count * ideas_per_theme` (i.e., 64 ideas). This is strict — partial completion is not complete.

Note from PITFALLS.md Performance Traps: "Lotus Blossom forced 64-idea expansion — stop expansion when consecutive ideas are semantic duplicates." Add a `min_useful_threshold` check: if the last 3 ideas from the same theme have content that starts with the same word OR is shorter than 3 words, add a prompt `'Explore a completely different angle for this theme'` to the TechniqueOutput.prompts field instead of forcing more duplicate ideas.

Import from `../../shared/types.js`. Use `crypto.randomUUID()`. No imports from den/, vtm/, knowledge/.
