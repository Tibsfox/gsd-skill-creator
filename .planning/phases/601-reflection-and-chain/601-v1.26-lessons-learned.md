## Key Lessons from v1.26

### L1: Recovery from Maintenance Trough (+0.960, Largest Delta in Chain)

The maintenance trough (3.70, 3.32) was structural — docs-only and stabilization releases lack evidence breadth for the 8-dimension scoring model. v1.26's 94 commits restore evidence in all 8 dimensions, producing the largest single-position recovery in the chain. The rolling average returns above 4.0 (4.074). **Key insight:** maintenance troughs are expected and healthy — they represent consolidation periods between feature development. The score model correctly identifies them as lower-evidence periods, not quality regressions.

### L2: Complete Standalone Vertical Slice as Extension Pack Template

v1.26 demonstrates the template for future extension packs: own module directory (`src/aminet/`), own type system (1,182 lines, 53 Zod schemas), own chipset YAML, 6 skills, 5 agents, 1 pipeline team, desktop panel, dashboard widget, and integration test suite. This is self-contained — no coupling to internal modules beyond the barrel export and standard integration points. Future packs (retro systems, educational domains, tool integrations) should follow this template.

### L3: Security as Genuine Multi-Layer Engineering

The virus scanning implementation is not checkbox security. Three independent detection strategies (signature pattern matching with 50+ patterns, structural heuristic analysis of hunk files and boot blocks, emulated execution via FS-UAE), a formal quarantine system with atomic moves and metadata sidecars, a scan gate enforcing 3 named security requirements (INS-07/08/09), path traversal prevention (Zip-Slip defense), integrity verification (SHA-256 + size), and Zod schema validation at every data boundary. Security scored highest (4.7/5.0).

### L4: Broad Horizontal vs Deep Vertical Composition

v1.26 (broad: ~30 modules across 7 phases) and v1.23 (deep: 17-layer pipeline with ICD contracts) represent two valid composition styles. The 8-stage install pipeline (download -> integrity -> scan -> gate -> extract -> map -> detect -> track) is the deepest vertical chain in v1.26, peaking at ~8 stages vs v1.23's 17. **Finding:** the chain's P6 metric should track both breadth and depth as separate composition signals rather than a single "composition" assessment.

### L5: FF-24 XSS Risk Persists in Desktop Panels

The Aminet browser panel renders user-contributed package descriptions (from Aminet .readme files) via `innerHTML` template literals in a Tauri webview. No visible sanitization. While the attack surface is limited to local execution (not a public web page), this is a correctness gap in an otherwise strong security release. **Action:** sanitize user-contributed description fields before rendering. Track in future reviews.

### L6: P9 N/A in Infrastructure Domains = Clean Assessment

When a release implements infrastructure (binary analysis, archive management, virus scanning, emulation) rather than evaluation/scoring, P9 (Scoring Duplication) naturally scores N/A. This is the correct assessment — infrastructure packs don't contain scoring formulas. The cleanest P9 result since the pattern was introduced confirms that scoring duplication is concentrated in specific module domains (calibration, budget, evaluation), not systemic.
