Implement optimization analysis to pass all tests.

**Algorithm structure -- three independent analyzers composed:**

1. **findBatchOpportunities(entries, manifests):**
   - Group entries by domain (exact string match)
   - For groups with 2+ entries: produce 'batch' suggestion, confidence = 0.6 + 0.05 * groupSize (cap 0.9)
   - Also check tag overlap: for each pair, compute Jaccard similarity of queueContext.tags. If overlap >= 0.4 and not already in a domain batch, produce 'batch' suggestion with confidence = overlap * 0.8
   - Include details.sharedTags or details.domain as appropriate

2. **findParallelLanes(entries, dependencyEdges):**
   - Build adjacency set from dependency edges (both directions -- if A->B, neither A nor B can parallel with each other)
   - Find maximal independent sets: entries with no edges between them
   - If independent set has >= 2 entries: produce 'parallel' suggestion
   - Confidence = 0.7 for completely independent, reduce by 0.1 for each shared resource in decomposition.sharedResources overlap
   - Include details.laneCount

3. **findSharedSetup(entries, manifests):**
   - For each pair, check:
     a. Shared skills: intersection of skillMatch names where status is 'ready' or 'recommended'. If >= 1 shared: suggestion with details.sharedSkills, confidence = 0.5 + 0.1 * sharedCount (cap 0.8)
     b. Shared external dependencies: intersection by name (case-insensitive). If >= 1 shared: suggestion with details.sharedDependencies, confidence = 0.5 + 0.1 * sharedCount (cap 0.8)
     c. Same topology: if both recommend same TopologyType. Suggestion with details.sharedTopology, confidence 0.4
   - Merge multiple shared setup signals for same pair into one suggestion (highest confidence, combined details)

**Composition:**
- `analyzeOptimizations` calls all three, concatenates results, sorts by confidence descending
- Handles gracefully: entries without manifests (skip manifest-dependent checks for those entries)
- DependencyEdge imported from dependency-detector.ts (type-only import)
- Pure function, no I/O
