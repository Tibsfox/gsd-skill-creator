**QueueManagerDeps** DI interface:
- appendAuditEntry: function to write audit entries
- readAuditLog: function to read audit log
- readFile: function to read files
- writeFile: function to write files
- mkdir: function to create directories

**createQueueManager(options: { basePath: string }, deps?: Partial<QueueManagerDeps>): QueueManager**

Returns a QueueManager object (not a class -- factory function returning a plain object with methods). The manager maintains an in-memory entries Map<string, QueueEntry> loaded from a `queue-state.json` file in the staging root (separate from queue.jsonl audit log).

**QueueManager interface methods:**

1. `addEntry(options: { filename, milestoneName, domain, tags, resourceManifestPath }): Promise<QueueEntry>`
   - Generates id: 'q-' + date portion + '-' + sequential counter (use a counter seeded from existing entries length)
   - Creates QueueEntry with state: 'uploaded', createdAt/updatedAt: now
   - Appends audit entry with action: 'create', fromState: null, toState: 'uploaded', actor: 'system'
   - Persists entries to queue-state.json
   - Returns created entry

2. `transition(entryId: string, toState: QueueState, actor: string, rationale: string): Promise<QueueEntry>`
   - Looks up entry by id (throws if not found)
   - Delegates to transitionQueueItem (validates transition)
   - Appends audit entry with action: 'transition', fromState, toState, actor, rationale
   - Updates in-memory map and persists to queue-state.json
   - Returns updated entry

3. `getEntry(entryId: string): QueueEntry | undefined`
   - Returns entry from in-memory map

4. `listEntries(filter?: { state?: QueueState }): QueueEntry[]`
   - Returns all entries, optionally filtered by state
   - Sorted by createdAt ascending

5. `analyzeQueue(): Promise<{ dependencies: DependencyGraph, optimizations: OptimizationSuggestion[] }>`
   - Loads manifests from disk for each entry (reads resourceManifestPath)
   - Calls detectDependencies with entries and manifests
   - Calls analyzeOptimizations with entries, manifests, and dependency edges
   - Returns combined analysis result

6. `getAuditLog(): Promise<QueueAuditEntry[]>`
   - Delegates to readAuditLog

**Tests (manager.test.ts):**
- addEntry creates entry with 'uploaded' state and records audit entry
- addEntry generates unique sequential ids
- transition moves entry to valid state and records audit entry
- transition throws for unknown entry id
- transition throws for invalid state transition (delegates to state-machine)
- getEntry returns entry by id, undefined for unknown
- listEntries returns all entries sorted by createdAt
- listEntries with state filter returns only matching entries
- analyzeQueue calls dependency detector and optimization analyzer
- analyzeQueue handles entries without manifests on disk (manifest read fails gracefully)
- getAuditLog returns all audit entries
- All operations persist state to queue-state.json
- DI: all deps can be overridden independently

Use mock deps for all tests (no real filesystem). Mock readFile for queue-state.json to return '[]' initially. Track writeFile calls to verify persistence. Track appendAuditEntry calls to verify audit logging.

Write tests first (RED), then implement (GREEN). Can be in same commit cycle since this is the integration plan.
