**types.ts:**
- `QueueState` type: union of 'uploaded' | 'checking' | 'needs-attention' | 'ready' | 'queued' | 'executing' | 'set-aside'
- `QUEUE_STATES` const array with all 7 values
- `VALID_QUEUE_TRANSITIONS` Record<QueueState, QueueState[]> with the transition map from tests
- `QueueEntry` interface:
  - id: string (format: 'q-YYYYMMDD-NNN')
  - filename: string (document filename in staging)
  - state: QueueState
  - milestoneName: string (human-readable milestone name)
  - domain: string (primary domain from resource analysis tags)
  - tags: string[] (from ResourceManifest.queueContext.tags)
  - resourceManifestPath: string (path to the .manifest.json in ready/)
  - createdAt: string (ISO 8601)
  - updatedAt: string (ISO 8601)
- `QueueAuditEntry` interface:
  - id: string (unique audit entry id, format: 'audit-YYYYMMDD-HHMMSS-NNN')
  - entryId: string (references QueueEntry.id)
  - action: string (human-readable action like 'transition', 'create', 'set-aside')
  - fromState: QueueState | null (null for creation)
  - toState: QueueState
  - actor: string (who triggered: 'system', 'user', specific agent name)
  - rationale: string (why this happened)
  - timestamp: string (ISO 8601)

**state-machine.ts:**
- `transitionQueueItem(entry: QueueEntry, toState: QueueState): QueueEntry`
  - Validates same-state check first
  - Validates transition against VALID_QUEUE_TRANSITIONS
  - Returns new object (spread entry) with updated state and updatedAt = new Date().toISOString()
  - Does NOT mutate original entry

**audit-logger.ts:**
- `AuditLoggerDeps` interface with appendFile, readFile, mkdir functions
- `appendAuditEntry(entry: QueueAuditEntry, options: { basePath: string }, deps?: AuditLoggerDeps): Promise<void>`
  - Uses STAGING_DIRS.queue to resolve path
  - Uses appendFile for atomic append (one JSON line + newline)
  - Cached mkdir (dirEnsured flag pattern from STATE.md decisions)
- `readAuditLog(options: { basePath: string }, deps?: AuditLoggerDeps): Promise<QueueAuditEntry[]>`
  - Reads entire file, splits by newline, JSON.parse each line
  - Skips empty lines
  - On parse error: calls optional onError callback, skips line
  - If file does not exist (ENOENT), returns empty array
  - Add onError to options: { basePath: string; onError?: (line: string, error: Error) => void }
