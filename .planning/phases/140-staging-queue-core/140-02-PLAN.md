8. **Multiple dependencies** -- Three entries where A -> B and A -> C. Graph has 2 edges.

  <action>
Implement dependency detection to pass all tests.

**Algorithm:**
1. For each pair of queue entries (i, j) where i !== j:
   a. Check for explicit dependency keywords in milestone names, domains, and vision summaries:
      - Forward keywords: "depends on", "requires", "needs", "after", "prerequisite"
      - Reverse keywords: "blocks", "enables", "provides for"
      - Match target milestone name or domain in the text surrounding the keyword
      - If found: add edge with type 'explicit', confidence 0.8-0.9 based on keyword specificity
   b. Check for implicit dependency via shared external dependencies:
      - Compare manifests[i].visionAnalysis.dependencies with manifests[j].visionAnalysis.dependencies
      - Match by dependency name (case-insensitive)
      - If shared deps found: add edge with type 'implicit', confidence = 0.3 + 0.1 * sharedCount (cap at 0.6)
   c. Check for implicit dependency via overlapping domain requirement categories:
      - Compare requirement categories between the two manifests
      - Use word-level Jaccard overlap (consistent with existing codebase pattern from skill-matcher)
      - If overlap > 0.3: add edge with type 'implicit', confidence = overlap * 0.6 (cap at 0.6)
2. Deduplicate edges: if both explicit and implicit exist between same pair, keep explicit (higher confidence).
3. Filter out self-referencing edges.
4. Return DependencyGraph with all edges and unique entry IDs.

Use pure functions, no I/O. Manifests passed as Map<string, ResourceManifest> keyed by QueueEntry.id.

Handle gracefully: entries without manifests in the map (skip dependency check for those entries, still include in entryIds).
  </action>
