## Key Questions Answered

### Q1: Desktop iteration or new domain?
**NEW DOMAIN.** v1.22 pivots entirely to infrastructure automation. Zero TypeScript changes. The bash/YAML/markdown paradigm is fundamentally different from prior releases. This is the project's first major multi-language expansion beyond TypeScript + Rust.

### Q2: Rust test coverage?
**NOT ADDRESSED.** Minor Rust changes (dead_code warning suppression) only. No new Rust tests.

### Q3: P-002 parameter governance?
**IMPROVED for infra domain.** Budget calculator has explicit documented rules. Template local-values pattern separates defaults from site-specific overrides. But CRT/boot parameters from v1.21 remain ungoverned.

### Q4: Dual composition track evolution?
**Third track added.** Now three composition paradigms:
1. CLI-side generator (~25 TS modules)
2. Desktop-side shell (~10 TS module groups)
3. Infrastructure-side automation (57 bash scripts + 8 shared libraries)

### Q5: P11 recovery?
**NO.** 2nd consecutive release with fix commit. P11 clean streak remains broken.
