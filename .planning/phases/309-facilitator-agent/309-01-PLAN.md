    - "blank slate", "brainstorm from scratch", "new app", "new product", "start fresh", "out of ideas" -> nature: 'open-ended', recommended_pathway: 'creative-exploration'
    - "why does", "root cause", "keep happening", "understand the problem", "abandon", "failing", "not working" -> nature: 'analytical', recommended_pathway: 'problem-solving'

    - "i want to use", "scamper session", "six hats", "rapid ideation", "just brainstorm", "let me" -> nature: 'explicit', recommended_pathway: 'free-form'

    When signals are mixed: default nature to 'mixed', recommended_pathway to 'problem-solving' (spec says mixed starts problem-solving).
    When no signals match: default to nature: 'mixed', recommended_pathway: 'problem-solving'.

    complexity field:
    - 'simple' when problem_statement length &lt; 50 chars or contains "quick", "simple", "small"
    - 'complex' when length > 200 chars or contains "architecture", "system", "enterprise", "comprehensive"
    - 'moderate' otherwise

    recommended_techniques: first 2-3 techniques from the pathway's technique_sequence.
    estimated_duration_ms: sum of default_duration_ms for recommended_techniques.
    rationale: short string explaining the mapping choice (e.g., "Problem contains analytical signals: 'why does', 'keep happening'").

    Cases (from spec test table):
    - "I need to brainstorm ideas for a new app" -> nature: 'open-ended', pathway: 'creative-exploration'
    - "Why do our users keep abandoning their carts?" -> nature: 'analytical', pathway: 'problem-solving'
    - "How can we make our onboarding better?" -> nature: 'improvement', pathway: 'product-innovation'
    - "Should we use React or Vue for the frontend?" -> nature: 'decision', pathway: 'decision-making'
    - "I want to do a SCAMPER session" -> nature: 'explicit', pathway: 'free-form'

    Part 2 — evaluateTransitionReadiness(session: SessionState): TransitionSignal | null

    Weighted confidence formula (from spec):
    transition_confidence =
      (timer_factor * 0.2) +           // ratio: (total_ms - remaining_ms) / total_ms
      (saturation_factor * 0.3) +      // idea velocity decline: (ideas_last_5min / ideas_first_5min); 1.0 if velocity &lt; 20% of peak
      (user_signal_factor * 0.4) +     // 1.0 if problem_statement contains "running low", "running dry", "done", "ready to move", "stuck"; 0.0 otherwise
      (minimum_threshold_factor * 0.1) // 1.0 if session has >= 5 ideas, 0.0 otherwise

    Returns null if confidence &lt; 0.5 (not yet worth signaling).
    Returns TransitionSignal with type based on dominant factor:
    - If saturation_factor caused rise: type = 'saturation_detected'
    - If user_signal_factor caused rise: type = 'user_request'
    - If timer_factor caused rise: type = 'timer_expiry'
    - Otherwise: type = 'natural_completion'

    Recommendation field: next phase (SessionPhase) after current phase per PHASE_ORDER.
    Returns null when session has no active_technique (no timer to track).

    Cases:
    - session with technique_timer 80% elapsed + 3 ideas in last 5 min vs 10 in first 5 min + problem_statement "running low" -> confidence > 0.7
    - session with technique_timer 10% elapsed + high velocity + no user signals -> null (confidence &lt; 0.5)
    - session with no active_technique -> null
