# GSD Skill Creator

## What This Is

A TypeScript + Rust + GLSL toolkit for creating, validating, and orchestrating Claude Code skills, agents, and teams — with a Tauri desktop application (GSD-OS), planning document dashboard, live terminal, staging layer, and information design system. Extended with a Minecraft Knowledge World, local cloud infrastructure, Amiga emulation, and formalized chipset definitions. Includes Project AMIGA mission infrastructure, a functioning Apollo AGC Block II simulator, RFC reference tooling, an Aminet archive management system, a GSD Den multi-agent coordination system, a comprehensive electronics educational pack, a vision-to-mission transformation pipeline, MCP integration (host + gateway + agent bridge), and an AI-facilitated brainstorming support system with 8 specialized agents and 16 techniques.

## Core Value

Skills, agents, and teams must match official Claude Code patterns so they work correctly when loaded by Claude Code — and the GSD ecosystem must provide spatial, visual, and operational tools that make complex system design tangible.

**Design Principles (proven):**
1. Calm by Default — Control surface quiet under nominal conditions
2. Human Decisions at Human Gates — System decides what it can; humans decide what matters
3. Isolation is Safety — Mission environments cannot affect the outside world without authorization
4. The Archive is Sacred — Every mission produces a complete, immutable record
5. Leverage over Power — Architectural intelligence over raw computation (The Amiga Principle)
6. Teach Through Use — Every mission is a learning opportunity for skill-creator
7. Fix or Amend — Code wrong: fix code; vision aspirational: amend doc with paper trail (v1.24)
8. Safety is Never Parallel — Safety warden on critical path, never parallelized with what it guards (v1.29)
9. Deny by Default — Unknown tools require admin scope; unknown MCP traffic blocked until trust established (v1.31)
10. No Criticism During Diverge — Osborn's rules enforced architecturally, not by policy; Critic agent blocked at instantiation level during non-Converge phases (v1.32)

## Constraints

- **Hardware minimum:** 16GB RAM, hardware virtualization support (VT-x/AMD-V)
- **Comfortable target:** 32GB+ RAM for full Knowledge World deployment
- **Open standards:** Every component must use documented, open interfaces
- **Legal compliance:** All Amiga software must be legally acquired (AROS ROM as default fallback)
- **Platform support:** Tier 1: CentOS Stream 9, Fedora 39+, Ubuntu 22.04+
- **Architecture:** Local-first, no cloud dependencies for core functionality
- **Strict module boundaries:** src/ never imports desktop/@tauri-apps/api; desktop/ never imports Node.js modules
