# examples/teams/

Organized into 4 category subfolders plus `deprecated/`. See [`../CATEGORIES.md`](../CATEGORIES.md) for taxonomy definitions.

## Categories

| Category | Directory | Description |
|----------|-----------|-------------|
| Code | [`code/`](code/) | Code review and security audit teams |
| Ops | [`ops/`](ops/) | DevOps pipeline, SRE operations, incident response teams |
| Infra | [`infra/`](infra/) | Infrastructure review, release management, platform onboarding teams |
| Migration | [`migration/`](migration/) | Migration and documentation generation teams |
| Deprecated | [`deprecated/`](deprecated/) | Superseded teams, preserved for reference |

Each team directory contains a `config.json` (member list, topology) and a `README.md` (purpose, workflow, use cases).

Per-category README files are populated during Stage 2 classification.

See [`../CHANGELOG.md`](../CHANGELOG.md) for the history of how teams arrived here.
