# release-and-deploy

Cut releases and get them to users or servers. Covers semantic versioning, changelogs, release notes, artifact packaging, deploy strategy (blue-green, canary, rolling), rollback plans, and the post-deploy verification checklist.

**Triggers:** `cut a release`, `ship it`, `semver bump`, `changelog`, `deploy to prod`, `rollback plan`, `canary`

**Affinity:** `sd-release-engineer`, `sd-tech-lead`
