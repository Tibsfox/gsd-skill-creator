# performance-and-profiling

Measure before optimizing. Covers microbenchmarks, profilers (CPU, allocation, IO), flame graphs, identifying the actual hot path, and optimizing only what the data justifies. Documents the before / after numbers so the win is auditable.

**Triggers:** `profile this`, `benchmark`, `hot path`, `flame graph`, `optimize performance`, `memory usage`, `measure first`

**Affinity:** `sd-developer`, `sd-tech-lead`
