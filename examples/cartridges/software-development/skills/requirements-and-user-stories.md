# requirements-and-user-stories

Turn a product idea or bug report into concrete, implementable work items. Write user stories with acceptance criteria, split epics into shippable increments, surface open questions before code is written, and keep the backlog honest about what is actually ready to pick up.

**Triggers:** `user story`, `acceptance criteria`, `backlog grooming`, `story points`, `slice the epic`, `ready for dev`, `requirements clarification`

**Affinity:** `sd-tech-lead`, `sd-developer`
