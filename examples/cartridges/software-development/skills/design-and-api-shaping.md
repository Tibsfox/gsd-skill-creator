# design-and-api-shaping

Shape module boundaries, data models, and public APIs before writing the implementation. Covers function signatures, type contracts, error models, naming, and the design-doc or ADR that records the decision. Design lives on paper long enough to get the shape right.

**Triggers:** `api design`, `design doc`, `adr`, `type contract`, `module boundary`, `interface shape`, `before i code this`

**Affinity:** `sd-tech-lead`, `sd-reviewer`
