# version-control-workflow

Operate git (or equivalent) as a first-class engineering tool. Covers branching strategy, conventional commits, clean history via rebase / squash, resolving conflicts on intent not just text, writing commit messages that explain the why, and opening pull requests that a reviewer can actually land.

**Triggers:** `git workflow`, `branch strategy`, `rebase`, `merge conflict`, `commit message`, `pull request`, `clean history`

**Affinity:** `sd-developer`, `sd-tech-lead`
