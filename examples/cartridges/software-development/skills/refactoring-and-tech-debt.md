# refactoring-and-tech-debt

Change the shape of existing code without changing its behavior. Small, test-backed steps — rename, extract, inline, move — applied continuously so tech debt does not compound. Knows when to refactor in place vs. when to open a dedicated cleanup PR.

**Triggers:** `refactor`, `clean up the code`, `tech debt`, `rename`, `extract function`, `simplify this`, `make it easier to change`

**Affinity:** `sd-developer`, `sd-reviewer`
