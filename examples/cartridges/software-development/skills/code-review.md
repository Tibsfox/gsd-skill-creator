# code-review

Review pull requests with discipline — correctness first, design next, style last. Covers leaving feedback that is specific and kind, distinguishing blocking from nit-level, and the author side of receiving review without defensiveness. A review is a conversation, not a gate.

**Triggers:** `code review`, `review this pr`, `leave feedback`, `request changes`, `approve pr`, `review comments`

**Affinity:** `sd-reviewer`, `sd-tech-lead`
