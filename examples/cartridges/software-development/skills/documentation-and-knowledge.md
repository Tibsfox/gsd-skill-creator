# documentation-and-knowledge

Write and maintain the documentation that outlives any one engineer — READMEs, API docs, architecture decision records, runbooks, and inline comments that explain intent when it is non-obvious. Defaults to no comment unless the why would surprise a reader.

**Triggers:** `write docs`, `readme`, `api docs`, `adr`, `runbook`, `update documentation`, `inline comment`

**Affinity:** `sd-developer`, `sd-tech-lead`
