# debugging-and-forensics

Investigate bugs and regressions via `/gsd debug`, `/gsd forensics`, and `/gsd health`. Runs scientific-method debug sessions (hypothesis → experiment → observation → refinement), manages checkpoints, and closes the loop by writing a DebugSession record into grove for future retrieval.

**Triggers:** `gsd debug`, `debug session`, `forensics`, `bug hypothesis`, `regression investigation`, `gsd health`, `scientific debugging`

**Affinity:** `gsd-capcom`, `gsd-debugger`
