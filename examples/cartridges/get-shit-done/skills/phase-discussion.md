# phase-discussion

Drive pre-plan phase discussion via `/gsd discuss-phase`. Surfaces gray-area decisions, collects advisor research, enumerates assumptions through the assumptions-analyzer, and leaves a decision log that feeds directly into the planner. Used when a phase is too ambiguous to plan directly and needs a design pass first.

**Triggers:** `discuss phase`, `gray area decisions`, `phase design discussion`, `list phase assumptions`, `advisor research`

**Affinity:** `gsd-capcom`, `gsd-advisor-researcher`, `gsd-assumptions-analyzer`
