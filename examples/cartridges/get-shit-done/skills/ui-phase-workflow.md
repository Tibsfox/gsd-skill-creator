# ui-phase-workflow

Drive the frontend-specific phase cycle via `/gsd ui-phase` and `/gsd ui-review`. Produces a UI-SPEC design contract, runs the 6-pillar visual audit, and writes UI-REVIEW.md with BLOCK/FLAG/PASS verdicts so frontend work stays contract-driven.

**Triggers:** `ui phase`, `ui review`, `ui spec`, `ui audit`, `design contract`, `visual audit`

**Affinity:** `gsd-capcom`, `gsd-ui-researcher`, `gsd-ui-checker`, `gsd-ui-auditor`
