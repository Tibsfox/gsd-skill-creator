# divide-and-conquer-bisection

Halve the search space at each step to localize a fault. Swap known-good components, disable half the configuration, roll back to a mid-point revision, or physically partition a system into halves and test each. The canonical strategy when the suspect set is large.

**Triggers:** `bisect`, `narrow it down`, `halve the search`, `swap components`, `git bisect`, `binary search for fault`, `isolate by halves`

**Affinity:** `comparative-analyst`, `hypothesis-tester`
