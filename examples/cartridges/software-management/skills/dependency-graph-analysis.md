# dependency-graph-analysis

Build and analyze the dependency graph — internal imports, external packages, transitive deps, cycles, orphan modules, and high-fan-in hotspots. Identifies coupling risks before a port, review, or refactor touches them.

**Triggers:** `dependency graph`, `import graph`, `circular deps`, `coupling analysis`, `fan in`, `orphan module`, `transitive deps`

**Affinity:** `sm-codebase-cartographer`
