# incident-rollback-and-recovery

Execute a clean rollback when a deploy goes bad — revert artifact, drain in-flight work, reconcile any forward-incompatible state (migrations, caches, queues), notify stakeholders, and capture the evidence needed for the post-incident review.

**Triggers:** `rollback`, `revert the deploy`, `back out`, `undo migration`, `bad deploy`, `drain traffic`, `emergency rollback`

**Affinity:** `sm-deploy-engineer`, `sm-lead`
