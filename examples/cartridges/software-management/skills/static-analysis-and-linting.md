# static-analysis-and-linting

Run and curate static analysis — linters, type checkers, security scanners, complexity metrics, dead-code detectors. Distinguishes signal from noise, suppresses at the right scope, and feeds new findings back into the review and porting workflows.

**Triggers:** `static analysis`, `lint the repo`, `type check`, `security scan`, `complexity metrics`, `dead code`, `suppress warnings`

**Affinity:** `sm-reviewer`, `sm-codebase-cartographer`
