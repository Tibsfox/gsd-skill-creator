# codebase-mapping-and-inventory

Walk an unfamiliar codebase and produce a structured inventory — languages, modules, entry points, build targets, test surfaces, generated files, and vendored dependencies. The first thing to do when you inherit a repo. Output feeds every downstream skill in this cartridge.

**Triggers:** `map the codebase`, `what's in this repo`, `inventory`, `unfamiliar repo`, `walk the tree`, `entry points`, `where does execution start`

**Affinity:** `sm-codebase-cartographer`, `sm-lead`
