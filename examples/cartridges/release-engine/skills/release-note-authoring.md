# release-note-authoring

Author the release-note package for a single release. Writes `docs/release-notes/<version>/README.md` with the mandatory sections — Summary, Key Features, Retrospective, Lessons Learned — and a full-body GitHub release text. Enforces the "docs are the story" rule (full verbosity) and the no co-author-line rule.

**Triggers:** `write release notes`, `release readme`, `github release body`, `docs/release-notes`, `release summary`, `release feature list`

**Affinity:** `release-capcom`, `release-engineer`
