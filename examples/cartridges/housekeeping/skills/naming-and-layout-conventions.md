# naming-and-layout-conventions

Enforce and evolve naming and directory-layout conventions. Covers file naming rules, directory structure templates, slugging policy, date-stamp conventions, and versioned-folder patterns (e.g. `v1.49.N/README.md`). Flags new files that violate convention so they can be renamed before they propagate, but never auto-renames without explicit go.

**Triggers:** `naming convention`, `layout convention`, `slugging`, `directory structure`, `file naming`, `rename policy`

**Affinity:** `housekeeping-capcom`, `curator`
