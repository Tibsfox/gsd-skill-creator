# repo-hygiene

Keep the working tree tidy without destroying work. Audits untracked files, dotfile cruft, stray scratch directories, orphaned lockfiles, and `.bg-shell/` style runtime state. Surfaces findings as a HousekeepingSweep record and proposes archive-or-ignore actions — never bulk-deletes. Respects the rule that `.planning/` and runtime state directories stay untracked by design.

**Triggers:** `clean up repo`, `repo hygiene`, `tidy working tree`, `untracked audit`, `dotfile cleanup`, `stray files`, `housekeeping sweep`

**Affinity:** `housekeeping-capcom`, `steward`
