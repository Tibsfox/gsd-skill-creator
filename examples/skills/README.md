# examples/skills/

Organized into 9 category subfolders plus `deprecated/`. See [`../CATEGORIES.md`](../CATEGORIES.md) for taxonomy definitions.

## Categories

| Category | Directory | Description |
|----------|-----------|-------------|
| GSD workflow | [`gsd/`](gsd/) | Get-Shit-Done workflow skills |
| Research | [`research/`](research/) | Research pipelines and data fidelity |
| Media | [`media/`](media/) | Audio, video, publishing, authoring |
| Dev | [`dev/`](dev/) | Developer productivity and language patterns |
| Ops | [`ops/`](ops/) | SRE, incident response, monitoring |
| Workflow | [`workflow/`](workflow/) | Loops, schedules, session awareness |
| Patterns | [`patterns/`](patterns/) | SQL, API design, Docker, CI/CD |
| Orchestration | [`orchestration/`](orchestration/) | Fleet mission, mayor coordinator, propulsion |
| State | [`state/`](state/) | Persistence and inter-agent channels |
| Deprecated | [`deprecated/`](deprecated/) | Superseded skills, preserved for reference |

Per-category README files are populated during Stage 2 classification.

See [`../CHANGELOG.md`](../CHANGELOG.md) for the history of how skills arrived here.
