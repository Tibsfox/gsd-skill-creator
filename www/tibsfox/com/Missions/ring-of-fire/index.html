How to Use This Package
  1. Open rimfire_sovereignty.pdf and review the Stage 1 Vision Guide with Tibsfox for alignment with Fox Infrastructure Group priorities before execution.
  2. Hand Stage 3 Mission Package to the GSD orchestrator in the gsd-skill-creator dev branch. The package is self-contained — zero additional context required.
  3. Begin Wave 0: use Haiku to generate the five schema templates (nation profile, legal framework, infrastructure model, source index, OCAP®/CARE checklist).
  4. Execute Wave 1 parallel tracks: Track A (Sonnet: Modules A + C), Track B (Opus: Module B governance profiles), Track C (Sonnet: Module E data sovereignty).
  5. CAPCOM gate after Wave 1: Tibsfox reviews governance profiles (Module B) for accuracy and cultural sensitivity before Opus synthesizes in Wave 2.
  6. Execute Wave 2: Opus produces Module D (cooperative infrastructure synthesis) and the FIG Sovereignty Stack mapping document.
  7. CAPCOM gate after Wave 2: Tibsfox reviews the FIG mapping for integration with Fox Infrastructure Group partnership architecture.
  8. Execute Wave 3: Sonnet assembles, VERIFY runs OCAP®/CARE audit, LOG produces commit messages and milestone summary.
