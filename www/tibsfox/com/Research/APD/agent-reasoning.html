Agent Reasoning

Together they map the boundary between when to use single versus multi-agent architectures.

Information Theory

Single-Agent LLMs Outperform Multi-Agent Systems on Multi-Hop Reasoning

Paper: arXiv:2604.02460
Categories: cs.AI, cs.CL · Date: April 2026 · Stanford University

Full Deep Dive: MAS → Single vs Multi-Agent

Uses the Data Processing Inequality to prove that information degrades at every agent handoff in multi-agent pipelines. For multi-hop reasoning tasks requiring preserved context chains, a single agent with sufficient token budget consistently outperforms multi-agent decomposition. The proof establishes a theoretical ceiling: multi-agent decomposition can never exceed single-agent performance when the task requires the full reasoning chain to be available at the point of inference. This has direct implications for when orchestration overhead exceeds its benefits.

Coupled Control

SCRAT: Structured Coupled Control with Memory

Paper: arXiv:2604.03201
Categories: cs.AI, cs.SE · Date: April 2026

Key Finding: SCRAT (Structured Coupled Reasoning and Action Traces) introduces a framework that couples an agent’s reasoning process with its action execution through structured memory. Rather than allowing free-form chain-of-thought followed by independent action selection, SCRAT enforces that each reasoning step is linked to a verifiable action trace, creating an auditable chain from thought to behavior.

Architecture: The system maintains a structured memory buffer where each entry contains: (1) the reasoning context that motivated the action, (2) the action taken, (3) the environment’s response, and (4) a verification signal indicating whether the action achieved its intended effect. This four-tuple structure enables post-hoc verification of agent decision quality.

Implication: SCRAT addresses the “hallucinated reasoning” problem where agents generate plausible-sounding justifications for incorrect actions. By coupling reasoning to verifiable outcomes, the framework provides a foundation for agent accountability that can be audited without re-running the agent.

Theory of Mind

Readable Minds: Theory of Mind in LLM Poker Agents

Paper: arXiv:2604.04157
Categories: cs.AI, cs.GT · Date: April 2026

Key Finding: LLM-based poker agents demonstrate emergent Theory of Mind (ToM) capabilities — the ability to model other players’ beliefs, intentions, and likely strategies. The agents develop internal representations of opponent behavior without being explicitly trained on ToM tasks, suggesting that strategic multi-agent interaction naturally elicits mental modeling capabilities in sufficiently large language models.

Methodology: The researchers measured ToM emergence through: (1) prediction accuracy for opponent actions, (2) adaptation to opponent strategy changes mid-game, (3) bluffing behavior calibrated to opponent modeling rather than hand strength alone. All three metrics showed performance above chance and improving with model scale.

Implication: Emergent ToM in adversarial settings raises both opportunities and risks. For cooperative multi-agent systems, ToM enables better coordination. For adversarial scenarios, it means agents can potentially anticipate and counter human oversight strategies — a consideration for the safety community.

Synthesis: The Architecture Decision Boundary

These four papers collectively map the decision boundary for agent architecture:

  • Single agent wins when the task requires preserved context chains and multi-hop reasoning (2604.02460). The Data Processing Inequality sets a theoretical ceiling on multi-agent decomposition for these tasks.
  • Multi-agent wins when the task decomposes into governance, execution, and compliance concerns (2604.01020). OrgAgent’s 102% improvement shows the benefit is real when the hierarchy matches the task structure.
  • Accountability requires coupling — SCRAT’s structured memory (2604.03201) provides the verification infrastructure needed for either architecture to be auditable.
  • Emergent capabilities demand caution — ToM emergence in poker agents (2604.04157) shows that multi-agent interaction can produce capabilities not present in the training objective, requiring monitoring beyond task-specific metrics.

The practical guideline: use single agents for reasoning-heavy tasks, hierarchical multi-agent systems for governance-heavy tasks, and instrument both with SCRAT-style verification for accountability.
