Papers

Context Optimization

Reflective Context Learning: Optimization Primitives for Context Space

Paper: arXiv:2604.03189
Categories: cs.CL, cs.AI · Date: April 2026

Key Finding: Introduces a formal framework that treats context as a mathematical space with optimization primitives. Rather than manually engineering prompts, reflective context learning enables agents to optimize their own context by: (1) measuring the information density of each context segment, (2) identifying which segments contribute to task completion, and (3) iteratively refining the context window to maximize task-relevant signal while minimizing noise.

Optimization Primitives: Three primitives defined: compress (reduce token count while preserving information), amplify (expand important context with retrieved detail), and reorder (arrange context to match the model’s attention patterns). These compose into higher-level context strategies.

Implication: Context engineering becomes a tractable optimization problem rather than an art. The primitives provide a vocabulary for describing and automating context manipulation that is currently done manually by prompt engineers.

Knowledge Graphs

Schema-Aware Planning for Reliable Knowledge Graphs

Paper: arXiv:2604.04190

Key Finding: LLM-based knowledge graph construction is unreliable when the model generates both the schema and the data in a single pass. Schema-aware planning separates these concerns: first, the agent generates or retrieves a domain-appropriate schema (entity types, relation types, cardinality constraints), then populates it using the schema as a structural constraint on extraction. This two-phase approach reduces entity type errors by 43% and relation hallucination by 56%.

Hybrid Toolset: The system combines LLM extraction with rule-based validation tools that enforce schema constraints. An entity extracted by the LLM is only committed to the graph if it satisfies the schema’s type constraints and passes consistency checks against existing entities.

Implication: Schema-first knowledge graph construction provides a structural defense against hallucination that complements the semantic defenses in the hallucination cluster. The 56% reduction in relation hallucination demonstrates that structural constraints are highly effective at catching fabricated connections.

RAG Evolution

Experience as Compass: Evolving Multi-Agent RAG Orchestration

Paper: arXiv:2604.00901

Key Finding: RAG orchestration that learns from past retrieval success and failure. Instead of static retrieval strategies, the system maintains a “compass” of past experiences: which retrieval methods worked for which query types, which sources provided accurate vs. hallucinated information, and which chunking strategies preserved the context needed for accurate answers. The compass evolves continuously as new queries are processed.

Architecture: Multi-agent RAG with specialized agents for: query decomposition, source selection, chunk ranking, and answer synthesis. Each agent’s strategy is parameterized and updated based on end-to-end answer quality feedback, creating a self-improving retrieval pipeline.

Implication: Static RAG pipelines degrade as document collections grow and query distributions shift. Experience-based evolution addresses both problems by continuously adapting retrieval strategy to match actual usage patterns. This connects directly to the Artemis mission’s pgvector deployment and research corpus management.

Data Management

EnviSmart: Robust Multi-Agent Environmental Data Management

Paper: arXiv:2604.01647

Key Finding: A multi-agent system designed specifically for environmental data management — sensor data ingestion, quality control, anomaly detection, and reporting for environmental monitoring networks. EnviSmart addresses the challenge of heterogeneous data sources (weather stations, air quality sensors, water quality monitors) that produce data at different frequencies, formats, and reliability levels.

Robustness Features: The system handles: (1) sensor dropout (automatic gap-filling from correlated stations), (2) calibration drift (statistical detection and correction), (3) unit inconsistencies (automatic normalization), and (4) temporal misalignment (resampling to common time grids). These are the practical data engineering challenges that most academic RAG systems ignore.

Implication: Environmental data management is a proving ground for real-world multi-agent data workflows. The robustness features — gap-filling, drift correction, normalization — are directly applicable to any multi-source data integration problem, including the Artemis mission’s weather monitoring network and research data pipeline.

Synthesis: From Context Engineering to Knowledge Infrastructure

These four papers trace a progression from individual context optimization to full knowledge infrastructure:

  1. Context as Math (2604.03189) — Reflective context learning provides the optimization primitives (compress, amplify, reorder) that formalize what prompt engineers do manually. This is the foundation layer.
  2. Structure as Constraint (2604.04190) — Schema-aware knowledge graphs add structural constraints that reduce hallucination by 56%. Structure is a force multiplier for accuracy.
  3. Experience as Adaptation (2604.00901) — RAG orchestration that learns from outcomes transforms static retrieval into self-improving knowledge access. The system gets better with use rather than degrading.
  4. Real Data as Test (2604.01647) — EnviSmart grounds knowledge management in the practical realities of heterogeneous, unreliable data sources. The robustness features (gap-filling, drift correction) are necessary for any production deployment.

The thread connecting these papers: context engineering is evolving from prompt crafting to knowledge infrastructure — with formal optimization, structural constraints, adaptive learning, and real-world robustness as the four pillars.
