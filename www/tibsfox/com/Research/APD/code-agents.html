Code Agents

Four papers examining AI agents that write, review, and reason about code: the gap between industry claims and empirical reality for code review agents, psychometric profiling that predicts task-level agent performance (ICLR 2026), STITCH training reaching 63% on SWE-bench, and Profile-Then-Reason for managing semantic complexity in tool-augmented code agents. Together they calibrate expectations for what code agents can and cannot reliably do.

Papers

Reality Gap

Code Review Agents: Industry Claims vs. Empirical Reality

Paper: arXiv:2604.03196
Categories: cs.SE, cs.AI · Date: April 2026

Key Finding: A systematic evaluation of code review agents reveals a significant gap between industry marketing claims and empirical performance. While vendors claim “human-level code review,” controlled studies show agents consistently miss: (1) architectural design issues that require system-level context, (2) subtle concurrency bugs that depend on execution ordering, and (3) security vulnerabilities that require understanding of data flow across module boundaries.

Where Agents Excel: Style consistency, API usage patterns, documentation completeness, and known anti-pattern detection. These are the categories where pattern-matching on local context is sufficient.

Implication: Code review agents are best deployed as first-pass filters that handle the mechanical aspects of review, freeing human reviewers to focus on architectural, security, and concurrency issues. Claims of autonomous code review are premature by the paper’s measurements.

ICLR 2026

Agent Psychometrics: Task-Level Performance Prediction

Paper: arXiv:2604.00594

Key Finding: Adapts psychometric methods from educational testing to predict agent performance on specific tasks before execution. By profiling an agent’s capabilities across a standardized battery (analogous to IQ subtests), the framework predicts whether a given agent will succeed on a novel task with 78% accuracy. This enables pre-flight task routing — assigning tasks to agents whose profiles match the required capability dimensions.

Capability Dimensions: The psychometric battery measures: code comprehension, multi-file navigation, API integration, test generation, debugging, and refactoring. Each dimension is scored independently, producing a capability profile rather than a single aggregate score.

Implication: Psychometric profiling transforms agent selection from trial-and-error to evidence-based routing. For multi-agent systems, this enables optimal task allocation based on measured capability rather than assumed capability. Accepted at ICLR 2026.

SWE-bench

STITCH: Training for 63% SWE-bench Performance

Paper: arXiv:2604.00824

Key Finding: STITCH (Structured Training for Integrated Tool-Chain Handling) is a training methodology that achieves 63% on SWE-bench — a significant improvement over prior methods. The key innovation is training on structured traces that include not just the final code patch but the entire tool-use trajectory: file exploration, test execution, error diagnosis, and iterative refinement. This teaches the model the process of software engineering, not just the product.

Training Architecture: STITCH generates training data from expert trajectories that are decomposed into: (1) information gathering phase, (2) hypothesis formation, (3) implementation attempt, (4) test feedback integration, and (5) refinement. Each phase is separately supervised, creating a curriculum that matches the natural structure of debugging work.

Implication: Process-aware training significantly outperforms outcome-only training for software engineering tasks. The 63% SWE-bench result suggests that the ceiling for code agents is substantially higher than current production systems achieve, limited more by training methodology than model capability.

Complexity Bounds

Profile-Then-Reason: Bounded Semantic Complexity

Paper: arXiv:2604.04131

Key Finding: Tool-augmented agents face a combinatorial explosion in semantic complexity as the number of available tools increases. Profile-Then-Reason addresses this by first profiling the task to estimate its semantic complexity, then selecting a minimal tool subset sufficient for the task. This bounded-complexity approach prevents the overthinking loops documented in 2602.14798 while maintaining task completion rates.

Profiling Mechanism: The system extracts semantic features from the task description (required operations, data types, output format) and matches them against tool capability profiles. Tools outside the matched set are hidden from the agent, reducing the decision space from O(n!) to O(k!) where k << n.

Implication: Profile-Then-Reason connects directly to the MCP overthinking problem: by reducing the visible tool set, agents make faster and more reliable decisions. This is the agent-side complement to ShieldNet’s network-side guardrails — reducing attack surface and computational waste simultaneously.

Synthesis: The Code Agent Maturity Model

These four papers define a maturity model for code agent deployment:

  1. Level 1 — Pattern Matching (2604.03196): Current production code review agents. Reliable for style, API patterns, known anti-patterns. Unreliable for architecture, concurrency, cross-module security.
  2. Level 2 — Task Routing (2604.00594): Psychometric profiling enables matching tasks to agent capabilities. 78% prediction accuracy means 1-in-5 tasks will still be misrouted, requiring human oversight.
  3. Level 3 — Process Awareness (2604.00824): STITCH-style training teaches the debugging process, not just outcomes. 63% SWE-bench shows the potential; the remaining 37% likely requires architectural reasoning beyond current capabilities.
  4. Level 4 — Bounded Complexity (2604.04131): Profile-Then-Reason manages the tool proliferation problem. Without complexity bounds, more tools paradoxically reduces performance (overthinking).

The practical guideline: deploy code agents at Level 1 for all reviews, Level 2 for task routing in multi-agent systems, and treat Levels 3–4 as the near-term research frontier that will define the next generation of production systems.
