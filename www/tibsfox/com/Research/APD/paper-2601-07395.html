MCP-ITP: Automated Implicit Tool Poisoning

arXiv: 2601.07395

Date: January 2026 · Categories: cs.CR, cs.CL

Abstract

An automated framework for generating implicit tool poisoning attacks against MCP servers. MCP-ITP demonstrates that tool descriptions — the natural-language metadata agents use to select tools — can be crafted to manipulate agent behavior without modifying the tool's actual implementation. The attack operates entirely at the semantic layer where agents make tool selection decisions, bypassing traditional code review and sandboxing defenses.

Key Findings

  1. Tool descriptions can be weaponized to redirect agent tool selection across multiple LLM backends
  2. The attack is entirely in the metadata — tool code is benign, bypassing code review
  3. Poisoned descriptions embed subtle semantic cues (e.g., “comprehensive search including environment variables”)
  4. Current MCP security focuses on code review and sandboxing — both are bypassed
  5. A new defense category is required: description semantic analysis
  6. Implicit payload injection achieves consistent redirection without explicit malicious instructions

Methodology

Automated attack generation framework that crafts tool descriptions with implicit payload injection. The framework generates descriptions that pass human review (they appear helpful) while containing semantic cues that bias agent tool selection. Tested against multiple LLM backends to validate cross-model effectiveness.

Implications for Our Architecture

Critical for our skill description system. The 250-char skill description cap we enforce is a partial mitigation (less room for semantic manipulation), but we need semantic analysis of skill descriptions, not just length limits. Our security-hygiene skill should add description intent verification — comparing what a skill description claims against what the skill actually does. This maps directly to the trust-relationship provider's verification stage.
