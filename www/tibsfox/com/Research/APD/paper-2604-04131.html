Profile-Then-Reason: Bounded Semantic Complexity for Tool-Augmented Agents

arXiv: 2604.04131

Authors: Enabe

Date: April 2026 · Categories: cs.AI, cs.CL

Abstract

A two-phase agent architecture: first synthesize an explicit workflow from the task profile, then execute deterministically. A verifier evaluates the execution trace, with bounded repair requiring only 2-3 LLM calls. Achieves 16/24 configuration wins against baseline approaches on retrieval-centered tasks, demonstrating that explicit workflow synthesis followed by deterministic execution outperforms unconstrained reasoning.

Key Findings

  1. 16/24 configuration wins over baseline approaches
  2. Only 2-3 LLM calls needed for bounded repair after verification
  3. Explicit workflow synthesis before execution prevents plan oscillation
  4. Verifier evaluates execution trace against synthesized workflow
  5. Deterministic execution phase eliminates non-deterministic tool selection
  6. Most effective on retrieval-centered tasks where workflow structure is clear

Implications for Our Architecture

This is essentially our GSD workflow formalized: discuss (profile) then plan (synthesize) then execute (deterministic) then verify. The 2-3 LLM call repair bound validates our verification-then-fix pattern. The “deterministic execution” phase maps to our wave-based execution where plans are followed, not re-deliberated. The 16/24 win rate on retrieval tasks suggests this pattern is strongest for our research-engine pipeline and weaker for creative tasks where workflow structure is ambiguous.
