ShieldNet: Network-Level Guardrails for Supply-Chain Injections

arXiv: 2604.04426

Authors: Yuan et al.

Date: April 2026 · Categories: cs.CR, cs.AI

Abstract

ShieldNet introduces network-level guardrails against supply-chain injections using SC-Inject-Bench, a benchmark of 10,000+ malicious tools across 25+ attack types. A man-in-the-middle proxy event extractor intercepts and classifies tool interactions in real time. Achieves F1=0.995 with only 0.8% false positive rate, making it practical for production deployment without significant interference to legitimate tool use.

Key Findings

  1. F1 = 0.995 detection accuracy for supply-chain injection attacks
  2. 0.8% false positive rate — low enough for production deployment
  3. 10,000+ malicious tools in the SC-Inject-Bench benchmark
  4. 25+ distinct attack types catalogued
  5. Man-in-the-middle proxy architecture enables transparent interception
  6. Event extractor classifies tool interactions without modifying the tool or the agent

Methodology

Man-in-the-middle proxy positioned between agent and MCP server intercepts all tool invocations. An event extractor analyzes request/response patterns against learned attack signatures. SC-Inject-Bench provides standardized evaluation with 10K+ malicious tools covering 25+ attack categories.

Implications for Our Architecture

Hook System Mapping

The MITM proxy pattern maps directly to our hook system — PreToolUse and PostToolUse hooks already intercept tool invocations at the boundary. ShieldNet's event extractor pattern could be implemented as a security hook that classifies tool interactions against known attack signatures.

Deployment Considerations

  • The 0.8% FP rate is acceptable for our use case where false positives generate warnings rather than hard blocks
  • SC-Inject-Bench should be added to our test suite for security validation
  • The 25+ attack type taxonomy provides a classification framework for our security-hygiene skill
  • Network-level interception catches attacks that code-level analysis misses (see MCP-ITP)
