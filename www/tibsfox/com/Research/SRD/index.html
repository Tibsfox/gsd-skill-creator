A protocol specification is the most honest thing in computing. It does not have opinions, does not defer to authority, does not explain itself in terms of what was intended. Security is not a property you add to a protocol — it is either present in the wire format or it is not. The spaces between are where protocols live.

Research Modules

M1: SSH Core Transport

SSH Track A · Transport layer

SSH binary packet format (RFC 4253 §6). Cipher negotiation, algorithm name-lists, key exchange (DH/ECDH/Curve25519). Protocol version exchange. Message type catalog. Block-size alignment and AEAD vs MAC-then-encrypt ordering.

M2: SSH Application Layer

SSH Track A · Authentication & channels

RFC 4252 authentication methods (publickey, password, keyboard-interactive). RFC 4254 connection protocol: channel lifecycle, flow control, port forwarding, X11 forwarding. SFTP subsystem message catalog.

M3: RDP Protocol

RDP Track B · Remote desktop

MS-RDPBCGR wire format. Complete 10-phase connection sequence. Static virtual channels (max 31) and dynamic channel architecture. Graphics pipeline variants. NLA/CredSSP/TLS security modes.

M4: VNC/RFB & SPICE

VNC SPICE Track C · Display protocols

VNC/RFB RFC 6143: framebuffer model, encoding registry (Raw, CopyRect, Hextile, Tight, ZRLE), security type registry. SPICE multi-channel architecture: QXL device model, TLS/SASL auth. X11 forwarding and NX/XPRA.

M5: Security & Hardening

Security Synthesis · Cross-protocol

Protocol security comparison: SSH vs RDP vs VNC vs SPICE. CVE landscape (BlueKeep, DejaBlue, SSH historical). Hardening guidance per IETF RFC 9142 and NIST SP 800-series. Cipher suite recommendations.

M6: Implementation Guide

Guide Synthesis · Practical reference

Protocol selection decision matrix. Performance characteristics and network overhead comparison. GSD Mesh applicability notes. SSH tunneling for mesh communications. Deployment architecture patterns.
