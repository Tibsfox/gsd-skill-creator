Fetch-Decode-Execute

The Fundamental CPU Loop — From Bits to Transistors, ALU, Pipeline, Speculation, JIT
Overview Encoding Execution Formats Microcode Security Rosetta Learn History
Every CPU runs the same loop: fetch the next instruction from memory, decode what it means, execute the operation, update state. On modern CPUs this is pipelined 20 stages deep, superscalar, speculative, and out-of-order — but the abstraction holds.

The Cycle

while (running) {
    instruction = memory[IP];           // FETCH
    (op, operands) = decode(instruction); // DECODE
    result = execute(op, operands);      // EXECUTE
    write_back(result);                  // WRITEBACK
    IP = next_address(op);              // UPDATE
}

From Bits to Electrical Signals

Machine code bits become electrical signals through layers of abstraction:

  1. Memory stores bits as charged/uncharged capacitors (DRAM) or as flip-flop states (SRAM/cache)
  2. The bus carries bits as voltage levels on wires
  3. The decoder (combinational logic: AND/OR/NOT gates) recognizes bit patterns and activates control lines
  4. The ALU (adders, shifters, multiplexers built from gates) performs the operation
  5. The register file (array of flip-flops) stores results
  6. The clock synchronizes everything: one tick = one pipeline stage advances

The ALU

The Arithmetic Logic Unit takes two operands and an operation selector, produces a result and flags (carry, zero, negative, overflow). A 64-bit adder is built from 64 full adders, optimized with carry-lookahead for speed. The ALU implements: add, sub, and, or, xor, not, shift left, shift right, compare.

The Pipeline

StageWork
FetchRead instruction bytes from I-cache at the instruction pointer
DecodeDetermine operation, source/dest registers, immediate values
RenameMap architectural registers to physical registers (eliminate false dependencies)
SchedulePlace instruction in a reservation station; wait for operands
ExecuteRun on an execution unit (ALU, FPU, load/store, branch)
WritebackWrite result to physical register
RetireCommit results in program order; free physical register

Modern x86: 15–20+ stages. Multiple instructions per stage (superscalar: 4–6 wide decode, 8–12 wide issue). Out-of-order: instructions execute when ready, not in program order. Retire in order to maintain the sequential abstraction.

Speculative Execution

The CPU guesses which way branches will go and executes speculatively. If the guess is correct (~97% of the time), the speculative results are committed. If wrong, they are discarded (pipeline flush, ~15–20 cycle penalty). Spectre exploits the fact that speculative loads leave observable cache side effects even when discarded.

JIT: Machine Code at Runtime

JIT compilers (V8 TurboFan, HotSpot C2, .NET RyuJIT, LuaJIT) write machine code bytes into executable memory at runtime. The CPU then fetches and executes those freshly-written bytes. This is the legitimate form of self-modifying code and the reason managed languages can approach native speed.
