Machine Code Security

Shellcode, ROP, Binary Analysis, Packers, Spectre/Meltdown, Code Signing
Overview Encoding Execution Formats Microcode Security Rosetta Learn History
Security vulnerabilities ultimately exist at the machine code level. Buffer overflows overwrite machine code addresses. ROP chains execute existing machine code out of order. Spectre leaks data through speculative machine code execution.

Shellcode

Hand-crafted machine code injected into a vulnerable process. Must be: position-independent (no absolute addresses), often null-free (null bytes terminate C strings), and small (fits in a buffer). Example: execve("/bin/sh") in ~27 bytes of x86-64 machine code.

Return-Oriented Programming (ROP)

When the stack is non-executable (DEP/NX), attackers chain small existing machine code sequences (“gadgets”) that end in ret (C3). By controlling the stack, they build arbitrary computation from the program’s own machine code. No code injection needed.

Defenses

CFIControl-Flow Integrity: restrict indirect branches to valid targets.
Shadow Stack (CET)Hardware-backed separate stack for return addresses. Intel CET, ARM BTI.
Code signingCryptographic signature on executable. OS refuses to run unsigned code (macOS, iOS).

Binary Analysis

  • Static analysis — reading machine code without running it (IDA Pro, Ghidra, Binary Ninja, radare2)
  • Dynamic analysis — running and observing (GDB, strace, ltrace, Valgrind, DynamoRIO, Pin)
  • Symbolic execution — exploring all paths mathematically (angr, KLEE)
  • Fuzzing — generating random inputs to find crashes (AFL, libFuzzer, honggfuzz)

Packers and Protectors

UPX compresses executables; machine code is decompressed at runtime. VMProtect and Themida convert machine code to a custom bytecode interpreted at runtime, making reverse engineering extremely difficult. Used legitimately (DRM) and maliciously (malware).

Spectre/Meltdown

Speculative execution side-channel attacks at the machine code level. The CPU speculatively executes machine code instructions that should not have been reached, and the cache effects of those speculative loads are measurable. The fundamental lesson: machine code execution has observable side effects even when the results are “discarded.”
