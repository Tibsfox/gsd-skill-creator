From Plugboards to CAFEBABE — The Stored-Program Concept and 80 Years of Executable Binary
Overview Encoding Execution Formats Microcode Security Rosetta Learn History
Before machine code, programs were hardware: rewired plugboards, rotary switches, punched sequences. The insight that a program is just data stored in memory — the stored-program concept — is the foundation of all modern computing.

Timeline

1936 Alan Turing publishes “On Computable Numbers” — the theoretical foundation. A universal machine that reads instructions from a tape.
1943 Colossus (Bletchley Park) — programmable but not stored-program. Configuration via plugboards and switches.
1945 ENIAC (UPenn, Eckert & Mauchly) — 18,000 vacuum tubes. Programmed by rewiring cables. No stored program.
1945 von Neumann’s “First Draft” on EDVAC — the stored-program concept. Instructions and data in the same memory. The birth of machine code as we know it.
Jun 1948 Manchester Baby (SSEM) — the FIRST machine to run a stored program. Tom Kilburn’s 17-instruction highest-proper-divisor program. 32-bit word, 32 words of Williams tube memory.
May 1949 EDSAC (Cambridge, Wilkes) — the first practical stored-program computer. Binary coded on 5-hole paper tape.
1951 UNIVAC I — first commercial computer. Machine code on magnetic tape.
1964 IBM System/360 — machine code standardized across a product family for the first time. The same binary runs on every S/360 model.
1978 Intel 8086 — the machine code format that would dominate for 47+ years (x86).
1985 ARM — Sophie Wilson’s instruction encoding: fixed 32-bit, the clean RISC alternative.
1999 ELF format standardized for Linux — the container for machine code on Unix systems.
2010 RISC-V — open-source instruction encoding. No licensing fees on the machine code format itself.
2017 WebAssembly 1.0 — a new “virtual machine code” for the browser, designed for safe execution of untrusted binary.
2018 Spectre/Meltdown — the discovery that machine code execution has observable side effects even when speculative results are “discarded.” Changed CPU design forever.

The Stored-Program Insight

Before 1948, a “program” was a physical configuration of the machine: plugboard connections (ENIAC), switch positions (Colossus), or cam patterns (Babbage’s Analytical Engine design). The stored-program concept — attributed to von Neumann, though Eckert/Mauchly and Turing had similar ideas — is that instructions are just numbers, stored in the same memory as data, and the CPU reads them sequentially.

This single insight made machine code possible: a program became a sequence of binary numbers, writable and modifiable as data. Self-modifying code, JIT compilation, and the entire software industry follow from this.
