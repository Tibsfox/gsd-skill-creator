Nobody learns machine code in isolation. You learn it by reading compiler output, by disassembling binaries, by stepping through code in a debugger, and by doing CTF challenges. The tools ARE the curriculum.

BookNotes

Tools as Curriculum

ToolWhat you learn
Compiler Explorer (godbolt.org)Write C/Rust/Go, see the machine code (shown as assembly). The bridge.
objdump -dDisassemble binaries. See hex bytes alongside assembly mnemonics.
xxd / hexdumpRaw hex dump of any file. See the actual bytes.
readelf -aELF structure: headers, sections, symbols, relocations.
GDB with layout asmStep through machine code one instruction at a time. See registers change.
GhidraReverse engineering: decompile machine code back to C-like pseudocode.
ImHexModern hex editor with pattern language for parsing binary formats.

A Learning Path

  1. Compiler Explorer: write int add(int a, int b) { return a + b; } in C. Read the assembly. Note the hex bytes on the left.
  2. gcc -c -o add.o add.c && objdump -d add.o — see the hex bytes that are the machine code
  3. xxd add.o | head -20 — see the raw bytes of the ELF file including the machine code
  4. CS:APP chapter 3 — systematic treatment of x86-64 machine-level programming
  5. A CTF challenge (pwn category) — forces you to understand machine code for exploitation
  6. Ghidra on a small binary — reverse engineering is the practical motivator
