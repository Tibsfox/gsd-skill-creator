Machine code is where all software converges. Every compiler, every JIT, every runtime eventually produces machine code. It is the universal output format.

The Complete Stack

  1. Source code (Python, Java, C++, Rust, Go, JavaScript, …)
  2. Intermediate representations (LLVM IR, Java bytecode, V8 bytecode, Wasm)
  3. Assembly language (human-readable) — ASM project
  4. Machine code (binary opcodes) — this project
  5. Microcode / micro-ops (CISC internal decomposition)
  6. Logic gates (AND, OR, NOT, XOR)
  7. Transistors (CMOS, FinFET, GAA)
  8. Physics (electron flow, quantum tunneling at 3nm)

WebAssembly as Virtual Machine Code

WebAssembly (.wasm) serves the same role as machine code — it’s the lowest-level target that developers compile to — but it is not real machine code. It is a portable, validated, sandboxed binary format that JIT compilers translate to real machine code (x86-64 or ARM64) at runtime. Wasm is “virtual machine code” in the same way that Java bytecode is “virtual machine code” for the JVM.
