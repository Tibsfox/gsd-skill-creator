Why Ada, and why concurrency as the focus? Ada is the only mainstream programming language that has had concurrency built into the language itself since its first standard in 1983. Four years before C++ had a standard and twelve years before Java existed, Ada already had tasks, rendezvous, and selective accept as first-class citizens. In 1995, Ada added protected objects — monitors with barrier synchronization that compile to efficient machine code while preserving semantic guarantees no library-based concurrency can match. Ada tasking is why fly-by-wire airliners are safe to fly. It is why driverless subway lines can run without collision. It is the quiet core of an enormous amount of the physical infrastructure you interact with every day.

The Safety-Criticality Ladder

Ada occupies a specific niche on the ladder of software assurance. Most languages sit on the first rung or two. Ada lives in the upper reaches:

  1. Scripting — Works most of the time. Good enough. Python, Bash, Perl. If it breaks, restart it.
  2. Application — Works reliably under normal conditions. Java, C#, Go. Exceptions are caught and logged.
  3. Systems — Works correctly on the metal. C, C++, Rust. Memory safety matters.
  4. Industrial real-time — Works within deadlines. Ada, real-time Java, C with RTOS. Timing is correctness.
  5. Safety-critical — Works or people die. Ada (DO-178C), MISRA C, SPARK. Mandatory certification.
  6. High-integrity — Proven correct. SPARK, Frama-C, seL4 (Isabelle/HOL). Zero undefined behavior.
  7. Formally verified — Proved, audited, certified to the highest assurance levels. CompCert, SPARK Platinum, seL4. Common Criteria EAL 6/7.

Ada plus SPARK is one of the few practical toolkits that covers rungs 4 through 7. Each rung costs more effort than the one below. Most software doesn’t need the higher rungs. When software does need them — airliners, trains, medical devices, nuclear control — Ada is where the professionals reach for.

Hello World, Ada Style

with Ada.Text_IO;

procedure Hello is
begin
   Ada.Text_IO.Put_Line ("Hello, world!");
end Hello;

Already more verbose than Python or C. The with clause names the package being used (like an import). The procedure name appears three times — once in the declaration, once after end, and (implicitly) as the file name. This is the point. Ada’s verbosity is not incidental; it is how the language ensures that every identifier is explicit, every dependency is visible, and every construct is scoped.

A Task Declaration

with Ada.Text_IO;

procedure Two_Tasks is
   task Printer;

   task body Printer is
   begin
      for I in 1 .. 5 loop
         Ada.Text_IO.Put_Line ("Hello from Printer" & Integer'Image (I));
         delay 0.1;
      end loop;
   end Printer;
begin
   for I in 1 .. 5 loop
      Ada.Text_IO.Put_Line ("Hello from Main" & Integer'Image (I));
      delay 0.1;
   end loop;
end Two_Tasks;

Declare a task, give it a body, and it starts running when the enclosing procedure begins. The procedure does not return until every task declared in its scope has finished — structured concurrency, decades before the term was coined. No thread pool, no library, no pthread_create. Just language primitives.

A Protected Object

protected Bounded_Counter is
   procedure Increment;
   entry Wait_For (N : Natural);
   function Current return Natural;
private
   Count : Natural := 0;
end Bounded_Counter;

protected body Bounded_Counter is
   procedure Increment is
   begin
      Count := Count + 1;
   end Increment;

   entry Wait_For (N : Natural) when Count >= N is
   begin
      null;  -- Caller unblocks when the barrier is true
   end Wait_For;

   function Current return Natural is
   begin
      return Count;
   end Current;
end Bounded_Counter;

A protected object is Ada’s answer to the question “what is a monitor, done right?” It has three kinds of operations: functions (read-only, multiple concurrent callers), procedures (exclusive writes), and entries (conditional, with a when barrier that callers queue behind). The language guarantees mutual exclusion, barrier semantics, and in the Real-Time Annex, ceiling-locked priority inheritance to prevent unbounded priority inversion. No library can give you this with comparable static guarantees.
