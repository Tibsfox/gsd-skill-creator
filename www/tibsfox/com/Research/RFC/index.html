On April 7, 1969, Steve Crocker wrote RFC 1 — seven pages describing host software for the ARPANET. He chose the name “Request for Comments” because he felt too junior to issue anything declarative. That modesty became an architectural decision: the entire Internet standards corpus inherits the posture that every document is provisional, every specification is an invitation to improve. More than 9,700 RFCs later, the conversation has never stopped. The series predates the IETF that now stewards it. Each RFC is immutable once published — if you made a mistake, you publish a new document and say so. No version drift. No ambiguity about which copy is authoritative. The Internet was built from small, precise, composable documents. This project maps the corpus, the process, and the toolchain for authoring new ones.

Research Modules

M1: RFC History & Governance

Governance Track A · Origins to present

RFC 1 (1969), Steve Crocker, ARPANET origins, Jon Postel and the RFC Editor, IETF formation (1986), governance architecture (ISOC/IAB/IESG/IRTF), rough consensus and running code, key RFCs that shaped the Internet, PNW early Internet connections.

M2: RFC Archive & Local Mirror

Archive Track A · Infrastructure

Corpus structure at rfc-editor.org, available formats (TXT/HTML/PDF/XML), rsync modules and bulk mirroring, rfc-index.xml parsing, local search indexing, cron maintenance, Obsoletes/Updates chains, storage planning.

M3: RFC Streams & Maturity Levels

Standards Track B · The standards track

Five publication streams (IETF/IAB/IRTF/Independent/Editorial), the standards track (Proposed Standard to Internet Standard), BCP subseries, Informational and Experimental status, STD numbering, stream selection decision tree.

M4: Authoring Tools & Workflow

Toolchain Track B · xml2rfc pipeline

xml2rfc v3 installation and output generation, RFCXML vocabulary (RFC 7991), idnits validation, author-tools.ietf.org web service, id2xml conversion, Markdown-to-RFC tools, GitHub integration and CI/CD builds.

M5: Internet-Draft Submission

Process Track C · Draft to RFC

I-D naming conventions, Datatracker submission workflow, 185-day expiry, Working Group engagement strategy, WG adoption, IETF Last Call, IESG review and ballot, AUTH48 final review, Independent Stream path.

M6: RFC Template & Style Guide

Literature Track C · Document craft

RFC 7322 required document structure, section-by-section guide, RFC 2119/8174 requirements language (MUST/SHOULD/MAY), boilerplate text by stream, reference management, IANA and Security Considerations, complete annotated xml2rfc v3 template, RFC as literature.
