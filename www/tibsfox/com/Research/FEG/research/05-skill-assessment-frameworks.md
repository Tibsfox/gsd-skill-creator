# Skill Assessment Frameworks

> **Domain:** Education Research & Competency Evaluation
> **Module:** 5 -- Assessment Design That Reveals Rather Than Conceals Gaps
> **Through-line:** *A test that passes everyone teaches you nothing. A test that fails everyone teaches you the wrong thing. Assessment is a measurement instrument. Its job is to produce an accurate signal about what the learner knows and does not know -- not to sort learners into bins, not to satisfy institutional requirements, but to reveal the truth. The gap-fill methodology depends on assessment that finds the seams. If the assessment conceals them, the gap-fill has nothing to work with.*

## 1. The Assessment Problem

Assessment in education serves multiple masters: diagnostic (what does the learner know?), formative (how is the learner progressing?), summative (has the learner achieved mastery?), and evaluative (is the program effective?). These purposes frequently conflict. An assessment designed to rank students (summative/evaluative) is poorly suited to identifying specific knowledge gaps (diagnostic). The FoxEdu Gap-Fill methodology requires assessment instruments optimized for gap detection [1, 2].

```
ASSESSMENT PURPOSE MATRIX
================================================================

  Purpose       Timing          Audience         FoxEdu Use
  ─────────     ──────          ────────         ──────────
  Diagnostic    Before learning  Learner + system Path construction
  Formative     During learning  Learner + mentor Pace adjustment
  Summative     After learning   Learner          Mastery certification
  Evaluative    Program-wide     Institution      Program improvement
```

The critical insight: most institutional assessment is summative (grades, certifications, degrees). The FoxEdu model prioritizes diagnostic and formative assessment because these directly feed the adaptive learning path (Module 4) and gap-fill content generation (Module 3) [1, 3].

> **SAFETY WARNING:** Assessment results that are used punitively (grade penalties, access denial, public ranking) suppress honest engagement. Learners who fear consequences for incorrect responses will avoid difficult questions, game the assessment, or disengage entirely. The FoxEdu assessment framework uses results exclusively for learning path optimization -- never for punishment, ranking, or gatekeeping beyond mastery verification [4].

## 2. Assessment Taxonomies

Multiple assessment classification systems exist in the education research literature. The FoxEdu framework draws primarily from Webb's DOK (1997), Bloom's revised taxonomy (Anderson & Krathwohl, 2001), and the competency-based education (CBE) movement's assessment typology [5, 6, 7].

### 2.1 Webb's Depth of Knowledge as Assessment Framework

| DOK Level | Assessment Type | Example Item (FoxEdu) | What It Reveals |
|-----------|----------------|----------------------|----------------|
| 1 - Recall | Fact retrieval | "What is the return period for a CSZ full-margin rupture?" | Whether the learner has read M06-A |
| 2 - Skill | Concept application | "Given this probability table, which hazard has the highest 50-year probability?" | Whether the learner can interpret the data |
| 3 - Strategic | Multi-step reasoning | "Compare Columbia Basin and coastal risk profiles for FoxEdu siting" | Whether the learner can synthesize M06-A data |
| 4 - Extended | Investigation | "Design a site selection matrix for a 90-bed FoxEdu campus using M06-A + M04-A data" | Whether the learner can create original analysis |

### 2.2 Assessment Type by Purpose

| Type | DOK Level | Diagnostic Value | Formative Value | Summative Value |
|------|-----------|-----------------|----------------|----------------|
| Multiple choice | 1-2 | Medium | Low | Medium |
| Short answer | 1-3 | Medium | Medium | Medium |
| Performance task | 2-4 | High | High | High |
| Portfolio | 3-4 | Low | High | High |
| Oral defense | 3-4 | High | Medium | High |
| Self-assessment | 1-4 | Medium | High | Low |

### 2.3 The DOK Alignment Rule

Assessment items must match the DOK level of the curriculum content they assess. A DOK 1 item (recall) cannot measure DOK 3 competency (strategic thinking). Webb's alignment methodology requires that at least 50% of assessment items match or exceed the DOK level of the corresponding curriculum standard [5, 8].

```
DOK ALIGNMENT CHECK
================================================================

  Curriculum DOK    Assessment DOK    Alignment?
  ──────────────    ──────────────    ──────────
  DOK 3             DOK 1             MISALIGNED -- item too easy
  DOK 3             DOK 2             MISALIGNED -- item below target
  DOK 3             DOK 3             ALIGNED
  DOK 3             DOK 4             ALIGNED -- above target is acceptable
  DOK 1             DOK 3             MISALIGNED -- item too hard for content
```

## 3. Competency-Based Assessment

Competency-Based Education (CBE) assesses what learners can do rather than what they have sat through. CBE emerged from vocational education (1960s), gained institutional traction through Western Governors University (1997), and was endorsed by the U.S. Department of Education for Title IV financial aid eligibility in 2013 [3, 9, 10].

### 3.1 CBE Principles

| Principle | Description | FoxEdu Application |
|-----------|------------|-------------------|
| Mastery-based | Advance on demonstration, not time | Mastery gates between modules |
| Transparent standards | Learners know exactly what is expected | Published rubrics for each module |
| Flexible pacing | Learn at own speed | Adaptive path adjusts timing |
| Multiple attempts | Failure is practice, not punishment | Re-assessment until mastery |
| Authentic assessment | Tasks mirror real-world application | Performance tasks from actual FoxEdu design problems |

### 3.2 Competency Definitions for FoxEdu Gap-Fill

Each module maps to specific competencies that the learner must demonstrate:

| Competency ID | Description | Module Source | DOK Level |
|--------------|-------------|--------------|-----------|
| FEG-C01 | Interpret PNW hazard probability tables | M06-A | 2 |
| FEG-C02 | Apply NIMS tier framework to emergency scenarios | M06-A | 3 |
| FEG-C03 | Describe Pacific Intertie topology and BPA grid structure | M06-B | 2 |
| FEG-C04 | Evaluate mesh communication redundancy for resilience | M06-B | 3 |
| FEG-C05 | Compare military housing cost models with amortization | M04-A | 3 |
| FEG-C06 | Analyze RCI funding mechanism for FoxEdu adaptation | M04-A | 3 |
| FEG-C07 | Evaluate Job Corps residential model as FoxEdu precedent | M04-B | 3 |
| FEG-C08 | Apply ANSI/OSHA housing standards to campus design | M04-B | 3 |
| FEG-C09 | Synthesize disaster risk + housing economics for site selection | Synthesis | 4 |
| FEG-C10 | Produce a board-ready FoxEdu campus financial summary | Synthesis | 4 |

### 3.3 Granularity: Competencies vs. Learning Objectives

| Level | Example | Grain Size | Assessment Method |
|-------|---------|-----------|------------------|
| Competency | "Evaluate Job Corps as FoxEdu precedent" | Broad | Performance task |
| Objective | "Compare cost per student per year across programs" | Medium | Short answer |
| Knowledge component | "Recall Job Corps annual enrollment figure" | Fine | Recognition item |

The adaptive system tracks at the knowledge component level (BKT parameters per component) but assesses at the competency level (performance tasks). This provides both the fine-grained tracking needed for path construction and the authentic assessment needed for mastery verification [6, 11].

## 4. Performance Tasks vs. Recognition Tests

The distinction between production tasks (generate an answer) and recognition tests (select an answer) is fundamental to assessment design that reveals gaps [1, 12].

### 4.1 Why Recognition Tests Conceal Gaps

Multiple-choice items allow correct responses through:
- Actual knowledge (the intended path)
- Test-wiseness (eliminating unlikely options)
- Guessing (25% probability on 4-option items)
- Recognition rather than recall (seeing the answer triggers memory without requiring generation)

A learner who selects "$180,000-$250,000" from options for military barracks cost may not be able to produce that figure in a board meeting. The recognition path and the production path are cognitively different [12, 13].

### 4.2 Performance Task Design

Performance tasks require the learner to produce a work product that demonstrates competency:

```
PERFORMANCE TASK: SITE SELECTION ANALYSIS
================================================================

  Context:
    You are preparing a site selection memo for the FoxEdu campus
    planning committee. Using M06-A hazard data and M04-A cost models:

  Task:
    1. Create a hazard comparison table for two candidate sites:
       (a) Columbia Basin (Grant County, WA)
       (b) South Puget Sound (Thurston County, WA)

    2. For each site, estimate total housing cost (build + 20-year ops)
       for a 90-bed campus using the modular prefab model.

    3. Write a 250-word recommendation with explicit reference to
       at least three hazard factors and two cost factors.

  Rubric:
    - Hazard table accurate and sourced (0-4 points)
    - Cost calculation correct with shown work (0-4 points)
    - Recommendation cites specific data (0-4 points)
    - Writing clarity and professionalism (0-3 points)
    Mastery threshold: >= 12/15 points
```

### 4.3 The Production-Recognition Spectrum

| Assessment Type | Production Demand | Diagnostic Value | Scalability |
|----------------|------------------|-----------------|------------|
| Multiple choice | None (recognition) | Low | High |
| Fill-in-the-blank | Low | Medium | High |
| Short answer | Medium | Medium | Medium |
| Calculation | Medium-high | High | Medium |
| Essay/memo | High | High | Low |
| Presentation/defense | Highest | Highest | Lowest |

The FoxEdu framework uses a mix: knowledge component tracking via short items (high volume, medium diagnostic value) and competency verification via performance tasks (lower volume, high diagnostic value) [1, 12].

## 5. Diagnostic Assessment for Gap Detection

Diagnostic assessment is the engine that drives the adaptive learning path. Its sole purpose is to accurately determine what the learner knows and does not know [1, 14].

### 5.1 Diagnostic Assessment Design Principles

| Principle | Rationale | Implementation |
|-----------|-----------|----------------|
| Cover the full domain | Gaps can exist anywhere | Items from every knowledge component |
| Minimize assessment time | Learners fatigue; time is finite | Adaptive item selection (KST-based) |
| Maximize information per item | Each response should update the knowledge model | Items chosen to maximally discriminate between knowledge states |
| No consequences | Accuracy requires honest engagement | Results used only for path construction |
| Immediate feedback | Learners learn from the diagnostic itself | Correct answer shown after each item |

### 5.2 KST-Based Diagnostic Assessment

Knowledge Space Theory provides a theoretically optimal diagnostic procedure. Rather than testing every concept independently, KST exploits the prerequisite structure to infer unmeasured concepts [15, 16].

```
KST DIAGNOSTIC PROCEDURE
================================================================

  Step 1: Start with median-difficulty concept
    Test: milcon-costs (medium prerequisite depth)

  Step 2: If correct, test concepts that have milcon-costs as prerequisite
    Test: rci-model OR modular-housing

  Step 3: If incorrect, test concepts that are prerequisites of milcon-costs
    Test: (milcon-costs has no prerequisites -- mark as unknown)

  Step 4: Use KST closure to infer:
    If milcon-costs = unknown, then rci-model, modular-housing,
    foxedu-mapping, site-selection are all unknown (by prerequisite)

  Result: One incorrect answer infers 4 additional unknowns.
          Assessment shortened by 4 items (~2 min saved).
```

### 5.3 Diagnostic Item Bank

For the FoxEdu Gap-Fill curriculum, a diagnostic item bank of 3-5 items per knowledge component provides sufficient coverage:

| Concept | Items | Avg. DOK | Avg. Time/Item |
|---------|-------|----------|---------------|
| hazard-prob | 4 | 1.5 | 45 sec |
| outage-tiers | 3 | 2.0 | 60 sec |
| network-topology | 4 | 1.5 | 45 sec |
| mesh-comm | 3 | 2.0 | 60 sec |
| energy-resilience | 3 | 2.5 | 90 sec |
| milcon-costs | 4 | 2.0 | 60 sec |
| rci-model | 3 | 2.5 | 75 sec |
| modular-housing | 3 | 2.0 | 60 sec |
| jobcorps-data | 4 | 2.0 | 60 sec |
| camp-models | 3 | 1.5 | 45 sec |
| foxedu-mapping | 4 | 3.0 | 120 sec |
| site-selection | 4 | 3.5 | 150 sec |
| **Total** | **42** | **2.1** | **~25 min** |

With KST-based adaptive item selection, the expected diagnostic length is 15-25 items (12-18 minutes), well within learner patience thresholds [15].

## 6. Formative Assessment and Feedback Loops

Formative assessment -- assessment embedded within the learning process that provides immediate feedback -- is the most powerful lever for learning improvement available to instructional design. Black and Wiliam's meta-analysis (1998) found effect sizes of 0.4-0.7 standard deviations for well-implemented formative assessment [17, 18].

### 6.1 Formative Assessment Principles

| Principle | Description | FoxEdu Implementation |
|-----------|------------|----------------------|
| Frequent | Every 10-15 minutes of instruction | Embedded check items in each module section |
| Low-stakes | No grade consequences | Results feed adaptive path only |
| Immediate feedback | Response within seconds | Correct answer + explanation shown immediately |
| Actionable | Feedback tells learner what to do next | "Review section 3.2" or "Try the calculation again" |
| Two-way | Both learner and system learn from the exchange | BKT updates + learner sees their progress |

### 6.2 Feedback Types

| Type | When | Example | Source |
|------|------|---------|--------|
| Verification | After response | "Correct" / "Incorrect" | System |
| Elaboration | After incorrect response | "The return period is 200-500 years, not 50-100" | System |
| Process | During task | "Try calculating build cost / amortization period first" | Mentor or system |
| Meta-cognitive | After task completion | "You consistently confuse annual and amortized costs" | System analysis |

### 6.3 The Formative-Diagnostic Loop

In the FoxEdu adaptive system, formative assessment data continuously updates the learner's knowledge model:

```
FORMATIVE-DIAGNOSTIC LOOP
================================================================

  Learner reads module section
       |
       v
  Embedded assessment item (formative)
       |
       v
  Learner responds
       |
       v
  BKT update (knowledge probability adjusted)
       |
       v
  Feedback provided
       |
       v
  Path adjustment if needed
       |       |
       v       v
  Next section  Review previous section
  (if P(L)>=0.8)  (if P(L)<0.5)
```

## 7. Rubric Design

Rubrics translate qualitative competency descriptions into measurable scoring criteria. For performance tasks (Module 4's mastery gates), well-designed rubrics are essential for reliable assessment [1, 19].

### 7.1 Rubric Types

| Type | Structure | Best For | Limitation |
|------|----------|----------|-----------|
| Holistic | Single score for overall quality | Quick assessment | Does not identify specific gaps |
| Analytic | Separate scores for each criterion | Diagnostic precision | More time to apply |
| Single-point | Describes expected performance only | Formative feedback | No graduated levels |

The FoxEdu framework uses analytic rubrics because they provide the diagnostic precision needed to identify which specific aspects of a competency the learner has not yet mastered [19, 20].

### 7.2 Analytic Rubric Template

For FoxEdu Gap-Fill performance tasks:

| Criterion | Exceeds (4) | Meets (3) | Approaching (2) | Beginning (1) |
|-----------|------------|-----------|-----------------|---------------|
| Data accuracy | All figures correct and sourced | Minor errors; all sourced | Some errors; most sourced | Major errors; unsourced |
| Analysis depth | DOK 4 synthesis present | DOK 3 analysis present | DOK 2 explanation only | DOK 1 recall only |
| Application | Novel FoxEdu-specific insight | FoxEdu mapping correct | Generic application | No application to FoxEdu |
| Communication | Board-ready professional quality | Clear and organized | Understandable but informal | Disorganized or unclear |

### 7.3 Inter-Rater Reliability

For rubrics applied by human assessors (mentors in the FoxEdu residential model), inter-rater reliability must be established through calibration sessions. Target: Cohen's kappa >= 0.70 (substantial agreement) [19, 21].

In practice, FoxEdu's small cohort size (60-120 learners) makes calibration feasible. With 3-5 mentors assessing performance tasks, a single calibration session per assessment cycle (reviewing 5-10 sample responses together) typically achieves the kappa target [21].

Integrating all assessment components into a coherent framework for the FoxEdu residential learning community [3, 5, 22].

### 8.1 Assessment Layers

```
FOXEDU ASSESSMENT ARCHITECTURE
================================================================

  LAYER 1: DIAGNOSTIC (Entry)
  ────────────────────────────
  KST-based adaptive diagnostic
  25-min max; identifies knowledge state
  Produces: initial learning path

  LAYER 2: FORMATIVE (Ongoing)
  ─────────────────────────────
  Embedded items in every module section
  Continuous BKT updates
  Produces: path adjustments, review scheduling

  LAYER 3: MASTERY GATES (Between Modules)
  ─────────────────────────────────────────

  Produces: advancement decisions

  LAYER 4: CAPSTONE (Synthesis)

  Produces: mastery certification for FoxEdu curriculum unit
```

## 9. Reliability and Validity

Assessment instruments must produce consistent results (reliability) that accurately measure the intended construct (validity) [1, 23].

### 9.1 Reliability Measures

| Measure | Meaning | Target | Measurement Method |
|---------|---------|--------|-------------------|
| Internal consistency (Cronbach's alpha) | Items within a test measure the same construct | >= 0.70 | Item-total correlation |
| Test-retest | Same learner gets similar score on different occasions | >= 0.80 | Repeat administration (2-week gap) |
| Inter-rater (Cohen's kappa) | Different raters give similar scores | >= 0.70 | Calibration sessions |
| Decision consistency | Same mastery/non-mastery decision on repeat | >= 0.85 | Classification agreement |

### 9.2 Validity Evidence

| Type | Question | Evidence for FoxEdu |
|------|---------|-------------------|
| Content validity | Do items cover the curriculum? | Coverage matrix mapping items to competencies |
| Construct validity | Do items measure the intended skill? | DOK alignment between items and competencies |
| Criterion validity | Do scores predict real-world performance? | Correlation with capstone project scores |
| Consequential validity | Are assessment consequences appropriate? | Results used for learning, not punishment |

### 9.3 Bias and Fairness

| Concern | Risk | Mitigation |
|---------|------|-----------|
| Cultural bias | Items assume specific cultural knowledge | Items reference FoxEdu-specific content, not cultural assumptions |
| Language bias | Complex language penalizes non-native speakers | Clear, direct language; technical terms defined |
| Prior knowledge bias | Items favor learners with specific backgrounds | Diagnostic assessment accounts for entry-level variation |
| Access bias | Technology requirements exclude some learners | Offline-capable assessment; paper fallback available |

> **SAFETY NOTE:** Assessment instruments in a FoxEdu context that serves Colville Confederated Tribes members must be reviewed for cultural appropriateness by tribal education authorities. This is not a generic diversity requirement -- it is a specific obligation to the specific sovereign nation whose territory the campus may occupy [22, 24].

## 10. Assessment in Residential Learning

The residential learning context changes assessment dynamics in ways that non-residential programs cannot replicate [25, 26].

### 10.1 Residential Assessment Advantages

| Feature | Non-Residential | Residential (FoxEdu) |
|---------|----------------|---------------------|
| Assessment timing | Scheduled class periods | Flexible; learner-chosen optimal times |
| Observation assessment | Limited to class time | 24/7 opportunities for authentic observation |
| Peer assessment | Coordination challenges | Natural peer interaction |
| Mentor access | Office hours only | On-site; informal assessment conversations |
| Practice environment | Home/dorm separation | Learning environment is living environment |

### 10.2 Authentic Assessment Opportunities

Residential living creates natural performance assessment contexts:

| Activity | Competency Assessed | Assessment Mode |
|----------|-------------------|----------------|
| Community meal planning | Budgeting, nutrition, logistics | Observation + product |
| Emergency drill | Disaster preparedness (M06-A) | Performance under conditions |
| Infrastructure maintenance | Building systems knowledge (M04) | Applied skill demonstration |
| Community governance meeting | Communication, analysis, synthesis | Oral participation |
| Peer tutoring | Depth of understanding (teaching = highest DOK) | Observation by mentor |

### 10.3 The Cohort Effect

In a 60-120 person residential cohort, assessment data from the group reveals curriculum-level patterns:

```
COHORT GAP DETECTION
================================================================

  If 70% of cohort fails the same item:
    -> Likely a curriculum gap (content not taught clearly)
    -> Trigger: remedial content review (Module 3)

  If 15% of cohort fails the same item:
    -> Likely individual knowledge gaps
    -> Trigger: adaptive path adjustment for those learners

  If item has high guess rate (many correct but low P(L)):
    -> Item may be poorly designed
    -> Trigger: item revision
```

## 11. The Verification Matrix as Assessment

The FoxEdu Gap-Fill verification matrix (from the mission specification) functions as a summative assessment of the gap-fill documents themselves. Each test in the matrix assesses whether the produced content meets its acceptance criteria [22, 27].

### 11.1 Verification Matrix as Rubric

The 40-test verification matrix maps directly to an analytic rubric for the gap-fill deliverables:

| Test Category | Count | Assessment of | Mastery Threshold |
|--------------|-------|--------------|-------------------|
| Safety-critical (SC-*) | 6 | Source quality, attribution, safety | 6/6 PASS required |
| Core functionality (CF-*) | 20 | Content completeness and accuracy | 18/20 PASS required |
| Integration (IN-*) | 10 | Cross-module coherence | 8/10 PASS required |
| Edge cases (EC-*) | 4 | Robustness | 2/4 PASS best-effort |

### 11.2 "The Gap Report"

The verification matrix produces a document called "The Gap Report" -- a summary of which success criteria are met, which are not, and what actions are required:

```
THE GAP REPORT -- VERIFICATION SUMMARY
================================================================

  Mission: FoxEdu Gap-Fill v1.0
  Date: [execution date]
  Modules: M06-A, M06-B, M04-A, M04-B + Synthesis Note

  SAFETY-CRITICAL:     _/6  PASS    [BLOCK if < 6]
  CORE FUNCTIONALITY:  _/20 PASS    [BLOCK if < 18]
  INTEGRATION:         _/10 PASS    [BLOCK if < 8]
  EDGE CASES:          _/4  PASS    [LOG only]

  OVERALL: _/40 PASS    SHIP / NO-SHIP decision

  Detailed results:
  SC-SRC: [PASS/FAIL] Source quality audit
  SC-NUM: [PASS/FAIL] Numerical attribution check
  SC-ADV: [PASS/FAIL] Policy advocacy scan
  SC-IND: [PASS/FAIL] Indigenous attribution review
  SC-SEC: [PASS/FAIL] Infrastructure security check
  SC-MOD: [PASS/FAIL] Module self-containment test
  ...
```

### 11.3 From Verification to Curriculum Assessment

The Gap Report pattern generalizes: any document produced by the GSD system can be assessed using the same verification matrix structure. The matrix is both a quality assurance tool (for documents) and an assessment framework (for the curriculum that produced them). If the documents fail, either the content generation process has a gap (Module 3) or the source material has a gap (Module 1). The verification matrix reveals which [22, 27].
