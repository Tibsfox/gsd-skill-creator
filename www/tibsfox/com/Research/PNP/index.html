Thompson and Ritchie gave us three words: file, process, pipe. From these three atoms, every piece of software running today was assembled. The pipe is the connective tissue of computing — the thing in the space between processes, carrying meaning from one conscious machine-fragment to another. The port is its network cousin: the numbered doorway that says this is where the outside comes in. Understanding these primitives at their specification level — the exact constants, the exact error codes, the exact atomicity guarantees — is not pedantry. It is the difference between an architecture that holds and one that fails silently at 2 AM when wave parallelism exceeds a kernel limit nobody wrote down.

Research Modules

M1: Pipe Mechanics

Kernel Track A · Anonymous pipes

pipe() and pipe2() syscall specification. Ring buffer internals, PIPE_BUF atomicity matrix, blocking/non-blocking behavior, O_DIRECT packet mode, splice/tee/vmsplice zero-copy operations, kernel tuning parameters.

M2: Named Pipes & FIFOs

Kernel Track A · Filesystem IPC

POSIX FIFO specification, mkfifo API, 7-state lifecycle with blocking behavior, blocking-on-open asymmetry. Windows named pipes 11-point contrast. Security considerations and TOCTOU race prevention.

M3: Port Taxonomy

Network Track B · IANA registry

Three IANA port ranges (RFC 6335). 23+ well-known port assignments. Ephemeral port kernel configuration. TIME_WAIT and port reuse (SO_REUSEADDR, SO_REUSEPORT). Port scanning methodology. Firewall rules by port.

M4: BSD Socket API

Network Track B · Socket lifecycle

POSIX.1-2024 socket interface. Server lifecycle (6 steps), client lifecycle (5 steps). sockaddr structs for IPv4/IPv6/Unix. 24-entry error code catalog. Unix domain sockets and SCM_RIGHTS fd passing.

M5: IPC Performance Spectrum

Performance Track C · Benchmarks

Four-mechanism throughput comparison at three block sizes. RTT latency profiles. Kernel path analysis (pipe_write vs tcp_sendmsg). Zero-copy paths. GSD IPC selection matrix. DACP atomicity audit.

M6: GSD Protocol Substrate

GSD Track C · Architecture

MCP transport mapping. Planning-bridge port policy. Subagent file descriptor budget. DACP bundle transport analysis. Cross-platform IPC strategy. Wave execution pipe architecture. The Amiga Principle at the pipe layer.
