Two TypeScript CLIs share the same gravity well. Caliber approaches the problem from the outside in: given any codebase, score its AI agent configuration quality, generate missing pieces, and keep everything synchronized as the code evolves. The gsd-skill-creator approaches from the inside out: given a developer's actual working patterns, observe what they do repeatedly, crystallize those patterns into reusable skills. They are not competitors. They are complements operating at different layers of the same stack. The integration lives in the spaces between them.

Research Modules

M1: Caliber Architecture Analysis

Scoring Track A · External system

Caliber's deterministic, LLM-free scoring engine. 100-point rubric across five categories: Existence, Quality, Grounding, Accuracy, Freshness. Session learning pipeline. Score regression guards. Structured fix data injection. Two-tier model system.

M2: GSD Skill-Creator Architecture

Architecture Track A · Internal system

DACP protocol three-part bundle structure. Skill lifecycle from detection through refinement. Bounded learning constraints: 20% cap, 3-correction minimum, 7-day cooldown. Module map of integration touch points across src/ directories.

M3: Integration Mapping

Bridge Track B · Port analysis

Three-tier port taxonomy: 6 direct ports (highest confidence), 5 adapted patterns (medium confidence), 3 GSD-native enhancements (new ground). Caliber TODOS roadmap analysis. Cross-repo skill registry patterns.

M4: Calibration Loops & Quality Metrics

Learning Track B · Feedback systems

Skill quality lifecycle: score, refine, re-score, gate. CALIBER_LEARNINGS taxonomy (6 categories). Session learning pipeline. Skill metrics JSONL schema. Trend analysis and ROI tracking. The self-improving skill ecosystem.

M5: Risk, Compatibility & The Bridge

Risk Track C · Synthesis

Scoring semantic mismatch. Token budget gating. PR #28 surgical partial merge analysis. Lock file race conditions. Compatibility wins: shared TypeScript/vitest/JSONL stack. The Amiga Principle: specialization as architectural strategy.
