Overview CUDA Streams LoRA Pipeline Dashboard Intent Router Hybrid Exec Lifecycle Mission Pack
The Amiga's custom chipset did not win because the 68000 was faster. It won because Agnus handled DMA, Denise handled display, and Paula handled audio — each operating with near-zero CPU involvement, each wired to the bus through documented register maps that programmers could reach directly. The GSD Silicon Layer makes this metaphor literal: skill-creator's observation pipeline maps to CUDA streams, patterns become kernel graphs, adapters become persistent SM partitions, and routing decisions become stream synchronization events. The chipset gets faster not because the hardware changes, but because the silicon becomes more specialized to the workload.

Research Modules

M1: CUDA Stream Orchestration

CUDA Track A · Silicon mapping

CUDA stream architecture for GSD's six pipeline stages. Kernel graphs and DAG execution. SM partitioning via persistent threads. MPS for concurrent contexts. Stream priority, preemption, and synchronization. RTX 4060 Ti profiling with Nsight Systems.

M2: LoRA Adapter Pipeline

LoRA Track B · Training pipeline

QLoRA distillation mechanics with NormalFloat4 quantization. Training pair extraction from session activity. Four-stage quality gate pipeline. VRAM budget management. Adapter hot-swap via llama-server API. Unsloth optimization on RTX 4060 Ti.

M3: CUDA/GL Dashboard

Dashboard Track C · Telemetry visualization

NVML telemetry pipeline via Rust nvml-wrapper. Filesystem event bus bridge (JSONL). WebGL shader pipeline for Denise panel. VRAM pressure heat map. Stream utilization display. Adapter activation timeline. Amiga register map parallel.

M4: Chipset Intent Router

Router Track A · Classification

DistilBERT-class intent classifier (CPU-resident, 67M params). Confidence scoring with Platt scaling calibration. Stream dispatch protocol. Two-stage gate: rule-based pre-filter plus learned classifier. Fallback chain design.

M5: Hybrid Execution Protocol

Hybrid Track B · Routing policy

Confidence model design with calibration requirements. Platt scaling and temperature scaling. Discrepancy detection via semantic similarity sampling. Continuous improvement loop. Three-signal degradation detection. Cost-benefit analysis.

M6: Adapter Lifecycle Governance

Governance Track C · Schema & policy

silicon.yaml v2 schema with lifecycle extensions. Adapter versioning (semver). Three-signal staleness detection. Community adapter quarantine pipeline. Integrity verification chain (SHA-256 + chipset.lock). VRAM eviction policy. Power management.
