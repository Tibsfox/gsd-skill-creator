Rust is named after a family of fungi that are, in Graydon Hoare’s words, “over-engineered for survival.” The name fits: the language is designed to produce software that outlasts its authors.

Origins (~2006–2010)

Graydon Hoare started Rust as a personal project around 2006 while working at Mozilla. The motivation was frustration with C and C++ — specifically, memory safety bugs in Firefox. The language drew from: ML and Haskell (type system, pattern matching, algebraic data types), Erlang (concurrency, message passing), C++ (systems-level control, zero-cost abstractions, RAII), Cyclone (safe C dialect, region-based memory), and Newsqueak/Limbo (channel-based concurrency).

Mozilla sponsored Rust development beginning in 2009. The Servo browser engine was started as a proving ground for the language.

Pre-1.0 — Dramatic Changes

Between 2010 and 2015, Rust changed more than most languages change in their entire lifetime. Features that were removed before 1.0:

  • Garbage collector — Rust originally had optional GC (@T managed pointers). Removed to commit fully to ownership.
  • Green threads — Rust had a green-thread runtime. Removed for the zero-cost principle (you can’t have a runtime if you target bare metal).
  • Typestate — an early experiment in encoding state machines in the type system. Too complex; removed.

Key people in this era: Patrick Walton, Niko Matsakis (borrow checker), Alex Crichton (Cargo, infrastructure), Brian Anderson (core team lead).

Rust 1.0 (May 15, 2015)

The stability guarantee: “stability without stagnation.” Once something is stable in Rust, it will never break. The 6-week release train (stable, beta, nightly) ships new features continuously while maintaining backward compatibility. Unstable features are gated behind nightly-only flags.

The Timeline

~2006 Graydon Hoare starts Rust as a personal project at Mozilla.
2009 Mozilla sponsors development. Servo browser engine work begins.
2010 Rust 0.1 — first public release.
~2013 Graydon Hoare steps away from the project. Community governance takes over.
May 15, 2015 Rust 1.0 — stability guarantee. 6-week release train begins.
Nov 2017 Firefox 57 Quantum ships with Servo’s Stylo CSS engine (Rust in production in a major browser).
Dec 2018 Rust 2018 Edition — Non-Lexical Lifetimes (NLL), module system reform, async foundations.
Nov 2019 Async/await stabilized (Rust 1.39).
Aug 2020 Mozilla layoffs — much of the Rust team is let go. Community concern about the language’s future.
Feb 2021 Rust Foundation formed (AWS, Google, Huawei, Microsoft, Mozilla). Ensures the language’s independence from any single company.
Oct 2021 Rust 2021 Edition — disjoint capture in closures, IntoIterator for arrays.
Mar 2021 Rust 1.51 — const generics (MVP) stabilized.
Nov 2022 Rust 1.65 — Generic Associated Types (GATs) stabilized. NSA guidance recommends Rust.
Dec 2022 Linux kernel 6.1 ships with Rust support — first kernel code in a language other than C and assembly in Linux’s 31-year history.
Feb 2024 White House ONCD report explicitly names Rust as a memory-safe language.
Feb 2025 Rust 2024 Edition (Rust 1.85). Gen blocks (preview), RPIT changes, unsafe improvements.

Editions

Rust’s edition system allows breaking syntax changes every 3 years while maintaining interoperability. Crates targeting different editions work together seamlessly.

EditionKey changes

The Safety Narrative

Since 2022, Rust has been named by three US government agencies as a recommended memory-safe alternative to C/C++:

  • NSA (November 2022): Software Memory Safety — names Rust, Go, Java, C#, Swift as memory-safe alternatives
  • CISA (2023): Shifting the Balance of Cybersecurity Risk — recommends migration to memory-safe languages
  • White House ONCD (February 2024): Back to the Building Blocks — explicitly names Rust as a memory-safe language for new development

This has accelerated adoption in critical infrastructure: the Linux kernel, Chromium, Android AOSP, Windows components, and AWS core services now include Rust code.

Governance

The Rust Foundation was formed in February 2021 with founding members AWS, Google, Huawei, Microsoft, and Mozilla. Executive Director: Rebecca Rumbul (since 2022). The Foundation handles infrastructure, trademark, and grants; it does not control the language design.

Language design is governed by teams: lang team (language features), compiler team, libs team, cargo team, moderation team. The 2022–2023 governance restructuring reorganized the core team and established a new leadership council.

Servo and Firefox

The Servo browser engine was Rust’s first major real-world test. Components that shipped in Firefox:

  • Stylo — the CSS engine (Firefox 57 Quantum, November 2017)
  • WebRender — GPU-accelerated compositing
  • URL parser

Servo was transferred to the Linux Foundation in 2020 after the Mozilla layoffs.
