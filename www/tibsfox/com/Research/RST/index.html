Why Rust? Rust is the language that answers the question every C and C++ programmer has asked for decades: “Can I have performance and safety?” The answer is yes — at the cost of a stricter compiler and a steeper learning curve. Rust’s ownership system and borrow checker eliminate use-after-free, double-free, dangling pointers, and data races at compile time, without a garbage collector, without a runtime, and without sacrificing performance. The result is a systems programming language that is as fast as C/C++ and as safe as Java, and has eaten into both of their territories: Linux kernel, Chromium, Android, AWS, Cloudflare, Discord, Figma, npm, and the entire modern build-tools ecosystem (esbuild, SWC, Turbopack, Biome, Rolldown).

The Three Promises

Memory Safety

No null pointers, no dangling references, no use-after-free, no double-free, no buffer overflows — all enforced at compile time by the ownership system and borrow checker. No garbage collector needed.

Fearless Concurrency

No data races in safe Rust. The type system (Send + Sync traits) proves at compile time that shared state is accessed correctly. Arc<Mutex<T>> is the pattern; the compiler catches everything else.

Zero-Cost Abstractions

Iterators, closures, generics, traits — all compile to the same machine code you’d write by hand. No runtime overhead. No virtual dispatch unless you ask for it (dyn Trait). The same promise as C++, but enforced.

No Runtime

No garbage collector. No VM. No green threads (since 1.0). Ahead-of-time compiled to native machine code via LLVM. Predictable performance. Suitable for kernels, firmware, and real-time systems.

The Timeline

~2006 Graydon Hoare starts Rust as a personal project. A fungus “over-engineered for survival.”
2009 Mozilla sponsors Rust development.
2010 First public release (Rust 0.1). Servo browser engine work begins.
May 2015 Rust 1.0 — stability guarantee. 6-week release train begins.
Nov 2017 Firefox 57 Quantum ships with Servo’s Stylo CSS engine (Rust in production in a browser).
Dec 2018 Rust 2018 Edition — NLL, module reform, async foundations.
Nov 2019 Async/await stabilized (Rust 1.39).
Aug 2020 Mozilla layoffs — much of the Rust team is let go. Community concern about governance.
Feb 2021 Rust Foundation formed (AWS, Google, Huawei, Microsoft, Mozilla).
Oct 2021 Rust 2021 Edition — disjoint capture, IntoIterator for arrays.
Dec 2022 Linux kernel 6.1 ships with Rust support (first kernel code in a language other than C and assembly).
Nov 2022 NSA guidance recommends Rust as a memory-safe alternative to C/C++.
Feb 2024 White House ONCD report names Rust explicitly as a memory-safe language.
Feb 2025 Rust 2024 Edition (Rust 1.85).
