Rust’s type system is where the power lives. Traits replace inheritance. Enums replace null. Result replaces exceptions. Pattern matching replaces type-checking if chains. Every abstraction compiles to the same code you’d write by hand.

Traits — Rust’s Answer to Interfaces

A trait defines shared behavior. Any type can implement any trait (subject to the orphan rule: either the trait or the type must be in your crate).

trait Summary {
    fn summarize(&self) -> String;

    // Default method
    fn preview(&self) -> String {
        format!("{}...", &self.summarize()[..20])
    }
}

struct Article { title: String, author: String, content: String }

impl Summary for Article {
    fn summarize(&self) -> String {
        format!("{} by {}", self.title, self.author)
    }
}

Trait bounds

// Function accepts any type implementing Display + Clone
fn print_twice<T: std::fmt::Display + Clone>(item: T) {
    let copy = item.clone();
    println!("{} {}", item, copy);
}

// Equivalent with where clause (cleaner for complex bounds)
fn print_twice<T>(item: T)
where
    T: std::fmt::Display + Clone,
{
    let copy = item.clone();
    println!("{} {}", item, copy);
}

impl Trait

// In argument position: sugar for a generic with a trait bound
fn notify(item: &impl Summary) {
    println!("Breaking: {}", item.summarize());
}

// In return position: the function returns "some type that implements Summary"
fn make_summary() -> impl Summary {
    Article { title: "...".into(), author: "...".into(), content: "...".into() }
}

Dynamic dispatch (dyn Trait)

// Trait object: runtime dispatch via vtable (like C++ virtual)
fn print_summary(item: &dyn Summary) {
    println!("{}", item.summarize());
}

// Box<dyn Trait> for owned trait objects
fn get_summarizer() -> Box<dyn Summary> {
    Box::new(Article { /* ... */ })
}

Generics

Rust generics are monomorphized: the compiler generates a specialized version for each concrete type used, exactly like C++ templates. Zero runtime cost. No type erasure (unlike Java).

struct Point<T> {
    x: T,
    y: T,
}

impl<T: std::ops::Add<Output = T> + Copy> Point<T> {
    fn sum(&self) -> T {
        self.x + self.y
    }
}

let p = Point { x: 3.0_f64, y: 4.0 };
println!("{}", p.sum()); // 7.0

Const generics (stable since 1.51)

struct Matrix<const ROWS: usize, const COLS: usize> {
    data: [[f64; COLS]; ROWS],
}

impl<const R: usize, const C: usize> Matrix<R, C> {
    fn zero() -> Self {
        Matrix { data: [[0.0; C]; R] }
    }
}

let m: Matrix<3, 4> = Matrix::zero();

Enums — Algebraic Data Types

Rust enums are sum types: each variant can carry different data. Combined with match, they replace inheritance hierarchies, visitor patterns, and null checks.

enum Shape {
    Circle { radius: f64 },
    Rectangle { width: f64, height: f64 },
    Triangle { base: f64, height: f64 },
}

fn area(shape: &Shape) -> f64 {
    match shape {
        Shape::Circle { radius } => std::f64::consts::PI * radius * radius,
        Shape::Rectangle { width, height } => width * height,
        Shape::Triangle { base, height } => 0.5 * base * height,
    }
}

Option and Result

Rust has no null. Instead:

TypeVariantsUse for
Option<T>Some(T) / NoneValues that might not exist
Result<T, E>Ok(T) / Err(E)Operations that can fail
 
// Option: no null
fn find_user(id: u64) -> Option<User> {
    users.iter().find(|u| u.id == id).cloned()
}

if let Some(user) = find_user(42) {
    println!("Found: {}", user.name);
}

// Result: no exceptions
fn read_config(path: &str) -> Result<Config, ConfigError> {
    let content = std::fs::read_to_string(path)?; // ? propagates errors
    let config: Config = toml::from_str(&content)?;
    Ok(config)
}

The ? operator

The ? operator is Rust’s answer to exceptions: it propagates errors up the call stack without boilerplate. If the Result is Ok, the value is unwrapped. If Err, the function returns early with that error (converting it via From if needed).

Error libraries

  • thiserror — derive macro for custom error types with Display and From
  • anyhow — boxed errors for applications (when you don’t need typed errors)

Pattern Matching

match value {
    0 => println!("zero"),
    1..=9 => println!("single digit"),
    n if n < 0 => println!("negative: {}", n),
    _ => println!("something else"),
}

// if let for single-pattern matching
if let Some(x) = maybe_value {
    println!("got: {}", x);
}

// let else (Rust 1.65) for early returns
let Some(x) = maybe_value else {
    return Err("missing value".into());
};

Box<T>Single owner, heapIf T: SendHeap allocation, recursive types, trait objects

Arc<T>Atomic ref countedYesShared ownership across threads

use std::sync::{Arc, Mutex};

let counter = Arc::new(Mutex::new(0));
let handles: Vec<_> = (0..10).map(|_| {
    let counter = Arc::clone(&counter);
    std::thread::spawn(move || {
        let mut num = counter.lock().unwrap();
        *num += 1;
    })
}).collect();

for h in handles { h.join().unwrap(); }
println!("Result: {}", *counter.lock().unwrap()); // 10

Iterators

Rust iterators are zero-cost: the compiler optimizes iterator chains into the same machine code as hand-written loops.

let sum: i32 = (1..=100)
    .filter(|n| n % 2 == 0)
    .map(|n| n * n)
    .sum();

let names: Vec<String> = users.iter()
    .filter(|u| u.active)
    .map(|u| u.name.clone())
    .collect();

String handling in Rust is “hard” because Rust enforces UTF-8 correctness. You cannot index into a String by byte position and get a character — because Unicode characters can be multi-byte. This catches bugs that every other language silently allows.

Derive Macros

#[derive(Debug, Clone, PartialEq, Eq, Hash)]
struct Point {
    x: i32,
    y: i32,
}

// Automatically gets:
// - Debug: {:?} formatting
// - Clone: .clone() method
// - PartialEq + Eq: == comparison
// - Hash: can be used as HashMap key
