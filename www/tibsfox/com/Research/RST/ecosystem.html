Cargo is the best package manager in systems programming. The ecosystem it enables — 150K+ crates on crates.io — is the largest argument for Rust after the language itself.

Cargo

Cargo is Rust’s build system and package manager rolled into one. It compiles code, downloads dependencies, builds libraries, runs tests, generates documentation, publishes crates, and manages workspaces. Shipped with Rust 1.0 in May 2015.

[package]
name = "my-service"
version = "0.1.0"
edition = "2024"

[dependencies]
serde = { version = "1.0", features = ["derive"] }
tokio = { version = "1", features = ["full"] }
axum = "0.8"
tracing = "0.1"
anyhow = "1.0"

[dev-dependencies]
criterion = { version = "0.5", features = ["html_reports"] }

[profile.release]
opt-level = 3
lto = true
codegen-units = 1
strip = true

// axum handler
use axum::{routing::get, Router, Json, extract::Path};
use serde::Serialize;

#[derive(Serialize)]
struct User { id: u64, name: String }

async fn get_user(Path(id): Path<u64>) -> Json<User> {
    Json(User { id, name: format!("user_{}", id) })
}

#[tokio::main]
async fn main() {
    let app = Router::new().route("/users/:id", get(get_user));
    let listener = tokio::net::TcpListener::bind("0.0.0.0:3000").await.unwrap();
    axum::serve(listener, app).await.unwrap();
}

Embedded Rust

Rust’s no_std mode strips the standard library to a bare-metal-compatible subset. The Embedded Working Group maintains:

  • embedded-hal — hardware abstraction layer traits (SPI, I2C, GPIO, UART, timers)
  • cortex-m — ARM Cortex-M support
  • esp-hal — Espressif ESP32 support
  • embassy — async embedded framework (async tasks on microcontrollers)
  • probe-rs — flash and debug embedded devices

WebAssembly (Wasm)

Rust is the premier language for compiling to WebAssembly:

  • wasm-pack — build pipeline: Rust → Wasm + JS bindings
  • wasm-bindgen — bridge between Rust and JavaScript
  • web-sys / js-sys — bindings to Web APIs and JS built-ins
  • Yew — Elm-like frontend framework in Rust/Wasm
  • Leptos — fine-grained reactive framework (signals)
  • Dioxus — React-like framework in Rust
  • Wasmtime (Bytecode Alliance) — Wasm runtime outside the browser
  • WASI — WebAssembly System Interface for server-side Wasm

Testing

Testing is built into the language and Cargo:

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn test_add() {
        assert_eq!(add(2, 3), 5);
    }

    #[test]
    #[should_panic(expected = "overflow")]
    fn test_overflow() {
        add(u32::MAX, 1);
    }
}

/// Adds two numbers.
///
/// ```
/// assert_eq!(my_crate::add(2, 3), 5);
/// ```
pub fn add(a: u32, b: u32) -> u32 {
    a.checked_add(b).expect("overflow")
}

Doc tests (examples in /// doc comments) are compiled and run as tests. Integration tests live in tests/. Benchmarks use criterion (stable) or #[bench] (nightly). Property testing via proptest or quickcheck.

Cross-Compilation

# Add a target
rustup target add aarch64-unknown-linux-gnu

# Build for that target
cargo build --target aarch64-unknown-linux-gnu --release

# Or use `cross` for Docker-based cross-compilation
cargo install cross
cross build --target aarch64-unknown-linux-musl --release

Rust supports dozens of targets: Linux, Windows, macOS, iOS, Android, WebAssembly, bare-metal ARM (Cortex-M, Cortex-A), RISC-V, and more.
