Era 1 — Machine & Assembly (1940s–1950s)

Before there were programming languages, there were switch positions and punch cards. The first programmers spoke directly to hardware — every bit placed by hand, every address calculated in the programmer's head. Understanding this era is understanding what a computer actually does when it executes your code.

Machine Language

Hardware Systems

Binary opcodes executed directly by the CPU. No translation, no abstraction — raw instructions the hardware understands. Von Neumann architecture: instructions and data share the same memory. Every language you have ever used compiles or interprets down to this.

Each CPU family has its own machine language. An x86 instruction looks nothing like ARM. The fetch-decode-execute cycle runs billions of times per second. Pipelining, branch prediction, out-of-order execution — the CPU is optimizing your machine code in real time.

1940s · direct hardware control · opcodes + operands · Von Neumann / Harvard architectures
Connects to: T55 (logic gates that make CPUs possible), LED (binary is voltage high/low), MPC (GPU machine code = SASS/PTX)

Assembly Language

Hardware Systems

Human-readable mnemonics for machine instructions. MOV, ADD, JMP, CMP — one line of assembly typically maps to one machine instruction. Registers are named (EAX, RSP, R0). Addressing modes let you reach into memory. The assembler translates to binary.

Still used today: bootloaders, OS kernels, cryptographic primitives, exploit development, reverse engineering. If you want to know why your code is slow, read the assembly output. Three major dialects dominate: x86-64 (Intel/AMD desktops), ARM (phones, Apple Silicon, Raspberry Pi), RISC-V (open-source, rising fast).

1950s · mnemonics · x86 / ARM / RISC-V / MIPS · assembler → machine code
Connects to: T55 (registers = flip-flop storage), EMG (interrupt-driven motor control), GPO (GPU assembly = PTX for CUDA)

Why the Metal Matters

Insight

Every abstraction has a cost. Every high-level feature compiles to sequences of loads, stores, branches, and arithmetic. Understanding the hardware means understanding why hash maps are O(1), why cache misses are expensive, why branch misprediction slows loops, and why memory alignment matters.

Our electronics research (LED controllers, 555 timers, sensor circuits) builds the intuition for how digital logic works at the transistor level. A CPU is millions of logic gates. A logic gate is a few transistors. A transistor is a voltage-controlled switch. It is switches all the way down.

abstraction cost · cache lines · branch prediction · SIMD · memory hierarchy
Connects to: LED (digital logic in hardware), T55 (timing circuits = clock generation), BPS (sensors = ADC = binary conversion)

Era 2 — The Foundational Languages (1950s–1970s)

The era that invented the idea of a programming language. Each of these languages introduced concepts that every language since has inherited, debated, or deliberately rejected. Seven languages that changed everything — most of them still in active production use.

FORTRAN

Language HPC

1957 — John Backus, IBM. The first high-level programming language. Created so scientists could write formulas instead of machine code. "FORmula TRANslation" — the name tells you everything.

Still the language of high-performance computing. CERN uses Fortran. Weather models run in Fortran. LAPACK and BLAS (the linear algebra libraries that NumPy calls under the hood) are Fortran. When you need every FLOP the CPU can give, Fortran's compiler optimizations remain unmatched for array-heavy numerical code.

1957 · John Backus, IBM · compiled · array-oriented · column-major memory layout
Connects to: CERN/WLCG (Monte Carlo simulations), MPC (numerical computing roots), Julia (spiritual successor for scientific computing)

LISP

Language AI Functional

1958 — John McCarthy, MIT. The original AI language. Code and data have the same structure (homoiconicity) — programs can rewrite themselves. S-expressions: (+ 1 2). Macros that generate code at compile time. Garbage collection was invented here.

Every functional programming concept traces back to LISP: first-class functions, closures, recursion as the primary control flow, immutable data. Emacs runs on Elisp. Clojure runs on the JVM. Paul Graham wrote the first web app (Viaweb/Yahoo Store) in LISP. The language that makes you think differently.

1958 · John McCarthy, MIT · homoiconic · macros · garbage collection · S-expressions
Connects to: OPEN (AI foundations), Unison (content-addressed = LISP's "code is data" taken further), Elixir (macro system lineage)

COBOL

Language Enterprise

1959 — Grace Hopper and the CODASYL committee. Designed to be readable by business managers. English-like syntax: MOVE AMOUNT TO TOTAL. The Department of Defense mandated it. Banks adopted it. Government adopted it.

An estimated 220 billion lines of COBOL are in production today. 95% of ATM transactions, 80% of in-person transactions touch COBOL. The IRS, Social Security, major banks — all COBOL. There are more COBOL transactions per day than Google searches. The language the world runs on, whether it admits it or not.

1959 · Grace Hopper / CODASYL · English-like syntax · 220B+ lines in production
Connects to: SQL (both declarative and business-data oriented), SYS (mainframe operations = systems administration's ancestor)

ALGOL

Language Paradigm

1960 — International committee (Backus, Naur, et al.). The language that influenced everything but was used by almost no one directly. Introduced: block structure, nested scopes, BNF grammar notation (Backus-Naur Form), structured programming.

ALGOL's syntax became the template for C, Pascal, Java, JavaScript, Python, Rust, Go, and virtually every imperative language. Its block-scoping rules are in every language you use. BNF notation is how we formally define every language's grammar. ALGOL did not win the market. It won the future.

1960 · Backus, Naur, et al. · block structure · BNF notation · lexical scoping · structured programming
Connects to: every language in Era 3-5 inherited ALGOL's syntax structure

C

Language Systems

1972 — Dennis Ritchie, Bell Labs. Created to write Unix. "Portable assembly" — close enough to hardware for an OS, abstract enough to compile on different machines. Pointers, manual memory management, minimal runtime, predictable performance.

The language that built the modern world. Linux kernel: C. Windows kernel: C. macOS kernel: C. PostgreSQL: C. SQLite: C. Python interpreter: C. Ruby interpreter: C. PHP interpreter: C. Nearly every language runtime you use is written in C. The most important language ever created, by influence and by the infrastructure that depends on it.

1972 · Dennis Ritchie, Bell Labs · compiled · manual memory · pointers · Unix · K&R
Connects to: Rust (C's memory model, made safe), SYS (Unix = C), our Tauri backend (Rust started as "C but safe")

Smalltalk

Language OOP

1972 — Alan Kay, Xerox PARC. "The best way to predict the future is to invent it." Objects all the way down — even numbers are objects, even true is an object. Message passing instead of function calls. The GUI was invented here. The mouse was invented here. Overlapping windows, copy-paste, undo.

Alan Kay's vision: a computer for children to learn with (the Dynabook, 1968 — he predicted the tablet 42 years before the iPad). Smalltalk's message-passing model influenced Objective-C, Ruby, Erlang, and the actor model. The IDE was invented in Smalltalk. Refactoring tools were invented in Smalltalk.

1972 · Alan Kay, Xerox PARC · message passing · GUI origin · Dynabook vision · image-based
Connects to: Ruby (Matz explicitly borrowed from Smalltalk), Erlang (actors = message passing), our agent system (message-passing patterns)

SQL

Language Data

1974 — Donald Chamberlin and Raymond Boyce, IBM. Based on Edgar Codd's relational algebra (1970). Declarative: you say what data you want, not how to get it. The query optimizer figures out the execution plan. SELECT * FROM missions WHERE status = 'active'.

50 years later, SQL is more dominant than ever. PostgreSQL, MySQL, SQLite, SQL Server, Oracle. Every relational database speaks SQL. NoSQL came and went; SQL won. Our pgvector database (foxy@localhost, schema artemis) adds vector similarity search to PostgreSQL — SQL extended for the embedding era.

1974 · Chamberlin & Boyce, IBM · declarative · relational algebra · Codd's 12 rules
Connects to: pgvector (SQL + embeddings), WLCG (ROOT = columnar, SQL-like queries), our artemis schema (6 tables, 1,087 pages)

Era 3 — Object-Oriented & Scripting (1980s–1990s)

The era of explosive growth. Object-oriented programming went mainstream. Scripting languages made programming accessible to non-specialists. The Web was born, and with it, three languages that would come to dominate everything: Java, JavaScript, and Python.

C++

Language Systems

1985 — Bjarne Stroustrup, Bell Labs. "C with Classes" became C++ — object-oriented programming bolted onto C. Templates, RAII, operator overloading, multiple inheritance. Zero-cost abstractions: high-level code that compiles to assembly as efficient as hand-written C.

Game engines (Unreal, Unity's native layer), browsers (Chrome, Firefox), databases (MySQL, MongoDB), compilers (Clang/LLVM), embedded systems, trading systems. When you need performance AND abstraction, C++ is the answer. The complexity cost is real — Rust exists because of C++'s footguns.

1985 · Bjarne Stroustrup, Bell Labs · compiled · RAII · templates · zero-cost abstractions
Connects to: Rust (ownership = RAII formalized), our desktop app (Tauri chose Rust over C++ for the same reasons)

Perl

Language Web

1987 — Larry Wall. "Practical Extraction and Report Language" (or "Pathologically Eclectic Rubbish Lister" — Larry doesn't mind either). The regex king. CPAN (175,000+ modules) was the first great package ecosystem. CGI scripts powered the early Web.

"There's more than one way to do it." Perl's philosophy of expressiveness over consistency. One-liners that do in 20 characters what takes 20 lines elsewhere. Text processing, system administration, bioinformatics (BioPerl). The language of the 1990s web, and still running the scripts that glue infrastructure together.

1987 · Larry Wall · regex · CPAN · CGI era · TMTOWTDI · sigils ($@%)
Connects to: Python (often positioned as "Perl, but readable"), SYS (sysadmin scripting), Ruby (borrows from Perl's expressiveness)

Python

Language AI/ML

1991 — Guido van Rossum. Named after Monty Python. Readability as a design principle — significant whitespace forces clean formatting. "Batteries included" standard library. "There should be one obvious way to do it."

The dominant language for machine learning (PyTorch, TensorFlow, scikit-learn), data science (Pandas, NumPy, Jupyter), automation, scripting, web (Django, Flask), and education. Our sweep.py runs hourly mission updates. pic2html.py recreates a 2007 tool. When you need something working in an hour, you reach for Python.

1991 · Guido van Rossum · dynamic typing · indentation syntax · PyPI · CPython (written in C)
Connects to: sweep.py (hourly automation), MPC (numpy/scipy backends), OPEN (ML research), Julia (Python's speed problem solved)

Ruby

Language Web

1995 — Yukihiro "Matz" Matsumoto. "Optimized for programmer happiness." Everything is an object (borrowed from Smalltalk). Blocks and closures everywhere. Metaprogramming as a feature, not an escape hatch. 5.times { puts "hello" } reads like English.

Ruby on Rails (2004, DHH) changed web development forever. Convention over configuration. MVC as standard. Scaffolding, migrations, ActiveRecord. GitHub is Rails. Shopify is Rails. Basecamp is Rails. The language that proved developer productivity can be a competitive advantage.

1995 · Yukihiro Matsumoto · blocks · mixins · open classes · RubyGems · Rails
Connects to: Smalltalk (direct philosophical ancestor), Elixir (Ruby syntax, Erlang semantics), our skill system (DSL-like conventions)

Java

Language Enterprise

1995 — James Gosling, Sun Microsystems. "Write once, run anywhere." The JVM compiles to bytecode that runs on any platform. Garbage collection, strict typing, interfaces, checked exceptions. Designed for reliability at enterprise scale.

Android's primary language (via Dalvik/ART). Enterprise backends everywhere. Hadoop, Kafka, Elasticsearch, Cassandra — the Big Data ecosystem is Java. Spring Boot powers millions of microservices. The JVM itself is a masterpiece: JIT compilation, escape analysis, garbage collectors that can handle terabytes of heap. 30 years old and still the #1 enterprise language.

1995 · James Gosling, Sun · JVM bytecode · GC · interfaces · Spring · Maven/Gradle
Connects to: Kotlin (modern Java), Scala (functional Java), Clojure (LISP on JVM), WLCG (some DIRAC components)

JavaScript

Language Web

1995 — Brendan Eich, Netscape. Written in 10 days. The language of the Web — every browser runs it. Originally just for form validation, now: frontend (React, Vue, Svelte), backend (Node.js), desktop (Electron, Tauri webview), mobile (React Native). The most deployed runtime in history.

Prototypal inheritance (not classical OOP). First-class functions. Event loop and async/await. JSON (JavaScript Object Notation) became the universal data format. npm is the largest package registry ever (2M+ packages). ES6+ transformed it from a toy language into a real one. TypeScript adds types. Our entire research site is hand-crafted JavaScript.

1995 · Brendan Eich, Netscape · prototype-based · event loop · V8 engine · npm · ES6+
Connects to: TypeScript (our 293K lines), our desktop app (Vite webview), p5.js (forest sim), every page on tibsfox.com

PHP

Language Web

1995 — Rasmus Lerdorf. "Personal Home Page" tools became a full language. Embedded in HTML: <?php echo "hello"; ?>. The Web's workhorse. Easy deployment (just upload .php files), shared hosting everywhere, MySQL integration out of the box.

WordPress (43% of all websites), Wikipedia (MediaWiki), Facebook (Hack/HHVM is PHP-derived), Laravel, Drupal, Magento. PHP gets mocked by developers who have never built a product that serves millions. Modern PHP (8.x) has types, enums, fibers, JIT compilation. The language survived every attempt to replace it.

1995 · Rasmus Lerdorf · server-side · embedded in HTML · WordPress · Composer
Connects to: our static site approach (we chose HTML over PHP for zero-dependency hosting), SYS (LAMP stack = Linux+Apache+MySQL+PHP)

Era 4 — Modern Systems & Functional (2000s–2010s)

A generation of languages designed to fix the problems of the previous generation. Memory safety without garbage collection. Concurrency without data races. Types without verbosity. The era where language design became a mature engineering discipline.

C#

Language Enterprise

2000 — Anders Hejlsberg, Microsoft. Created for .NET as Microsoft's answer to Java. Properties, events, delegates, LINQ (SQL-like queries in code), async/await (C# had it before JavaScript). Evolved from "Java clone" to one of the most feature-rich languages.

Unity game engine (the most popular game engine by project count). Enterprise backends via ASP.NET. Cross-platform via .NET Core (now just .NET). Xamarin for mobile. Blazor for WebAssembly. Anders Hejlsberg also designed TypeScript — the same mind shaped two of the most important modern languages.

2000 · Anders Hejlsberg, Microsoft · .NET / CLR · LINQ · async/await origin · Unity
Connects to: TypeScript (same designer: Anders Hejlsberg), Java (direct competitor), Rust (both have pattern matching and enums)

Go

Language Cloud

2009 — Rob Pike, Ken Thompson, Robert Griesemer, Google. Designed for Google-scale infrastructure. Goroutines (lightweight threads, 2KB stack) and channels (typed pipes between goroutines). Compiles to a single binary. No inheritance. No generics (until 1.18). Aggressively simple.

Docker: Go. Kubernetes: Go. Terraform: Go. etcd: Go. Prometheus: Go. The entire cloud-native infrastructure stack is written in Go. Fast compilation (the whole standard library in seconds). Built-in formatting (gofmt — no style debates). The language that said "less is more" and meant it.

2009 · Pike, Thompson, Griesemer, Google · goroutines · channels · single binary · gofmt
Connects to: K8S (written in Go), SYS (infrastructure tooling), our GUPP protocol (Go's concurrency model informs agent patterns)

Rust

Language Systems Safety

2010 — Graydon Hoare, Mozilla. Memory safety without garbage collection. The ownership system: every value has exactly one owner. Borrowing rules enforced at compile time. No null. No data races. If it compiles, it won't segfault.

Our Tauri backend is Rust. Firefox's Servo engine: Rust. The Linux kernel is adopting Rust. Cloudflare, Discord, Dropbox, AWS (Firecracker) — all using Rust for performance-critical systems. Voted "most loved language" 8 years running. The language that proved you don't have to choose between safety and speed.

2010 · Graydon Hoare, Mozilla · ownership · borrow checker · no GC · Result<T,E> · Cargo
Connects to: our Tauri desktop app (src-tauri/), C (what Rust replaces), WASM (Rust compiles to WebAssembly), GPO (GPU compute kernels)

Kotlin

Language Mobile

2011 — JetBrains. Modern Java that actually respects your time. Null safety built in (String? vs String). Data classes, extension functions, coroutines, sealed classes. 100% interop with Java — call any Java library, use any Java framework.

Google's preferred language for Android (2019). Multiplatform: share code between Android, iOS, web, and desktop. Spring Boot supports Kotlin first-class. Gradle build scripts in Kotlin (replacing Groovy). The language that showed you can fix Java without breaking Java's ecosystem.

2011 · JetBrains · JVM · null safety · coroutines · Android official · multiplatform
Connects to: Java (runs on JVM), TypeScript (similar "add types and modernize" approach to its base language), Swift (similar design philosophy)

TypeScript

Language Web Our Stack

2012 — Anders Hejlsberg, Microsoft. Static types layered onto JavaScript. Gradual typing: start loose, tighten as you go. Interfaces, generics, union types, mapped types, conditional types. The type system is Turing-complete (yes, you can compute at the type level).

Our primary language: 293,531 lines across 2,829 files. Skills, agents, chipsets, CLI, library code — all TypeScript. 21,298 tests via Vitest. The entire GSD orchestration system. Every skill, every command, every hook. The language we think in.

2012 · Anders Hejlsberg, Microsoft · gradual typing · tsc · .d.ts · 293K lines in our codebase
Connects to: our entire codebase (src/), JavaScript (compiles to JS), C# (same designer), Vite (build tool), Vitest (test runner)

Elixir

Language Functional Distributed

2012 — Jose Valim. Ruby-like syntax on the Erlang VM (BEAM). Pattern matching, pipe operator, immutable data, supervision trees. Processes are dirt cheap (millions per node). "Let it crash" philosophy — supervisors restart failed processes automatically.

Phoenix LiveView: real-time web without JavaScript. WhatsApp (Erlang, same VM) handled 2M connections per server. Discord uses Elixir for their real-time infrastructure. The BEAM VM was built by Ericsson for telephone switches that must never go down. Fault tolerance is not a feature — it is the foundation.

2012 · Jose Valim · BEAM VM · OTP · supervision trees · pattern matching · pipe operator
Connects to: our agent system (supervision = agent orchestration), CMH (distributed compute), Smalltalk (message passing ancestor via Erlang)

Swift

Language Apple

2014 — Chris Lattner, Apple. Replaces Objective-C for iOS/macOS development. Optionals for null safety. Protocol-oriented programming (protocols > inheritance). Value types (structs) preferred over reference types (classes). Playgrounds for interactive development.

All modern iOS and macOS apps are Swift. SwiftUI for declarative interfaces. Server-side Swift (Vapor) exists but is niche. Chris Lattner also created LLVM (the compiler infrastructure that Rust, C++, and dozens of other languages use) and later created Mojo. One person's influence spans three language eras.

2014 · Chris Lattner, Apple · optionals · protocols · value types · ARC · SwiftUI
Connects to: Rust (similar safety goals, different approach), Kotlin (parallel evolution for mobile), Mojo (same creator: Chris Lattner)

Era 5 — Domain-Specific & the AI Era (2010s–2026)

Languages designed for specific domains: GPUs, statistics, scientific computing, systems without hidden costs. And the emergence of natural language as a programming interface — where the line between "writing a prompt" and "writing a program" dissolves.

GLSL / HLSL

Language GPU

2004 / 2002 — OpenGL ARB / Microsoft. Shader languages that run on the GPU. GLSL for OpenGL/Vulkan, HLSL for DirectX. Each invocation runs on a different pixel or vertex simultaneously. The GPU does not loop through pixels — it runs all of them at once.

Our shader pipeline uses GLSL on the RTX 4060 Ti. Ray marching, signed distance functions, procedural textures, compute shaders. The GPU thinks in parallel: no branches, no pointers, no allocation. Write a function that transforms one pixel. The hardware runs it 2 million times per frame. A fundamentally different model of computation.

2002-2004 · vertex + fragment + compute shaders · massively parallel · SPIR-V intermediate
Connects to: our shaders/ directory, GPO (GPU orchestration), VAV (voxel rendering), MPC (compute shaders for math), forest sim (visual effects)

R

Language Statistics

1993 — Ross Ihaka and Robert Gentleman, University of Auckland. A language built by statisticians for statisticians. Vectorized operations, data frames, ggplot2 (the gold standard for statistical graphics), Bioconductor (genomics), CRAN (19,000+ packages).

Where Python dominates machine learning, R dominates statistical analysis, biostatistics, and academic research. The tidyverse (dplyr, tidyr, ggplot2) is the most elegant data manipulation API in any language. R Markdown and Shiny for reproducible research and interactive dashboards. The language that made data science visual.

1993 · Ihaka & Gentleman · vectorized · data frames · ggplot2 · CRAN · tidyverse
Connects to: Python (complementary in data science), Julia (R's speed problem solved), our weather data analysis pipeline

Julia

Language HPC Science

2012 — Jeff Bezanson, Stefan Karpinski, Viral Shah, Alan Edelman, MIT. "We want the speed of C with the dynamism of Ruby. We want a language that's homoiconic, with true macros like Lisp. We want something as usable for statistics as R, as natural for string processing as Perl."

Multiple dispatch as the core paradigm (not OOP, not functional — dispatch). JIT compilation via LLVM. Calls C and Fortran with zero overhead. Used at the Federal Reserve, BlackRock, NASA, and the climate modeling community. Solves the "two-language problem" — prototype in Julia, ship in Julia. No rewrite needed.

2012 · MIT · multiple dispatch · JIT/LLVM · zero-overhead C FFI · Pkg · DifferentialEquations.jl
Connects to: FORTRAN (same domain, modern approach), MPC (mathematical computing), CERN (particle physics simulations)

Zig

Language Systems

2016 — Andrew Kelley. A C replacement that does not hide anything. No hidden control flow, no hidden allocators, no hidden casts. Comptime: arbitrary code execution at compile time (generics without generics). Cross-compilation to 39 architectures out of the box. Drop-in C compiler replacement.

Bun (the JavaScript runtime) is written in Zig. TigerBeetle (distributed financial database) is Zig. Used as a cross-compiler for C projects (zig cc). The philosophy: "No hidden cost. Every operation is explicit. If the code looks simple, it IS simple." The anti-C++ — simplicity as a feature, not a limitation.

2016 · Andrew Kelley · comptime · no hidden allocators · cross-compile · C interop · Bun runtime
Connects to: C (direct replacement target), Rust (different safety philosophy), our build pipeline (Bun is a Zig project)

Unison

Language Functional Distributed

2019 — Paul Chiusano and Runar Bjarnason. Content-addressed code: every function is identified by a hash of its syntax tree, not its name. Renaming a function changes nothing. No builds, no dependency conflicts, no broken imports. Code is immutable and globally unique.

Distributed computing as a language primitive: Remote.fork sends a computation to another node. No serialization, no RPC stubs, no protobuf — the runtime handles it. Our UNI research project explores these ideas: 18 files, 6,142 lines. Content-addressing is LISP's "code is data" taken to its logical extreme.

2019 · Chiusano & Bjarnason · content-addressed · structural typing · abilities (algebraic effects)
Connects to: UNI research project (18 files), LISP (homoiconicity lineage), our content-addressed patterns, CMH (distributed compute)

Mojo

Language AI-Native Hardware

2023 — Chris Lattner, Modular. Python superset built on MLIR (the compiler infrastructure for ML hardware). Write Python, get C++ speed. Ownership model (borrowed from Rust). Compiles to any hardware: CPU, GPU, TPU, custom accelerators. No Python rewrite needed.

35,000x faster than Python on benchmarks. The same person who created LLVM, Clang, and Swift now building the AI-native language. Targets the "two-language problem" from the other direction: instead of making a fast language easy (Julia), make the easy language fast (Mojo). If AI workloads are the future, Mojo is designed to be their language.

2023 · Chris Lattner, Modular · Python superset · MLIR · ownership · hardware-agnostic · 35,000x Python
Connects to: Python (superset), Rust (ownership model), Swift (same creator), GPO (GPU acceleration), MPC (math compute acceleration)

The Paradigm Map

Languages are tools. Paradigms are worldviews. Each paradigm answers the question "how should we think about computation?" differently. Most modern languages are multi-paradigm — but understanding the pure forms makes you fluent in all of them.

Imperative

C, Assembly, FORTRAN, COBOL, Go

Step-by-step instructions. "Do this, then this, then this." Mutable state, explicit control flow, loops, assignments. The model closest to how hardware actually executes. You tell the machine exactly what to do, in what order. Maximum control, maximum responsibility.

Object-Oriented

Java, C++, C#, Smalltalk, Python, Ruby, Kotlin, Swift

The world modeled as objects that hold state and respond to messages. Encapsulation (hide internals), inheritance (share behavior), polymorphism (same interface, different implementations). Design patterns (Gang of Four) emerged here. SOLID principles. The dominant paradigm for 30 years — now sharing the stage with functional.

Functional

LISP, Haskell, Elixir, Clojure, Erlang, OCaml, F#, Unison

Computation as the evaluation of mathematical functions. Immutable data, no side effects, first-class functions, recursion over loops, pattern matching, algebraic data types. Pure functions are easy to test, easy to parallelize, easy to reason about. The paradigm that took 50 years to go mainstream.

Declarative

SQL, CSS, HTML, Prolog, Terraform, Make, Nix

Describe what you want, not how to get it. SQL: "give me all users over 18" (the optimizer decides the execution plan). CSS: "this element is red" (the browser decides the rendering). The opposite of imperative. Higher abstraction, less control, often more concise. Our HTML research pages are declarative.

Concurrent / Actor

Erlang, Elixir, Go, Scala (Akka), Pony, BEAM languages

Computation as independent processes that communicate by passing messages. No shared mutable state — each actor/process owns its memory. The actor model (Carl Hewitt, 1973) predates the internet but perfectly describes distributed systems. Erlang's "let it crash" philosophy: supervision trees restart failed processes. Our agent system uses these patterns.

Logic

Prolog, Datalog, miniKanren, core.logic

Define facts and rules. Ask questions. The runtime finds answers by searching and backtracking. Unification, horn clauses, resolution. "Socrates is a man. All men are mortal. Is Socrates mortal?" — the program deduces the answer. Used in expert systems, natural language processing, theorem proving, and type inference engines.

Array / Matrix

APL, MATLAB, Julia, NumPy, R, J, K

Operate on entire arrays at once, not element by element. A single expression transforms a matrix. APL (1966, Kenneth Iverson) used mathematical notation as code: +/v sums a vector. MATLAB is the language of engineering simulation. NumPy brought array programming to Python. The paradigm behind every neural network library.

Cross-Domain Connections

Every language connects to our research. The Rosetta insight: the same patterns appear in hardware, software, mathematics, and nature. Different notation, same truth.

Languages ↔ Electronics

Assembly maps to logic gates. C's memory model maps to physical address lines. PWM is a for-loop in hardware. Our 555 timer generates clock signals that drive CPUs. Our LED controllers run C on microcontrollers. The abstraction layers between software and hardware are thinner than most programmers realize.

Languages ↔ Mathematics

LISP is lambda calculus with parentheses. SQL is relational algebra with keywords. Haskell is category theory with syntax. Julia is numerical methods with JIT. Our Math Co-Processor bridges them: symbolic math in TypeScript calling CUDA kernels on the GPU. The language is the interface; the math is eternal.

Languages ↔ Agent Systems

Erlang's actor model is our agent orchestration. Go's goroutines are our parallel executors. Elixir's supervision trees are our error recovery. Prompt engineering is our skill system. The GSD architecture is a programming language for AI agents, with plans as programs and executors as runtimes.

Languages ↔ Natural Systems

DNA is a programming language (4-symbol alphabet, codon = instruction, ribosome = CPU). Neural networks are a programming language (weights = parameters, layers = functions, backprop = compilation). Physarum slime mold solves optimization problems without code. Nature compiled first.
