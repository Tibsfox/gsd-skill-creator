1. p5.js / Processing

Processing (2001, Casey Reas & Ben Fry at MIT Media Lab) was designed to teach visual programming to artists. p5.js is its JavaScript descendant — everything runs in the browser, no installation needed.

The core loop: setup() runs once. draw() runs 60 times per second. Everything in between is your canvas.

function setup() {
  createCanvas(800, 600);
  background(4, 6, 12);    // our --bg color
}

function draw() {
  // A circle that follows the mouse
  fill(76, 175, 80, 40);   // semi-transparent green
  noStroke();
  circle(mouseX, mouseY, 30);

  // Fade the background slightly each frame — creates trails
  fill(4, 6, 12, 10);
  rect(0, 0, width, height);
}

That’s 12 lines of code and it creates an interactive painting tool with motion trails. p5.js is immediate: no build step, no compilation, no framework. Write code, see pixels.

Our forest ecosystem simulation is 1,297 lines of p5.js. It renders weather-driven trees (L-systems), bird flocking (Boids), Kuramoto firefly synchronization, Physarum slime mold networks, and astronomically accurate sun/moon positions — all in real time, all driven by live KPAE weather data. Every frame is a function of the current state of the world.

2. Generative Art

L-Systems: Grammar as Growth

An L-system (Aristid Lindenmayer, 1968) is a rewriting grammar. Start with an axiom (a string). Apply rules (character substitutions) iteratively. Interpret the result as drawing commands. Trees, ferns, fractals, coral — all from a few characters and rules.

// L-system: Fractal tree
// Axiom: F
// Rule: F → F[+F]F[-F]F
// F = draw forward, + = turn right 25°, - = turn left 25°
// [ = save position, ] = restore position

var axiom = "F";
var rules = { "F": "F[+F]F[-F]F" };
var angle = 25;
var iterations = 4;  // 4 iterations produces a convincing tree

function generate(str, n) {
  for (var i = 0; i < n; i++) {
    var next = "";
    for (var j = 0; j < str.length; j++) {
      next += rules[str[j]] || str[j];
    }
    str = next;
  }
  return str;
}
// Result at iteration 4: 533 characters, encoding a branching tree

Noise Fields

Perlin noise (Ken Perlin, 1983, for Tron) generates smooth random values. Unlike pure random, adjacent points have similar values — creating natural-looking textures, terrain, and flow fields.

function draw() {
  for (var x = 0; x < width; x += 10) {
    for (var y = 0; y < height; y += 10) {
      var angle = noise(x * 0.005, y * 0.005, frameCount * 0.01) * TWO_PI * 2;
      stroke(255, 40);
      var len = 8;
      line(x, y, x + cos(angle) * len, y + sin(angle) * len);
    }
  }
}

This creates a flow field — a vector field where each point has a direction derived from noise. Drop particles into a flow field and they trace organic, river-like paths. This is how generative art pieces like Tyler Hobbs’s “Fidenza” work.

Cellular Automata

Conway’s Game of Life (1970): a grid of cells, each alive or dead. Four rules: underpopulation (die with <2 neighbors), survival (live with 2-3), overcrowding (die with >3), reproduction (dead cell with exactly 3 becomes alive). From these four rules emerges: gliders, oscillators, spaceships, Turing-complete computation.

Langton’s Ant (1986): an ant on a grid. On a white cell: turn right, flip color, move forward. On a black cell: turn left, flip color, move forward. Two rules. After ~10,000 steps of apparent chaos, the ant spontaneously constructs a highway — an infinite diagonal pattern. Order from chaos.

  2. Implement Conway’s Game of Life in p5.js. Start with a random grid. Apply the four rules each frame. Watch structures emerge, stabilize, and interact.
  3. Build an L-system renderer. Start with the fractal tree above. Try different rules and angles. Document the relationship between grammar and visual form.

3. Data Visualization

Data visualization turns numbers into insight. The best visualizations are not decorative — they reveal patterns that tables cannot.

D3.js: Data-Driven Documents

D3 (Mike Bostock, 2011) binds data to DOM elements and applies transformations. It’s not a charting library — it’s a visualization framework. You describe what the data should look like; D3 makes it happen.

// SVG bar chart from weather data
var data = [
  {hour: "6am", temp: 5.0}, {hour: "9am", temp: 11.0},
  {hour: "noon", temp: 14.0}, {hour: "3pm", temp: 13.8},
  {hour: "6pm", temp: 10.5}, {hour: "9pm", temp: 7.5}
];

var svg = d3.select("body").append("svg").attr("width", 400).attr("height", 200);
var y = d3.scaleLinear().domain([0, 20]).range([180, 10]);
var x = d3.scaleBand().domain(data.map(d => d.hour)).range([40, 380]).padding(0.2);

svg.selectAll("rect")
  .data(data)
  .join("rect")
  .attr("x", d => x(d.hour))
  .attr("y", d => y(d.temp))
  .attr("width", x.bandwidth())
  .attr("height", d => 180 - y(d.temp))
  .attr("fill", "#4CAF50");

Our radar charts, timeline visualizations, and the Artemis II trajectory canvas are all hand-coded SVG/Canvas. No charting libraries. When you understand the primitives (lines, arcs, bezier curves, gradients), you can build any visualization.

4. Shaders & GPU

A shader is a program that runs on the GPU — massively parallel, thousands of cores executing the same function on different data simultaneously. The GPU thinks differently than the CPU: not “do this, then this, then this” but “do this to everything at once.”

GLSL: The Language of the GPU

// Fragment shader — runs once per pixel, every frame
precision mediump float;
uniform float u_time;
uniform vec2 u_resolution;

void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution;  // normalize to 0-1

  // Signed distance function: circle
  float d = length(uv - 0.5) - 0.3;

  // Color based on distance
  vec3 color = mix(
    vec3(0.3, 0.7, 0.3),   // green inside
    vec3(0.02, 0.03, 0.05), // dark outside
    smoothstep(0.0, 0.01, d)
  );

  // Animated glow
  color += vec3(0.1) * (1.0 / (1.0 + abs(d) * 40.0)) * (0.5 + 0.5 * sin(u_time));

  gl_FragColor = vec4(color, 1.0);
}

This runs on every pixel simultaneously. On a 1920×1080 display, that’s 2,073,600 parallel executions per frame, 60 frames per second. The GPU handles it effortlessly. The same parallel architecture powers machine learning (matrix multiplications), cryptocurrency mining (hash computations), and scientific simulation (fluid dynamics, particle physics).

Our math co-processor runs CUDA kernels on the RTX 4060 Ti for FFT, SVD, Monte Carlo, and matrix operations. Same GPU, different shading language (CUDA vs GLSL), same massively parallel execution model. The ALICE experiment’s Run 3 upgrade uses GPUs for real-time track reconstruction at 50 kHz — 500,000 tracks per second on GPU.

5. Interactive Simulations

A simulation is a model of a system that evolves over time. Set the rules. Set the initial conditions. Press play. Watch.

Agent-Based Models

Each entity (bird, fish, firefly, car, person) follows simple local rules. Global behavior emerges from local interactions. Craig Reynolds’s Boids (1987): three rules produce flocking behavior — separation (don’t crowd), alignment (match neighbors’ heading), cohesion (steer toward center of group).

Our Simulations

Everything we build is interactive and runs in the browser:

  • Forest ecosystem — L-system trees, Boid birds, Kuramoto fireflies, Physarum slime mold. Weather-driven from KPAE METAR. Astronomically accurate sun/moon.
  • Kuramoto oscillators — Coupled oscillators synchronizing. Demonstrates the phase transition from disorder to synchronization as coupling strength increases.
  • Orbital mechanics — Two-body Newtonian gravity. Elliptical orbits, Hohmann transfers, escape velocity.
  • Tidal prediction — Harmonic constituents from NOAA data. Predicts Everett tide heights.
  • Heat shield reentry — Thermal protection system physics for Orion’s 11 km/s direct entry.
  • Circadian rhythms — Beat frequencies between biological clocks. 24h vs 24.5h lunar tidal period.

The Creative Coding Philosophy

Creative coding is not about making pretty pictures. It’s about using code as a medium for exploration. A simulation is a hypothesis rendered visible. A generative artwork is a constraint set made tangible. A shader is a mathematical function you can see.

The tools are free. p5.js runs in any browser. GLSL runs on any GPU. The canvas is infinite. The iteration cycle is instant: change a number, see the result. This is the fastest feedback loop in all of programming. Use it.
