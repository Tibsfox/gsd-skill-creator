1. HTML & CSS

Every web page is a tree of elements (HTML) with visual rules applied (CSS). That’s it. A heading, a paragraph, a link, a list, an image — these are the building blocks. CSS says what they look like: color, size, spacing, layout.

The Document Tree

HTML is a tree structure. The browser parses your markup into a Document Object Model (DOM) — a tree of nodes that JavaScript can traverse and modify. Every element is a node. Every node has a parent (except the root). This is the same tree structure from Pathway 1.

<!DOCTYPE html>
<html>
  <head>
    <title>My Page</title>
  </head>
  <body>
    <h1>Hello</h1>
    <p>A paragraph with a <a href="https://example.com">link</a>.</p>
  </body>
</html>

That’s 8 elements. The browser renders them top-to-bottom, left-to-right. CSS selectors target elements by tag name (h1), class (.card), ID (#nav), or position in the tree (main > p:first-child).

CSS: The Cascade

CSS stands for Cascading Style Sheets. The cascade is the algorithm that resolves conflicts when multiple rules apply to the same element. Order of precedence: inline styles > ID selectors > class selectors > element selectors. When in doubt, the more specific selector wins. When specificity ties, the last rule wins.

:root {
  --bg: #04060c;          /* CSS custom property (variable) */
  --text: #c8d0e0;
}
body {
  background: var(--bg);   /* Use the variable */
  color: var(--text);
  font-family: Georgia, serif;
  line-height: 1.7;
}
.card {
  display: grid;           /* CSS Grid for layout */
  grid-template-columns: repeat(auto-fill, minmax(300px, 1fr));
  gap: 1rem;
}

Modern CSS has two layout systems: Flexbox (one-dimensional: row or column) and Grid (two-dimensional: rows and columns). Every page on this site uses both. The Research index cards use Grid. The header nav uses Flexbox.

This site IS the project. Every page you’re reading is hand-coded HTML & CSS. View source on any page to see the patterns. The CERN pages use the same :root variables, the same .card class structure, the same responsive grid.

Try This

  1. Right-click any page on this site and choose “View Page Source.” Read the HTML. Find the <style> block. Match CSS rules to what you see.
  2. Open browser DevTools (F12). Click the Elements tab. Hover over DOM nodes — the browser highlights them on the page. Edit CSS values live and watch the page change.
  3. Create a file called hello.html. Write a heading, a paragraph, and a link. Open it in your browser. Add a <style> block. Change the background color, font size, and margins.
  4. Inspect the ALICE page. Note: no build system, no framework, no dependencies. Just HTML and CSS in a single file.

2. JavaScript

JavaScript started as a language for animating web pages. It is now the most widely deployed programming language in history. It runs in every browser, on every server (Node.js), on desktop (Electron), and on mobile (React Native). One language, every platform.

The Basics

// Variables and types
const name = "Artemis";     // string (immutable binding)
let distance = 140000;       // number (mutable binding)
const crew = ["Wiseman", "Glover", "Koch", "Hansen"];  // array

// Functions
function milesToKm(miles) {
  return miles * 1.60934;
}

// Arrow function (shorthand)
const formatMET = (hours) => `MET +${Math.floor(hours/24)}d ${hours%24}h`;

// Objects
const mission = {
  name: "Artemis II",
  phase: "translunar",
  crew: crew,
  distanceMi: distance,
  distanceKm() { return milesToKm(this.distanceMi); }
};

console.log(mission.distanceKm());  // 225307.6
console.log(formatMET(46.92));      // MET +1d 22h

The DOM: Where JavaScript Meets HTML

JavaScript can read and modify the DOM tree. This is how every interactive web page works — from a button click handler to a full application.

// Find an element
const heading = document.querySelector('h1');

// Change its content
heading.textContent = 'Updated!';

// Listen for events
document.querySelector('.btn').addEventListener('click', (e) => {
  console.log('Button clicked at', e.clientX, e.clientY);
});

// Fetch data from an API
const response = await fetch('https://api.weather.gov/stations/KPAE/observations/latest');
const data = await response.json();
console.log(data.properties.temperature.value);  // current temp in C

async/await is how JavaScript handles operations that take time (network requests, file reads, timers). The await keyword pauses execution until the promise resolves. Under the hood, JavaScript is single-threaded with an event loop — it never blocks, it just schedules callbacks.

Try This

  1. Open DevTools Console (F12 → Console). Type document.querySelector('h1').style.color = 'red' and press Enter. The heading turns red.
  2. Type fetch('https://api.weather.gov/stations/KPAE/observations/latest').then(r => r.json()).then(d => console.log(d.properties.temperature.value)). That’s a live weather API call from your browser.
  3. Read the source of our forest simulation. It’s 1,297 lines of JavaScript using p5.js. Find the draw() function — it runs 60 times per second.

3. HTTP & APIs

HTTP (Hypertext Transfer Protocol) is the language browsers and servers speak. Every web interaction is a request-response pair.

MethodPurposeExample
GETRetrieve dataGET /stations/KPAE/observations/latest
POSTCreate dataPOST /api/sweeps with JSON body
PUTReplace dataPUT /api/status with full object
PATCHUpdate part of dataPATCH /api/mission with partial update
DELETERemove dataDELETE /api/sweep/v1.3.14

Status Codes

The server responds with a status code. The first digit tells you the category:

RangeMeaningCommon Codes
2xxSuccess200 OK, 201 Created, 204 No Content
3xxRedirect301 Moved Permanently, 304 Not Modified
4xxClient error400 Bad Request, 401 Unauthorized, 404 Not Found
5xxServer error500 Internal Error, 502 Bad Gateway, 503 Unavailable

REST

REST (Representational State Transfer) is a convention for organizing APIs around resources. A resource is a noun — a mission, a sweep, a weather observation. The HTTP method is the verb. GET /missions/artemis-ii retrieves the mission. POST /missions/artemis-ii/sweeps creates a new sweep.

The NWS Weather API we use follows REST conventions: GET /stations/KPAE/observations/latest returns the latest observation for station KPAE. The URL is the resource identifier. The response is JSON.

Try This

  1. Open DevTools Network tab (F12 → Network). Reload any page. Watch the HTTP requests flow — HTML first, then CSS, JS, images. Click any request to see headers, status code, response body.
  2. Open a new browser tab. Navigate to https://api.weather.gov/stations/KPAE/observations/latest. The browser makes a GET request and shows you raw JSON.
  3. Install curl and run: curl -s https://api.weather.gov/stations/KPAE | python3 -m json.tool. Same API, from the command line.

4. TCP/IP & DNS

When you type a URL, dozens of protocols cooperate to deliver the page. The Internet Protocol Suite (TCP/IP) is organized in layers:

LayerProtocolWhat It Does
ApplicationHTTP, FTP, SMTP, DNSWhat the user sees. Web pages, email, file transfer.
TransportTCP, UDPReliable delivery (TCP) or fast delivery (UDP). Port numbers.
InternetIP (v4, v6)Addressing and routing. Every device gets an IP address. Routers forward packets toward the destination.
LinkEthernet, WiFi, LTEPhysical transmission. Bits on a wire, radio waves, fiber optic pulses.

DNS: The Internet’s Phone Book

When you visit tibsfox.com, your computer asks a DNS resolver: “What IP address is tibsfox.com?” The resolver queries a hierarchy of nameservers (root → .com → tibsfox.com’s authoritative server) and returns an IP address like 216.222.199.72. Your browser connects to that IP on port 443 (HTTPS).

The entire process takes milliseconds. It happens before a single byte of your web page is fetched. DNS is caching at every level — your browser, your OS, your router, your ISP’s resolver. This is the same hierarchical caching pattern we see in CPU caches (L1 → L2 → L3 → RAM → disk) and in the WLCG computing grid (Tier 0 → Tier 1 → Tier 2).

TCP: Reliable Delivery

TCP guarantees that data arrives in order and without corruption. It does this through a three-way handshake (SYN, SYN-ACK, ACK), sequence numbers, acknowledgments, and retransmission. The cost: latency overhead. The benefit: you can build HTTP on top of it and trust that every byte arrives.

UDP skips the reliability guarantees for speed. Used for video streaming, gaming, DNS queries, and VoIP — applications where a dropped packet is better than a delayed one.

The same TCP/IP stack runs over fiber optic, WiFi, ham radio (see DRP zero-trust comms), and satellite links (how Artemis II communicates through TDRS). The protocol doesn’t care about the physical medium.

Try This

  1. Run dig tibsfox.com to see DNS resolution. Note the answer section (IP address) and the query time (milliseconds).
  2. Run traceroute api.weather.gov to see every router hop between you and the NWS server. Each line is a node in the network graph.
  3. Run curl -v https://tibsfox.com 2>&1 | head -30. The -v flag shows the TLS handshake, HTTP headers, and TCP connection details.

5. Databases & State

The web is stateless — each HTTP request is independent. Databases give it memory. When the browser closes, when the server restarts, the data persists.

SQL: The Language of Relational Databases

-- Create a table
CREATE TABLE weather_samples (
  id SERIAL PRIMARY KEY,
  station TEXT NOT NULL,
  timestamp TIMESTAMPTZ NOT NULL,
  temp_c NUMERIC(4,1),
  wind_kmh NUMERIC(5,1),
  conditions TEXT
);

-- Insert data (from our KPAE METAR)
INSERT INTO weather_samples (station, timestamp, temp_c, wind_kmh, conditions)
VALUES ('KPAE', '2026-04-03T21:15:00Z', 12.0, 5.5, 'SCT035 BKN090');

-- Query: what was the max temperature today?
SELECT MAX(temp_c) FROM weather_samples
WHERE station = 'KPAE' AND timestamp::date = '2026-04-03';

-- Join: match weather samples with sweep versions
SELECT s.version, w.temp_c, w.conditions
FROM sweeps s
JOIN weather_samples w ON s.timestamp = w.timestamp
ORDER BY s.version;

PostgreSQL is the database we use for the Artemis II mission. The artemis schema holds weather samples, forecast scores, space weather, river flow, and sweep logs. pgvector adds embedding columns for semantic search across 1,092+ research pages.

Key Concepts

ConceptWhat It Means
ACIDAtomicity, Consistency, Isolation, Durability. A transaction either fully succeeds or fully rolls back. No partial writes.
IndexA data structure that speeds up queries. Like a book index — without it, the database reads every row (table scan).
NormalizationOrganizing data to eliminate redundancy. Each fact stored once, referenced by keys.
Foreign keyA column that references another table’s primary key. Enforces relationships.
MigrationA versioned change to the database schema. Applied in order. Reversible.

Beyond SQL

Redis: In-memory key-value store. Microsecond reads. Used for caching, session state, rate limiting. MongoDB: Document store. JSON-like documents without fixed schemas. SQLite: Embedded database in a single file. No server. Used in mobile apps, browsers, and embedded systems. Every iPhone has multiple SQLite databases.

Try This

  1. Install SQLite: sudo apt install sqlite3. Run sqlite3 test.db. Create a table, insert rows, query them. The database is a single file.
  2. If PostgreSQL is running: psql -h localhost -U postgres -d tibsfox -c "SELECT count(*) FROM artemis.weather_samples;"
  3. Read the WLCG page’s section on data management. CERN stores 1.5 exabytes across 170 sites. Same SQL underneath, different scale.

The Full Stack

Put it all together: a user types a URL. DNS resolves the domain. TCP connects to the server. TLS encrypts the channel. HTTP requests the page. The server queries a database, formats the result as HTML, and sends it back. The browser parses the HTML into a DOM tree, applies CSS rules, and executes JavaScript. The user sees a web page.

Every layer is independent. You can swap PostgreSQL for SQLite. You can swap TCP for QUIC. You can swap HTML for JSON. The interfaces between layers are the contracts. As long as the contract is honored, the implementation is free to change. This is the same layered architecture that CERN’s computing grid uses, that our trust system uses, and that the Internet itself is built on.
