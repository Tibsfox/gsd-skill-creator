1. Data & Statistics

Every ML project starts and ends with data quality. A model trained on bad data produces confident wrong answers. Statistics is the foundation.

import pandas as pd
import numpy as np

# Load our weather comparison data
df = pd.read_csv('weather_samples.csv')

# Basic statistics
print(df['kpae_temp'].describe())  # count, mean, std, min, 25%, 50%, 75%, max

# Correlation: how strongly do two variables move together?
print(df[['kpae_temp', 'ksea_temp']].corr())  # should be high (~0.95+)

# Error metrics
mae = np.mean(np.abs(df['model_temp'] - df['kpae_temp']))  # Mean Absolute Error
rmse = np.sqrt(np.mean((df['model_temp'] - df['kpae_temp'])**2))  # Root Mean Square Error
print(f"MAE: {mae:.2f}°C, RMSE: {rmse:.2f}°C")

2. Supervised Learning

Given labeled examples (input → output), learn the mapping. The two types:

TypeOutputExampleAlgorithm
RegressionContinuous numberPredict tomorrow’s temperature from today’s conditionsLinear regression, random forest, gradient boosting
ClassificationCategoryWill it rain? (yes/no). What type of cloud?Logistic regression, SVM, decision tree, neural network

The Bias-Variance Tradeoff

Underfitting (high bias): The model is too simple. It misses real patterns. A straight line through sinusoidal temperature data.

Overfitting (high variance): The model memorizes training data, including noise. It performs perfectly on training data and terribly on new data. A polynomial that hits every training point but oscillates wildly between them.

The goal: Find the complexity that captures real patterns without memorizing noise. Cross-validation measures this: train on 80% of data, test on 20%. Repeat 5 times with different splits. The average test error is your estimate of real-world performance.

from sklearn.ensemble import RandomForestRegressor
from sklearn.model_selection import cross_val_score

# Predict KPAE temperature from: hour, day, wind_speed, humidity, pressure
X = df[['hour', 'day', 'wind_kmh', 'humidity', 'pressure_hpa']]
y = df['kpae_temp']

model = RandomForestRegressor(n_estimators=100, max_depth=8)
scores = cross_val_score(model, X, y, cv=5, scoring='neg_mean_absolute_error')
print(f"Cross-validated MAE: {-scores.mean():.2f} ± {scores.std():.2f}°C")

3. Neural Networks

A neural network is layers of simple functions composed together. Each layer transforms its input through a weighted sum and a nonlinear activation function. The magic is in the training: backpropagation computes how to adjust each weight to reduce the error.

ArchitectureInputStrengthExample
MLP (dense)Fixed-size vectorTabular data, simple patternsWeather prediction from sensor readings
CNNGrid (image, spectrogram)Spatial patterns, translation invarianceSatellite cloud classification, particle tracks
RNN/LSTMSequenceTemporal patterns, variable-lengthTime-series forecasting, speech
TransformerSequenceLong-range dependencies, parallelizableLanguage (GPT, Claude), protein folding

The Transformer Revolution

“Attention Is All You Need” (Vaswani et al., 2017) replaced recurrence with self-attention: each token attends to every other token, weighted by learned relevance scores. This unlocked parallelism (unlike sequential RNNs) and enabled scaling to billions of parameters.

The key equation: Attention(Q, K, V) = softmax(QKT / √dk) × V. Queries, keys, and values are linear projections of the input. The softmax creates a probability distribution over positions. The weighted sum of values produces the output. Multi-head attention runs this in parallel across multiple “heads,” each learning different types of relationships.

4. Large Language Models

An LLM is a transformer trained on a large fraction of human text to predict the next token. From this simple objective emerges: reasoning, coding, translation, summarization, tool use, and instruction following.

ConceptWhat It Means
TokenizationText is split into subword tokens (BPE). “Artemis” might be one token; “microcontroller” might be two. Claude uses ~100K token vocabulary.
Context windowHow many tokens the model sees at once. Claude Opus 4: 200K tokens. More context = more information, but also more computation.
TemperatureControls randomness. T=0 is deterministic (always picks most likely token). T=1 is creative. T>1 is increasingly random.
System promptInstructions that shape the model’s behavior for a conversation. CLAUDE.md is our system prompt.
Tool useThe model can call external functions (read files, run code, fetch web pages) and incorporate results into its reasoning.
Chain of thoughtPrompting the model to “think step by step” improves accuracy on complex tasks. The extended thinking block.

5. Agent Orchestration

An agent is an LLM with tools and autonomy. A multi-agent system is agents coordinating to solve problems no single agent can handle.

PatternDescriptionOur Implementation
Mayor-CoordinatorOne agent dispatches work, doesn’t execute itGastown northbridge: creates convoys, monitors progress
Polecat-WorkerEphemeral agent: receives work, executes, commits, terminatesGSD executor subagents, one per phase plan
Witness-ObserverMonitors other agents for health, stalls, failuresHawk watches for what’s NOT there
ConvoyGroup of workers dispatched together as a unitWave-based parallel execution (3-4 agents per wave)
GUPPGet Up and Push Protocol — interrupt-driven vs pollingAgents don’t wait to be asked, they push results

The key insight from our 50+ project research series: agent orchestration patterns converge with distributed computing patterns. The WLCG’s DIRAC pilot jobs are GUPP workers. Kubernetes pods are polecat executors. The scheduler is a mayor-coordinator. These aren’t metaphors — the math is identical.

MCP: Model Context Protocol

MCP standardizes how LLMs connect to external tools and data sources. An MCP server exposes capabilities (tools, resources, prompts) over a transport (stdio, HTTP). The model calls tools by name with structured arguments.

The AI Landscape in 2026

The field moves fast. The principles don’t. Data quality matters more than model size. Evaluation is harder than training. The gap between a demo and a production system is enormous. And the most important skill isn’t building models — it’s knowing when a model is the wrong tool.

Most problems don’t need AI. A SQL query, a regex, a lookup table, a simple heuristic — these solve 90% of data problems faster, cheaper, and more reliably than any model. AI is for the remaining 10%: where the pattern is too complex to specify, where the input is too variable to enumerate, where the relationship between input and output is learned rather than programmed.
