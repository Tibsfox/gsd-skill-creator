How to Run Part F

  1. Confirm Part B complete: v1.50.{MISSION_SLUG} with populated crew_roles, mission_phases, and fault_scenarios.
  2. Part F runs in parallel with Parts C, D, and E — all three consume Part B independently.
  3. Run with gsd-skill-creator:
    gsd run skill-creator --template nasa_partf_template.pdf --input {MISSION_SLUG}.ck.yaml --mode operations
  4. Wave 0 classifies the mission type (Crewed/Autonomous/Both) and verifies the NASA 3D model URL is live.
  5. Wave 1: five parallel tracks begin simultaneously. TOPO (Bear) monitors for overruns and reallocates if any track exceeds budget.
  6. Wave 2: three simulation sub-tracks (Minecraft, Blender, Unreal/Hardware) run after Wave 1 skill outputs are available. All Python scripts are executed before Wave 3 proceeds.
  7. Wave 3 (Opus): AI CAPCOM architecture, career profiles, cross-module integration, long-duration strategy review.
  8. Wave 4 deposits 7 new skills to /mnt/skills/user/ and mission-control-chipset.yaml to .college/chipsets/.
  9. Approve CAPCOM Wave 4B gate. Release tagged v1.50.{MISSION_SLUG}.OPS.
