Engineering Notebook — The RTG Thermal Trade

RTGs produce far more heat than electrical power (efficiency ~6%). Most of that heat must go somewhere that doesn't overheat electronics. The engineering insight: at Saturn, where the spacecraft needs heat to stay warm, the "waste" from the RTG becomes the primary heating system. The engineer who recognized this reframing saved mass, simplified the design, and solved two problems with one thermal network. This kind of systems thinking — seeing heat as a resource rather than a problem — is a signature skill of experienced spacecraft thermal engineers.

How to Run Part D

  1. Confirm Part B is complete: v1.50.{MISSION_SLUG} release exists with a populated hardware_record in the CK YAML.
  2. Part D runs in parallel with Part C — both consume the Part B output independently. No Part C dependency.
  3. Hand the template and Part B hardware record to gsd-skill-creator:
    gsd run skill-creator --template nasa_partd_template.pdf --input {MISSION_SLUG}.ck.yaml --mode engineering
  4. Wave 0 validates the hardware record. BLOCK fires if key engineering fields are missing — escalate to CAPCOM for manual data entry from NTRS sources.
  5. Waves 1A, 1B, 2A, 2B run in parallel: structural/vibration/mechanisms, materials, math workbook, requirements/manufacturing.
  6. Wave 3 (Opus): career profiles, tutor scripts, math verification, legacy analysis — all in one Opus context window.
  7. Approve CAPCOM Wave 4B gate. Release tagged v1.50.{MISSION_SLUG}.ENG.
