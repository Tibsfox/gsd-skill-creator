From Sakichi Toyoda asking “why?” five times on the Toyota factory floor in the 1930s to Microsoft Research’s AgentRx pipeline achieving +23.6% fault localization in 2026, root cause analysis has evolved from a single-method shop-floor discipline into a multi-paradigm investigation science spanning healthcare sentinel events, aviation black boxes, cloud telemetry, and AI agent trajectory debugging. This research digest traces that entire arc — eight decades of methodology, 10 modules, 879 cross-references across a 246-project corpus.

Module Overview

Classical

Classical Methods

Five Whys (Toyota/Toyoda, 1930s), Ishikawa Fishbone Diagrams (6M categories), Fault Tree Analysis (Boolean AND/OR gates), FMEA (Risk Priority Numbers), Barrier Analysis (Swiss Cheese model). The foundational toolkit that shaped every subsequent method.

Systems

Systems-Theoretic Approaches

STAMP (Leveson, MIT), STPA (proactive hazard analysis), CAST (retrospective), HFACS (aviation), Swiss Cheese Model (Reason), Safety-III. The paradigm shift from component failure to emergent system-level control breakdowns.

SRE

Software Engineering & SRE

Google blameless postmortems, observability (logs/metrics/traces), error budgets, DORA metrics, chaos engineering, cascading failure analysis, circuit breakers. How software engineering adapted RCA for continuous deployment.

AI Agents

AI Agent Debugging

AgentRx (Microsoft, 4-stage pipeline, +23.6% localization), OpenRCA (335 failures, 68GB telemetry), AgentDebug (+24% accuracy), 9-category failure taxonomy. The frontier of RCA applied to non-deterministic reasoning chains.
