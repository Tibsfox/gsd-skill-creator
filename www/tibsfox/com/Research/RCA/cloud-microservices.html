In microservice architectures with hundreds of interdependent services, a latency spike in one service can cascade into full outages across the system. Cloud-native RCA demands automated causal reasoning over service dependency graphs, distributed traces, and multi-modal telemetry at scales no human investigator can process manually.

DynaCausal — Dynamic Causality-Aware Fault Propagation

DynaCausal addresses a fundamental limitation of static dependency graphs: in real microservice systems, causal relationships between services change dynamically based on load, configuration, and deployment state. A service that is an upstream dependency under normal conditions may become a downstream consumer during failover.

Key innovations:

  • Dynamic causal graph construction: Continuously updated from distributed traces rather than static service manifests
  • Cascading fault propagation modeling: Tracks how failures propagate through the dynamic graph, accounting for circuit breakers, retries, and fallback paths
  • Temporal causality: Distinguishes between correlation and causation using time-lagged analysis — service B degraded after service A, but did A cause B’s degradation?
  • Multi-hop root cause identification: Traces the causal chain back through multiple service hops to the originating failure

CHASE — Causal Hypergraph for Microservice Systems

CHASE extends traditional causal graph approaches by modeling microservice interactions as hypergraphs — where a single edge can connect multiple nodes simultaneously, capturing the reality that a failure often involves multiple interacting services rather than simple pairwise dependencies.

CHASE processes multimodal data: metrics (CPU, memory, latency), logs (error patterns, stack traces), and traces (request paths). Each modality provides different causal signals, and CHASE fuses them through a learned attention mechanism that weights each modality based on its informativeness for the specific failure type.

MetaRCA — Generalizable Production Fault Analysis

MetaRCA tackles the generalization problem: most cloud RCA systems are trained on specific microservice deployments and fail when applied to new systems. MetaRCA uses meta-learning to extract transferable patterns from fault incidents across multiple production environments.

  • Few-shot fault diagnosis: Given only 3–5 examples of a new failure type, MetaRCA can accurately identify root causes
  • Cross-system transfer: Patterns learned from one microservice deployment transfer to architecturally different systems
  • Incident embedding: Represents each incident as a vector in a learned space where similar root causes cluster together

Service Dependency Graphs & Trace Analysis

The foundation of all cloud-native RCA is the service dependency graph — a directed graph where nodes are services and edges represent communication channels (HTTP, gRPC, message queues, shared databases).

Anomaly Correlation Across Service Meshes

Modern cloud RCA systems combine multiple signals through anomaly correlation:

  1. Detect: Identify anomalous metrics/logs/traces across all services simultaneously
  2. Cluster: Group related anomalies by temporal proximity and causal connectivity
  3. Rank: Score each service by its likelihood of being the root cause, using graph centrality and temporal precedence
  4. Validate: Cross-check the top candidate against change events, deployment timelines, and known failure patterns
  5. Explain: Generate a human-readable causal chain from the root cause through intermediate services to the user-facing symptom

DynaCausal — Full Results

Deep extraction from Zhang et al. (arXiv:2510.22613, Oct 2025) provides complete benchmark results for DynaCausal’s dynamic causality-aware RCA framework.

Three-Stage Methodology

  1. Multi-Modal Data Alignment: Integrates metrics, logs, and traces into unified multivariate time series. Constructs dynamic call graphs with edge weights: ei,j(t) = σ(α · Norm(C) + (1-α) · Norm(R))
  2. System-Interaction-Aware Representation Learning: Temporal encoding via Transformer encoders with multi-head self-attention; Hybrid-Aware GAT combining standard graph attention with pre-calculated dynamic edge weights modeling fault propagation causality
  3. Causal Representation Discrimination (CRD): Temporal Causal Disentanglement (TCD) via contrastive learning; Spatial Causal Ordering (SCO) via pairwise ranking constraints

Combined Loss Function

Ltotal = LCE + λ1 · LTCD + λ2 · Lrank

Where TCD enforces temporal disentanglement (anomalous vs normal service embeddings) and ranking loss implements causal ordering priorities (root causes must score above downstream affected services).

Cascading Fault Case Study

Existing methods incorrectly ranked PaymentService and AdService above the actual root cause CheckoutService. CheckoutService showed only subtle latency increase while downstream services exhibited pronounced memory shifts. DynaCausal’s SCO constraint correctly forced root causes to rank above affected downstream services, identifying the subtle-but-causal originator.

Goal-Driven RCA Framework — Fang et al.

Deep extraction from Fang et al. (arXiv:2510.19593, Oct 2025) reframes RCA through objectives rather than data types. The survey covers 135 papers (2014–2025) and argues that “the goal of localizing a faulty service for rapid triage is fundamentally different from identifying a specific functional bug.”

Formal RCA Definition

F: O → G where:

  • Input Space (O): Logs, metrics, traces, events, supplementary data (code, configs, docs)
  • Output Space (G): Incident Propagation Graph (DAG) with three node types: root cause (zero in-degree), trigger (optional catalyst), symptom (observable failure)

The Effectiveness-Data-Cost Triangle

Fundamental trade-off identified across all surveyed methods: improving effectiveness requires richer data, but richer data increases cost. This three-way competition forces strategic compromises — no method optimizes all three simultaneously. This is our token-budget constraint in different clothes.

Critical Gaps in Current Research

  • Current research focuses on “point-finding” (identifying root cause nodes) rather than “graph-building” (constructing complete incident propagation graphs)
  • No existing work generates comprehensive propagation paths with validated causal edges
  • SRE seeking rapid mitigation (MTTR) needs different RCA than developer pursuing permanent fixes (MTBF) — this distinction is absent from conventional surveys
