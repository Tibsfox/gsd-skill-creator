Start with ILP — understand why basic blocks are too small. Then read Fisher. Then see what modern compilers actually do. Then map it to AI orchestration.

Staged Learning Path

  1. Read Chapter 4 of Hennessy & Patterson — Computer Architecture: A Quantitative Approach. Covers instruction-level parallelism, dynamic scheduling (Tomasulo’s algorithm), branch prediction, and speculation. This is the foundation: understand why ILP matters and how hardware exploits it before studying how compilers do the same job statically.
  2. Read Fisher’s 1981 paper — “Trace Scheduling: A Technique for Global Compaction.” The founding document. Only 12 pages. Introduces the core ideas: basic blocks are too small (4–7 instructions on average), traces follow the most-likely execution path, compaction fills VLIW slots, and bookkeeping repairs off-trace paths. Everything in this project descends from this paper.
  3. Read Ellis’s 1985 Bulldog thesis — or the 1986 book version, Bulldog: A Compiler for VLIW Architectures. The first complete implementation of trace scheduling. Covers the engineering details Fisher’s paper left abstract: DAG construction, resource tables, code motion rules, compensation code generation.
  4. Read Hwu, Mahlke, and Chen (1992) — “The Superblock: An Effective Technique for VLIW and Superscalar Compilation.” Superblocks simplify trace scheduling by eliminating join points (single entry, multiple exits). Easier bookkeeping, nearly the same ILP extraction. The practical successor to full trace scheduling.
  5. Study the TI C6x architecture — Read the TI C6000 Programmer’s Guide. The C6x is a modern, commercially successful VLIW DSP with 8 functional units and 256-bit instruction packets. It makes VLIW concepts concrete: you can see the fetch packets, execute packets, NOP slots, and software pipeline loops. A living reference.
  6. Read the LLVM scheduling infrastructure — Start with the LLVM documentation on MachineScheduler and ScheduleDAG. Study how LLVM represents scheduling units, latencies, and resource conflicts. Trace the path from SelectionDAG → MachineInstr → ScheduleDAGInstrs → MachineScheduler. This is where trace scheduling’s descendants live in production compilers.
  7. Explore GCC’s scheduling passes — -fschedule-insns (before register allocation) and -fschedule-insns2 (after register allocation). Study the interaction with profile-guided optimization (-fprofile-generate / -fprofile-use), which gives the compiler the branch probability data that trace selection needs.
  8. Read the AI Crossover module — See how trace scheduling concepts map to modern AI orchestration: context windows as VLIW slots, model tiers as functional units, token budgets as instruction count limits. The formal mapping table makes the structural analogy precise and actionable.

Canonical References

Foundational Papers and Books

ReferenceAuthor(s)YearNotes
Trace Scheduling: A Technique for Global Compaction Joseph A. Fisher 1981 The founding paper. Introduced trace selection, trace compaction, and bookkeeping. 12 pages that launched a field.
Bulldog: A Compiler for VLIW Architectures John R. Ellis 1985 PhD thesis (Yale); published as book in 1986. First complete trace scheduling compiler implementation.
The Superblock: An Effective Technique for VLIW and Superscalar Compilation Hwu, Mahlke, Chen et al. 1992 Simplified trace scheduling by eliminating join points. Single entry, multiple exits. IMPACT compiler, UIUC.
Sentinel Scheduling for VLIW and Superscalar Processors (Hyperblocks) Mahlke et al. 1992 Extended superblocks with if-conversion (predicated execution). Requires hardware predication support.
Iterative Modulo Scheduling B. Ramakrishna Rau 1994 The definitive software pipelining algorithm. Complementary to trace scheduling — targets loops instead of irregular control flow.
Embedded Computing: A VLIW Approach to Architecture, Compilers, and Tools Fisher, Faraboschi, Young 2005 Fisher’s textbook-length treatment. Covers VLIW architecture, trace scheduling, and embedded applications. The most complete single reference.

Textbooks

BookAuthor(s)Notes
Computer Architecture: A Quantitative Approach Hennessy & Patterson THE architecture textbook. Chapter 4 covers ILP, dynamic scheduling, VLIW, and EPIC. 6th edition (2019) or later.
Advanced Compiler Design and Implementation Steven S. Muchnick 1997. The most complete compiler textbook. Chapters on instruction scheduling, software pipelining, and global optimization.
Compilers: Principles, Techniques, and Tools (Dragon Book) Aho, Lam, Sethi, Ullman 2006 (2nd edition). Chapter 10 covers instruction-level parallelism and code scheduling. The classic compiler textbook.

Architecture References

DocumentPublisherNotes
TMS320C6000 Programmer’s Guide Texas Instruments Practical VLIW programming. 8 functional units, 256-bit fetch packets. Software pipelining examples. Freely available from TI.
Intel IA-64 Architecture Software Developer’s Manual Intel Corporation The EPIC reference. 128-bit bundles, predication, speculative loads, register rotation. 4 volumes. Historical now, but the design ideas are instructive.
Qualcomm Hexagon Programmer’s Reference Manual Qualcomm Modern VLIW DSP in billions of smartphones. 4-wide VLIW packets. LLVM-based toolchain. The most commercially successful VLIW architecture alive today.

Free Resources

  • Fisher’s papers — Many are freely available through Yale’s digital library and CiteSeerX. The 1981 paper is widely accessible.
  • MIT 6.823 (Computer System Architecture) — Lecture slides covering ILP, VLIW, dynamic scheduling, and memory hierarchy. Available on MIT OpenCourseWare.
  • LLVM documentation — The MachineScheduler, ScheduleDAG, and target-specific scheduling model documentation. Start at llvm.org/docs.
  • GCC internals documentation — The insn-scheduling and profile-feedback sections of the GCC Internals Manual. Available at gcc.gnu.org.
  • Trimaran compiler — Open-source research compiler for VLIW/EPIC from the IMPACT group (UIUC). Includes an HPL-PD machine model. Available at trimaran.org. The closest thing to a hands-on trace scheduling laboratory.
