1 What Is This?

GSD + Skill Creator turns Claude Code into a production system. It layers project management (GSD) and adaptive learning (Skill Creator) on top of an AI assistant, so you can plan work, execute it with parallel agents, and verify the results — all inside your terminal.

The proof is in what it produces. This system has autonomously generated 189+ research projects, a 329-page textbook, and 91,000 words of research documentation — each with full source, citations, and HTML/PDF output. It is not a demo; it is a running production pipeline.

3 Installation (5 minutes)

You need Node.js 18+ and an Anthropic API key. Then:

# Install Claude Code (the AI assistant)
npm install -g @anthropic-ai/claude-code

# Install GSD (the project management framework)
npm install -g get-shit-done

# Clone the reference implementation
git clone https://github.com/Tibsfox/gsd-skill-creator.git
cd gsd-skill-creator

# Install project-specific Claude config (skills, agents, hooks)
node project-claude/install.cjs
Note: GSD (get-shit-done) is the upstream framework that provides project management structure. The gsd-skill-creator repo is the reference implementation that adds 69 skills, 39 agents, and 57 commands on top of it. You can use GSD alone, or with the full Skill Creator stack.

4 Your First 10 Minutes

Open Claude Code in the project directory and try these commands. Skills load automatically based on context — no configuration needed.

  1. Open Claude Code in the project directory: 
    claude
  2. See all available GSD commands: 
    /gsd:help
  3. See project statistics — skills, tests, docs, phases: 
    /gsd:stats
  4. Check current progress and what to work on next: 
    /gsd:progress
  5. Try the session briefing to see active skills and budget: 
    /sc:start

That is the full onboarding. You are now inside a system with 69 auto-activating skills, deterministic hooks for commit validation and session state, and a full project management lifecycle. Everything else happens by asking Claude Code to do work.

5 The Three Layers

The system is built from three layers, each with a clear job:

Skill Creator
Adaptive learning layer. 69 skills that load automatically based on context. Agents, teams, chipsets. Observes your work patterns and suggests improvements. This repo is the reference implementation.
GSD (Get Shit Done)
Project management framework. Phases, plans, execution, verification. Handles the lifecycle: discuss → plan → execute → verify → ship. 57 commands covering everything from /gsd:new-project to /gsd:autonomous.
Claude Code
The AI assistant underneath. Reads your codebase, writes code, runs tests, manages git. GSD and Skill Creator sit on top as skills and hooks — they extend Claude Code without modifying it.

6 Five Things You Can Do Right Now

Command What It Does
/gsd:new-project Start a new project with deep context gathering. Creates PROJECT.md, roadmap, and requirements through interactive questioning.
/gsd:plan-phase Plan a phase of work. Produces a detailed PLAN.md with verification criteria, then validates it in a review loop before execution.
/gsd:execute-phase Execute all plans in a phase with wave-based parallel agents. Multiple Claude instances working simultaneously on independent tasks.
/gsd:verify-work Validate what was built against requirements. Conversational UAT that checks every acceptance criterion.
/research-engine Produce a research document on any topic. Autonomous pipeline from topic to structured HTML/PDF output with citations and cross-references. Proven at scale (28 docs, 91K words for the HEL project alone).

For the full command list, run /gsd:help inside Claude Code, or explore the command definitions on GitHub.
