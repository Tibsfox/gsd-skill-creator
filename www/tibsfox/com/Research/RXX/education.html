The Canon

TitleAuthorYearWhy It Matters
The REXX Language: A Practical Approach to ProgrammingMike Cowlishaw1985 (2nd ed. 1990)THE book. Written by the language creator. The title IS the philosophy. Prentice-Hall. ISBN 0-13-780735-X (2nd ed). Our porting reference starts here.
Modern Programming Using REXXRobert O’Hara & David Gomberg1988Practical REXX programming with real-world examples. Prentice-Hall. Good complement to Cowlishaw’s reference-oriented approach.
The REXX HandbookGabriel Goldberg & Philip Smith (eds.)1992Collection of articles by REXX experts. Covers VM/CMS, OS/2, and TSO REXX. McGraw-Hill. Excellent for understanding the ecosystem.
Programming in REXXCharles Daney1992McGraw-Hill. Tutorial-oriented. Good for learning from scratch.
REXX Programmer’s ReferenceHoward Fosdick2005Wiley. The most recent comprehensive reference. Covers classic REXX, ooRexx, and NetRexx. ISBN 0-7645-7996-8.

  factorial = factorial * i
END

Learning Path — From Zero to ARexx Port Developer

  1. Level 1: Basic REXX — Install Regina REXX. Write SAY, PULL, IF/THEN/ELSE, DO loops. Read Cowlishaw chapters 1-4. Understand that everything is a string.
  2. Level 2: String Mastery — Master the built-in functions: SUBSTR, WORD, WORDS, LEFT, RIGHT, POS, TRANSLATE, STRIP. Use NUMERIC DIGITS for arbitrary precision. Write programs that process text files.
  3. Level 3: PARSE — The killer feature. Learn template-based parsing: word patterns, positional patterns, literal patterns. Replace regex with PARSE for structured text. Read Cowlishaw chapter 6.
  4. Level 4: Stems & Structures — Compound variables (stem.tail). Build arrays, hash tables, trees. Learn PROCEDURE EXPOSE for scope control. Implement a real data structure.
  5. Level 5: ARexx Concepts — Understand ADDRESS, host commands, the port system. Read the ARexx Reference Manual. Study how applications implement ARexx support. Read the exec.library message port docs.
  6. Level 6: Implementation Study — Read Regina REXX and BRexx source code. Understand the interpreter architecture: lexer, parser, evaluator. Study how PARSE and INTERPRET are implemented.
  7. Level 7: Port Architecture — Design the TypeScript interpreter. Map ARexx ports to MCP tools. Implement the PARSE engine. Build the string-based value model with NUMERIC DIGITS. Test against Regina for compatibility.
  8. Level 8: Native & GPU — x86-64 assembly interpreter. CUDA thread model. The escalation ladder complete. ARexx reborn in GSD-OS.

The secret: REXX is one of the most readable languages ever designed. If you can read English, you can read REXX. The learning curve is gentle. The depth is in the PARSE instruction, the arbitrary precision arithmetic, and the port system — and those are exactly the features that matter for our porting project.
