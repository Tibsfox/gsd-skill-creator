PARSE vs Regular Expressions

The fundamental difference: Regex describes what to match. PARSE describes what to extract. Regex finds patterns in text. PARSE decomposes text into named variables according to a template. For structured data (logs, configs, CSV, fixed-width), PARSE is often clearer and faster than regex.

Extract Fields from a Log Line

LanguageCode
REXXPARSE VAR line ip ' - - [' timestamp '] "' method ' ' path ' ' . '" ' status ' ' size
Perlmy ($ip, $ts, $method, $path, $status, $size) = $line =~ /^(\S+) - - \[([^\]]+)\] "(\S+) (\S+) [^"]*" (\d+) (\d+)/
Pythonm = re.match(r'^(\S+) - - \[([^\]]+)\] "(\S+) (\S+) [^"]*" (\d+) (\d+)', line)
Bash + awkecho "$line" | awk '{print $1, $4, $6, $7, $9, $10}'

The REXX PARSE version reads like a description of the format. The regex versions encode the same information as an opaque pattern string.

Parse a URL

/* REXX: parse a URL into components */
url = "https://api.example.com:8080/v2/users?id=42"
PARSE VAR url protocol '://' host ':' port '/' path '?' query

/* Result: protocol="https" host="api.example.com"
           port="8080" path="v2/users" query="id=42" */
# Python equivalent
from urllib.parse import urlparse
p = urlparse("https://api.example.com:8080/v2/users?id=42")
# p.scheme, p.hostname, p.port, p.path, p.query

Common Operations Across Languages

Variable Assignment & Types

LanguageCodeType System
REXXname = "Alice"; age = 30Typeless (everything is a string)
Pythonname = "Alice"; age = 30Dynamic (str, int are different types)
Perl$name = "Alice"; $age = 30;Dynamic (scalar context)
Bashname="Alice"; age=30Typeless (everything is a string)
TypeScriptlet name = "Alice"; let age = 30;Static (string, number)

REXX and Bash share the “everything is a string” model. Python and TypeScript distinguish types. This is the core porting challenge: REXX’s "42" + 1 = "43" must work transparently.

Associative Arrays / Dictionaries

LanguageCode
REXX (stems)color.sky = "blue"; color.grass = "green"; SAY color.sky
Pythoncolor = {"sky": "blue", "grass": "green"}; print(color["sky"])
Perlmy %color = (sky => "blue", grass => "green"); print $color{sky};
Bashdeclare -A color; color[sky]="blue"; echo ${color[sky]}
TypeScriptconst color: Record<string, string> = {sky: "blue"}; console.log(color.sky);

REXX stems are unique: student.1.name, student.1.grade — multi-level compound keys as a single dot-separated string. No other language has this natively.

Arbitrary Precision Arithmetic

LanguageCode
REXXNUMERIC DIGITS 100; SAY 2**256
Pythonprint(2**256) (built-in arbitrary precision integers)
Perluse Math::BigInt; print Math::BigInt->new(2)->bpow(256);
JavaScriptconsole.log(2n ** 256n) (BigInt, integers only)
Bashecho "2^256" | bc (external tool)

REXX does arbitrary precision decimal arithmetic, not just integers. NUMERIC DIGITS 100; SAY 1/3 gives 100 digits. Python integers are arbitrary but floats are IEEE 754. This is why REXX’s 0.1 + 0.2 = 0.3 (correct) while Python’s 0.1 + 0.2 = 0.30000000000000004.

The MCP connection: ARexx’s ADDRESS 'IMAGEFX' "LOAD FILE picture.iff" is structurally identical to an MCP tool call: invoke a named service with a string command, wait for a string result. ARexx scripts orchestrating multiple applications through ports ARE agents coordinating tools through MCP. The protocol is the same. The names have changed.
