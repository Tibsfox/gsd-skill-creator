Stage 1: TypeScript Interpreter

Why TypeScript? Already in our stack (gsd-skill-creator). V8 JIT provides good performance. async/await maps naturally to ARexx port waits. MCP integration is native. Largest pool of developers. Stage 1 is about correctness and ecosystem integration, not raw performance.

Stage 2: x86-64 Assembly Interpreter

Why assembly? The Amiga ethos: know your machine. A carefully written x86-64 REXX interpreter can be faster than any high-level port. Zero dependencies. Direct syscall interface. Understanding at the instruction level is core to our escalation ladder philosophy.

Memory Layout

+------------------+  High address
| Stack            |  PROCEDURE frames, DO nesting
+------------------+
| Heap             |  Dynamic allocation (stems, results)
+------------------+
| Port buffers     |  Shared memory for IPC
+------------------+
| Variable pool    |  Hash tables for variables
+------------------+
| String pool      |  Immutable string storage
+------------------+
| Code segment     |  REXX bytecode / threaded code
+------------------+  Low address

Stage 3: CUDA GPU Runtime

The vision: 10,000 concurrent REXX scripts on GPU. One thread block = one script instance. Shared memory = variable pool. Global memory = port buffers. The math co-processor already uses CUDA (18 tools, 125 tests). This extends it with REXX scripting.

The INTERPRET Challenge

Cannot dynamically compile on GPU. Solution: pre-compile all possible INTERPRET targets at launch time into a dispatch table. INTERPRET becomes an indirect device function call. For scripts with truly dynamic INTERPRET (constructing code from data), fall back to CPU execution for those clauses.

Batch REXX Execution — The Real Power

/* Launch 10,000 variants of a template script on GPU */
/* Each block processes different data with the same logic */

/* Template: Monte Carlo pi estimation */
NUMERIC DIGITS 20
inside = 0
DO i = 1 TO iterations
  x = RANDOM(0, 1000000) / 1000000
  y = RANDOM(0, 1000000) / 1000000
  IF (x*x + y*y) <= 1 THEN inside = inside + 1
END
pi = 4 * inside / iterations
ADDRESS 'RESULTS' pi        /* send result to collection port */

10,000 thread blocks, each running this script with different random seeds. Results collected via the GPU port system (global memory buffers read by CPU). Total execution: milliseconds for what would take minutes sequentially.

Cross-Stage Integration

  1. Stage 1 (TypeScript) runs in gsd-skill-creator. MCP integration. Development and testing platform. ANSI compliance suite passes here first.
  2. Stage 2 (x86-64) standalone native binary. Used for performance-critical paths: data processing pipelines, system automation, real-time scripting.
  3. Stage 3 (CUDA) extends the math co-processor. Batch execution of REXX scripts on GPU. Monte Carlo, parameter sweeps, data transformation at scale.
  4. All three share the MCP port bridge. A script in any stage can ADDRESS any MCP tool. The port system is the universal IPC layer.
