Why Rexx? Why now? Because ARexx is the soul of the Amiga, and the Amiga is the soul of this project. ARexx’s port system — where any application can script any other application through named message ports — is the most elegant inter-process communication model ever deployed on a personal computer. It predated COM, AppleScript, D-Bus, and MCP by years. It ran on a machine with 512KB of RAM. And its principles are exactly what GSD-OS needs: a universal scripting layer that connects tools, agents, and co-processors through message passing.

This is not just a research project. This is the architectural blueprint for bringing ARexx back — first as a modern language implementation, then as native x86-64 for performance, then as CUDA kernels running on the GPU. The escalation ladder, applied to a language.

Research method: Primary sources (Cowlishaw’s book, ANSI X3.274, ARexx documentation, Regina/ooRexx source), cross-reference against 289-project PNW Research Series corpus, Rosetta translation mapping, porting feasibility analysis

Mission period: April 2026 · Artemis II branch · Research Co-Processor v2

The ARexx Port System — IPC Done Right

/* The ARexx port system: any application talks to any application */

[ARexx Script] → ADDRESS 'IMAGEFX' → "LOAD FILE picture.iff"
                                        ← RC=0 RESULT="loaded"

[ARexx Script] → ADDRESS 'GOLDED.1' → "OPEN FILE readme.txt"
                                        ← RC=0 RESULT="42 lines"

[ARexx Script] → ADDRESS 'DOPUS.1' → "COPY FROM ram: TO work:"
                                        ← RC=0 RESULT="5 files copied"

/* One script. Three applications. Zero shared libraries. */
/* Just named ports, string messages, and string results. */

The insight: ARexx ports are exactly MCP tools. A named endpoint that accepts string commands and returns string results. The Model Context Protocol (2024) reinvented what Commodore shipped in 1990. Our implementation will make this explicit: ARexx ports ARE MCP tools. ADDRESS 'TOOL_NAME' IS a tool call. The script IS the agent. The port IS the server.

The Three-Stage Porting Pipeline

  1. Stage 1: Modern Language Port — TypeScript implementation within gsd-skill-creator. Full REXX interpreter: lexer, parser, AST, evaluator. ARexx ports mapped to MCP tool calls. PARSE engine, arbitrary precision NUMERIC DIGITS, stem variables. Integrates with existing infrastructure. Goal: run real ARexx scripts against real MCP tools.
  2. Stage 2: x86-64 Native Assembly — Hand-written x86-64 NASM interpreter. String operations via SSE4.2 (PCMPESTRI). Arbitrary precision via multi-register arithmetic. Port system via Unix domain sockets and shared memory. Zero dependencies. Goal: the fastest REXX interpreter on Earth.
  3. Stage 3: CUDA GPU Port — REXX interpreter running on GPU as part of the GSD math co-processor. One thread block = one script instance. Warp-cooperative string operations. Shared memory = variable pool. Global memory = port buffers. INTERPRET becomes dispatch table lookup. Goal: 10,000 concurrent REXX scripts on GPU for batch processing, Monte Carlo, parameter sweeps.

The vision: In GSD-OS, every MCP tool is an ARexx port. Every ARexx script is an agent. The ADDRESS instruction is how agents invoke tools. The PARSE instruction is how they process results. Arbitrary precision NUMERIC DIGITS powers the math co-processor. And when you need 10,000 concurrent agents, you launch them on the GPU.

Commodore got there first. We’re bringing it back.

A Taste of REXX

/* Classic REXX: readable, practical, human-friendly */
SAY "Enter a number:"
PULL number

IF DATATYPE(number, 'N') THEN DO
  NUMERIC DIGITS 100              /* 100-digit precision */
  factorial = 1
  DO i = 1 TO number
    factorial = factorial * i
  END
  SAY number"! =" factorial
END
ELSE
  SAY "That's not a number."
/* ARexx: orchestrating applications through ports */
ADDRESS 'IMAGEFX'                 /* talk to ImageFX */
"LOAD FILE 'photo.iff'"          /* load an image */
"SCALE 640 480"                   /* resize it */
"FILTER SHARPEN 50"               /* apply sharpening */

ADDRESS 'DOPUS.1'                 /* switch to Directory Opus */
"COPY FROM 'ram:photo.iff' TO 'work:output/'"

ADDRESS 'GOLDED.1'                /* switch to GoldED text editor */
"OPEN FILE 'work:output/log.txt'"
"INSERT 'Processed photo.iff at' DATE() TIME()"

SAY "Done. Three applications. One script."
