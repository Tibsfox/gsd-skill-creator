The Amiga’s preemptive multitasking and message-passing IPC created a unique opportunity. No other personal computer of the late 1980s could do what ARexx did: make every application scriptable and every application able to talk to every other application.

Why the Amiga Was Different

In 1990, MS-DOS ran one program at a time. The Mac used cooperative multitasking (one stuck program froze everything). Windows 3.0 was barely multitasking. The Amiga had preemptive multitasking since 1985, with a message-passing IPC system built into the kernel. This wasn’t an add-on — it was the foundation. Device drivers, GUI events, audio, graphics, and inter-process communication all used the same message port system.

When William Hawes built ARexx on top of this, he didn’t need to invent IPC. He inherited exec.library’s proven, fast, universal message system and added a REXX scripting layer on top. The result was unprecedented.

The Amiga’s flat memory space (no memory protection) enabled zero-copy message passing: you literally passed pointers. A RexxMsg from an ARexx script to ImageFX contained direct pointers to strings in the script’s memory space. This was fast and dangerous — exactly the Amiga ethos.

The depth of integration: ImageFX alone had over 200 ARexx commands. Directory Opus had its own ARexx scripting guide. GoldEd’s entire macro system was ARexx. MUI’s GUI toolkit could be driven entirely from ARexx scripts. This wasn’t token support — these applications were designed to be scripted.

The Deeper Connection

This project is not nostalgia. The Amiga’s architecture was right. Message-passing IPC, scriptable applications, typed memory pools, lightweight tasks, and a universal scripting language that connected everything — these are exactly the primitives that modern AI agent systems need. The Amiga had them in 1990 on a 7 MHz 68000 with 512KB of RAM.

GSD-OS inherits these principles. ARexx is the scripting layer. exec.library’s message ports are MCP tools. The Amiga’s memory model maps to our tiered storage (RAM, NVMe, GPU). The library system maps to our skill/agent hot-loading. We’re not recreating the Amiga. We’re completing what it started.
