The PARSE Engine

PARSE is REXX’s killer feature. It describes what to extract, not what to match. Where regex says “find this pattern,” PARSE says “the data looks like this — put the pieces in these variables.”

The PARSE Instruction Forms

FormSourceExample
PARSE ARGFunction/subroutine argumentsPARSE ARG name, age, city
PARSE VAR nameNamed variablePARSE VAR line word1 word2 rest
PARSE PULLQueue or stdinPARSE PULL command args
PARSE VALUE expr WITHExpression resultPARSE VALUE DATE() WITH day month year
PARSE SOURCEProgram invocation infoPARSE SOURCE system how name
PARSE VERSIONInterpreter versionPARSE VERSION lang ver date

Note: ARG (without PARSE) and PULL (without PARSE) uppercase the input. PARSE ARG and PARSE PULL preserve case.

Template Patterns

1. Word Patterns (Default)

Variables separated by spaces consume one whitespace-delimited word each. The last variable gets everything remaining.

line = "Alice 30 Seattle WA"
PARSE VAR line name age city state
/* name="Alice" age="30" city="Seattle" state="WA" */

line = "one two three four five"
PARSE VAR line first second rest
/* first="one" second="two" rest="three four five" */

2. Literal Patterns

A quoted string in the template marks a delimiter. Text before the delimiter goes into the preceding variable.

line = "name=Alice;age=30;city=Seattle"
PARSE VAR line 'name=' name ';age=' age ';city=' city
/* name="Alice" age="30" city="Seattle" */

url = "https://api.example.com:8080/v2/users?id=42"
PARSE VAR url protocol '://' host ':' port '/' path '?' query
/* protocol="https" host="api.example.com" port="8080"
   path="v2/users" query="id=42" */

csv = "Alice,30,Seattle,WA"
PARSE VAR csv name ',' age ',' city ',' state
/* name="Alice" age="30" city="Seattle" state="WA" */

3. Positional Patterns (Absolute)

A number specifies a column position (1-based). Text from the current position to that column goes into the preceding variable.

/* Fixed-width record: name(1-20) dept(21-30) salary(31-40) */
record = "Alice Smith         Engineering 85000     "
PARSE VAR record name 21 dept 31 salary
/* name="Alice Smith         " dept="Engineering" salary="85000     " */

4. Positional Patterns (Relative)

A signed number (+n or -n) moves the cursor relative to its current position.

data = "ABCDEFGHIJ"
PARSE VAR data first +3 second +4 third
/* first="ABC" second="DEFG" third="HIJ" */

/* Skip 5 characters, take 3 */
PARSE VAR data . +5 middle +3 .
/* middle="FGH" (dot is a placeholder variable) */

5. Variable Patterns

A parenthesized variable name is evaluated and used as a positional pattern.

start = 5
len = 3
PARSE VAR data . +(start) piece +(len) .
/* piece = characters starting at position 5, length 3 */

Combining Patterns

/* Apache log line — complex real-world parsing */
log = '192.168.1.1 - frank [10/Oct/2000:13:55:36 -0700] "GET /index.html HTTP/1.0" 200 2326'

PARSE VAR log ip ' - ' user ' [' timestamp '] "' method ' ' path ' ' proto '" ' status ' ' size

/* ip="192.168.1.1" user="frank" timestamp="10/Oct/2000:13:55:36 -0700"
   method="GET" path="/index.html" proto="HTTP/1.0" status="200" size="2326" */
/* Parse a key=value config file */
line = "  database.host = db.example.com  "
PARSE VAR line key '=' value
key = STRIP(key)
value = STRIP(value)
/* key="database.host" value="db.example.com" */
/* Parse an email address */
email = "foxy@tibsfox.com"
PARSE VAR email user '@' domain
/* user="foxy" domain="tibsfox.com" */

The PARSE Algorithm

For implementers: The PARSE template is processed left-to-right. Each pattern element adjusts a cursor position and assigns text to a variable:

  1. Start with cursor at position 1 of the source string.
  2. For each template element:
    • Variable name: Will receive text from cursor to next delimiter.
    • Literal string: Search forward from cursor. Assign preceding variable text up to (but not including) the match. Advance cursor past the match.
    • Absolute position n: Assign preceding variable text from cursor to position n-1. Set cursor to n.
    • Relative position +n: Assign text from cursor through cursor+n-1. Advance cursor by n.
    • Dot (.): Placeholder — same as a variable but discards the result.
  3. The last variable gets everything from cursor to end of string.
  4. If a literal is not found, the rest of the string goes to the preceding variable and remaining variables get empty strings.

Implementation Notes for Porting

TypeScript (Stage 1): Implement as a function that takes a source string and a compiled template (array of pattern elements). Each element is a discriminated union: {type: 'word', name}, {type: 'literal', value}, {type: 'absolute', position}, {type: 'relative', offset}. The parser compiles templates once, the evaluator runs them per-call.

x86-64 (Stage 2): String scanning via REPSCASB or SSE4.2 PCMPESTRI for literal pattern matching. Positional patterns are simple pointer arithmetic. The template is a bytecode array consumed by a tight loop.

CUDA (Stage 3): Warp-cooperative PARSE: 32 threads in a warp each test one character position for literal matches simultaneously. Positional patterns are trivial (thread 0 handles). Word splitting via parallel scan for whitespace.
