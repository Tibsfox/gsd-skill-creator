Research Modules

M1: Dataset Catalog Integration

Data Track A · DCAT/Schema.org

W3C DCAT vocabulary, DCAT-US federal metadata schema, schema.org/Dataset for Google Dataset Search, JSON-LD encoding patterns, CKAN API integration, Zenodo/CERN/USDA/NIST portal adapters, dataset freshness tracking, and version management.

M2: Learning Pathway Architecture

Pathway Track A · xAPI/LRS

xAPI (IEEE 9274.1.1-2023) statement vocabulary, Learning Record Store schema, concept dependency graphs in JSON-LD, the eight-layer mathematical progression, Bloom's taxonomy tracking, Zone of Proximal Development engine, and cross-department pathway bridges.

M3: Pedagogical Verbosity Engine

Pedagogy Track A · Literate programming

Three-tier annotation architecture (glance/read/study), Bloom-scaffolded comment templates per panel family, symbol-to-code mapping patterns, fill-in-the-blank and Target Practice scaffolding, complexity ceiling rules. The Knuth-Wall-Jupyter lineage.

M4: Rosetta Panel Code Standards

Panel Track B · Seven-language verbosity

Panel verbosity contracts for Python, C++, Java, Perl, Lisp, Pascal, and Fortran. Real-dataset integration patterns, auto-grade hooks for LRS tracking, cross-panel consistency rules, and the flagship exponential decay example across all seven families.

M5: Department Seeding Protocol

Lifecycle Track B · Onboarding/Maintenance

CPAN-inherited structural invariants, department lifecycle (proposed through maintained), concept-to-dataset mapping workflow, cross-department link registration API, stale-link detection skill, CAPCOM quality gates. Math and Cooking pilot departments.
