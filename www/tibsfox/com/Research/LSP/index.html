λThe Building Block

Why Lisp deserves the full treatment. This is not just another programming language project. Lisp is the second-oldest high-level language still in use (after Fortran), but more importantly it is the language from which the idea of “artificial intelligence” was launched. McCarthy coined the term in 1955 for the 1956 Dartmouth workshop and then, in 1958, invented Lisp to do the work. Every symbolic AI program from 1960 to 1990 was written in some Lisp. The expert systems that sold for $100K apiece, the theorem provers, the natural-language understanders, the first attempts at common sense — all of them. And when gradient descent ran away with the modern deep-learning era, the Lisp ideas quietly re-emerged in new clothing: automatic differentiation, computational graphs, program synthesis, and the rising interest in neurosymbolic reasoning. The loop closes.

The Central Thesis

// The Loop Closes //

In 1958, John McCarthy described a language in which programs were represented as lists, a list was a parseable tree, and a tiny function called eval could interpret any such tree as a running program. One page of definitions. That tiny function, reimplemented in Lisp itself, was the first meta-circular interpreter: a language that could describe itself.

For the next thirty years, nearly every attempt at artificial intelligence was written in some dialect of this language — because symbols and trees were what AI researchers needed, and because the ability to write programs that wrote programs was what made their research tractable. The expert systems era, the Lisp machines, MYCIN and CYC and MACSYMA, all of it was the same idea playing out at successively larger scales.

Then the symbolic era ended. Gradient descent won. Deep learning migrated to Python, tensors replaced lists, GPUs replaced the CADR. The Lisp world went quiet.

But the ideas didn’t die. Automatic differentiation is a program transformation — a macro in disguise. Computational graphs are S-expressions that happen to run on hardware. JAX traces functions and rewrites them — which is what Lisp programmers have been doing since 1960. Neurosymbolic AI is the acknowledgment that the symbolic tradition was always half of the answer. Code-generation LLMs read and write abstract syntax trees — which is what S-expressions already were.

The programmer of 2026 working with LLMs, autodiff, and program synthesis has more in common with the MIT AI Lab of 1975 than with the web developer of 2015. The loop closes. The parentheses were never the problem. They were the point.

A Taste — Factorial in Five Lisps

The same algorithm across the family, demonstrating dialect-specific flavor:

Common Lisp

(defun factorial (n)
  (if (<= n 1)
      1
      (* n (factorial (- n 1)))))

Scheme (R7RS)

(define (factorial n)
  (if (<= n 1)
      1
      (* n (factorial (- n 1)))))

Clojure

(defn factorial [n]
  (if (<= n 1)
    1
    (* n (factorial (dec n)))))

Racket

#lang racket
(define (factorial n)
  (if (<= n 1)
      1
      (* n (factorial (- n 1)))))

Emacs Lisp

(defun factorial (n)
  (if (<= n 1)
      1
    (* n (factorial (- n 1)))))

Note what’s different: almost nothing. Note what’s the same: the shape of the program is the shape of the tree the compiler sees. That identity of source text and data structure is the entire secret.
