Common Lisp — The Industrial Standard

The definitive text was Common Lisp: The Language (CLtL1, 1984) by Guy L. Steele Jr. (Digital Press). CLtL2 (1990) expanded to incorporate CLOS, the condition system, and the LOOP macro. ANSI X3.226-1994 became the formal standard in December 1994 — approximately 1,300 pages. It remains, as of 2026, the most recent and still-authoritative specification of any major Lisp dialect. There is no ANSI Common Lisp 2, and there does not need to be.

Common Lisp vs Scheme. Common Lisp bet on a single massive standard that froze the language. Scheme bet on a series of small standards (RnRS) that evolved continuously. Common Lisp is a working industrial language today, 32 years after its standard, because the spec is so complete that no implementation needs to extend it to be useful. Scheme is a research language today because each implementation extends the tiny spec in incompatible directions.

Quicklisp: The Turning Point

Before 2010, installing Common Lisp libraries meant downloading tarballs from random FTP sites, figuring out ASDF paths by hand, and debugging compatibility issues with no index. The ecosystem was tangled and demoralizing.

Quicklisp, released October 2010 by Zach Beane (Xach), changed everything. A single (ql:quickload :alexandria) fetches, resolves dependencies, compiles, and loads any library. Quicklisp distributes a curated set of libraries tested monthly against SBCL on Linux. As of 2025, Quicklisp tracks approximately 2,200 libraries in the monthly dist.

Scheme — The Elegant Minimalist

In 1975, Gerald Sussman and Guy Steele at the MIT AI Lab wrote a small Lisp interpreter to explore Carl Hewitt’s Actor model. The surprise: once they implemented closures and tail calls correctly, the distinction between actors and lambda expressions dissolved. That insight led to the Lambda Papers: “Lambda: The Ultimate Imperative,” “Lambda: The Ultimate Declarative,” “Lambda: The Ultimate GOTO.”

Scheme introduced or popularized: lexical scoping as the default (older Lisps used dynamic scoping); proper tail calls required by the language spec; first-class continuations via call-with-current-continuation.

Scheme was small. The entire language could be described in about 50 pages. The Revised Report on the Algorithmic Language Scheme — RnRS — went through R2RS, R3RS, R4RS, R5RS (1998, the classic), the controversial R6RS (2007) that split the community, and R7RS (2013) which tried to unify small (teaching) and large (production) variants.

Racket — Languages as Libraries

Racket, formerly PLT Scheme, is the PLT Group’s descendant of Scheme, rebranded in 2010. The central innovation: Racket is a language for making languages. The #lang system lets you ship a new language as a regular library. Users include #lang my-language at the top of their file.

DrRacket (formerly DrScheme) is considered one of the best pedagogical programming environments ever built. How to Design Programs (HtDP) by Felleisen, Findler, Flatt, and Krishnamurthi is the design-recipe textbook that guides students through increasingly complex Racket programs.

For the deep dive into how Racket’s macro system enables languages as libraries, see Metaprogramming.

Clojure — The Modern Pragmatist

Rich Hickey, an independent consultant in New York, took a sabbatical from 2005 to 2007 to design and build Clojure. He wanted a Lisp for the JVM because that’s where the enterprise work was, and the JVM’s libraries were overwhelming compared to anything Common Lisp had. He released Clojure in October 2007 as open source.

Core Design Choices

  1. Immutability as default — data structures are persistent (Hash Array Mapped Tries based on Phil Bagwell’s 2000 paper). Mutation is opt-in.
  2. Functions over methods — dispatch on protocols, not on a single receiver’s class hierarchy.
  3. Hosted language — Clojure is a guest on the JVM (and later JS, CLR, BEAM, Dart). It embraces host interop.
  4. Data over code — prefer data structures (maps, vectors) for configuration and messaging over classes and interfaces. “Data is the API.”
  5. Concurrency-safe primitives — STM, agents, atoms, refs. Four reference types sorted by (sync/async) × (coordinated/uncoordinated).
  6. Values over places — Hickey’s 2012 talk “The Value of Values” argues that programming should manipulate immutable values rather than mutable locations.

Functional Relatives (Not Lisps, But Family)

The ML family descends culturally from Lisp via the typed lambda calculus research program. These are functional cousins, not Lisps:

  • ML (1973, Robin Milner, Edinburgh) — originally the “metalanguage” for LCF
  • Standard ML (1990 formal definition), OCaml (1996, INRIA)
  • Haskell (1990, Peyton Jones et al.)
  • F# (2005, Don Syme, Microsoft)
  • Elm (2012, Evan Czaplicki)
  • PureScript (2013, Phil Freeman)
  • Unison (2014, Paul Chiusano, Rúnar Bjarnason) — content-addressed code

These share Lisp’s commitment to expressions over statements, first-class functions, recursion as the basic control construct, and immutability as a default. They do not share parentheses, macros, or homoiconicity.

Julia: Not a Lisp, But a Lisp in Disguise

Julia (2012, Bezanson, Karpinski, Shah, Edelman) has an Algol-like surface syntax. But underneath, Julia is a Lisp. The creators are explicit. Julia’s parser produces an S-expression-shaped AST, manipulated by macros before compilation. Julia has multiple dispatch (directly inspired by CLOS), macros that operate on the AST, metaprogramming through Expr objects (which are Lisp cons cells with a different name), and homoiconicity in the sense that Julia code is expressible as Julia data.

Jeff Bezanson has described Julia as “Scheme with types and Ruby-ish syntax.” Stefan Karpinski has said that Julia’s multiple dispatch comes directly from CLOS and Dylan.
