1. CMU ML in Production — 16-Lecture Study Guide

Intermediate CML AGT

A structured guide through Carnegie Mellon’s 16-lecture series on deploying machine learning systems safely in production. The core thesis: safety does not equal reliability.

  • STPA (Systems-Theoretic Process Analysis): Leveson’s hazard analysis for complex systems

2. OWASP Application Security Learning Path

Intermediate AGT FAIS

A learning path through OWASP application security fundamentals, extended with AI-specific threat models from the Artemis II security research.

Learning Path

  1. OWASP Top 10 (Web): injection, broken auth, XSS, SSRF, security misconfiguration
  2. OWASP Top 10 for LLMs: prompt injection, training data poisoning, model denial of service
  3. Indirect injection taxonomy: tool poisoning, context manipulation, output steering
  4. Agent-specific threats: DDIPE attacks, skill marketplace supply-chain risks
  5. Defensive patterns: input sanitization, output validation, multi-model verification

Practical Exercises

  • Run OWASP ZAP against a local test application
  • Build an input sanitizer for LLM-generated SQL queries
  • Implement the multi-model verification ensemble from the AI Programming module
  • Analyze a skill definition for hidden injection patterns

3. CppCon Systems Programming Track

Advanced CSP HPC

A curated track through CppCon 2025 systems programming talks, covering memory models, VLIW architecture, and high-frequency trading optimizations.

Talk Sequence

  1. Memory Models & Concurrency: C++ memory model, atomics, lock-free data structures
  2. VLIW “Impossible” Computers: trace scheduling, explicitly parallel architectures
  3. Bloomberg Ticker Plant: 55x optimization via cached index snapshots at ~4–5 cycles/record
  4. Lime Trading: 2M msg/sec zero-loss fair distribution on 4 cores via work contracts
  5. Memory-Centric Computing: 50%+ CPU time waiting for memory, 60–90% energy in data movement

Key Concepts

  • Cache-oblivious algorithms: minimize memory access regardless of cache size
  • Zero-copy networking: kernel bypass with DPDK or io_uring
  • NUMA-aware allocation: bind threads to cores near their memory
  • Compile-time computation: constexpr and template metaprogramming for zero-runtime-cost abstractions

4. Corporate Finance — Damodaran Pathway

Intermediate ACC

A structured pathway through Aswath Damodaran’s corporate finance and accounting principles, synthesized from Artemis II business research.

Study Path

  1. Financial statements: balance sheet, income statement, cash flow — reading and connecting them
  2. Valuation fundamentals: DCF, relative valuation, intrinsic value vs market price
  3. Cost of capital: WACC, risk-free rate, equity risk premium
  4. Corporate governance: agency costs, board structure, shareholder rights
  5. IP strategy for tech companies: patent vs trade secret, AI-generated artifact ownership

Practical Exercises

  • Value a small tech company using DCF with three growth scenarios
  • Calculate WACC for a SaaS startup vs an established hardware company
  • Analyze the IP implications of AI-generated code in an open-source project

A certification-prep guide covering the five configuration management primitives discovered during the PLM engineering research.

  • BOM chaos: PLM-to-ERP-to-MES drift as primary configuration management failure mode
  • Drift as primary risk: organizational training and continuous audit prevent configuration erosion

6. Structural Engineering — Stiffness Method Tutorial

Advanced BCM SOP

A tutorial on the direct stiffness method for structural analysis, connecting to the engineering research on hover brackets and impact dynamics.

Learning Path

  1. Truss analysis: pin-jointed structures, axial force only
  2. Beam elements: bending, shear, and moment distribution
  3. Frame analysis: combined axial and bending with rigid joints
  4. Assembly: element stiffness matrices into global stiffness matrix
  5. Solution: apply boundary conditions, solve Ku = F

Practical Exercises

  • Analyze a simple truss by hand using the stiffness method
  • Implement a 2D frame solver in Python using NumPy
  • Validate against a known solution (cantilever beam deflection)
  • Design a hover bracket for 150–3,000 lb capacity using the stiffness method results

7. Legal Technology & IP Strategy Primer

Beginner ACC

An introduction to intellectual property strategy for technology companies, with special attention to AI-generated artifact ownership.

Key Topics

  • IP types: patent, trade secret, copyright, trademark — when to use each
  • AI-generated content: current legal landscape for AI-authored code and artifacts
  • Open source licensing: MIT, Apache 2.0, GPL, AGPL — implications for AI projects
  • Trade secret protection: NDAs, access controls, and documentation requirements
  • Fintech compliance: zero-tolerance financial software in regulatory environments

Practical Exercises

  • Audit an open-source project for license compatibility
  • Draft a basic IP assignment clause for contractor agreements
  • Create an access control matrix for trade secret materials
