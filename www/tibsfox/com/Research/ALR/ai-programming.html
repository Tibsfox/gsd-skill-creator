1. Claude Code Skill Development Tutorial

Intermediate SAA GSD

Build a custom Claude Code skill from scratch using the YAML-based skill definition format. Skills auto-activate based on context and provide domain-specific guidance without explicit invocation.

What You Build

  • A .claude/skills/ skill definition with activation triggers, instructions, and tool recommendations
  • Context-aware activation using file patterns, conversation keywords, and project state
  • Integration with GSD workflow for automated phase execution

Key Concepts

  • Skill anatomy: name, description (250-char cap), instructions, activation conditions
  • Auto-activation: skills load based on file context, not explicit calls
  • Token budget: each skill consumes context window — monitor with sc:status
  • Security hygiene: the security-hygiene skill validates self-modifying operations

Prerequisites

Claude Code CLI installed, a project with .claude/ directory, familiarity with YAML

2. MCP Server Building Guide

Intermediate AGT FAIS

Build a Model Context Protocol server in both Python and TypeScript. MCP servers expose tools, resources, and prompts to AI agents via a standardized JSON-RPC protocol.

What You Build

  • A Python MCP server using the mcp SDK with tool definitions and resource handlers
  • A TypeScript MCP server using @modelcontextprotocol/sdk with streaming support
  • Tool input validation using JSON Schema and Zod

Key Concepts

  • Protocol layers: transport (stdio/SSE), session, tool/resource/prompt primitives
  • 110M SDK downloads as of April 2026 — MCP is the standard agent integration layer
  • Security model: tools execute with host permissions; validate all inputs
  • 3,600x adoption growth since November 2024 (MCP Dev Summit data)

Prerequisites

Python 3.11+ or Node.js 20+, understanding of JSON-RPC, API design basics

3. Agent Orchestration Patterns

Advanced MAS AGT

Implement multi-agent orchestration using the patterns proven during Artemis II: convoy dispatch, worker pools, hierarchical supervision, and result aggregation.

What You Build

  • A convoy-based orchestrator that dispatches work items to parallel agent workers
  • A supervisor pattern (meta-harness) where one agent monitors others for stalls and errors
  • Result aggregation with conflict resolution and merge strategies

Key Concepts

  • OrgAgent pattern: 102.73% improvement over flat MAS with 74.52% token reduction (arXiv:2604.01020)
  • Data Processing Inequality: single-agent outperforms multi-agent on multi-hop reasoning (arXiv:2604.02460)
  • Right-sizing: use multi-agent for embarrassingly parallel work, single-agent for deep reasoning
  • 12 agentic primitives: QUERY, COMPUTE, OBSERVE, COMPARE, CONNECT, PUBLISH, COMMIT, VERIFY, plus four more

4. LightRAG Integration Hands-On

Intermediate AGT CSP

Integrate LightRAG — a graph layer above vector embeddings — for knowledge retrieval that understands relationships, not just similarity.

What You Build

  • A document ingestion pipeline that builds both vector and graph indices
  • Query routing that selects vector search, graph traversal, or hybrid based on query type
  • Integration with pgvector for persistent embedding storage

Key Concepts

  • Graph + vector: LightRAG adds entity-relation graphs above standard embeddings
  • Context engineering formula: define output first, then construct start state to reach it
  • Token optimization: context compression reduces cost without losing retrieval quality
  • all-MiniLM-L6-v2: embedding model used in the Artemis II pgvector pipeline

5. Tool Poisoning Detection Lab

Advanced AGT FAIS

Build a detection system for DDIPE (Dynamic Data-Driven Indirect Prompt Engineering) attacks against AI coding agents, based on arXiv:2604.03081.

Security research context: This lab teaches defensive techniques. The 631,813 skills in the SkillsMP marketplace have no mandatory security review. Understanding attack vectors is essential for building safe agent systems.

What You Build

  • A static analyzer that scans skill/tool definitions for injection patterns
  • A runtime monitor that detects anomalous tool call sequences
  • A multi-model verification ensemble that cross-checks outputs across Claude, GPT, and Gemini

Key Concepts

  • 11.6–33.5% DDIPE bypass rates across Claude Code, OpenHands, Codex, Gemini CLI
  • 1.6% payloads bypass all three models simultaneously — cross-model defense works
  • 15 MITRE ATT&CK categories covered by 1,070 adversarial skill payloads
  • Zero-trust agent architecture: assume every tool call is potentially compromised

6. Multi-Model Verification Ensemble

Advanced AGT MAS

Set up a verification ensemble where multiple LLMs cross-check each other’s outputs, reducing hallucination and catching fabricated content.

What You Build

  • A routing layer that sends queries to multiple models in parallel
  • A consensus algorithm that identifies agreement and flags divergence
  • Integration with the MIRAGE benchmark for grounding verification

Key Concepts

  • MIRAGE problem: models fabricate visual descriptions for absent images at >60% rate (arXiv:2603.21687)
  • Epistemic symmetry: LLMs exhibit bidirectional Q&A capability — test both directions
  • Cost management: 658x cost inflation from MCP overthinking loops (arXiv:2602.14798)
  • Consensus threshold: require 2-of-3 agreement before accepting output

7. Context Compression Techniques

Intermediate AGT CSP

Implement practical context compression for long-context applications, reducing token usage while preserving retrieval quality and reasoning capability.

What You Build

  • A prompt compressor that identifies and removes redundant information
  • A cached index snapshot system for repeated queries (~4–5 cycles/record, 55x optimization)
  • A context window budget allocator that prioritizes high-value information

Key Concepts

  • Context engineering as formula: output = f(start_state) — work backwards from desired result
  • Cognitive load theory: applied to AI tool design for human-in-the-loop systems
  • Token budget tracking: enforce hard limits with pre-execution checking per convoy
  • DAG context resolution: resolve dependencies to build minimal sufficient context
