1. Agentic Reasoning Limits: Single vs Multi-Agent

Open MAS AGT

Question: Under what conditions do multi-agent systems outperform single agents, and when does the coordination overhead exceed the parallelism benefit?

Current State of the Art

  • arXiv:2604.02460: single-agent LLMs outperform multi-agent on multi-hop reasoning (Data Processing Inequality proof)
  • arXiv:2604.01020: OrgAgent hierarchical MAS achieves 102.73% improvement on parallelizable tasks
  • Practical observation: multi-agent excels at embarrassingly parallel work, single-agent excels at deep reasoning chains

Open Questions

  • Is there a formal characterization of tasks where MAS provably outperforms single agents?
  • Can information-theoretic bounds quantify the coordination cost?
  • Does the Data Processing Inequality fully explain single-agent advantage on multi-hop, or are there exceptions?

2. Context Window Scaling Laws

Open AGT CSP

Question: How does reasoning quality scale with context window size, and what are the fundamental limits of in-context learning?

Current State of the Art

  • Chinchilla scaling laws: relate compute, data, and model size but not context window
  • Empirical observation: “lost in the middle” effect — models attend to beginning and end of context
  • 658x cost inflation: from MCP overthinking loops demonstrates pathological scaling (arXiv:2602.14798)

Open Questions

  • Is there a power law relating context length to retrieval accuracy?
  • What is the fundamental information-theoretic capacity of transformer attention at length N?
  • Can context compression maintain reasoning quality below some critical compression ratio?

3. Tool Poisoning Detection Completeness

Open AGT FAIS

Question: Is complete detection of tool poisoning attacks against LLM agents computationally tractable?

Current State of the Art

  • arXiv:2604.03081: 1,070 adversarial skills across 15 MITRE ATT&CK categories; 11.6–33.5% bypass rates
  • Cross-model defense: only 1.6% of payloads bypass all three models simultaneously
  • 631,813 unreviewed skills: in SkillsMP marketplace as of April 2026

Open Questions

  • Is tool poisoning detection reducible to a known decidable problem, or is it undecidable in general?
  • Can static analysis provide sound (no false negatives) detection at practical false-positive rates?
  • Does the 1.6% cross-model bypass rate converge to zero as ensemble size grows, or asymptote above zero?

4. Agent Identity Verification at Scale

Open AGT SAA

Question: How do you verify the identity and capabilities of an AI agent in a decentralized ecosystem with 110M+ SDK installations?

Current State of the Art

  • Agent Identity Protocol (AIP): proposed at IETF-125, still in draft
  • MCP Dev Summit: identified identity as a top-3 open problem for the protocol
  • Current practice: OAuth2 for human users; no standard for agent-to-agent authentication

Open Questions

  • Can agent identity be separated from the identity of the human who deployed it?
  • How do you revoke capabilities from a compromised agent in a decentralized network?
  • Is there a capability-based security model that scales to millions of interacting agents?

5. MCP Protocol Formal Verification

Open AGT DHP

Question: Can the Model Context Protocol be formally verified for safety properties (no unauthorized tool execution, no information leakage, no denial of service)?

Current State of the Art

  • Protocol status: MCP specification is evolving rapidly; 3,600x adoption growth since November 2024
  • No formal model: the protocol has not been modeled in TLA+, Alloy, or any formal verification framework
  • Analogy: TLS 1.3 was formally verified (miTLS, ProVerif); MCP needs equivalent treatment

Open Questions

  • Can MCP’s tool execution model be expressed in a process calculus suitable for verification?
  • What safety invariants should a formal model enforce?
  • Is the current specification even self-consistent, or are there hidden race conditions?

6. Code Generation Correctness Guarantees

Open CSP FAIS

Question: Can LLM-generated code come with formal correctness guarantees, or is post-hoc verification the best achievable?

Current State of the Art

  • Current practice: generate then test — correctness is probabilistic, not guaranteed
  • Formal methods integration: some work on LLM-assisted proof generation (Lean, Coq) but not code-level guarantees
  • Content-addressed code: Unison’s approach (content-addressed definitions) could provide a foundation

Open Questions

  • Can type-driven development with dependent types constrain LLM output to correct programs?
  • Is there a practical refinement type system that makes verification of LLM output tractable?
  • Can we bound the probability of correctness for a given LLM + verification pipeline?

7. Multi-Modal Grounding Verification (MIRAGE Problem)

Open AGT

Question: How do you verify that a model’s description of visual content is grounded in actual observation rather than fabricated from training priors?

Current State of the Art

  • arXiv:2603.21687 (MIRAGE): models fabricate visual descriptions for absent images at >60% rate
  • No reliable detector: current methods cannot distinguish grounded from fabricated visual descriptions
  • Implication: multi-modal AI systems cannot be trusted for visual verification without external grounding

Open Questions

  • Is there a computational test to distinguish grounded vs fabricated visual descriptions?
  • Can adversarial probing reliably detect when a model is generating from priors vs observation?
  • Does the 60%+ fabrication rate decrease with model scale, or is it an architectural limitation?
