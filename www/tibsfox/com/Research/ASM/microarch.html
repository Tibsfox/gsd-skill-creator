The ISA is the contract between software and hardware. The microarchitecture is how the hardware fulfills that contract. Two CPUs implementing the same ISA can have wildly different performance.

The Classic 5-Stage Pipeline

  1. Fetch — read the next instruction from memory (instruction cache)
  2. Decode — determine what the instruction does, which registers it uses
  3. Execute — perform the operation (ALU, FPU, address calculation)
  4. Memory — load from or store to memory (data cache)
  5. Writeback — write the result back to a register

Pipelining lets the CPU work on 5 instructions simultaneously (one per stage), increasing throughput. Modern CPUs have 15–20+ stage pipelines.

Cache lines are 64 bytes on x86. Every memory access loads a full cache line. False sharing (two threads writing to the same cache line) is a common performance bug visible only at the assembly level.

Out-of-Order Execution

Modern CPUs execute instructions out of program order to fill pipeline bubbles. Tomasulo’s algorithm (1967): reservation stations hold instructions waiting for operands; the reorder buffer (ROB) ensures results retire in order. Register renaming maps architectural registers to a larger set of physical registers, eliminating false dependencies (WAR, WAW).

Branch Prediction

Branches are the enemy of pipelining. Modern predictors (TAGE, perceptron-based) are 97%+ accurate. A misprediction costs a full pipeline flush (~15–20 cycles on modern x86). This is why if vs switch vs cmov (conditional move) can matter in hot loops.

Virtual Memory

The CPU translates virtual addresses to physical addresses via page tables. Default page size: 4 KB. Hugepages: 2 MB / 1 GB. The TLB (Translation Lookaside Buffer) caches translations. A TLB miss costs ~10–100 cycles for a page walk. ASLR randomizes addresses for security.

Interrupts and Exceptions

  • Hardware interrupts: timer, I/O, inter-processor interrupts
  • Exceptions: page faults, divide by zero, general protection faults
  • Software interrupts: syscall (x86-64), svc (ARM64), ecall (RISC-V)

The Interrupt Descriptor Table (IDT, x86) maps interrupt numbers to handler addresses.

Spectre and Meltdown (2018)

Speculative execution side-channel attacks discovered in January 2018. Spectre: bounds check bypass via speculative loads leaks data through cache timing. Meltdown: speculative kernel memory reads from userspace. Mitigations: KPTI (kernel page-table isolation), retpoline, IBRS, STIBP, MDS buffers. Performance impact: 2–30% depending on workload. Changed how CPUs are designed.
