The “Compiler Is Usually Smarter” Rule

Modern optimizing compilers (GCC -O2, Clang -O2, MSVC /O2) produce assembly that is better than most humans can write. The compiler knows the microarchitecture, the pipeline, the register allocation constraints, and the instruction scheduling rules. Hand-written assembly is justified only when:

  • The compiler cannot use the instruction you need (AES-NI, CPUID, special system registers)
  • The code must be constant-time (crypto) and the compiler might optimize away your timing protections
  • You need to control exact register state (context switches, interrupt handlers)
  • You are writing a JIT compiler that emits machine code at runtime
  • You are doing reverse engineering or exploit development

SIMD in Practice

; AVX2: multiply 8 floats in parallel
vmovups ymm0, [rdi]          ; load 8 floats from array A
vmovups ymm1, [rsi]          ; load 8 floats from array B
vmulps  ymm2, ymm0, ymm1    ; multiply all 8 in parallel
vmovups [rdx], ymm2          ; store 8 results

This single AVX2 sequence processes 8 multiplications in one instruction. The equivalent C loop would take 8 iterations. SIMD is why video decoding, image processing, and machine learning inference run as fast as they do.

Security: Assembly-Level Attacks

  • Buffer overflow — write past a buffer to overwrite the return address on the stack. Classic exploitation technique (Morris worm, 1988).
  • ROP (Return-Oriented Programming) — chain small instruction sequences (“gadgets”) ending in ret to build arbitrary computation without injecting code.
  • Shellcode — hand-crafted assembly injected into a vulnerable process. Must be position-independent, often avoid null bytes.
  • Spectre/Meltdown — microarchitectural side-channel attacks that leak data via speculative execution (cache timing).

Defenses: Stack canaries, ASLR, DEP/NX (non-executable stack), Control-Flow Integrity (CFI), Shadow Stack (Intel CET), retpoline.

WebAssembly — The Virtual Assembly

WebAssembly (Wasm) is a stack-based virtual ISA designed for the browser (and increasingly for server-side). It is NOT real assembly — it’s an intermediate representation that JIT compilers translate to real assembly (x86-64 or ARM64) at runtime. But it occupies the same conceptual role: “the lowest level the programmer can target.”
