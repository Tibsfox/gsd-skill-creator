x86-64 Architecture

Registers, Instructions, Calling Conventions, SIMD — The Dominant Desktop/Server ISA
Overview x86-64 ARM64 Microarch Tools Modern Rosetta Learn History
x86-64 (AMD64) is the ISA running most of the world’s servers, desktops, and laptops. It’s a CISC architecture with a 46-year lineage from the Intel 8086 (1978), extended to 64-bit by AMD in 2003.

Hello World (Linux, Intel syntax, NASM)

; hello.asm — x86-64, Linux, NASM
section .data
    msg db "Hello, world!", 10    ; 10 = newline
    len equ $ - msg

section .text
    global _start

_start:
    mov rax, 1          ; syscall: write
    mov rdi, 1          ; fd: stdout
    mov rsi, msg        ; buffer
    mov rdx, len        ; length
    syscall

    mov rax, 60         ; syscall: exit
    xor rdi, rdi        ; status: 0
    syscall
# Assemble and link:
nasm -f elf64 hello.asm -o hello.o
ld hello.o -o hello
./hello

System V AMD64 Calling Convention

Arguments: 1st–6th integer/pointer in rdi, rsi, rdx, rcx, r8, r9. Floats in xmm0–xmm7. Remaining on stack.

Return: rax (integer), rdx (2nd return), xmm0 (float).

Caller-saved: rax, rcx, rdx, rsi, rdi, r8–r11, xmm0–xmm15.

Callee-saved: rbx, rbp, r12–r15.

Stack: 16-byte aligned before call. Red zone: 128 bytes below rsp that leaf functions can use without adjusting the stack pointer.

A function in assembly

; int add(int a, int b) — System V AMD64
add:
    ; a in edi, b in esi (32-bit args, zero-extended)
    lea eax, [edi + esi]    ; eax = a + b
    ret                      ; return value in eax
