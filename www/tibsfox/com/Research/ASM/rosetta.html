Every language in this research series — C, C++, Rust, Go, Java, JavaScript, Python, Perl — eventually becomes assembly. Assembly is the final representation before the transistors switch.

How Each Language Becomes Assembly

The ABI — The Contract Between Languages and Assembly

The Application Binary Interface is the contract that makes cross-language interop possible. It defines: register usage, calling conventions, stack alignment, struct layout, name mangling, exception handling tables, and thread-local storage. The C ABI (System V AMD64 on Linux, AAPCS64 on ARM) is the lingua franca — every language that calls C (via FFI) must speak the C ABI.

C Was Designed as “Portable Assembly”

Dennis Ritchie designed C at Bell Labs (1972) specifically to be close enough to assembly that a systems programmer could predict what the compiler would emit, while being portable enough to run on different architectures. The result: C maps to assembly more directly than any other high-level language. The CEE research project covers this lineage.

The Escalation Ladder (Complete)

  1. Bash — interpreted by a C program (compiled to assembly)
  2. Perl / Python / JavaScript — interpreted or JIT-compiled; the runtime is C/C++ (compiled to assembly)
  3. Java / Go / C# — compiled to bytecode, then JIT-compiled to assembly
  4. Rust / C++ — compiled to assembly via LLVM or GCC
  5. C — compiled to assembly (“portable assembly”)
  6. Assembly — you are here. One step above machine code.
  7. Machine code — raw binary opcodes the CPU decodes
  8. Microarchitecture — how the CPU executes the machine code (pipeline, OOO, caches)
  9. Transistors — the physics layer
