Assembly language is not one language but a family: one dialect per CPU architecture, each a human-readable representation of the machine code that CPU actually executes. Every other language in the PNW Research Series eventually compiles down to this.

Why assembly language in 2026? You will never write a production application in assembly. But you will read it — in compiler output (Compiler Explorer), in debugger backtraces, in kernel crash dumps, in security advisories, in performance profiles. Understanding assembly is what separates “I write code that runs fast” from “I understand why my code runs fast.” Every language in this series — C, C++, Rust, Go, Java (via JIT), JavaScript (via V8) — eventually becomes assembly. This project covers what it looks like when it gets there.

Research method: Intel SDM, ARM Architecture Reference Manual, RISC-V spec, Patterson & Hennessy, Agner Fog’s optimization manuals, Compiler Explorer, Linux kernel source, and cross-reference against the 286-project PNW Research Series corpus.

The Three Major Architectures

x86-64 (AMD64)

The dominant desktop and server ISA. CISC (Complex Instruction Set). Variable-length instructions (1–15 bytes). 16 general-purpose registers. Intel and AMD. The x86-64 module covers registers, instructions, calling conventions, and SIMD.

ARM64 (AArch64)

The dominant mobile, embedded, and increasingly server ISA. RISC (load/store architecture). Fixed 32-bit instructions. 31 general-purpose registers. Apple M-series, AWS Graviton, Qualcomm, every smartphone. The ARM64 module covers the instruction set, NEON/SVE, and AAPCS64.

RISC-V

The open-source ISA from UC Berkeley (2010). No licensing fees. Modular design (base + extensions). Growing in embedded, education, and research. The future competitor to ARM. Covered in the ARM64 module alongside ARM.

Microarchitecture

What happens inside the CPU: pipelines, out-of-order execution, branch prediction, caches, TLB, speculative execution. The ISA is the contract; the microarchitecture is the implementation. The microarchitecture module covers the hardware.
