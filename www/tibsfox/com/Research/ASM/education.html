The best way to learn assembly in 2026: write C, read the assembly output in Compiler Explorer, and trace what the compiler did. Start with x86-64 or ARM64, not historical ISAs.

Online Resources

  • Compiler Explorer (godbolt.org) — THE primary learning tool. Write C, see assembly.
  • x86 and amd64 instruction reference (felixcloutier.com) — searchable x86 instruction reference extracted from Intel SDM
  • CS:APP (Computer Systems: A Programmer’s Perspective, Bryant & O’Hallaron) — the systems programming textbook, includes extensive x86 assembly
  • Ben Eater’s YouTube — building a CPU from scratch, 8-bit computer series
  • LiveOverflow — binary exploitation, CTF walkthroughs, reverse engineering
  • pwn.college — ASU’s free cybersecurity course, heavy on assembly and exploitation

A Reading Order That Works

  1. Compiler Explorer — write simple C functions, read the assembly output with -O2
  2. Patterson & Hennessy (COD, RISC-V edition) — chapters 1–5 for ISA and pipeline
  3. NASM + Linux — write hello world, a function that adds numbers, a loop
  4. GDB — step through your assembly programs instruction by instruction
  5. CS:APP chapters 3, 5, 6 — machine-level programming, optimization, memory hierarchy
  6. Agner Fog’s manuals — when you want to understand why one instruction sequence is faster
  7. Ghidra or IDA — reverse-engineer a small binary for practical assembly reading
  8. Read kernel source — Linux arch/x86/entry/entry_64.S is the canonical real-world assembly
