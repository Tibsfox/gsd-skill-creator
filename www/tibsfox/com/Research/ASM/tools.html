The assembly toolchain: assemblers turn mnemonics into machine code, linkers combine object files, debuggers let you step through instructions, and disassemblers turn binary back into readable assembly. Compiler Explorer changed how everyone learns.

Linkers and Object Formats

LinkerNotes

Object file formats: ELF (Linux/BSD), Mach-O (macOS), PE/COFF (Windows). Sections: .text (code), .data (initialized data), .bss (uninitialized), .rodata (read-only constants).

Debuggers

# GDB: step through assembly
gdb ./myapp
(gdb) break _start
(gdb) run
(gdb) disassemble
(gdb) stepi                # step one instruction
(gdb) info registers       # show all registers
(gdb) x/16xb 0x7fffffffde00  # examine 16 bytes of memory
(gdb) set $rax = 42        # modify a register

Compiler Explorer (godbolt.org)

Matt Godbolt (2012). Paste C/C++/Rust/Go/Java code, see the assembly output from dozens of compilers, with color-coded source-to-assembly mapping and optimization flags. THE tool for learning assembly in 2026. Write C, read the assembly output. Changed how an entire generation thinks about compilers.

Inline Assembly

// GCC extended asm (C)
uint64_t rdtsc(void) {
    uint32_t lo, hi;
    asm volatile ("rdtsc" : "=a"(lo), "=d"(hi));
    return ((uint64_t)hi << 32) | lo;
}

// Rust asm! macro (stable since 1.59)
use std::arch::asm;
let result: u64;
unsafe {
    asm!("rdtsc", out("eax") _, out("edx") _, lateout("rax") result);
}
