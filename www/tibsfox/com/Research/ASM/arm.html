ARM64 (AArch64) runs every smartphone, every Apple Mac since 2020, and an increasing share of cloud servers. RISC-V is the open-source challenger. Both are RISC: load/store architecture, fixed-width instructions, large register files.

ARM64 Hello World

// hello.s — AArch64, Linux
.data
msg:    .ascii "Hello, world!\n"
len =   . - msg

.text
.global _start
_start:
    mov x8, #64         // syscall: write
    mov x0, #1          // fd: stdout
    ldr x1, =msg        // buffer
    mov x2, #len        // length
    svc #0              // syscall

    mov x8, #93         // syscall: exit
    mov x0, #0          // status: 0
    svc #0

NEON and SVE

NEON: 32 × 128-bit vector registers (v0–v31). Packed integer and floating-point SIMD. Available on all AArch64.

SVE (Scalable Vector Extension): Vector-length-agnostic (128–2048 bits). Predicate registers for per-lane masking. Gather/scatter. SVE2 adds cryptographic and fixed-point operations. ARM Neoverse V-series implements SVE.

RISC-V

The open-source ISA. Designed at UC Berkeley (2010) by Krste Asanovic, Andrew Waterman, and Yunsup Lee. Free: no licensing fees, no royalties. Governed by RISC-V International.

  • 32 integer registers (x0–x31; x0 hardwired to zero)
  • Modular: base ISA (RV32I/RV64I) + standard extensions: M (multiply), A (atomic), F (float), D (double), C (compressed 16-bit), V (vector)
  • Privileged modes: Machine (M), Supervisor (S), User (U)
  • System call: ecall
