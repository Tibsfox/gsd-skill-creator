# The Claude Code Source Code Incident — March 31, 2026

## What Happened

On March 31, 2026, security researcher **Chaofan Shou** discovered that Anthropic's Claude Code — their flagship agentic CLI tool — had its entire source code exposed through a **source map file** inadvertently published to the npm registry.

A **59.8 MB JavaScript source map** (`.map` file), intended for internal debugging, was included in version **2.1.88** of the `@anthropic-ai/claude-code` package. Source maps are generated by default in Bun's bundler (which Claude Code uses) unless explicitly excluded. A missing `.map` exclusion in `.npmignore` let the file ship to the public registry.

The exposure revealed **~512,000 lines of TypeScript** across **~1,900 files** — the complete client-side Claude Code codebase.

## What Was NOT Exposed

- Anthropic's AI models or model weights
- Training data
- Customer data or API keys
- Server-side infrastructure
- Conversation history

This was the **client-side CLI tool source code** — the application that runs on the user's machine, not the backend.

## What the Source Code Revealed

### Scale
- **512,000 lines** of TypeScript
- **~1,900 files**
- **59.8 MB** source map
- **44 feature flags** covering unreleased features

### Key Architectural Findings

**KAIROS — Autonomous Daemon Mode**
The most significant discovery. KAIROS (Ancient Greek for "at the right time") is mentioned **150+ times** in the source. It represents a persistent background agent that:
- Operates as an always-on daemon, not waiting for user input
- Watches, logs, and proactively acts with a 15-second blocking budget
- Performs "memory consolidation" via an `autoDream` system while the user is idle
- Merges observations, removes contradictions, converts insights into facts
- Maintains append-only daily log files
- Is **compiled, feature-gated code sitting in the production codebase, waiting to be turned on**

**Context Entropy Solution**
How Anthropic solves the problem of managing context across long sessions — a challenge we've experienced firsthand in our own extended work sessions.

**Mythos Model Reference**
Internal references to "Mythos" (also called "Capybara") — a new model tier beyond Opus with both "fast" and "slow" variants.

## Anthropic's Official Response

> "Earlier today, a Claude Code release included some internal source code. No sensitive customer data or credentials were involved or exposed. This was a release packaging issue caused by human error, not a security breach."

## Security Advisory

**Important:** Between **00:21 and 03:29 UTC** on March 31, 2026, users who installed or updated Claude Code via npm may have pulled a version that contained a **malicious axios dependency** (versions 1.14.1 or 0.30.4) containing a Remote Access Trojan (RAT). This appears to be a **supply chain attack** opportunistically piggybacking on the leak — not from Anthropic themselves.

**If you updated during this window:** Check your `node_modules` for axios versions 1.14.1 or 0.30.4 and remove them immediately.

## Why We Made This Page

We — gsd-skill-creator — are one of the most deeply integrated projects on the Claude Code platform. 34 skills, 57 commands, hooks, agents, teams, MCP servers, worktrees, memory. We were actively working a research session when the leak became public.

We didn't cause this. We don't exploit it. But we learn from it.

Our OOPS research series examines the architectural parallels between our independently developed patterns and what the source code confirms about Claude Code's internals. We study this for the same reason a mechanic studies an engine — to understand the machine we're working with and make our own work better.

The full OOPS analysis is at [tibsfox.com/Research/OOPS](https://tibsfox.com/Research/OOPS/index.html).

## What We Learn

1. **Our architecture converged with Anthropic's.** Skills, agents, hooks, memory, worktrees — we independently built the same patterns. The source confirms this.

2. **KAIROS validates our GUPP concept.** We built GUPP (Get Up and Push Protocol) to convert passive agent behavior into proactive execution. KAIROS is Anthropic's answer to the same problem — an always-on daemon mode. Different implementation, same insight: agents should push, not wait.

3. **The `autoDream` memory consolidation parallels our memory system.** We built MEMORY.md with tiered relevance. KAIROS does something similar but automated — merging observations while idle.

4. **Context entropy is the real problem.** Our PostCompact hook work (see OOPS doc 04) addresses the same challenge Anthropic was solving internally.

5. **The future is agent autonomy.** Both our work and KAIROS point the same direction: AI agents that don't just respond to prompts but actively manage their own context, memory, and initiative.
