Overview Core libc Compilers Systems ABI Rosetta Learn History
C occupies a unique position in the Rosetta translation matrix: it is not one language among many, it is the language the others are written in. When you translate from Perl to Python to C, you are not translating between peers — you are descending toward the machine. C is the last rung before assembly, and the first rung above the metal. Everything above it calls down to it; nothing meaningful exists below it that isn’t already architecture-specific.

C’s Place on the Escalation Ladder

From the Perl research project’s escalation ladder — the sysadmin’s philosophy of problem-solving:

5Machine code / microcodeWhen assembly is too high-level

C is rung 3 — the last rung before leaving high-level languages entirely. When you graduate from Perl/Python/PHP to C, you’re making a specific trade: you give up garbage collection, runtime errors, and dynamic typing in exchange for direct access to the machine. Every subsequent rung (assembly, machine code) is a more extreme version of the same trade.

Where Is the C Archive?

The research series has now covered four language archives:

PackagistPHP2012400K+ packages

C has no canonical archive. This is not an oversight. It is a philosophical statement. C code lives in:

  • System package managers — apt, yum, pacman, Homebrew, pkgsrc. libc, zlib, libpng, OpenSSL all shipped by the OS
  • Source tarballs on project websites — the classic ./configure && make && make install workflow
  • Git repositories — kernel.org, git.savannah.gnu.org, GitHub for newer projects
  • Conan (JFrog) and vcpkg (Microsoft) — modern C/C++ package managers, but they never achieved CPAN/PyPI scale dominance
  • The distribution repositories — Debian’s APT has ~75,000 packages, Arch’s AUR has 90,000+, most of them C libraries or C-based tools

Why no CPAN for C? Because C predates the concept. By the time PAUSE and CPAN appeared in 1995, C libraries had been distributed via Usenet, FTP, and tarballs for 20 years. The distribution model was already “use the system package manager.” C’s ubiquity is both its strength and why it never needed its own archive.

C as a Compile Target

Many languages compile to C rather than to native code directly. C becomes a portable intermediate representation — you get the reach of every C compiler for free.

LanguageRelationship to C
NimCompiles to C (or C++, or JS). Gets GCC/Clang optimization for free
ValaC#-like syntax, compiles to C with GObject runtime
HaxeCan target C++, C#, JS, PHP, Python, Lua — and C via hxcpp
Chicken SchemeScheme-to-C compiler
Eiffel (EiffelStudio)Bertrand Meyer. Compiles to C for portability
V languageModern compile-to-C language (2019)
CythonPython superset compiled to C. Gives Python speed via C extension modules
Early C++Bjarne Stroustrup’s original Cfront (1983) was a C++-to-C translator
Early Rust (2010)The first Rust compiler (rustboot) was written in OCaml and emitted C before LLVM

Why compile to C? Portability. If your C code compiles, it runs on every architecture with a C compiler — which is every architecture. You inherit 53 years of compiler optimization work for free. The downside: your generated C is often unreadable, your error messages reference C line numbers, and your debugger sees C, not your source language. But for portability, nothing beats it.

Rust — The Challenger (2015+)

Rust is the most serious attempt in 50 years to replace C for systems programming. Graydon Hoare started Rust at Mozilla in 2006; Rust 1.0 shipped May 2015. Its pitch: memory safety without garbage collection. The borrow checker enforces at compile time what C leaves to the programmer’s discipline.

Rust in the Linux kernel (2022+): Kernel 6.1 (Dec 2022) accepted the first Rust code. The “Rust for Linux” project, championed by Miguel Ojeda, lets kernel drivers be written in Rust while the core stays in C. As of 2026, several production drivers ship in Rust.

The February 2024 White House ONCD report explicitly called for adoption of memory-safe languages (Rust, Go, C#, Swift, Java, Python) in new code. The report did not say “stop writing C.” It said “prefer memory-safe where possible.” The C/C++ community’s response has been mixed: Bjarne Stroustrup and others have pushed “Safe C++” profiles; WG14 has added C23 features with safety hints; CHERI (Cambridge) offers capability-based hardware enforcement.

The bet: Rust coexists with C for decades. The Linux kernel will be C for the foreseeable future. Embedded systems will stay C. The C ABI will remain the universal FFI target. But new code for networked services, browsers, and security-critical components is increasingly Rust.
