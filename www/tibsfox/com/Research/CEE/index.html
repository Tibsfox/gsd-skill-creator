#include <stdio.h>
int main(void) { printf("hello, world\n"); return 0; }

Those eight lines, or something close, appeared in the 1978 first edition of Kernighan & Ritchie’s The C Programming Language. They taught a generation to program and became the first program in almost every programming language tutorial ever written. Dennis Ritchie created C at Bell Labs in 1972 to rewrite Unix — and in doing so, gave the world a portable assembler that became the substrate on which every operating system, every language runtime, every database, every web server, and every embedded device is built. When you write Python, you’re running C. When you run a web browser, you’re running C. When your phone boots, it boots into C. The language is 53 years old and it is still the universal ground truth.

Hello, World — Where It All Started

#include <stdio.h>

int main(void) {
    printf("hello, world\n");
    return 0;
}

This program appeared in Kernighan & Ritchie’s The C Programming Language (1978). Brian Kernighan had first used “hello, world” in his 1972 Bell Labs tutorial for the B language. It became the canonical first program in virtually every programming language tutorial ever written since — not because K&R declared it so, but because generations of programmers copied K&R’s first example into their own books. It’s one of the most widely replicated cultural artifacts in computing.

Why C Became the Substrate

1. Unix’s portability proved C’s portability. When Thompson and Ritchie ported Unix from the PDP-11 to the Interdata 8/32 in 1977, they proved that an operating system could be written in a portable high-level language and moved to a new architecture without rewriting in assembly. That was revolutionary. Every prior OS was tied to its hardware.

2. The C ABI became the universal calling convention. Once enough systems spoke C, every new language needed to call C libraries. Rather than inventing new FFI mechanisms, languages adopted the C calling convention. Python ctypes, Ruby FFI, Lua’s C API, Rust’s extern "C", Go’s cgo, Node’s N-API — all speak C. C is the Esperanto of programming languages.

3. The performance ceiling is C + assembly. When Python needs speed, it drops to C (NumPy, PyTorch). When JavaScript needs speed, it drops to C (V8’s Torque, wasm). When any high-level language needs the metal, it either is C or calls C. The only thing below C is assembly, and C is already 95% of the way there.

4. The standard library is the least common denominator. libc is present on every Unix-like system. If you write pure C with only libc, your code will compile on Linux, macOS, BSD, embedded RTOS, Windows (with MinGW), everywhere. Nothing else has this reach.

C sits underneath the other three. Perl’s interpreter is written in C. PHP’s Zend Engine is C. CPython is C. When any of those languages calls a native extension, the ABI is C’s. When they allocate memory, it’s ultimately malloc/free in C. C is the language the other languages are written in, and the language they call when they need to talk to each other.
