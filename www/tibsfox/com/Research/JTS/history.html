History & Evolution

From 10 Days in May 1995 to 2 Million npm Packages — 30 Years of JavaScript and TypeScript
Overview JavaScript TypeScript Runtimes Frameworks Ecosystem Rosetta Learn History
“I was recruited to do ‘Scheme for the browser.’ I was under orders to make the language look like Java.” — Brendan Eich, describing how JavaScript was born under contradictory constraints.

Origins (1995)

Brendan Eich was hired at Netscape Communications in April 1995. Netscape needed a scripting language for its browser to complement Java applets — something lightweight that “non-programmers” could use. In 10 days in May 1995, Eich wrote the first prototype, originally called Mocha.

The design drew from five sources: Scheme (closures, first-class functions), Self (prototype-based inheritance), Java (C-family syntax, the new operator), HyperTalk (event-driven model), and AWK (type coercion). The result was a language that was simultaneously functional, object-oriented, and imperative — and pleased nobody’s purists.

Mocha was renamed LiveScript in September 1995, then JavaScript in December 1995. The “Java” name was a trademark licensing deal with Sun Microsystems to ride the Java hype wave. It is the most expensive naming confusion in the history of computing.

JavaScript first shipped in Netscape Navigator 2.0 (March 1996).

Browser Wars and Standardization (1996–1999)

Microsoft reverse-engineered JavaScript as JScript for Internet Explorer 3 (August 1996). Incompatibilities between the two were brutal: document.all vs document.getElementById, layers vs divs, incompatible event models. The DHTML era was a nightmare of browser sniffing.

To escape vendor lock-in, Netscape submitted JavaScript to Ecma International for standardization. The TC39 committee was formed with delegates from Sun, Netscape, Microsoft, IBM, and others.

Jun 1997 ECMA-262 1st edition — ECMAScript 1. Guy Steele as editor.
Jun 1998 ECMAScript 2 — editorial alignment with ISO/IEC 16262.
Dec 1999 ECMAScript 3 — regex, try/catch, better string handling. Waldemar Horwat as editor. The stable base for a decade.

The Lost Years (1999–2005)

ECMAScript 4 was ambitious: classes, modules, types, generators, optional typing. The committee split into camps. Mozilla + Adobe (ActionScript 3 was based on the ES4 draft) pushed forward. Microsoft + Yahoo (represented by Douglas Crockford) opposed. By 2008, ES4 was politically dead — the committee agreed to “reset” and ship ES3.1 as ES5 instead.

Crockford published JavaScript: The Good Parts (O’Reilly, May 2008) — a thin, opinionated book that argued JavaScript had a beautiful language hiding inside its ugly parts. The book rehabilitated JavaScript’s reputation among serious programmers. Crockford also invented JSON as a subset of JavaScript’s object literal syntax, giving the web its dominant data interchange format.

AJAX and the Rebirth (2005)

In February 2005, Jesse James Garrett published an essay coining the term AJAX (Asynchronous JavaScript + XML). Google Maps and Gmail demonstrated the pattern: fetch data from the server without reloading the page, update the DOM in place, and give web applications the feel of desktop software.

jQuery 1.0 (John Resig, August 2006) normalized the DOM across browsers: $(selector).method() replaced hundreds of lines of browser-sniffing code. jQuery was the dominant front-end library from 2006 to ~2013. At its peak, it was loaded by over 70% of the top million websites.

ECMAScript 5 (December 2009)

The successful follow-up to the failed ES4. Edited by Allen Wirfs-Brock. Added: "use strict", native JSON, Object.create, Array iteration methods (map, filter, reduce, forEach, some, every), property descriptors, getters and setters.

Node.js (November 2009)

Ryan Dahl presented Node.js at JSConf EU in November 2009. The pitch: non-blocking I/O with JavaScript’s callback model, powered by Google’s V8 engine and a new cross-platform I/O library called libuv. JavaScript broke out of the browser.

npm (Isaac Schlueter, 2010) gave Node a package manager that would grow to become the world’s largest software registry.

TypeScript (October 2012)

Anders Hejlsberg at Microsoft (previously Turbo Pascal, Delphi, C#) publicly announced TypeScript on October 1, 2012. TypeScript 0.8 was the first public release. The goal: add optional static typing to JavaScript without breaking compatibility. All valid JS is valid TS.

Early adoption was slow. The pivotal moment came when Google chose TypeScript for the Angular 2 rewrite (announced 2015, shipped 2016). By 2018, TypeScript was the fastest-growing language on GitHub. By 2024, most new projects are TypeScript-first.

ECMAScript 6 / ES2015 (June 2015)

The biggest language upgrade since the original. Edited by Allen Wirfs-Brock. Took 6 years to ship. Added: let/const, arrow functions, classes, template literals, destructuring, default parameters, rest/spread, Promise, Map/Set/WeakMap/WeakSet, Symbol, iterators/generators, and ES Modules (import/export).

The committee simultaneously adopted an annual release cadence: ES2016, ES2017, ES2018, … with a fixed cut date. No more multi-year delays.

The Module Wars

The decade-long transition from CommonJS (require) to ES Modules (import) has been one of the most painful ecosystem migrations in any language:

  • 2009: CommonJS (require/module.exports) ships with Node.js
  • 2015: ES Modules spec published; no runtime supports it yet
  • 2017–2019: Browsers gain native <script type="module">
  • 2019–2021: Node.js adds experimental ESM support; dual CJS/ESM packages emerge
  • 2022+: Node.js "type": "module" in package.json; dual-package hell with .mjs/.cjs extensions

The TC39 Process

StageMeaning
0 — StrawmanAnyone can propose an idea
1 — ProposalTC39 agrees to explore it; has a champion
2 — DraftFormal spec text written
3 — CandidateComplete spec; needs implementation feedback
4 — FinishedTwo implementations ship; will be in the next annual release

The TC39 meets roughly every two months with delegates from browser vendors, framework authors, and the broader community. Proposals can take years to move from Stage 1 to Stage 4. Some never make it — the pipeline proposal, the decorators saga, and the module blocks proposal are all examples of long-lived Stage 2+ proposals.
