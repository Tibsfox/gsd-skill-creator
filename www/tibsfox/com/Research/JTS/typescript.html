TypeScript Type System

Structural Typing, Generics, Utility Types, Conditional Types, Mapped Types, Discriminated Unions
Overview JavaScript TypeScript Runtimes Frameworks Ecosystem Rosetta Learn History
Anders Hejlsberg announced TypeScript at Microsoft in October 2012. It adds a gradual, structural type system over JavaScript and compiles to plain JS. Any valid JavaScript is valid TypeScript (with implicit any), so adoption is incremental. Types exist at compile time only — the compiler checks them, then emits JavaScript with types stripped.

The Erasure Model

Types are erased at compile time. No runtime type info, no reflection, no generic dispatch. The TypeScript compiler (tsc) checks types, then emits JavaScript with all type annotations removed.

// TypeScript input
function greet(name: string): string { return `hi ${name}`; }

// JavaScript output
function greet(name) { return `hi ${name}`; }

Structural Typing

“Duck typing at compile time.” Two types are compatible if their shapes match. No need for shared interfaces or inheritance hierarchies. This is what makes TypeScript feel “light” compared to Java or C#.

interface Duck { quack(): void; }
class Mallard { quack() { console.log("quack"); } }
class RubberDuck { quack() { console.log("squeak"); } }

function feed(d: Duck) { d.quack(); }
feed(new Mallard());        // OK
feed(new RubberDuck());     // OK — no shared ancestor needed
feed({ quack() {} });       // OK — plain object works too

Generics

// Generic function
function identity<T>(x: T): T { return x; }
identity(42);     // T inferred as number

// Generic class
class Box<T> {
  constructor(public value: T) {}
  map<U>(f: (t: T) => U): Box<U> { return new Box(f(this.value)); }
}

// Constraints
function longest<T extends { length: number }>(a: T, b: T): T {
  return a.length >= b.length ? a : b;
}

// Default type parameters
interface Request<Body = unknown, Query = Record<string, string>> {
  body: Body;
  query: Query;
}

// Generic interface (Repository pattern)
interface Repository<T> {
  get(id: string): Promise<T | null>;
  save(item: T): Promise<void>;
}

interface User { id: string; name: string; email: string; }

type UpdatePayload = Partial<Omit<User, "id">>;
// { name?: string; email?: string; }

type UserView = Pick<User, "id" | "name">;
// { id: string; name: string; }

Conditional Types

type IsString<T> = T extends string ? true : false;
type A = IsString<"hi">;   // true
type B = IsString<42>;     // false

// Distributive: naked type parameters distribute over unions
type ToArray<T> = T extends any ? T[] : never;
type X = ToArray<string | number>;   // string[] | number[]

Mapped Types

type Nullable<T> = { [K in keyof T]: T[K] | null };

// Key remapping (TS 4.1+)
type Getters<T> = {
  [K in keyof T as `get${Capitalize<string & K>}`]: () => T[K]
};

Template Literal Types (TS 4.1)

type EventName = `on${Capitalize<"click" | "focus" | "blur">}`;
// "onClick" | "onFocus" | "onBlur"

type PathParam<Path extends string> =
  Path extends `${infer _}:${infer Param}/${infer Rest}`
    ? Param | PathParam<Rest>
    : Path extends `${infer _}:${infer Param}`
      ? Param
      : never;

type P = PathParam<"/users/:id/posts/:postId">;
// "id" | "postId"

Discriminated Unions

The idiomatic TS pattern for type-safe algebraic data types:

type Circle = { type: "circle"; r: number };
type Square = { type: "square"; side: number };
type Triangle = { type: "triangle"; base: number; height: number };
type Shape = Circle | Square | Triangle;

function area(s: Shape): number {
  switch (s.type) {
    case "circle":   return Math.PI * s.r ** 2;
    case "square":   return s.side ** 2;
    case "triangle": return 0.5 * s.base * s.height;
  }
  // TypeScript knows this is exhaustive — no default needed
}

Type Guards

// typeof guard
function process(x: string | number) {
  if (typeof x === "string") return x.toUpperCase();
  return x.toFixed(2);
}

// User-defined type predicate
function isCircle(s: Shape): s is Circle {
  return s.type === "circle";
}

// Assertion function
function assertDefined<T>(val: T | null): asserts val is T {
  if (val === null) throw new Error("null");
}

satisfies (TS 4.9)

const palette = {
  red: [255, 0, 0],
  green: "#00ff00",
  blue: [0, 0, 255],
} satisfies Record<string, string | number[]>;

// palette.red is number[] (not string | number[])
// palette.green is string (not string | number[])
// Type is narrowed while still validating against the constraint

Runtime Validation

Because types erase, you need runtime validation at system boundaries (user input, API responses, config files). The modern TS-first options:

LibraryAuthorCharacter
ZodColin McDonnellTS-first schema validation; infer types from schemas
Valibot—Modular alternative to Zod; smaller bundles
ArkType—TypeScript syntax inside strings: type("string | number")
io-tsGiulio Cantifp-ts family; functional-style codecs
Yup—Older; Formik’s default schema library
 
import { z } from "zod";

const UserSchema = z.object({
  id: z.string().uuid(),
  name: z.string().min(1),
  email: z.string().email(),
  age: z.number().int().min(0).optional(),
});

type User = z.infer<typeof UserSchema>;
// { id: string; name: string; email: string; age?: number }

const result = UserSchema.safeParse(input);
if (result.success) {
  const user: User = result.data;
}
