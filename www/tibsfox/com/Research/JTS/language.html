Brendan Eich was hired at Netscape in April 1995 to put “Scheme in the browser.” The business side wanted Java-like syntax instead. The result was a language with Scheme’s semantic core and Java’s surface, thrown together in ten days.

Design Heritage

  • Scheme — first-class functions, closures, lexical scope, garbage collection, dynamic typing
  • Self (Smalltalk descendant) — prototype-based objects: no classes, just objects pointing at other objects
  • Java — C-family syntax, curly braces, if/while/for, the new operator
  • HyperTalk — the event-driven model: onclick, onload, addEventListener
  • AWK — automatic type coercion: toString/valueOf conversions

The result: a multi-paradigm language that is simultaneously functional, prototype-OO, and imperative.

number is IEEE 754 double-precision. There is no integer type. Exact integers go up to 253. bigint handles arbitrary precision. Everything non-primitive is an object: plain objects, arrays, functions, dates, regex, Map, Set, WeakMap, WeakSet, typed arrays.

typeof quirks: typeof null === "object" is a famous day-one bug that has never been fixed because too much code depends on it. typeof function(){} === "function" is a special case — functions are objects but get their own typeof string.

Type Coercion

JavaScript has loose equality (==) and strict equality (===). Loose equality coerces types; strict equality does not. The universal advice: always use ===.

"" == false;     // true  (both coerce to 0)
"" === false;    // false (different types)
0 == null;       // false (special case)
null == undefined; // true (special case)
NaN === NaN;     // false (IEEE 754 rule)

Gary Bernhardt’s WAT talk (CodeMash 2012) remains the canonical comedy routine demonstrating JavaScript’s coercion behavior.

Functions — The Heart of JavaScript

// Function declaration (hoisted)
function greet(name) { return `Hello, ${name}!`; }

// Function expression (not hoisted)
const greet = function(name) { return `Hello, ${name}!`; };

// Arrow function (ES2015) — inherits `this` from enclosing scope
const greet = (name) => `Hello, ${name}!`;

// Closure: inner function captures outer variable
function makeCounter() {
  let count = 0;
  return () => ++count;
}
const next = makeCounter();
next(); // 1
next(); // 2

this rules

  • Default — undefined in strict mode; globalThis in sloppy mode
  • Implicit — the object to the left of the dot: obj.method()
  • Explicit — call/apply/bind set this directly
  • new — creates a new object and binds this to it
  • Arrow — does not have its own this; uses the enclosing lexical this

Prototypes and Classes

JavaScript is prototype-based: every object has a [[Prototype]] slot pointing to another object. Property lookups walk the prototype chain until they find the property or reach null.

// ES2015 class (syntactic sugar over prototypes)
class Animal {
  #name;  // private field (ES2022)

  constructor(name) { this.#name = name; }
  speak() { return `${this.#name} makes a noise.`; }
}

class Dog extends Animal {
  speak() { return `${super.speak()} Woof!`; }
}

const d = new Dog("Rex");
d.speak(); // "Rex makes a noise. Woof!"

Under the hood, class defines a constructor function, sets up prototype links, and uses Object.setPrototypeOf for extends. Private fields (#name, ES2022) are truly private — not accessible outside the class body, not even via reflection.

Promises and async/await

// Promise (ES2015)
fetch("/api/user")
  .then(res => res.json())
  .then(user => console.log(user.name))
  .catch(err => console.error(err));

// async/await (ES2017) — syntactic sugar over promises
async function loadUser() {
  try {
    const res = await fetch("/api/user");
    const user = await res.json();
    console.log(user.name);
  } catch (err) {
    console.error(err);
  }
}

// Promise combinators
const [users, posts] = await Promise.all([
  fetch("/api/users").then(r => r.json()),
  fetch("/api/posts").then(r => r.json()),
]);

Promise.all (all or nothing), Promise.allSettled (all results regardless), Promise.race (first to settle), Promise.any (first to fulfill). Top-level await (ES2022) works in ES Modules.

// Destructuring
const { name, age = 25, ...rest } = user;
const [first, , third] = items;

// Optional chaining (ES2020)
user?.address?.city;
user?.getName?.();
arr?.[0];

// Nullish coalescing (ES2020)
const count = input ?? 0;  // 0 only for null/undefined

// Logical assignment (ES2021)

obj.y ||= "fallback";

// Template literals

// Spread
