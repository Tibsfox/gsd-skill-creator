Scientific Computing

NumPy, SciPy, Pandas, Matplotlib, Jupyter, SymPy — How Python Became the Lingua Franca of Science
Overview Core PyPI Science ML/AI Web Rosetta Learn History
Python did not start as a scientific language. It became one through a deliberate, community-driven effort spanning three decades: Numeric (Jim Hugunin, 1995) → Numarray (STScI, 2001) → NumPy (Travis Oliphant, 2006) → the entire SciPy stack built on top. By 2015, Python had displaced MATLAB in university physics labs, R in statistics departments, and IDL in astronomy. The combination of a readable scripting language with C-performance numerical libraries was unbeatable.

NumPy — The Foundation

Travis Oliphant, 2006. In 1995, Jim Hugunin wrote Numeric, the first array library for Python. By 2001, the Space Telescope Science Institute — needing to process Hubble data — built Numarray to handle large astronomical datasets. The community fragmented. Travis Oliphant, then at Brigham Young University, spent 2005 unifying both libraries into NumPy. The 1.0 release in October 2006 ended the split and gave Python a single canonical n-dimensional array. Without that consolidation, none of the rest of this ecosystem exists.

NumPy 2.0 (June 2024) — The Big Cleanup

After 18 years on the 1.x line, NumPy 2.0 was a deliberate breakage to clean up the API: new scalar promotion rules (NEP 50), variable-width UTF-8 string dtype, ~10% of the public namespace pruned, Array API standard compliance. Downstream libraries (PyTorch, Pandas, scikit-learn, SciPy) coordinated to land 2.0 support over the same window.

SciPy — Scientific Algorithms

Eric Jones, Travis Oliphant, Pearu Peterson — 2001. Combined multipack (wrapper for FITPACK, MINPACK, ODEPACK, QUADPACK) with other modules into a single library. SciPy 1.0 arrived in October 2017 — 16 years to call it stable, a community in-joke.

Pandas — The Data Analysis Workhorse

Wes McKinney — 2008. Created at AQR Capital to wrangle financial time series. Open-sourced 2009. The DataFrame and Series abstractions brought R’s data.frame ergonomics to Python. Pandas 2.0 (April 2023) introduced optional Apache Arrow backends for 10-100x speedups on string and datetime operations.

Polars — The Rust Successor

Ritchie Vink, 2020. Rust-based DataFrame library with a lazy query engine (like Spark), 10-100x faster than Pandas for many workloads. Growing rapidly; PyArrow integration; the next generation of dataframes. Pandas and Polars will likely coexist for years.

Matplotlib & The Visualization Stack

John Hunter — 2003. John Hunter was a neurobiologist at University of Chicago who needed better plots for his EEG data than MATLAB offered. Matplotlib’s API deliberately mimicked MATLAB’s to ease migration. Hunter passed away in 2012; the project continued under a steering council. Matplotlib is the universal baseline — everything else plugs into it or replaces it.

Jupyter — The Scientific IDE

Fernando Pérez — 2001 (IPython) → 2014 (Jupyter Project). Started as IPython, a better Python REPL with tab completion, magic commands, and shell escapes. In 2011, the IPython Notebook introduced browser-based literate programming. In 2014, the project forked into Jupyter (named for Julia, Python, R) to signal language-agnostic focus. Today, Jupyter notebooks are the de facto interface for scientific computing, data analysis, and ML research.

SymPy — Symbolic Mathematics

Ondřej Čertík, 2006. Pure-Python computer algebra system. Calculus, algebra, discrete math, number theory, physics. Runs inside Jupyter. Used as a teaching tool and as a Mathematica/Maple alternative. Integrates with NumPy via lambdify to convert symbolic expressions to fast numerical functions.

from sympy import symbols, integrate, diff, solve, Eq

x, y = symbols('x y')

# Symbolic integration
integrate(x**2 * y, x)             # x**3*y/3

# Symbolic differentiation
diff(x**3 + 2*x**2 + x, x)        # 3*x**2 + 4*x + 1

# Solve equations
solve(Eq(x**2 - 4, 0), x)          # [-2, 2]

Why Python Won Scientific Computing

Three factors converged:

  1. NumPy as the lingua franca. Every library could exchange ndarrays. No data conversion tax between libraries.
  2. C extension API. Slow Python + fast C means you can prototype in Python and drop to C (via Cython, ctypes, cffi, or raw CPython API) for hot paths. The language doesn’t fight you.
  3. Scripting ergonomics. MATLAB is great for algorithms but weak at scripting and data plumbing. Python handles the plumbing and the math.

The result: by 2015, Python had displaced MATLAB in university physics labs, R in statistics departments, and IDL in astronomy. By 2020, “the scientific Python stack” (NumPy + SciPy + Pandas + Matplotlib + Jupyter + scikit-learn) was the default toolkit for any kind of data analysis worldwide.

NumFOCUS — The Foundation Behind the Stack

NumFOCUS is a 501(c)(3) non-profit founded in 2012 that acts as the fiscal sponsor for most of the scientific Python ecosystem: NumPy, SciPy, Pandas, Matplotlib, Jupyter, scikit-learn, and many more. It provides infrastructure, grants, and governance support that individual projects couldn’t sustain alone. The PyData conference series is a NumFOCUS initiative.
