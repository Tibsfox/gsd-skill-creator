Django, Flask, FastAPI — From WSGI to ASGI, the Long Road to Async
Overview Core PyPI Science ML/AI Web Rosetta Learn History
Python’s web story is one of pragmatism, layered abstractions, and a long, painful migration from synchronous WSGI to asynchronous ASGI. Django made Python web development serious. Flask made it minimalist. FastAPI made it modern. Instagram, running one of the largest Django deployments ever, proved that Python could serve billions of requests per day.

Django — The Batteries-Included Workhorse

Adrian Holovaty & Simon Willison, July 2005. Created at Lawrence Journal-World newspaper in Lawrence, Kansas. Named after jazz guitarist Django Reinhardt. Tagline: “The web framework for perfectionists with deadlines.” Open-sourced under BSD. Current: Django 5.1 (Aug 2024), 5.2 LTS (Apr 2025).

Batteries-Included Philosophy

Django shipped with everything a newspaper website needed out of the box: ORM, templating, forms, authentication, sessions, internationalization, and most famously the Django admin interface — an auto-generated CRUD backend derived from your models. The admin alone sold Django to thousands of organizations.

Django REST Framework (DRF)

Tom Christie, 2011. The de facto Django REST library. Serializers, ViewSets, routers, browsable API, token/JWT/OAuth auth, throttling, pagination. Practically synonymous with “Django API.”

Notable Django Users

Instagram — one of the largest Django deployments ever, billions of requests/day. Pinterest, Disqus, The Washington Post, The Guardian, NASA, Mozilla, Eventbrite, Bitbucket (originally), National Geographic.

Flask — The Microframework That Started as a Joke

Armin Ronacher, April 1, 2010. Released as an April Fool’s joke — a “single-file framework” parody that the community took seriously. Ronacher also created Jinja2, Werkzeug, Click, MarkupSafe — the Pallets Projects suite. Current: Flask 3.1 (Dec 2024).

The Microframework Philosophy

Provide URL routing, request/response handling, and a templating integration — and nothing else. Want a database? Pick one. Want forms? Pick one. Want auth? Pick one. The result was an ecosystem of “Flask-X” extensions.

FastAPI — Modern Async + Type Hints

Sebastian Ramírez, December 2018. Modern async Python web framework built on Starlette and Pydantic. Uses type hints as the interface contract: function parameters define validation, serialization, and OpenAPI documentation. Performance approaching Node.js. The fastest-growing Python web framework.

Why FastAPI Took Off

from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

class User(BaseModel):
    id: int
    name: str
    email: str | None = None

@app.post("/users/")
async def create_user(user: User) -> User:
    # user is validated, parsed, and typed
    # OpenAPI docs auto-generated at /docs
    return user

Three lines to get:

  • Request validation (400 errors for bad input)
  • Response serialization
  • Auto-generated OpenAPI/Swagger docs at /docs
  • Auto-generated ReDoc at /redoc
  • IDE autocomplete throughout
  • Type checker validation

FastAPI Users

Microsoft, Uber, Netflix (parts), Explosion AI (spaCy), Hugging Face (various APIs), countless ML inference services (FastAPI + PyTorch is a canonical pair).

WSGI → ASGI — The Async Migration

WSGI (2003, PEP 333 / PEP 3333)

Web Server Gateway Interface. Phillip J. Eby, December 2003. The contract between Python web applications and Python web servers. Synchronous, request-per-thread model:

def application(environ, start_response):
    start_response('200 OK', [('Content-Type', 'text/plain')])
    return [b'Hello, World!\n']

Every Python web framework before 2018 spoke WSGI: Django, Flask, Pyramid, Bottle, web.py, Pylons.

ASGI (2018)

Asynchronous Server Gateway Interface. Andrew Godwin, 2018. WSGI’s successor: supports async, WebSockets, long-lived connections, and server push. The protocol is a set of three awaitable callables: application(scope, receive, send). ASGI lets a single process handle thousands of concurrent connections.

Instagram — Python at Planetary Scale

Instagram is the most-cited proof that Python (and Django) can serve at planetary scale. Their story:

  • Founded 2010 on Django. Decision to use Python for speed of iteration.
  • Scaled to billions of requests/day on Django + PostgreSQL + memcached + Cassandra
  • Early Python 2 → 3 migration — published detailed blog posts on how they did it
  • Cinder — Instagram’s internal CPython fork with JIT, strict modules, and shadowcode. Open-sourced May 2021
  • Cinder features influenced Faster CPython, per-interpreter GIL work, and 3.13’s experimental JIT
  • Instagram still runs mostly Python in 2026. Proved that the bottleneck is engineering discipline, not the language
