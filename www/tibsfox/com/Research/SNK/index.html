Python was born over a Christmas break in December 1989 when Guido van Rossum, bored and looking for a hobby project, started building a successor to the ABC language he had worked on at CWI in Amsterdam. He named it after Monty Python’s Flying Circus, not the snake. Thirty-seven years later, Python is the most-taught programming language in the world, the lingua franca of scientific computing and machine learning, and the language that finally answered Perl’s TMTOWTDI with a different philosophy: there should be one— and preferably only one —obvious way to do it.

Why Python? Python won three revolutions in a row. It won teaching (replacing Java and C in intro CS). It won scientific computing (NumPy, Pandas, Jupyter). And it won machine learning (PyTorch, TensorFlow, HuggingFace). No other language has been this dominant across this many domains simultaneously. The reasons are simple: readable syntax, one obvious way, a C extension API that made high-performance libraries possible, and a community that valued teaching over cleverness.

The Zen of Python

Tim Peters, PEP 20 (2004). Type import this in any Python interpreter to read it. This is Python’s answer to Perl’s TMTOWTDI — a manifesto for a different kind of language:

  1. Beautiful is better than ugly.
  2. Explicit is better than implicit.
  3. Simple is better than complex.
  4. Complex is better than complicated.
  5. Flat is better than nested.
  6. Sparse is better than dense.
  7. Readability counts.
  8. Special cases aren’t special enough to break the rules.
  9. Although practicality beats purity.
  10. Errors should never pass silently.
  11. Unless explicitly silenced.
  12. In the face of ambiguity, refuse the temptation to guess.
  13. There should be one— and preferably only one —obvious way to do it.
  14. Although that way may not be obvious at first unless you’re Dutch.
  15. Now is better than never.
  16. Although never is often better than right now.
  17. If the implementation is hard to explain, it’s a bad idea.
  18. If the implementation is easy to explain, it may be a good idea.
  19. Namespaces are one honking great idea — let’s do more of those!

Why Python Won — Three Revolutions

Revolution 1: Teaching (2003–Present)

Python replaced Java and C as the dominant language for introductory computer science courses. By 2014, eight of the top ten US computer science programs taught Python as their first language. The reasons: readable syntax (no semicolons, no braces, meaningful whitespace), fast feedback loops (REPL, no compilation), and a gentle learning curve. You could write meaningful programs in minutes, not days.

Revolution 2: Scientific Computing (2006–Present)

NumPy (2006, Travis Oliphant) unified the scientific Python ecosystem around a single n-dimensional array type. SciPy, Pandas, Matplotlib, and Jupyter followed. By 2015, Python had displaced MATLAB in university physics and engineering labs, R in statistics departments, and IDL in astronomy. The combination of a readable scripting language with C-performance numerical libraries (via NumPy’s ufunc system) was unbeatable.

Revolution 3: Machine Learning & AI (2015–Present)

TensorFlow (Google, 2015) and PyTorch (Facebook AI Research, 2016) both chose Python as their primary interface. Not Java, not C++, not R — Python. The Python scientific stack was too entrenched to ignore, and the combination of Python’s ergonomics with GPU-accelerated tensor operations became the dominant paradigm for deep learning. By 2022, every major LLM (GPT, Llama, Claude, Gemini) was trained using Python. HuggingFace Transformers made pre-trained models accessible in three lines of code.

The three languages represent three different philosophies of “scripting.” Perl optimized for expressiveness, PHP optimized for deployment, Python optimized for readability. All three succeeded in their niches, but Python’s readability-first philosophy proved most durable as codebases grew and teams diversified. When you’re reading code you didn’t write, TSBOWTDI is easier than TMTOWTDI.
