scikit-learn, TensorFlow, PyTorch, JAX, HuggingFace — How Python Won the AI Revolution
Overview Core PyPI Science ML/AI Web Rosetta Learn History
TensorFlow (Google, 2015) and PyTorch (Facebook AI Research, 2016) both chose Python as their primary interface. Not Java, not C++, not R — Python. The Python scientific stack was too entrenched to ignore. By 2022, every major LLM (GPT, Llama, Claude, Gemini) was trained using Python. HuggingFace Transformers made pre-trained models accessible in three lines of code. Python won the AI revolution because NumPy was already there.

scikit-learn — Classical Machine Learning

David Cournapeau, 2007 (GSoC). Started as scikits.learn, one of several SciPy toolkits. Dormant for two years until Fabian Pedregosa, Gaël Varoquaux, and Alexandre Gramfort at Inria’s Parietal team (Saclay, France) — neuroimaging researchers — rebooted it in 2010. First public release February 2010. Now the gold standard for classical ML.

The Consistent API That Won

The single biggest reason scikit-learn dominated classical ML is its uniform estimator API. Every algorithm implements the same interface:

estimator.fit(X, y)
estimator.predict(X_new)
estimator.transform(X)         # for preprocessors
estimator.fit_predict(X, y)
estimator.score(X, y)

Every algorithm — from LogisticRegression to RandomForestClassifier to KMeans to PCA — implements the same four-or-five method interface. Combined with Pipeline, ColumnTransformer, GridSearchCV, and cross_val_score, this turns ML workflows into Lego.

What scikit-learn Deliberately Doesn’t Do

  • No GPUs — pure CPU on NumPy/SciPy/Cython. (cuML from NVIDIA RAPIDS is the GPU port.)
  • No deep learning — that’s PyTorch/TensorFlow/JAX territory
  • No distributed training — joblib parallelism only; use Dask-ML or Spark MLlib for distributed

This restraint is why scikit-learn remains a stable, trustworthy foundation rather than chasing every trend.

TensorFlow — Google’s Deep Learning Platform

Google Brain (Jeff Dean, Rajat Monga, Sanjay Ghemawat, et al.), November 9, 2015. Predecessor: DistBelief (2011), used to train the famous “cat neuron” paper and Inception. TensorFlow was open-sourced under Apache 2.0 — a watershed moment that legitimized open-source deep learning at industrial scale.

TensorFlow 1.x → 2.x: The Big Pivot

TF 1.x (2015–2019): static graphs. Define-then-run model with tf.Graph and tf.Session. Fast and parallelizable but painful to debug. PyTorch’s eager execution started winning hearts.

TF 2.0 (September 2019): Eager execution by default, tf.function for JIT tracing, Keras as the official high-level API. The 1.x cruft was removed.

Keras — The High-Level API

François Chollet (Google), March 2015. Originally a Theano frontend, added TensorFlow backend mid-2015. Folded into TF as tf.keras in TF 1.4 (2017); TF 2.0 made it official. Keras 3.0 (November 2023) is multi-backend again — the same keras.Model can train on JAX, TensorFlow, or PyTorch. Chollet’s API now runs on top of competing backends.

Status in 2026

PyTorch has decisively won the research market. TensorFlow remains strong in production deployment, mobile/edge (LiteRT), and inside Google. Keras 3 hedged Google’s bet across all three backends.

PyTorch — The LLM Era’s Workhorse

Adam Paszke, Sam Gross, Soumith Chintala (Facebook AI Research), January 2017. Predecessor: Lua Torch (Ronan Collobert, Koray Kavukcuoglu, Clément Farabet, ~2002, NYU/IDIAP). Paszke was a Warsaw undergraduate intern at FAIR when he rewrote Torch with a Python frontend using define-by-run dynamic computation graphs.

Why Define-by-Run Won

The dynamic graph model felt natively Pythonic:

  • You could print(tensor) mid-forward-pass
  • You could use for loops, if statements, recursion
  • Debugging worked with standard Python tools
  • Especially NLP researchers (where sequence lengths vary and trees/graphs are common) switched immediately

By 2019, PyTorch had overtaken TensorFlow in academic publications.

PyTorch 2.0 — torch.compile (March 2023)

torch.compile(model) — a JIT compiler that traces the model with TorchDynamo (Python bytecode-level graph capture), lowers via AOTAutograd and PrimTorch, then codegens with TorchInductor (Triton-based) to fast CUDA kernels. The promise: 30–200% speedups with one line of code, no model rewrite. By 2024–2025 it became the default mode for most production training.

PyTorch Foundation (September 2022)

Meta transferred PyTorch governance to the PyTorch Foundation under the Linux Foundation. Founding members: Meta, Microsoft, Google, NVIDIA, AWS, AMD. PyTorch was no longer “Meta’s framework” but cross-industry infrastructure, like Linux or Kubernetes.

Why PyTorch Won the LLM Era

Every major open LLM family — Llama, Mistral, Falcon, Qwen, DeepSeek, Phi, Gemma — ships PyTorch checkpoints first. The HuggingFace ecosystem is PyTorch-native. NVIDIA optimizes PyTorch as a first-class target. The compounding effect is overwhelming: researchers use PyTorch → papers ship PyTorch code → models ship PyTorch checkpoints → production ships PyTorch inference.

JAX — Functional NumPy with Autodiff

Roy Frostig, Matthew James Johnson, Chris Leary, Dougal Maclaurin, Adam Paszke, et al. (Google Research), December 2018. Spiritual successor to Autograd (Maclaurin, Duvenaud, Adams, Harvard 2014). Adds XLA compilation for GPU/TPU and a richer set of functional transformations. No 1.0 yet — deliberately cautious.

Composable Function Transformations

JAX is functional: arrays are immutable, functions are pure, and the library exposes composable transformations:

TransformationWhat it does
jitXLA compilation of a Python function into a fused kernel
gradReverse-mode automatic differentiation of any pure function
vmapAuto-vectorization — write for one example, vectorize over a batch
pmap / pjit / shard_mapMulti-device SPMD sharding
vjp / jvpVector-Jacobian and Jacobian-vector products
hessianSecond derivatives: jacfwd(jacrev(f))

These compose. jit(grad(vmap(f))) is a perfectly normal expression: vectorize f over a batch, take its gradient, JIT-compile the result. JAX is what you get when Haskell people do deep learning.

HuggingFace, 2018. Started as a chatbot company, pivoted to become the GitHub of machine learning. transformers library made pre-trained models accessible in three lines of code.

from transformers import pipeline
classifier = pipeline("sentiment-analysis")

Why Python Won the AI Revolution

Three factors compounded:

  1. NumPy was already there. GPU frameworks needed a tensor abstraction, and NumPy’s ndarray had been the scientific Python lingua franca for a decade. PyTorch and TensorFlow both mirror the NumPy API.
  2. C extension API. Python can call C and CUDA efficiently via cffi, Cython, and the CPython C API. This let ML frameworks offload heavy lifting to GPU kernels while keeping the user-facing code in Python.
  3. Readable researchers’ code. When papers publish code, it needs to be readable. Python’s TSBOWTDI philosophy and clean syntax make research code comprehensible across institutions. Perl research code would be obfuscated by default; Python research code is a de facto standard.

The result: every major AI lab — OpenAI, Anthropic, DeepMind, Meta AI, Google Brain, Mistral, Stability — runs on Python. Every LLM checkpoint ships as PyTorch weights. Every fine-tuning tutorial uses HuggingFace. Python is the universal compiler-target of AI research.
