PyPI launched in 2003, originally nicknamed “the Cheese Shop” after the Monty Python sketch (“This is the most tested cheese shop in the district, sir.”). Richard Jones proposed it in PEP 301. It was hosted at cheeseshop.python.org, then moved to pypi.python.org, and finally to pypi.org in 2018 when Warehouse went live. Today it is the second-largest language-specific package registry in the world.

Warehouse — The Modern PyPI

Warehouse is the codebase powering pypi.org today. The project began in 2011 by Donald Stufft as a from-scratch rewrite of the legacy PyPI codebase. It went live as the production PyPI on April 16, 2018.

  • Stack: Pyramid web framework, PostgreSQL, Elasticsearch, Redis
  • CDN: Fastly (hundreds of TB/day bandwidth)
  • License: MIT, open source at github.com/pypi/warehouse
  • Funding: Mozilla Open Source Support, Facebook, Sloan Foundation, individual sponsors

The Packaging Crisis (2010–2018)

For roughly 2010–2018, the Python packaging story was infamous. Common gripes:

  • distutils was minimal and quietly hostile to anything beyond the trivial case
  • setuptools monkey-patched distutils and accreted features unpredictably
  • easy_install could install but not uninstall, and pulled .egg files into a shared location with version conflicts
  • pip solved install/uninstall but for a decade did not have a real dependency resolver — it would happily install incompatible dependency sets
  • Build configuration was a Python script (setup.py) that ran at install time — arbitrary code execution from any package’s installer
  • Multiple competing tools (pipenv, poetry, hatch, pdm, pip-tools) split the community
  • C extensions on Windows were a nightmare, partially fixed by wheels and manylinux PEPs

xkcd #1987 (“Python Environment”) from 2018 became the canonical illustration.

Modern Packaging Tools

pip — The Canonical Installer

pip install requests              # install from PyPI
pip install requests==2.31.0      # specific version
pip install -r requirements.txt   # from file
pip install -e .                  # editable install from local dir
pip install package --index-url https://custom.index/simple
pip freeze > requirements.txt     # snapshot current env
pip show requests                 # show package info
pip uninstall requests

venv — Virtual Environments (Built-in)

python -m venv .venv              # create virtual environment
source .venv/bin/activate         # activate (Unix)
.venv\Scripts\activate            # activate (Windows)
pip install -r requirements.txt   # install into isolated env
deactivate                        # exit

Poetry — Modern Dependency Manager

poetry new my-project             # create project with pyproject.toml
poetry add requests               # add dependency, update lock file
poetry add --group dev pytest     # dev dependency
poetry install                    # install from lock file
poetry run python script.py       # run in project env
poetry build                      # build wheels + sdists
poetry publish                    # upload to PyPI

uv — The Rust-Based Rocket (2024+)

uv venv                           # create venv (100x faster than venv)
uv pip install requests           # install (10-100x faster than pip)
uv pip sync requirements.txt      # sync exactly
uv tool install ruff              # install CLI tools
uv run script.py                  # run with inline deps
uv sync                           # install from pyproject.toml + lock

uv is Astral’s Rust-based replacement for pip, virtualenv, pip-tools, pipx, and poetry. Created by Charlie Marsh (also the creator of the ruff linter). It is 10-100x faster than pip, supports the full Python packaging standards, and has been adopted rapidly since its February 2024 release. By 2025, uv had surpassed Poetry in download metrics for many project types.

conda — Scientific Computing Alternative

conda create -n myenv python=3.14 numpy scipy pandas
conda activate myenv
conda install -c conda-forge pytorch
conda env export > environment.yml
mamba install tensorflow          # mamba = fast conda (C++)

conda handles binary dependencies (BLAS, LAPACK, CUDA, MKL) that pip historically struggled with. Popular in data science and ML via the Anaconda distribution. mamba and miniforge are community-driven, faster alternatives.

Supply Chain Security

PyPI has had its share of supply chain incidents. The community has responded with layered defenses:

  • PEP 458 — Surviving a compromise of PyPI (TUF: The Update Framework)
  • Sigstore integration — Keyless signing via Fulcio, transparency via Rekor
  • 2FA mandatory — For all PyPI accounts with upload privileges (2023+)
  • Trusted Publishers — OIDC-based publishing from GitHub Actions, no long-lived tokens
  • Typosquatting detection — Names similar to popular packages get flagged
  • PyPI Security Working Group — Dedicated response team

Known incidents: colorama typosquatting (2017), ctx takeover (2022), various crypto-miner packages. Every incident drove improvements to the security model.

How to Publish to PyPI

# Modern workflow with pyproject.toml
[build-system]
requires = ["hatchling"]
build-backend = "hatchling.build"

[project]
name = "my-package"
version = "0.1.0"
description = "My awesome package"
authors = [{name = "Your Name", email = "you@example.com"}]
dependencies = ["requests>=2.31"]
requires-python = ">=3.10"

# Build
python -m build                   # creates dist/*.whl and dist/*.tar.gz

# Upload
twine upload dist/*               # legacy
# OR
uv publish                        # modern
