Why logic programming now? Prolog has a reputation as yesterday’s AI language — shelved when statistical ML took over in the 1990s. But the declarative idea refused to die. Today Datalog powers CodeQL (GitHub’s security engine) and Souffle (Oracle Labs’ program analyzer). Answer Set Programming runs bioinformatics and combinatorial planning via clingo. The Rust borrow checker’s Polonius is a Datalog engine. IBM Watson used Prolog in its Jeopardy-winning NLP pipeline. And the 2020s renaissance in neuro-symbolic AI — DeepProbLog, Scallop, Logic Tensor Networks — is bringing logic programming back as the symbolic counterpart to neural networks. The pendulum swings.

The Reasoning Ladder

Prolog belongs on a different ladder than the sysadmin’s escalation from bash to machine code. Logic programming is about how you describe the problem, not how close you are to the metal:

  1. Imperative — Tell the computer what to do, step by step. Assign, branch, loop, mutate. The dominant model since FORTRAN (1957).
  2. Functional — Tell the computer how to compute a value from its inputs. Map, fold, compose. Pure, referentially transparent. Lisp (1958), ML, Haskell.
  3. Object-oriented — Tell the computer which entities exist and how they interact. Message-passing, encapsulation, inheritance. Simula (1967), Smalltalk, Java.
  4. Logic — Tell the computer what is true. Facts and rules. Ask it questions. Let unification and search do the rest. Prolog (1972), Datalog, ASP.
  5. Constraint — Tell the computer what must hold in a solution. Let a specialized solver find it. CLP(FD), CLP(R), SAT, SMT, Z3.
  6. Probabilistic — Tell the computer what is likely. Condition on evidence. Stan, Pyro, ProbLog. A logic programming relative.
  7. Neural — Show the computer examples. Let gradient descent find the function. PyTorch, JAX. The dominant model since ~2012.
  8. Neuro-symbolic — Combine them. Use neural networks for perception, logic programming for reasoning. DeepProbLog, Scallop. The frontier.

Every rung is still useful. Every rung has its own languages, tools, and culture. Prolog occupies rung four — but as this research series shows, the surrounding rungs are increasingly its collaborators.

The Famous Four-Line Quicksort

No introduction to Prolog is complete without this. It is declarative — it says what a sorted list is, not how to sort.

quicksort([], []).
quicksort([Pivot|T], Sorted) :-
    partition(Pivot, T, Less, Greater),
    quicksort(Less, LSorted),
    quicksort(Greater, GSorted),
    append(LSorted, [Pivot|GSorted], Sorted).

partition(_, [], [], []).
partition(P, [H|T], [H|L], G) :- H =< P, partition(P, T, L, G).
partition(P, [H|T], L, [H|G]) :- H >  P, partition(P, T, L, G).

This is the naive version (not in-place, not using accumulators). A production version uses difference lists and is still shorter than the C equivalent. Ask Prolog ?- quicksort([3,1,4,1,5,9,2,6], S). and it replies S = [1,1,2,3,4,5,6,9]. But it can also run backwards in certain dialects — a glimpse of the bidirectional magic that makes the language feel unique.

Why Logic Programming Refuses to Die

The short version: There are problems for which the declarative specification is dramatically shorter than any operational implementation. When you have such a problem — scheduling, constraint satisfaction, program analysis, symbolic reasoning — a logic language beats every other approach by 10× or more in code size, and often in time-to-working-solution.

The long version: Modern industry has rediscovered several of these problems:

  • Program analysis — GitHub CodeQL, Oracle Souffle, Rust’s Polonius. Static analysis is recursive fact-base querying, which is exactly Datalog.
  • Cloud network verification — Amazon, Microsoft, and Google use Datalog-style tools to verify firewall and routing configurations at scale.
  • Combinatorial optimization — Airline crew scheduling (SICStus), factory planning, logistics. CLP(FD) and ASP remain competitive with CP-SAT and MiniZinc.
  • Knowledge representation — Description logics, OWL, Wikidata. The semantic web winter ended, but the survivors run on logic-programming kin.
  • Neuro-symbolic AI — 2020s research combining deep networks with symbolic reasoning. DeepProbLog, Scallop, Logic Tensor Networks. LLMs generating Prolog/ASP as a reasoning layer.

The punchline: Prolog the language is a specialist tool with a steep learning curve. But the ideas of logic programming — unification, declarative rules, fixed-point evaluation, constraint propagation — are foundational infrastructure across computing. Learning Prolog is learning the family of ideas that keep getting reinvented.
