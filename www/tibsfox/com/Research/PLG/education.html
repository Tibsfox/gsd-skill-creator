The hardest thing about learning Prolog is unlearning imperative reflexes. Every textbook has a moment where the student asks “but how do I assign to a variable?” and the answer is you don’t. The answer is that you state relationships and let unification do the work. Getting to that moment takes about two weeks. The aha is worth it.

A Recommended Learning Path

  1. Install SWI-Prolog. It’s free, BSD-licensed, cross-platform, and has the friendliest online REPL (SWISH). Start there unless you have a strong reason otherwise.
  2. Read “Learn Prolog Now!” (LPN!). Free, online, interactive. Blackburn, Bos, Striegnitz. Covers core Prolog in twelve lessons with exercises. This is the undisputed fastest path from zero to “I can write a program.”
  3. Read Bratko’s “Prolog Programming for Artificial Intelligence.” The 4th edition (2011) is still the best general-purpose Prolog textbook. Bratko is patient with the beginner and deep with the expert.
  4. Watch Markus Triska’s “The Power of Prolog” videos. Free, on YouTube and the power-of-prolog.com site. Strict ISO Prolog, clpfd, and a deep emphasis on declarative correctness. Triska is the modern successor to Richard O’Keefe’s school.
  5. Read O’Keefe’s “The Craft of Prolog” (1990). Dated examples, timeless wisdom. This is the book that teaches you when (and why) to use cut, when to use accumulators, and what “elegance” means in this language.
  6. Read Sterling & Shapiro’s “The Art of Prolog” (2nd ed, 1994). The MIT Press classic. More formal than Bratko, with beautiful treatments of metainterpreters, program transformation, and the expert-system tradition.
  7. Do the classic exercises. Zebra puzzle, Einstein’s riddle, N-queens (both naive and with clpfd), Tower of Hanoi, tic-tac-toe, a graph coloring problem, a small DCG for arithmetic, the Towers of Hanoi, simple map coloring. They are classics because each one teaches a different piece of the Prolog mental model.
  8. Branch out to CLP(FD) and ASP. Once Prolog feels comfortable, start using library(clpfd) for constraint problems. Then try clingo to see Answer Set Programming — a related but distinct paradigm that solves some problems Prolog can’t.
  9. Learn Datalog via Souffle. If you’re interested in program analysis, Souffle is where Datalog is the most industrial. The tutorial is short and the payoff is large.
  10. Read papers. Colmerauer’s “The Birth of Prolog,” Kowalski’s “Algorithm = Logic + Control,” Warren’s WAM paper, Lally et al. on Watson’s Prolog component. Primary sources are short and shocking — the foundational ideas are simpler than the textbooks make them look.

Your First Session

Assuming SWI-Prolog is installed. On Debian/Ubuntu: sudo apt install swi-prolog. On macOS: brew install swi-prolog. On Windows: download from swi-prolog.org.

Start the REPL:

$ swipl
Welcome to SWI-Prolog (threaded, 64 bits, version 9.2.x)
...
?-

Enter a few facts and rules directly:

?- assert(likes(alice, prolog)).
true.

?- assert(likes(bob, datalog)).
true.

?- assert((programmer(X) :- likes(X, prolog) ; likes(X, datalog))).
true.

?- programmer(X).
X = alice ;
X = bob.

Now write a file family.pl:

parent(tom, bob).
parent(tom, liz).
parent(bob, ann).

grandparent(X, Z) :- parent(X, Y), parent(Y, Z).

And consult it:

?- [family].
true.

?- grandparent(tom, Who).
Who = ann.

Congratulations — you have now written more Prolog than 95% of the world. The next step is to go read LPN! and Bratko and spend two weeks rewiring your brain.

Courses and Academic Programs

Prolog is most commonly taught in:

  • Programming language courses — Prolog as a paradigm example alongside Lisp/Haskell/ML. Often one or two weeks.
  • Artificial intelligence courses — The classical AI half (search, knowledge representation, expert systems) still uses Prolog. Bratko’s textbook is the common choice.
  • Logic courses — CS-department logic courses sometimes include Prolog as the “logic you can run.”
  • Databases — Datalog appears in graduate database theory courses (Ullman, Abiteboul).
  • Compilers and program analysis — Datalog (via Souffle or Doop) appears in modern static-analysis courses.

Notable public course materials:

  • CS386: Logic Programming (Stanford, Gensler) — classic materials available online.
  • CS 2800: Discrete Structures (Cornell) — uses miniKanren-style relational thinking.
  • Peter Norvig’s AI courses — available on GitHub, with Prolog-adjacent exercises in Python.
  • Markus Triska’s TU Wien materials — on power-of-prolog.com, complete with videos.

The Community

Small, old, friendly. The logic programming community is perhaps a few thousand active practitioners worldwide, centered on academic research but with a practical edge. It has the feel of a specialist guild: everyone knows everyone’s papers, and questions from beginners are almost always welcomed.

The flagship organizations:

  • Association for Logic Programming (ALP) — since 1986. Runs the ICLP conference, publishes TPLP, maintains the newsletter.
  • ICLP (International Conference on Logic Programming) — annual since 1982. The main research venue. Proceedings published by OASIcs / Cambridge.
  • LP/NMR (Logic Programming and Nonmonotonic Reasoning) — biennial, ASP-focused.
  • Prolog User Groups — informal, regional, low-volume.

Places to ask questions:

  • SWI-Prolog discourse forum — fastest response for SWI-specific questions.
  • Stack Overflow, tag ‘prolog’ — well-curated, with regular experts.
  • Reddit r/prolog — low-volume, patient.
  • ##prolog IRC on Libera.Chat — real-time, smaller.

Tips for Surviving the Learning Curve

  1. Do not fight the paradigm. If you’re reaching for loops, mutation, or explicit state, you are not writing Prolog yet. Rewrite the problem as “what is true about the solution.”
  2. Start with pure Prolog. No cut, no assert/retract. Get comfortable with unification and recursion first. Add impurities later, deliberately.
  3. Trace every confusing query. The Byrd-box tracer is Prolog’s gift. Watch each step of the search tree. Understanding comes from seeing the choices.
  4. Name your variables clearly. _ for “don’t care,” descriptive names for everything else. L, L1, L2 is a common anti-pattern.
  5. Read other people’s Prolog. Triska, O’Keefe, the SWI-Prolog library sources. Good Prolog style is a learned thing and it is very different from good Python style.
  6. Use clpfd for arithmetic problems. Don’t use is/2 unless you must. CLP(FD) will give you bidirectional, pure, declarative arithmetic.
  7. Avoid cut until you understand it. When you must use cut, read O’Keefe’s chapter on green vs red cuts first. Most cuts are avoidable.
  8. Do the Zebra puzzle. Really. It’s the rite of passage.
