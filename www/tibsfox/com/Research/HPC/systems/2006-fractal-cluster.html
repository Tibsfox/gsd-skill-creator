System Overview

This is where the HPC archive begins — not as a formal build spec, but as a working render cluster assembled to produce fractal composites at resolutions that no single machine could handle. The workstation plus three render nodes ran Ultra Fractal 4's network rendering across 6 threads.

The Fractal Work

The cluster produced 7.1 gigapixel composite images, tiled at 2400 DPI for physical 30"x40" poster prints. Individual render regions took tens to hundreds of hours of compute time.

Historical Roots

The fractal rendering referenced earlier systems going back to the early 1990s — a 486 with a 387 math coprocessor, Amiga systems. The Mandelbrot set formula z → z² + c in the journal title is not decorative. It is the literal computation that drove the need for more cores.

"More Fun with Fractals" — 13 fractal visualizations, references rendering on 486/387/Amiga — LJ #3223, March 2006

Computing Context: Late 2006

Intel Core 2 Duo had just launched (July 2006). AMD Athlon 64 X2 represented the affordable dual-core option. Multi-core was new territory for consumer hardware — quad-core wouldn't arrive until Intel's Core 2 Quad Q6600 in January 2007. Building a cluster from commodity parts was the only way to get parallel compute at this scale.

Where This Fits

This is System #1 of 13 in the archive. It establishes the pattern that every subsequent build follows: identify the workload, price the parts, calculate $/core, build it from commodity components. The target of 10 nodes at $250 each set the economic framework — the render farm scales horizontally, not vertically.
