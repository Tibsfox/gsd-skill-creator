Contents

  • Domain map and module architecture (ASCII diagrams)
  • 5 research modules: Formation, Licensing, Taxation, Employment, Funding & Support
  • 12 testable success criteria

Stage 2 — Research Reference

  • M1: Entity comparison matrix (5 structures), formation procedure, UBI workflow
  • M2: State business license triggers, local licensing, industry-specific permits
  • M3: B&O tax classifications with 2026 rates, sales tax expansion (Oct 2025), city B&O
  • M4: Minimum wage (state + Everett tiers), overtime thresholds, PFML (1.13%), workers' comp, WA Cares
  • M5: SBA programs, SSBCI ($163.4M), Flex Fund, CDFIs, Snohomish County resources
  • Complete source bibliography (government + professional)

Stage 3 — Mission Package

  • Squadron activation profile (12 roles)
  • 4-wave execution plan with 2 parallel tracks
  • 9 deliverables with acceptance criteria
  • 30 tests (5 safety-critical, 14 core, 6 integration, 5 edge case)
  • Verification matrix mapping all 12 success criteria to tests
  • ~67,500 token budget across ~15 context windows

Usage Instructions

  1. Open the PDF and review the Vision Document (Stage 1) to confirm scope and module structure.
  2. Check the Research Reference (Stage 2) for source quality and data currency — all figures are current through March 2026.
  3. Hand the complete package to gsd-skill-creator via Claude Code for execution.
  4. The Mission Package (Stage 3) provides the wave plan, component specs, and test plan the orchestrator will follow.
  5. CAPCOM checkpoints occur at wave boundaries — the orchestrator will pause for human approval.

Recompiling the LaTeX

  1. Ensure you have XeLaTeX installed with the DejaVu font family.
  2. Run three compilation passes:
    xelatex -interaction=nonstopmode mission.tex
    xelatex -interaction=nonstopmode mission.tex
    xelatex -interaction=nonstopmode mission.tex
  3. Three passes resolve the table of contents, cross-references, and page count.
