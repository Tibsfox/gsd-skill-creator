Overview Core Architecture Instruments Hardware Extensibility Education Mission Pack
Ableton Live is not a conventional digital audio workstation. It is a philosophical statement about the relationship between improvisation and composition — the space between a free-flowing jam and a committed arrangement. The Tab key that switches between Session View and Arrangement View is among the most philosophically loaded keystrokes in music software.

Research Modules

M1: Core Architecture

Architecture Track A · Audio engine

Session View vs. Arrangement View duality. The dual-timeline paradigm, clip launching, scene triggering, Back to Arrangement mechanics. Audio engine internals: sample rate handling, latency compensation, warping algorithms (Complex Pro, Beats, Tones, Texture, Re-Pitch).

M2: Instruments & Sound Design

Synthesis Track A · Sound design

Complete taxonomy of all 20 Live 12 instruments: Analog, Operator, Wavetable, Meld, Drift, Collision, Tension, Electric, Granulator III, Sampler, Simpler, and more. Synthesis architectures mapped: subtractive, FM, wavetable, physical modeling, granular, MPE-native.

M3: Hardware Ecosystem

Hardware Track B · Controllers

Push 3 (standalone and tethered), Ableton Move, Ableton Link protocol. The evolution from Push 1 through Push 3's ARM-based standalone mode. Hardware-software co-design philosophy. Link's peer-to-peer tempo synchronization across devices.

M4: Extensibility Layer

Extension Track B · APIs

Max for Live (M4L) visual patching with full Live Object Model access. Python MIDI Remote Scripts for control surface integration. AbletonOSC for network-based parameter control. LOM hierarchy documented to three levels. Decision tree: when to use which API.

M5: Education & Ecosystem

Education Synthesis · Community

Ableton Certified Trainer program, institutional partnerships, Learning Synths and Learning Music web apps. Loop Summit and community events. The ecosystem of third-party Max for Live developers, Pack creators, and the Ableton user community.
