The Through-Line

Ableton Live's dual-view architecture is the Amiga Principle applied to music production: two specialized execution paths (Session and Arrangement) sharing the same data (tracks, clips, devices) but offering radically different relationships with time. The Session View holds open possibility; the Arrangement View commits to sequence. The Tab key is the boundary between them — the space between improvisation and composition.

The extensibility layer (Max for Live, Python, AbletonOSC) mirrors the GSD ecosystem's own architecture: multiple specialized interfaces to the same underlying model, each optimized for a different relationship with the system. M4L is visual and immediate; Python is deterministic and batch-capable; OSC is networked and real-time. The Live Object Model that unifies them is the equivalent of a DACP bundle schema — a structured contract that allows different agents to interact with the same state.
