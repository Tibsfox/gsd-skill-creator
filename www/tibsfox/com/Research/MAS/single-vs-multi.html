Single-Agent LLMs Outperform Multi-Agent Systems on Multi-Hop Reasoning Under Equal Thinking Token Budgets

Paper: arXiv:2604.02460
Authors: Dat Tran, Douwe Kiela (Stanford University)
Date: April 2, 2026
Categories: cs.CL (Computation & Language), cs.MA (Multiagent Systems)

Recent work reports strong performance from multi-agent LLM systems, but these gains are often confounded by increased test-time computation. When computation is normalized, single-agent systems can match or outperform MAS, yet the theoretical basis and evaluation methodology behind this comparison remain unclear.

1. Theoretical Foundation: Data Processing Inequality

The paper’s central argument rests on information theory. When a multi-agent system receives messages M derived from full context C, the Data Processing Inequality guarantees:

I(Y; C) ≥ I(Y; M)

The mutual information between the correct answer Y and the full context C is always greater than or equal to the mutual information between Y and any derived messages M. In plain language: every time agents pass messages to each other, information is lost. Under a fixed token budget with perfect context utilization, single-agent systems are provably more information-efficient.

4. When MAS Wins: Context Degradation

The theoretical prediction identifies a precise boundary. When single-agent context utilization degrades sufficiently, MAS decomposition recovers information that the single agent loses:

Corruption Level (α)Noise TypeWinner
α = 0.3 (mild)Masking, substitutionSAS leads
α = 0.5 (moderate)Masking, substitutionComparable
α = 0.7 (heavy)Masking, substitutionMAS clearly better

Critical insight: MAS advantage emerges specifically when information is corrupted rather than merely removed. Filtering and decomposition stabilize reasoning under noise. Deletion and distractors show weaker trends — SAS advantage persists across all levels.

6. Methodological Discoveries

Gemini API Budget Artifact

Gemini’s visible thought text plateaus below requested budgets for SAS, while MAS surfaces more visible thinking via multiple calls. This creates misleading apparent MAS advantages in published benchmarks. The paper introduces SAS-L (structured pre-analysis prompts) to encourage deeper internal reasoning without additional compute.

Benchmark Memorization

Paraphrasing ablation: light paraphrasing decreased accuracy (0.331→0.326) but deep semantic paraphrasing improved it (0.331→0.358). This suggests original questions suffer from memorization or overfitting from pretraining.

8. Implications for Our Architecture

  • Plan quality checking, architectural decisions: Single-agent Opus is the right call — DPI says full context beats message-passing
  • Fleet-mission pattern (parallel builds, research): Well-justified — these are parallelizable tasks, not reasoning chains
  • Explore agents for large codebases: Validated — large/noisy context is exactly where MAS decomposition helps
  • Convoy/mayor coordination: Justified as orchestration (2604.01020), not reasoning — consistent with this paper
  • Benchmark evaluation: Must control for thinking tokens when comparing architectures; published benchmarks may overstate MAS benefits
