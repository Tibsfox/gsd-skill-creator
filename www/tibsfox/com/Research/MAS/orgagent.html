OrgAgent: Organize Your Multi-Agent System like a Company

Paper: arXiv:2604.01020
Authors: Yiru Wang, Xinyue Shen, Yaohui Han, Michael Backes, Pin-Yu Chen, Tsung-Yi Ho
Date: April 1, 2026
Categories: cs.MA (Multiagent Systems), cs.AI

While large language model-based multi-agent systems have shown strong potential for complex reasoning, how to effectively organize multiple agents remains an open question. OrgAgent separates collaboration into governance, execution, and compliance layers.

1. Three-Layer Architecture

Layer A: Governance

RoleFunctionOur Equivalent
CEOStrategic alignment — determines what needs to be donelab-director
CTOTechnical soundness — ensures approach is validgsd-planner
COOResource efficiency — optimizes cost and throughputmayor-coordinator

The governance layer determines the execution mode for each task based on complexity assessment, routing simple tasks to DIRECT mode and complex tasks to FULL MAS.

Layer B: Execution

RoleFunctionOur Equivalent
DrafterGenerates initial answersgsd-executor
ReviewerExamines quality, provides feedbackgsd-verifier
SpecialistTargeted domain expertisesecurity-reviewer, performance-profiler, etc.

Three execution modes:

  • DIRECT — single generation, cheapest (simple factual questions)
  • LIGHT MAS — with review loop (moderate complexity)
  • FULL MAS — with specialist support (complex multi-step reasoning)

Layer C: Compliance

Workers are assigned dynamically from a pool of reusable skill profiles rather than fixed to specific benchmarks:

Stronger models (GPT-5 mini, GPT-OSS-120B) developed sharper specialization under hierarchy; weaker models (Llama-3.1-8B) showed diffuse skill distribution.

5. Key Findings

Hierarchy Improves “Knowing What You Don’t Know”

Abstention rates on unanswerable questions increased up to 39.78% under hierarchy. The compliance layer (CSO/CCO) catches cases where the execution layer generates confident but wrong answers. This maps to our gsd-verifier catching false completions.

When Hierarchy Works Best

  • Stable skill assignment — workers keep their specialization across tasks
  • Controlled information flow — governance decides what context each worker sees
  • Layered verification — every answer passes through review before compliance

When Hierarchy Doesn’t Help

MuSR (multistep soft reasoning) showed mixed results. Tasks requiring creative synthesis rather than structured decomposition may not benefit from corporate-style hierarchy. This aligns with 2604.02460’s finding that single agents handle deep reasoning better.

6. Design Principles Extracted

  1. Separate planning from execution — governance decides mode before work begins
  2. Match execution mode to task complexity — DIRECT for simple, FULL for complex
  3. Compliance is not optional — format validation + answer validation are distinct roles
  4. Dynamic skill assignment — workers draw from a skill pool, not fixed personas
  5. Hierarchy saves tokens — routing simple tasks to DIRECT mode avoids unnecessary MAS overhead
  6. Strong models specialize; weak models don’t — hierarchy amplifies model capability
