Supply-Chain Poisoning Attacks Against LLM Coding Agent Skill Ecosystems

Paper: arXiv:2604.03081
Authors: Yubin Qu, Yi Liu, Tongcheng Geng, Gelei Deng, Yuekang Li, Leo Yu Zhang, Ying Zhang, Lei Ma
Date: April 3, 2026
Categories: cs.CR (Cryptography & Security), cs.AI, cs.CL

LLM-based coding agents extend their capabilities via third-party agent skills distributed through open marketplaces without mandatory security review. Unlike traditional packages, these skills are executed as operational directives with system-level privileges, so a single malicious skill can compromise the host.

1. The Attack: DDIPE

Document-Driven Implicit Payload Execution embeds malicious logic within legitimate technical structures of skill documentation — code examples and configuration templates — rather than explicit instructions. Agents copy-paste what they treat as “official examples” from documentation during in-context learning, executing the payload without anyone asking.

Two Embedding Strategies

StrategyMechanismExploitation
Code Example PoisoningMalicious logic in Markdown code blocks (```python, ```bash)Models reproduce “official examples” verbatim
Config Template PoisoningBackdoored YAML/JSON/Makefile templatesAgents apply configuration files without scrutiny

Three Camouflage Techniques

  • Functional compliance disguise: Package data theft as “environment telemetry” or “security auditing”
  • Silent exception suppression: Wrap malicious calls in try/except: pass for uninterrupted execution
  • False authority suggestion: Add compliance certification language; name C2 endpoints to resemble internal services

4. Key Insights

Brevity Evades Safety

Executed payloads average 1,850 bytes vs. 2,214 bytes for refused. A universal pip configuration breach — 479 bytes, 9-line Python script — was executed by all three models under Claude Code, interpreted as routine environment setup.

Alignment + Architecture Asymmetry

Model AlignmentClaude Code DEROpenHands DERAmplificationSleeper Payloads
Sonnet 4.6 (strong)251154.6×5 (4.3%)
GLM-4.7 (weak)2629011.3×219 (75.8%)

Implication: Strong alignment makes architecture nearly redundant. Weak alignment makes architecture critical.

Cross-Model Diversity as Defense

Only 17 samples (1.6%) bypass all three models simultaneously on Claude Code. Heterogeneous model deployment compresses attack surface from 13–20% to below 2%.

6. Defense Recommendations

  1. Multi-model verification ensemble — deploy heterogeneous models as security barriers; compresses attack surface from 20% to <2%
  2. Semantic intent analysis — audit skill semantics beyond lexical patterns; current static analysis catches 90.7% but 2.5% evade both detection AND alignment
  3. Per-skill permission scoping — grant execution privileges at individual skill level, not blanket authorization
  4. Architecture calibrated to alignment — prioritize model updates for strong models, robust sandboxing for weak ones
  5. Explicit trust-boundary annotations — mark repository-provided content as untrusted in system prompts (validated by OpenHands fix)
  6. Supply-chain integrity verification — mandatory security review for marketplace skills (631,813 with none currently)
  7. Treat skill documentation as executable code — signature verification and content-integrity checks before context injection
