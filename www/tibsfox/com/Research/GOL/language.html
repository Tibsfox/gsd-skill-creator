“Go is a language designed by people who write C and aimed at people who write Java and work at Google.” — A community quip that captures the design center: C’s performance instincts with Java’s safety expectations, minus both languages’ complexity.

Every type has a zero value. Numeric types are 0, booleans are false, strings are "", pointers/slices/maps/channels/interfaces are nil. The Go Proverb: “Make the zero value useful.”

Variables

var age = 42                  // type inferred

)

Functions

// Multiple return values
func divide(a, b float64) (float64, error) {
    if b == 0 {
        return 0, fmt.Errorf("division by zero")
    }
    return a / b, nil
}

// Named return values
func swap(a, b int) (x, y int) {
    return b, a
}

// Variadic
func sum(nums ...int) int {
    total := 0
    for _, n := range nums { total += n }
    return total
}

// First-class / closures
func makeAdder(base int) func(int) int {
    return func(x int) int { return base + x }
}

// Defer: runs when the enclosing function returns (LIFO order)
func readFile(path string) ([]byte, error) {
    f, err := os.Open(path)
    if err != nil { return nil, err }
    defer f.Close()
    return io.ReadAll(f)
}

Structs

type User struct {
    ID    int64  `json:"id"`
    Name  string `json:"name"`
    Email string `json:"email"`
}

// Method with pointer receiver (can modify the struct)
func (u *User) SetName(name string) { u.Name = name }

// Method with value receiver (read-only copy)
func (u User) String() string { return fmt.Sprintf("%s <%s>", u.Name, u.Email) }

// Embedding (composition, not inheritance)
type Admin struct {
    User           // promoted fields and methods
    Level string
}

Interfaces

Implicit satisfaction: A type implements an interface simply by having the right methods. No implements keyword. This is Go’s version of duck typing at compile time.

type Reader interface { Read(p []byte) (n int, err error) }
type Writer interface { Write(p []byte) (n int, err error) }
type ReadWriter interface { Reader; Writer }  // interface embedding

// io.Reader is the canonical example:
// *os.File, *bytes.Buffer, *strings.Reader, *http.Response.Body
// all implement Reader without declaring it.

// The empty interface (any) accepts all types
func printAnything(v any) { fmt.Println(v) }

// Type assertion
if s, ok := v.(string); ok {
    fmt.Println("string:", s)
}

// Type switch
switch v := x.(type) {
case string:  fmt.Println("string:", v)
case int:     fmt.Println("int:", v)
default:      fmt.Println("other")
}

Generics (Go 1.18)

// Generic function with type constraint
func Map[T, U any](s []T, f func(T) U) []U {
    result := make([]U, len(s))
    for i, v := range s { result[i] = f(v) }
    return result
}

// Type constraint using interface
type Number interface {
    ~int | ~int64 | ~float64
}

func Sum[T Number](nums []T) T {
    var total T
    for _, n := range nums { total += n }
    return total
}

// Usage
doubled := Map([]int{1, 2, 3}, func(n int) int { return n * 2 })

Error Handling

// The error interface
type error interface { Error() string }

// The "if err != nil" pattern
result, err := doSomething()
if err != nil {
    return fmt.Errorf("doSomething failed: %w", err) // wrap with %w
}

// Error wrapping and checking (Go 1.13+)
if errors.Is(err, os.ErrNotExist) { /* sentinel error check */ }

var pathErr *os.PathError
if errors.As(err, &pathErr) { /* type-specific error handling */ }

// Custom error type
type ValidationError struct {
    Field   string
    Message string
}
func (e *ValidationError) Error() string {
    return fmt.Sprintf("%s: %s", e.Field, e.Message)
}

Why if err != nil? Go chose explicit error returns over exceptions because exceptions hide control flow. Rob Pike: “Errors are values.” You can store them, pass them, compose them. The verbosity is the trade-off for clarity.

Slices, Arrays, Maps

// Array (fixed size, value type)
var a [5]int

// Slice (dynamic, reference to underlying array)
s := []int{1, 2, 3}
s = append(s, 4, 5)
sub := s[1:3] // [2, 3] — shares underlying array

// Map
m := map[string]int{"a": 1, "b": 2}
m["c"] = 3
v, ok := m["d"]  // ok is false if key missing
delete(m, "a")

// Range
for i, v := range s { fmt.Println(i, v) }
for k, v := range m { fmt.Println(k, v) }

Pointers

x := 42
p := &x          // p is *int, points to x
fmt.Println(*p)  // 42 (dereference)
*p = 100         // x is now 100

// No pointer arithmetic in Go (unlike C)
// Pointers are for: avoiding copies, modifying values, nil-able types

Control Flow

// for is the ONLY loop keyword
for i := 0; i < 10; i++ { }    // C-style
for i < 10 { }                  // while-style
for { }                          // infinite
for i, v := range collection { } // range

// if with init statement
if err := doThing(); err != nil { return err }

// switch (no fallthrough by default!)
switch day {
case "Mon", "Tue", "Wed", "Thu", "Fri":
    fmt.Println("weekday")
default:
    fmt.Println("weekend")
}

require (
    github.com/gin-gonic/gin v1.10.0
    go.uber.org/zap v1.27.0
)

// import
import (
    "fmt"
    "net/http"
    "github.com/gin-gonic/gin"
)

Capitalized names are exported (public). Lowercase names are unexported (package-private). No public/private keywords needed.

Defer, Panic, Recover

  • defer — schedules a call to run when the function returns (LIFO). Used for cleanup (close files, unlock mutexes, flush buffers).
  • panic — unrecoverable error. Unwinds the stack, running deferred functions. NOT for normal error handling.
  • recover — called inside a deferred function, catches a panic and returns the value. Used in libraries to prevent panics from crashing the caller.

Testing

// math_test.go
func TestAdd(t *testing.T) {
    tests := []struct {
        a, b, want int
    }{
        {1, 2, 3},
        {0, 0, 0},
        {-1, 1, 0},
    }
    for _, tt := range tests {
        t.Run(fmt.Sprintf("%d+%d", tt.a, tt.b), func(t *testing.T) {
            got := Add(tt.a, tt.b)
            if got != tt.want {
                t.Errorf("Add(%d, %d) = %d, want %d", tt.a, tt.b, got, tt.want)
            }
        })
    }
}

Table-driven tests are the idiomatic Go testing pattern. No assertion library needed (though testify is popular).
