Go is one of the easiest languages to learn. The spec fits in a few dozen pages. The standard library is production-grade. “A Tour of Go” takes an afternoon. You can be productive in a week.

The Go Proverbs

Rob Pike’s “Go Proverbs” talk (GopherCon 2015) distills Go’s philosophy:

  • “Don’t communicate by sharing memory, share memory by communicating.”
  • “Concurrency is not parallelism.”
  • “Channels orchestrate; mutexes serialize.”
  • “The bigger the interface, the weaker the abstraction.”
  • “Make the zero value useful.”
  • “A little copying is better than a little dependency.”
  • “Clear is better than clever.”
  • “Errors are values.”
  • “Don’t panic.”
  • “Documentation is for users.”

A Reading Order That Works

  1. A Tour of Go (go.dev/tour) — 2–4 hours
  2. Go by Example (gobyexample.com) — reference as you code
  3. Effective Go (go.dev/doc/effective_go) — read once, return often
  4. Build something: an HTTP API with net/http, a CLI tool with cobra, or a concurrent pipeline
  5. The Go Programming Language (Donovan & Kernighan) — the comprehensive reference
  6. 100 Go Mistakes (Harsanyi) — for leveling up from beginner to idiomatic
  7. Concurrency in Go (Cox-Buday) — when you need deep concurrency patterns
  8. Read Kubernetes or Docker source — production Go at scale
