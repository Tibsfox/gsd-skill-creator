Overview IP Layer TCP Core Congestion Control Companion Protocols Transport Evolution Mission Pack
In 1978, Jon Postel split what had been a single Transmission Control Program into two: IP for routing packets, TCP for reliable delivery. That split — a separation of concerns executed with surgical precision — is what the Amiga Principle looks like at civilizational scale. The spaces between packets are where TCP lives. The spaces between agents are where GSD lives.

Research Modules

M1: IP Layer

IP Track A · Internet layer

Complete IPv4 header (13 fields, bit-level). IPv6 fixed header and extension header chain. Fragmentation mechanics, PMTUD, ARP for IPv4, NDP/SLAAC for IPv6. CIDR addressing, special blocks, dual-stack operation.

M2: TCP Core

TCP Track B · Transport layer

Complete TCP segment format per RFC 9293. All control flags, options registry (SACK, timestamps, window scale). Three-way handshake, ISN generation. Full 11-state FSM with simultaneous close, half-open recovery, RST handling.

M3: Congestion Control

Congestion Track B · Flow dynamics

Van Jacobson's slow start and congestion avoidance. CUBIC cubic window formula. BBR model-based algorithm (BtlBw/RTprop). SACK block format, ECN mechanics, RTO computation. Fairness analysis between CUBIC and BBR flows.

M4: Companion Protocols

Protocols Track A · Supporting stack

UDP 8-byte header and semantics. ICMP v4 and ICMPv6 message type catalogues. ARP mechanics and spoofing surface. NDP, SLAAC, DAD for IPv6. DNS query/response format. DHCP lease lifecycle. DoH, DoT, DoQ extensions.

M5: Transport Evolution

Evolution Synthesis · Next-gen transport

TLS 1.3 over TCP: 1-RTT and 0-RTT handshake. QUIC: UDP-based, TLS-integrated, stream-multiplexed, connection-migrating. HTTP/3 and QPACK. TCP Fast Open, Multipath TCP. Protocol ossification and QUIC's architectural response.
