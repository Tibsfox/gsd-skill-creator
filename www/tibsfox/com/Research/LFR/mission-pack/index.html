Contents

Stage 1 — Vision Document: Narrative vision for a 1:1-scale Minecraft simulation of Weyerhaeuser's 2.5M-acre PNW timberlands. Five problem statements, six research modules (Terrain Genesis, Forest Canopy, Climate Bridge, Fire & Disturbance, Mesh Telemetry, Living World Engine), chipset configuration, 12 success criteria.

Stage 2 — Research Reference: Sourced findings from USGS 3DEP, NASA FIRMS, NOAA NCEI, USFS FIA, Weyerhaeuser SEC filings, Arnis world generation engine, OpenWeatherMap, and NIFC fire perimeters. Safety and sensitivity considerations. Complete bibliography.

Stage 3 — Mission Package: Fleet activation profile (17 roles). Wave execution plan with parallel terrain and data tracks. 8 deliverables, 36 tests (7 safety-critical), verification matrix, token budget (~385K across 45–55 context windows), dependency graph.

Usage

  1. Review the PDF to understand scope, modules, and wave structure
  2. Feed the mission package to gsd-skill-creator via Claude Code
  3. GSD orchestrator decomposes into waves and assigns to agents
  4. Wave 0 (Foundation) runs first, producing schemas consumed by all subsequent waves
  5. Waves 1A and 1B run in parallel — terrain generation and data plugin development
  6. Wave 2 (Synthesis) integrates all streams into the Living World Engine
  7. Wave 3 (Publication) produces educational content, server config, and tests

Recompilation

The LaTeX source requires XeLaTeX with DejaVu fonts:

xelatex -interaction=nonstopmode living-forest-mission.tex (three passes)
