North-south vs east-west

  • North-south — traffic entering the cluster from outside. Handled by API gateways. Rate limiting, auth, transformation, public SSL.
  • East-west — traffic between services within the cluster. Handled by service meshes. mTLS, retries, circuit breaking.

The line between gateway and mesh has blurred. Envoy Gateway (2022) is the CNCF project unifying the two under Gateway API.

The unification

OpenTelemetry (2019) merged OpenCensus (Google) and OpenTracing (CNCF). It provides a vendor-neutral API, SDK, wire protocol (OTLP), and collector. The W3C Trace Context spec (2020) standardizes how trace IDs propagate across service boundaries via the traceparent HTTP header. CNCF graduated in 2024.

In 2026, the modern observability stack is: OpenTelemetry SDKs in your code, OTLP over the wire, a collector in the cluster, and any backend you want (Jaeger/Tempo/Honeycomb for traces, Prometheus/VictoriaMetrics for metrics, Loki/Elastic for logs). Vendor-neutrality is the point.

The serverless trade-off

Pro: pay per invocation, scale to zero, no server management. Con: cold starts, observability is harder, vendor lock-in is high, debugging is painful, long-running or stateful workloads are awkward. Best fit: event-driven glue, scheduled jobs, low-traffic APIs, webhooks.

8. Durable Execution and Workflow Engines

A modern category: runtimes that let you write “just code” for long-running, fault-tolerant workflows. The runtime persists execution state, so your workflow survives crashes, restarts, and long waits (days or months) without you having to build state machines manually.

10. Platform Engineering & Internal Developer Platforms

The 2020s organizational answer to microservices operational complexity: build an Internal Developer Platform (IDP) that abstracts the infrastructure so application teams don’t have to become Kubernetes experts. Treat the platform as a product, not a project.

  • Backstage — Spotify’s open-source developer portal, released 2020, CNCF incubating. The de facto standard.
  • Port — commercial IDP
  • Humanitec — commercial IDP
  • Cortex — service catalog and scorecards
  • OpsLevel, Effx, Roadie — competing catalogs

Matthew Skelton and Manuel Pais’s Team Topologies (2019) is the canonical text: stream-aligned, platform, enabling, and complicated-subsystem teams. The platform team’s job is to reduce cognitive load on stream-aligned teams by providing “golden paths.”

13. Getting Started in 2026

A reasonable stack to stand up a new service-oriented system in 2026:

  1. Containers: Docker for local dev, containerd in production.
  2. Orchestration: Kubernetes via a managed provider (EKS / GKE / AKS) unless you have strong reasons to self-manage.
  3. Service-to-service: gRPC for internal (Buf Connect for the ergonomics); REST + OpenAPI for external.
  4. Async: Kafka or Redpanda; AsyncAPI for event schemas.
  5. API gateway: Envoy Gateway or Kong at the edge.
  6. Service mesh (optional): Linkerd if you need mTLS and retries but not the full Istio feature set. Skip if your scale doesn’t need it.
  7. Observability: OpenTelemetry SDKs + OTLP + Prometheus + Grafana + Tempo. Honeycomb or Datadog if you want SaaS.
  8. CI/CD: GitHub Actions or GitLab CI + Argo CD for GitOps.
  9. Secrets: Vault or External Secrets Operator with your cloud KMS.
  10. Durable workflows: Temporal for long-running or multi-step processes; Argo Workflows for batch.
  11. Developer portal: Backstage once you have >5 services.
  12. Platform engineering: treat the platform as a product, not a project.

Or, if your team is small and your scale is modest: a single Rails/Django/FastAPI monolith on a managed Postgres instance, deployed via a single CI pipeline. This is the 2026 answer too. The best architecture is the one you can actually run.

14. The CNCF Landscape

landscape.cncf.io is the visual map of the cloud-native ecosystem, maintained by the Cloud Native Computing Foundation since 2016. As of 2026 it tracks over 1,600 projects and products across every layer of the stack: provisioning, runtime, orchestration, app definition, observability, security, storage, networking, serverless, machine learning, databases, messaging, and on. The landscape is both a map and a mirror — it shows the territory and it shows the excess.

The CNCF project lifecycle: Sandbox (early stage) → Incubating (production deployments documented) → Graduated (mature, widely adopted). As of 2026, graduated projects include Kubernetes, Prometheus, Envoy, etcd, containerd, Fluentd, Jaeger, Helm, OpenTelemetry, Linkerd, Istio, CoreDNS, Vitess, Cilium, Argo, Flux, CNI, Falco, OPA, Notary, Rook, SPIFFE/SPIRE, TUF, and several more.

15. Tooling Gaps in 2026

  • Testing. Contract testing and chaos engineering are still under-adopted. Pact works but is rarely integrated into CI.
  • Developer experience across languages. Buf has made this better for Protobuf but OpenAPI tooling is still fragmented.
  • Cross-cluster service discovery. Multi-region and multi-cluster remain painful.
  • Service mesh simplification. Istio is still too complex; Cilium and Linkerd are simpler but not fully equivalent.
  • Cost observability. “Which service is costing us how much?” is still surprisingly hard to answer.
