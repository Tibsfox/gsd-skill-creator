OpenStack Cloud Platform

NASA Engineering Rigor · IaaS Self-Hosting · OpenStack · Kolla-Ansible · Cloud Primitives
Overview Architecture NASA SE Rigor IaaS Self-Hosting Networking & Orchestration Cloud Comparison Mission Pack
In 2010, NASA's Nebula cloud platform and Rackspace's Cloud Files merged into OpenStack. The decision was not a business strategy — it was a statement about the nature of infrastructure knowledge. NASA had built something that worked. They could have contracted it exclusively, licensed it commercially, or kept it internal. Instead, they open-sourced it, because the code — like the Systems Engineering Handbook, like every mishap report — was created with public investment for public benefit. Understanding OpenStack means understanding every cloud platform on earth, because OpenStack names the primitives that every other platform obscures behind product branding.

Research Modules

M1: OpenStack Architecture & Core Services

Infrastructure Track A · Service primitives

Keystone identity, Nova compute, Glance images, Horizon dashboard. Service catalog and API contracts. Containerized deployment via Kolla-Ansible. The integration dependency chain from identity through orchestration. Message queue architecture with RabbitMQ and database backends.

M2: NASA Systems Engineering Applied to Cloud Ops

NASA SE Track A · Engineering rigor

The SE Engine's 17 processes mapped to cloud deployment. Life-cycle reviews from MCR through PLAR as deployment gates. ConOps, SEMP, and RVM document types. NPR 7123.1D requirements traceability. Tailoring NASA methodology for lab-scale infrastructure. Verification methods: Test, Analysis, Inspection, Demonstration (TAID).

M3: IaaS Self-Hosting & Ceph Integration

Storage Track B · Data sovereignty

Cinder block storage with LVM and Ceph backends. Swift object storage ring architecture. Ceph RADOS integration: OSDs, MONs, CRUSH maps. Storage replication and erasure coding. Why self-hosted storage matters for data sovereignty. Backup strategies and disaster recovery procedures.

M4: Neutron Networking & Heat Orchestration

Networking Track B · SDN + orchestration

Neutron ML2 plugin architecture. OVS vs. OVN backends. Software-defined networking: tenant isolation, security groups, floating IPs, DHCP. Heat orchestration templates (HOT format), stack lifecycle, auto-scaling. Network design patterns for single-node and multi-node topologies.

M5: Cloud Comparison & Digital Sovereignty

Operations Track C · Strategic analysis

OpenStack vs. AWS vs. GCP vs. Azure: the primitive translation table. When self-hosted makes sense and when it does not. TCO analysis frameworks. Kubernetes and OpenStack: complementary layers. The GSD-OS cloud layer. Digital sovereignty, vendor lock-in economics, and the open infrastructure commons.
