How to Run This Mission

  1. Hand the PDF to your GSD orchestrator in Claude Code. The document is self-contained — no additional context required.
  2. Start with Wave 0 (Haiku / PLAN agent). Let it produce the terminology schema and source index before launching the parallel tracks.
  3. CAPCOM gate after Wave 0: review the terminology schema before proceeding. This is where you can adjust scope or add terms.
  4. Launch Waves 1A and 1B in parallel. Track A handles MOD-1 (Foundations) and MOD-2 (Shannon). Track B handles MOD-3 (Hierarchy) and MOD-4 (Limits). Both use the Wave 0 schemas as shared prefix — cache hit guaranteed.
  5. CAPCOM gate after Wave 2: review the GSD Architectural Synthesis and integration pass before publication. This is the judgment-heavy output.
  6. Wave 3 publication: Sonnet assembles, verifies sources against the bibliography, runs the test suite, and produces the final document.

Recompile the LaTeX

Three XeLaTeX passes required to resolve TOC, cross-references, and page count:

xelatex -interaction=nonstopmode mission.tex
xelatex -interaction=nonstopmode mission.tex
xelatex -interaction=nonstopmode mission.tex

Requires XeLaTeX with DejaVu fonts (pre-installed on Ubuntu). Packages: fontspec, xcolor, titlesec, fancyhdr, tabularx, colortbl, booktabs, tcolorbox, hyperref, lastpage.
