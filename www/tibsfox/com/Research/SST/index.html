States, Symbols & Tape

Turing Machines · Shannon Fungibility · Chomsky Hierarchy · Undecidability · GSD Synthesis
Overview Foundations Shannon Hierarchy Limits GSD Synthesis Mission Pack
The insight arrived as a vibe: a finite state machine with infinite tape is the same as an infinite state machine with finite tape. Shannon turned this exact intuition into a theorem in 1956. States and tape alphabet richness are fungible. The complexity budget of computation can be denominated in either currency. But tape length is a different resource entirely — restrict it and you lose real computational power, provably and permanently.

Research Modules

M1: Formal Foundations

Theory Track A · Turing 1936

Turing's original computing machine: finite state control, one-way infinite tape, read-write head, transition function. Configuration triples, Church-Turing thesis, deterministic vs. nondeterministic machines. Finite state machines as the no-tape special case.

M2: Shannon Trade-off

Shannon Track A · Fungibility

Shannon (1956): both directions of the fungibility theorem. The UTM size frontier from (15,2) to (2,18). Minsky's 7-state 4-symbol machine. Wolfram's (2,3) conjecture and Smith's 2007 weak-universality proof. The complexity budget metaphor.

M3: Tape-Length Hierarchy

Hierarchy Track B · Chomsky 1956

Chomsky hierarchy Type 0 through Type 3. Linear bounded automata: Myhill 1960, Landweber 1963. Configuration space argument: bounded tape implies finite configurations implies decidability. The LBA open problem.

M4: Limits & Undecidability

Limits Track B · Turing 1936

Halting problem by diagonalization. Why infinite tape is necessary for undecidability. Rice's theorem: no non-trivial property of TM output is decidable. Contrast with bounded-tape decidability. Gödel's incompleteness connection.

M5: GSD Architectural Synthesis

GSD Synthesis · Opus

Context window as bounded tape. LLMs as bounded-tape machines. Fresh-context subagents as tape-reset strategy. DACP three-part bundle as valid state serialization. The Amiga Principle as Shannon's theorem applied to agent architecture.

Computational Hierarchy

COMPUTATIONAL POWER HIERARCHY
=======================================================
  TURING MACHINE                [Infinite tape]
  Finite states + unbounded write tape
  Recognizes: Recursively Enumerable languages (Type 0)
  Halting: Undecidable
     |
     | restrict tape to O(n) cells (input length)
     v
  LINEAR BOUNDED AUTOMATON      [Tape ~ input length]
  Finite states + tape = f(|input|)
  Recognizes: Context-Sensitive languages (Type 1)
  Halting: Decidable per input
     |
     | restrict to stack or O(log n) scratch tape
     v
  PUSHDOWN / LOG-SPACE          [Bounded aux tape]
  Finite states + auxiliary bounded tape
  Recognizes: Context-Free languages (Type 2)
     |
     | remove writable tape entirely
     v
  FINITE STATE MACHINE          [No tape]
  Finite states + read-only left-to-right scan
  Recognizes: Regular languages (Type 3)
=======================================================
  Shannon (1956): within ANY row, states <-> symbols
  Moving DOWN: permanent, provable power loss
=======================================================
