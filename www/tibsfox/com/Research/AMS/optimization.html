Optimization theory formalizes the tradeoffs inherent in every agentic system — more agents vs fewer tokens, accuracy vs cost, breadth vs depth. Pareto frontiers reveal the set of solutions where no objective can improve without degrading another.

Module Overview

This module examines five optimization problems from the agentic technology research: Pareto hypervolume for multi-objective agent orchestration, token budget optimization under hard constraints, cost-accuracy tradeoffs in financial document processing, the iron triangle as a constraint-based engineering framework, and greedy scheduling vs optimal assignment in parallel execution.

Key Mathematical Concepts

Pareto Optimality and Hypervolume

x* is Pareto optimal if ∄ x such that:
fi(x) ≤ fi(x*) for all i, and
fj(x) < fj(x*) for some j

Hypervolume HV(S) = λ({q : ∃ p ∈ S, p dominates q, q dominates r})

OrgAgent (arXiv:2604.01020) achieves a 102.73% improvement by finding a Pareto-optimal point that simultaneously reduces token usage (74.52% reduction) and improves accuracy. The Pareto frontier for agent orchestration has at least three objectives: accuracy, latency, and token cost. Hypervolume measures the “volume” of objective space dominated by the solution set.

Token Budget Optimization

maximize quality(x) subject to:
∑i tokens(xi) ≤ B (budget constraint)
xi ≥ xmin (minimum context per agent)

This is a variant of the knapsack problem.

Given a fixed token budget B, the optimizer must allocate tokens across agents to maximize output quality. Each agent requires minimum context (xmin) to function. The MCP overthinking loop problem (658x cost inflation, arXiv:2602.14798) shows what happens when this constraint is violated — unbounded token consumption with diminishing returns.

Cost-Accuracy Tradeoffs

minimize cost(x) subject to accuracy(x) ≥ α
or equivalently:
maximize accuracy(x) subject to cost(x) ≤ C

Financial document processing demands both low cost and high accuracy. The dual formulation shows these are equivalent problems — you either minimize cost for a given accuracy floor or maximize accuracy for a given budget ceiling. The 658x overthinking multiplier demonstrates that naive optimization (just keep trying) fails catastrophically.

Iron Triangle Constraints

The project management iron triangle — scope, time, cost — maps to a constrained optimization problem. Fixing any two determines the third. In agent orchestration, the three constraints become: task complexity (scope), latency budget (time), and token/compute cost. The Gastown convoy model fixes scope per convoy and optimizes the time-cost tradeoff.

Greedy vs Optimal Scheduling

Greedy scheduling assigns each task to the first available agent. Optimal scheduling (NP-hard in general) considers global task dependencies. For DAG-structured workloads, the gap between greedy and optimal narrows because topological constraints limit choices. The wave model is a practical middle ground: optimal within each wave, greedy across waves.
