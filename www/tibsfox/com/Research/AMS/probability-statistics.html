Agentic systems are inherently non-deterministic. The same prompt yields different outputs 10–15% of the time. Probability and statistics provide the tools to measure, model, and account for this variability — from Monte Carlo benchmarking to Bayesian hallucination detection.

Module Overview

This module formalizes five statistical concepts from the agentic research: the 10–15% non-determinism rate as a measurable property of LLM systems, Monte Carlo evaluation as the correct benchmarking methodology, Bayesian hallucination detection (HaloProbe), Cohen’s kappa for inter-rater agreement in agent evaluation, and confidence intervals for security bypass rates.

Key Mathematical Concepts

Non-Determinism in LLM Systems

P(output1 = output2 | same input) ≈ 0.85–0.90

Variance: Var(output) > 0 even with temperature = 0
(floating-point non-determinism, batching effects)

Even with temperature set to zero, LLM outputs vary 10–15% due to floating-point non-determinism in GPU computation, batching effects, and stochastic sampling in attention layers. Any evaluation that runs a benchmark once and reports a single number is statistically meaningless. This is why Monte Carlo evaluation is necessary.

Monte Carlo Evaluation

μ̂ = (1/N) ∑i=1N f(xi)

Standard error: SE = σ / √N
95% CI: μ̂ ± 1.96 · SE

Agent benchmarks require multiple independent runs (N ≥ 30) to establish reliable statistics. The standard error shrinks as 1/√N, so quadrupling runs halves the error. The MIRAGE benchmark (arXiv:2603.21687) found models fabricate visual descriptions at >60% rate — a finding that required Monte Carlo methodology to establish with confidence.

Bayesian Hallucination Detection (HaloProbe)

P(hallucination | evidence) = P(evidence | hallucination) · P(hallucination) / P(evidence)

Prior: P(hallucination) ≈ 0.10–0.60 (task-dependent)
Update with each verification signal.

HaloProbe uses Bayesian updating to estimate the probability that a given output is hallucinated. The prior depends on the task domain (factual retrieval has lower base rates than creative generation). Each piece of verification evidence — source citation found, calculation confirmed, entity exists — updates the posterior downward. Outputs that survive multiple evidence checks have low hallucination probability.

Cohen’s Kappa

κ = (po - pe) / (1 - pe)

po = observed agreement
pe = expected agreement by chance

κ = 0.88 ⇒ “almost perfect” agreement (Landis & Koch)

When two raters (or two agents, or an agent and a human) evaluate the same set of outputs, Cohen’s kappa measures agreement beyond chance. The 0.88 score from the agentic benchmark evaluation indicates near-perfect inter-rater reliability. Values below 0.40 are “fair”; the gap between 0.40 and 0.88 represents the quality improvement from structured evaluation protocols.

Confidence Intervals for Bypass Rates

p̂ = x/n (sample proportion)
95% CI: p̂ ± 1.96 √(p̂(1-p̂)/n)

DDIPE bypass: 11.6–33.5% across platforms
Cross-model: 1.6% (all three simultaneously)

Security bypass rates from arXiv:2604.03081 range from 11.6% to 33.5% across Claude Code, OpenHands, Codex, and Gemini CLI. The cross-model rate (1.6% bypass all three simultaneously) provides the critical defense metric. These are sample proportions with confidence intervals — the true population rate lies within the interval with 95% probability.
