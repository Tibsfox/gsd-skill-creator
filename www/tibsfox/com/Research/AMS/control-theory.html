Control theory treats agentic systems as plants to be regulated — with feedback loops correcting errors, PID controllers maintaining setpoints, and STPA identifying hazardous control actions. Every guardrail is a control system boundary. Every drift is a control failure.

Module Overview

This module formalizes five control-theoretic concepts from the agentic technology research: feedback loops in meta-agentic bootstrapping, PID-analogous error correction in agent validation, STPA (Systems-Theoretic Process Analysis) for ML safety, guardrails as control system boundaries, and drift detection and correction in enterprise AI systems.

Key Mathematical Concepts

Feedback Loops in Meta-Agentic Systems

y(t) = G(s) · [r(t) - H(s) · y(t)]

Closed-loop transfer: Y/R = G/(1 + GH)
Stability: all poles of (1 + GH) in left half-plane

Meta-agentic bootstrapping uses feedback: an agent’s output is evaluated by a supervisor agent, and the error signal drives correction. The p3nGu1nZz bootstrapping methodology is a feedback system where the plant G is the learning agent and H is the validation harness. Stability requires that the feedback loop does not amplify errors — the system must be BIBO (bounded-input, bounded-output) stable.

PID-Analogous Error Correction

u(t) = Kpe(t) + Ki∫0te(τ)dτ + Kd(de/dt)

Kp = proportional gain (immediate correction)
Ki = integral gain (accumulated error)
Kd = derivative gain (rate of change)

Agent validation follows PID logic. The proportional term corrects the current error (this output is wrong). The integral term addresses accumulated drift (this agent has been consistently biased). The derivative term anticipates trends (error is growing, intervene early). The GSD verify-work skill implements all three: immediate test failures (P), retrospective pattern analysis (I), and trend detection across sessions (D).

STPA for ML Safety

Hazardous control action types:
1. Control action not provided (causes hazard)
2. Control action provided (causes hazard)
3. Too early, too late, out of order
4. Stopped too soon, applied too long

Nancy Leveson’s STPA (2011) identifies safety hazards by analyzing the control structure rather than individual component failures. Applied to ML systems: a guardrail that fails to fire (type 1) or fires incorrectly (type 2) creates a hazard. The CMU ML-in-Production lecture series (Kastner) establishes that safety ≠ reliability — a reliable system can still be unsafe if its control actions are inappropriate for the operating context.

Guardrails as Control Boundaries

Constraint: xmin ≤ x(t) ≤ xmax for all t

Saturation: u(t) = clip(uraw, umin, umax)
Anti-windup: reset integrator when saturated

Guardrails in agentic systems are hard constraints on the control output. When an agent attempts an action outside bounds, the guardrail clips it — analogous to actuator saturation in physical control systems. The anti-windup problem is real: if the integral term keeps accumulating error while the output is saturated, the system overshoots when the constraint is lifted. RLHF/CAI alone are insufficient (CMU finding) because they lack formal saturation handling.

Drift Detection and Correction

Drift: |y(t) - ytarget| > ε for t > T

Detection: CUSUM, EWMA, or hypothesis testing
Correction: re-calibrate model, update parameters

Enterprise AI drift occurs when the system’s output gradually deviates from the target — the model was trained on data that no longer represents the current distribution. Detection uses statistical process control methods (CUSUM, EWMA). The AmpGPT research on organizational training identifies drift as the primary risk: axiom-based training decays over time as people forget or reinterpret the axioms.
