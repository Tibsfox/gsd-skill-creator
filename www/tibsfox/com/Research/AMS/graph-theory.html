Graph theory provides the structural backbone for agentic systems — from DAG-based task scheduling to knowledge graphs that power retrieval-augmented generation. Every dependency chain, every agent hierarchy, and every trust boundary is a graph.

Module Overview

This module covers five graph-theoretic structures that appear throughout the agentic technology research: directed acyclic graphs for wave-based task execution, topological sorting for dependency resolution, knowledge graphs in LightRAG’s entity voting system, hierarchical agent organization as DAGs, and ShieldNet’s network guardrail graph filtering.

Key Mathematical Concepts

Directed Acyclic Graphs (DAGs)

G = (V, E) where E ⊆ V × V
Acyclic: no path v1 → v2 → ... → v1

Wave scheduling: Wk = {v ∈ V : all predecessors ∈ ∪i<k Wi}

Wave-based execution in gsd-skill-creator partitions a DAG into waves. Each wave contains tasks whose predecessors all completed in earlier waves. This is the mathematical foundation of the Gastown convoy model — 50+ projects can execute in parallel within a wave because they share no dependencies.

Topological Sort

Given DAG G = (V, E), find ordering σ : V → {1,...,|V|}
such that (u,v) ∈ E ⇒ σ(u) < σ(v)

Algorithm: Kahn’s (BFS) or DFS-based, O(|V| + |E|)

Topological sorting linearizes task dependencies into a valid execution order. The GSD workflow uses this to determine which phases can run and which are blocked. Multiple valid topological orderings exist for most DAGs — wave-based scheduling selects the one that maximizes parallelism.

Knowledge Graphs and Entity Voting

KG = (E, R, T) where:
E = entities, R = relations, T = triples (e1, r, e2)

LightRAG voting: score(e) = ∑t ∈ T(e) relevance(t, query)

LightRAG builds a knowledge graph layer above vector embeddings. Entities vote on relevance through their relation triples — an entity with many high-relevance connections to the query wins. This graph structure captures semantic relationships that flat vector similarity misses.

Agent Hierarchy as DAG

OrgAgent’s three-layer corporate hierarchy (executive → department → worker) is a DAG where edges represent delegation. The 102.73% improvement over flat MAS on SQuAD 2.0 comes from structured information routing through the graph rather than broadcast to all agents. Each level in the hierarchy filters and focuses the task.

ShieldNet Graph Filtering

Network guardrails model trust relationships as a graph where nodes are agents and edges carry trust scores. ShieldNet filters communication by requiring minimum trust weight on the path between sender and receiver. Graph cut algorithms identify the minimum set of edges to sever to isolate a compromised agent.
