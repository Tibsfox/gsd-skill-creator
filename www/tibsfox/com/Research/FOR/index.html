Why Fortran? Because it is the only programming language that has been in continuous production use since the Eisenhower administration. Code written in 1970 still runs — not just runs, but runs in production at NASA, NCAR, ECMWF, ORNL, and every major scientific institution on Earth. It is the language of climate models, weather forecasts, computational chemistry, computational fluid dynamics, dense linear algebra, and the supercomputer benchmarks that determine the TOP500 rankings.

Fortran is also the language that proved a high-level language could be efficient. In 1953, when John Backus began the project at IBM, the prevailing wisdom was that compiled code would always be too slow — that real performance required hand-written assembly. The first FORTRAN compiler shipped in 1957 after 18 person-years of work, and it produced code competitive with hand-tuned assembly. That single demonstration changed everything.

FORTRAN, As It Looked in 1957

[col 1] [col 6] [col 7…72]
C     COMPUTE THE SUM OF SQUARES FROM 1 TO N
      PROGRAM SUMSQ
      INTEGER I, N, TOTAL
      N = 100
      TOTAL = 0
      DO 10 I = 1, N
         TOTAL = TOTAL + I*I
   10 CONTINUE
      WRITE (6,20) TOTAL
   20 FORMAT (1H , 'SUM OF SQUARES =', I10)
      STOP
      END

Fixed-form FORTRAN: column 1 was reserved for C comment markers, columns 1-5 for statement labels, column 6 for continuation (any non-blank character indicated continuation from the previous card), and columns 7-72 for the actual statement. Columns 73-80 were for sequence numbers — in case you dropped your deck of punch cards on the floor.

Modern Fortran (2003+) Looks Like This

program sumsq
  use, intrinsic :: iso_fortran_env, only: int64
  implicit none

  integer(int64) :: i, n, total
  n = 100
  total = sum([(i*i, i=1, n)])    ! array constructor + intrinsic sum
  print '(a, i0)', 'Sum of squares = ', total
end program sumsq

Free form. Implicit none. Modules from iso_fortran_env. Array constructors. The sum intrinsic operating on a generated array. Modern Fortran is barely recognizable as the same language — and yet the 1957 program above will still compile and run with any modern Fortran compiler.

The Fortran Timeline — 69 Years of Standards

  1. 1957 — FORTRAN I — John Backus and team ship the first compiler at IBM after 18 person-years. Runs on IBM 704. Proves compiled code can rival hand-written assembly.
  2. 1958 — FORTRAN II — Subroutines, functions, COMMON blocks. Programs can be modular for the first time.
  3. 1962 — FORTRAN IV — LOGICAL data type, logical IF. Cleanup of FORTRAN II quirks. Becomes the basis for the first standard.
  4. 1966 — FORTRAN 66 — First ANSI standard (X3.9-1966). Codifies the language. The version that ran on early IBM 360s.
  5. 1978 — FORTRAN 77 — Structured programming. CHARACTER data type. Block IF/ELSE. The version most legacy code was written in. Reigns for 13 years.
  6. 1991 — Fortran 90 — The major modernization. Free-form source. Modules. Array operations. Recursion. Pointers. Dynamic allocation. Mixed case (Fortran instead of FORTRAN). The language reborn.
  7. 1997 — Fortran 95 — Minor revision. FORALL, PURE functions. Cleanup of F90.
  8. 2004 — Fortran 2003 — Object-oriented programming. ISO_C_BINDING for C interoperability. IEEE arithmetic. Type-bound procedures. The biggest revision since F90.
  9. 2010 — Fortran 2008 — Coarrays (native parallel programming). Submodules. DO CONCURRENT. The parallel revolution.
  10. 2018 — Fortran 2018 — Enhanced coarrays. Atomic operations. Teams. Better C interop.
  11. 2023 — Fortran 2023 — Generics. Conditional expressions. Tokenization improvements. The latest standard.

Why Fortran Still Matters in 2026

ArgumentDetail
PerformanceCompilers can vectorize Fortran more aggressively than C/C++ because of stricter aliasing rules. Array semantics enable SIMD optimizations that hand-written loops cannot match.
LongevityCode written in 1970 still compiles and runs. NASA mission planning software, NCAR climate models, ECMWF weather forecasts — legacy investments measured in millions of lines.
EcosystemBLAS, LAPACK, ScaLAPACK, NetCDF, HDF5, MPI — the foundations of computational science. Every numerical Python package eventually calls Fortran code.
IEEE arithmeticStandard floating-point semantics, well-defined behavior, no surprises. Modern Fortran has IEEE_ARITHMETIC module for direct control.
Array firstArrays are first-class. c = a + b works on entire arrays. No need for NumPy, no broadcasting confusion.
Parallel nativeCoarrays in the language standard. DO CONCURRENT for GPU and SIMD. OpenMP directives. MPI bindings.
Domain dominanceClimate, weather, CFD, computational chemistry, dense linear algebra, oil & gas, finance. The scientific computing default.
Renaissancefpm, stdlib, LFortran, Flang — modern tooling for the first time in decades. New community, new energy.
