Build Tools, IDEs, Testing, Logging, and the JVM as a Polyglot Runtime
Overview Language OOP JVM Frameworks Ecosystem Rosetta Learn History
Build tools in the Java world evolved in three generations: XML-driven task runners (Ant), convention-over-configuration declarative builds (Maven), and programmable incremental build engines (Gradle, Bazel). Each generation solved real pain from the previous one, and all three still ship software in 2026.

Maven Central

Maven Central (repo.maven.apache.org) is the largest open-source artifact repository in any language ecosystem, with 10 million+ artifacts. Every Java dependency lives somewhere in the groupId:artifactId:version coordinate space. The ~/.m2/repository local cache is the cultural constant of Java development.

Scopes: compile, provided, runtime, test, system, import (for BOMs).

Log4Shell (CVE-2021-44228, December 2021)

The December 2021 Log4Shell vulnerability in Log4j 2 was one of the most impactful security incidents in Java’s history. A JNDI lookup feature allowed remote code execution from a single log message. The ubiquity of Log4j in the Java ecosystem meant nearly every enterprise Java codebase had to be patched within days. It catalyzed a broader conversation about supply-chain security, SBOMs, and dependency scanning in Java.

// JUnit 5 + AssertJ
import org.junit.jupiter.api.Test;
import static org.assertj.core.api.Assertions.*;

class UserServiceTest {

    @Test
    void findByEmail_returns_user_when_present() {
        var repo = mock(UserRepository.class);
        when(repo.findByEmail("foxy@example.com"))
            .thenReturn(Optional.of(new User(1L, "foxy", "foxy@example.com")));

        var service = new UserService(repo);
        var user = service.findByEmail("foxy@example.com");

        assertThat(user).isPresent()
            .get()
            .extracting(User::name)
            .isEqualTo("foxy");
    }
}

Logging

  • Log4j 2 (Apache) — modern rewrite of the 1999-era Log4j 1
  • SLF4J — Simple Logging Facade for Java (Ceki Gülcü, who also wrote Log4j 1 and Logback). The standard facade.
  • Logback — SLF4J’s native implementation
  • java.util.logging (JUL) — built-in, less popular
  • tinylog — lightweight alternative

The idiomatic pattern: log through SLF4J, bind to Logback or Log4j 2 at deploy time. This decouples library authors (who code against the facade) from application deployers (who choose the backend).

Other JVM Languages

One of Java’s most consequential gifts to the world is the JVM as a multi-language platform. invokedynamic (Java 7) was added specifically to make dynamic languages fast on the JVM.

LanguageCreatorYearCharacter
ScalaMartin Odersky2004Pure FP + OO hybrid; rich type system. Twitter, LinkedIn, Databricks (Spark).
KotlinJetBrains2011 / 1.0 in 2016Pragmatic, null-safe, interop-first. Android official, Spring, Ktor.
ClojureRich Hickey2007Lisp; immutable persistent data structures; STM. Datomic, Nubank.
GroovyJames Strachan2003Dynamic, scripting, Java-friendly. Gradle (original DSL), Grails, Jenkins pipelines.
JRubyCharles Nutter et al.2001Ruby on the JVM. Logstash, Foreman.
JythonJim Hugunin1997Python on the JVM. Now largely inactive.
CeylonRed Hat2011–2018Alternative static language. Archived.

Plus the Truffle framework under GraalVM, which hosts JavaScript, Python, R, Ruby, LLVM bitcode, and WebAssembly in the same runtime. Every interpreter benefits from the same JIT.

SDKMAN — Managing JDKs

SDKMAN! (sdkman.io) is the de facto tool for managing multiple JDK installations on the same machine. It knows about all major distributions (Temurin, Corretto, Liberica, GraalVM, Zulu, Oracle) and lets you switch between them per-shell or per-project:

sdk install java 21.0.4-tem      # install Temurin 21
sdk install java 24-graalce      # install GraalVM CE 24
sdk use java 21.0.4-tem          # switch this shell

Code Quality Tools

  • Checkstyle — style conformance (formatting, naming)
  • PMD — source code analyzer, bug patterns
  • SpotBugs (formerly FindBugs) — bytecode-level static analysis
  • Error Prone (Google) — compile-time checks via an annotation processor
  • SonarQube — code quality dashboard; widely used in CI
  • JaCoCo — code coverage
  • Spotless — source code formatter enforcement (Google Java Format, Eclipse, Prettier)

Containers and Cloud

  • Jib (Google) — container image builder for Java without a Dockerfile
  • Cloud Native Buildpacks — CNB, supports Java via Paketo
  • Distroless (Google) — minimal container base images for Java
  • GraalVM Native Image — produces native Linux binaries that start in milliseconds and use tens of MB
  • CRaC (Coordinated Restore at Checkpoint) — checkpoint a warm JVM, restore later for fast startup
