Spring Framework — The Defining Framework

Rod Johnson introduced Spring in his 2002 book Expert One-on-One J2EE Design and Development (Wrox). The book was a sharp critique of EJB 2.x — overly heavyweight, hard to test, dependent on app servers. Spring was the working alternative: a plain-old-Java-object (POJO) framework with dependency injection, AOP, and transactional abstractions.

Twenty years later, Spring is the single most influential Java library ever written. Spring Boot (2014) made it convention-over-configuration, production-ready, and microservice-friendly — and became the default way to start a Java service.

Spring Boot — REST controller

@RestController
@RequestMapping("/api/users")
public class UserController {

    private final UserService userService;

    public UserController(UserService userService) {
        this.userService = userService;
    }

    @GetMapping("/{id}")
    public ResponseEntity<UserDto> getUser(@PathVariable Long id) {
        return userService.findById(id)
            .map(ResponseEntity::ok)
            .orElse(ResponseEntity.notFound().build());
    }

    @PostMapping
    public ResponseEntity<UserDto> create(@Valid @RequestBody CreateUserRequest req) {
        UserDto created = userService.create(req);
        return ResponseEntity
            .created(URI.create("/api/users/" + created.id()))
            .body(created);
    }
}

Jakarta EE — The Standard Stack

J2EE (1999) → Java EE (2006) → Jakarta EE (2018, after Oracle donated Java EE to the Eclipse Foundation). The trademark change meant the javax.* package prefix was renamed to jakarta.* in Jakarta EE 9 — a disruptive but necessary step.

APIPurpose
Servlets / JSP / JSFWeb tier: HTTP handling, templating, component framework
JPA (Jakarta Persistence)ORM standard. Hibernate is the reference implementation.
EJBEnterprise JavaBeans. Legacy; still used but rarely chosen for new projects.
JMS (Jakarta Messaging)Messaging abstraction; backed by ActiveMQ, IBM MQ, etc.
JAX-RS (Jakarta REST)REST APIs with annotations
CDI (Contexts and Dependency Injection)Standard DI, Weld reference implementation
JTA (Jakarta Transactions)Distributed transaction coordination
Bean ValidationJSR 303 — declarative validation (@NotNull, @Size, @Email)

Modern Lightweight Frameworks

The 2018–2020 wave brought a new generation of frameworks optimized for microservices, fast startup, and GraalVM Native Image:

Quarkus

Red Hat, 2019. “Supersonic subatomic Java.” Kubernetes-native, GraalVM Native Image first-class. Build-time metadata resolution makes Native Image painless.

Micronaut

Object Computing, 2018. Compile-time DI instead of runtime reflection. Fast startup, low memory.

Helidon

Oracle, 2018. MicroProfile compliant. Helidon SE (reactive functional) and Helidon MP (MicroProfile/Jakarta EE).

Dropwizard

2011. Opinionated “operations” framework. Combines Jetty + Jersey + Jackson + Metrics + Guava. Production-ready out of the box.

Plus: Javalin (lightweight Kotlin/Java web framework), Spark Framework (Sinatra-inspired micro, not to be confused with Apache Spark), Vert.x (Eclipse, reactive, polyglot).

Android — Java’s Second Life

Android Inc. was founded in 2003 and acquired by Google in 2005. Android 1.0 shipped in September 2008. Java was the primary language from the start, via the Dalvik VM (later replaced by ART, Android Runtime). Android uses Java source compiled to Java bytecode, then transformed to Dalvik bytecode (.dex).

  • Google v. Oracle (2010–2021) — the decade-long lawsuit over Android’s use of Java APIs. US Supreme Court ruled April 5, 2021 that Google’s use was fair use.
  • Android Studio (IntelliJ-based, 2013) — the official IDE
  • Kotlin as official Android language since 2017, but Java is still widely used
  • Jetpack libraries — the modern Android standard library

Why so much big data is Java: the JVM’s excellent multi-threading, mature GC, huge library ecosystem, cross-platform operation, enterprise acceptance, deep profiling toolchain, and a decade of production experience running long-lived daemons under load.

Financial / Trading

  • LMAX Disruptor — Martin Thompson’s low-latency ring buffer, used in trading exchanges
  • Chronicle suite (Peter Lawrey) — high-performance messaging and persistence
  • Aeron — efficient messaging (also Martin Thompson)
  • Azul Zing / Prime — C4 pauseless GC for HFT workloads

Java in HFT is a distinct world: deterministic GC, off-heap allocation, lock-free data structures, busy-wait spinlocks. Not Jane Street (OCaml), but much of the rest of HFT outside FPGA paths.

Messaging and Integration

  • Kafka — the de facto event streaming standard
  • RabbitMQ — AMQP, general-purpose message broker (Erlang, but Java clients everywhere)
  • ActiveMQ (Apache), Artemis — JMS brokers
  • Spring Integration, Apache Camel — enterprise integration pattern implementations
  • gRPC-Java — Google’s RPC framework
  • graphql-java, DGS (Netflix) — GraphQL servers

Observability
