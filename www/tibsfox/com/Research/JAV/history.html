History & Evolution

From the Green Project to Java 25 — 35 Years of People, Standards, Lawsuits, and Cadence

“The Java language was not intended to be a radical advance in programming language theory. We wanted to build something that just worked, that was reliable, that would let you write programs for devices you’d hand to your grandmother.” — James Gosling

The Green Project (1991–1995)

Java’s genesis lies in a small Sun Microsystems skunkworks called the Green Project, chartered in December 1990 by Patrick Naughton (who had been preparing to leave Sun for NeXT). Sun co-founder Scott McNealy asked him to write a memo describing everything he thought was wrong with Sun, and to propose a project that would fix it.

The founding trio was James Gosling, Patrick Naughton, and Mike Sheridan, with additional contributions from Bill Joy, Arthur Van Hoff, Jonathan Payne, Ed Frank, Chris Warth, and about thirteen engineers total. The team worked out of an office on Sand Hill Road in Menlo Park, physically separated from Sun’s main campus.

Their target was not Sun’s workstation market but the emerging world of consumer electronic devices — set-top boxes, PDAs, cable TV receivers, handhelds. The proliferation of CPU architectures in consumer devices (SPARC, MIPS, 68k, various microcontrollers) created demand for a portable, safe, network-aware environment that C and C++ couldn’t deliver cleanly.

The *7 (Star Seven) was the team’s hardware demonstrator: a handheld wireless PDA with a SPARC chip, touchscreen, and an animated agent named Duke (drawn by Joe Palrang) — who would become Java’s mascot. The device ran a small window system called SunDew and a new language called Oak, named after a tree outside Gosling’s office.

The pivot to the Web

Gosling originally tried to build what he needed as a C++ extension but abandoned it after concluding the memory model, template system, and header-file mechanics would never be safe for downloadable code on resource-constrained devices. So he started fresh.

The “white paper” on Oak (August 1991) laid out eleven buzzwords: simple, object-oriented, distributed, interpreted, robust, secure, architecture-neutral, portable, high-performance, multi-threaded, dynamic. These words would appear on Sun marketing slides for a decade.

A Time Warner interactive-TV bid in 1993 was lost to Silicon Graphics. A 3DO deal failed. By mid-1994, the Green spin-out (FirstPerson, Inc.) was effectively dissolved — and Oak was looking for a new purpose.

That purpose appeared when Bill Joy, Naughton, and Gosling recognized that the nascent World Wide Web had exactly the properties Oak was designed for: heterogeneous machines, unreliable networks, executable content, and a security model assuming untrustworthy sources. In summer 1994, Naughton and Payne wrote WebRunner (soon renamed HotJava), a browser that could download and execute Oak bytecode embedded in web pages. The first applets.

Oak → Java (1995)

Oak could not ship under that name — it was trademarked by Oak Technology, a graphics-chip company. The standard story: a Sun marketing team led by Kim Polese brainstormed names at a Palo Alto coffee shop, and Java — a nod to the coffee Silicon Valley programmers were fueled by — beat out alternatives including “DNA,” “Silk,” and “Lyric.”

The Public Launch (May 23, 1995)

Sun unveiled Java and HotJava at the SunWorld conference on May 23, 1995 in San Francisco. The most consequential news that day came from a separate announcement: Marc Andreessen, then head of Netscape, declared that Netscape Navigator would include a Java runtime and execute applets inline in web pages.

That single endorsement transformed Java from a Sun research project into what appeared to be the programming language of the Web. Within six months, virtually every major software vendor had announced a Java strategy.

Sun’s John Gage coined “The network is the computer” around this time — a slogan Sun had used before but which the Java launch gave genuine content. Programs and computation itself would move fluidly across networks; Java was the execution fabric.

JDK 1.0 shipped January 23, 1996. It contained about 212 classes in eight packages (java.lang, java.io, java.net, java.util, java.awt, java.awt.image, java.awt.peer, java.applet), an interpreter, a bytecode compiler (javac), an applet viewer, and the HotJava browser.

The Licensing Saga

Sun’s original model (1995–2006): source available under a commercial license. 2006–2007: Sun GPLs the JDK, establishing OpenJDK. Red Hat backs IcedTea. 2010: Oracle acquires Sun and continues open-source development alongside Oracle JDK. 2018: Oracle changes licensing — Oracle JDK becomes commercial-only; OpenJDK remains free under GPLv2 with Classpath Exception. 2021: Oracle reverses course — Oracle JDK 17 and later are free under the Oracle No-Fee Terms and Conditions (NFTC).

Google v. Oracle (2010–2021)

One of the longest-running and most consequential software lawsuits in history. Oracle sued Google in August 2010, arguing that Android’s reimplementation of the Java standard library APIs was copyright infringement. The case wound through the Federal Circuit twice, with wildly different rulings:

  • 2012 — District court: APIs not copyrightable. Oracle appeals.
  • 2014 — Federal Circuit: APIs are copyrightable. Google appeals.
  • 2016 — Second trial: Google’s use is fair use. Oracle appeals.
  • 2018 — Federal Circuit: not fair use. Google petitions Supreme Court.
  • April 5, 2021 — US Supreme Court rules 6-2 that Google’s use of the Java SE API was fair use.

The ruling shaped the software industry: API reimplementation is largely protected, and the risk of cloning a popular API is reduced.

The 6-Month Cadence (2017–present)

Mark Reinhold announced the new cadence in September 2017 alongside Java 9. Every six months: a new feature release. Every two years: an LTS release with long-term support from Oracle and other distributors (Adoptium, Amazon Corretto, Azul Zulu, BellSoft Liberica).

Criticism

  • Verbosity — Hello World has a class, a public static void main, and String[] args. Java 21 relaxed this with unnamed classes (preview).
  • XML configuration hell — the J2EE and early Spring era. Annotations replaced it.
  • Performance misconceptions — pre-HotSpot Java was slow; post-HotSpot Java is fast, but the reputation stuck.
  • “Enterprise Java” as a cultural stereotype — FizzBuzzEnterpriseEdition.
  • Null pointer exceptions — Tony Hoare’s billion-dollar mistake.
  • Oracle’s stewardship — periodic concerns about licensing and openness.
  • Slow startup — addressed by AOT, GraalVM Native Image, CRaC, Project Leyden.
