Java in Context — Cross-Language Translation, JVM Polyglot, Android, and the C++ Lineage
Overview Language OOP JVM Frameworks Ecosystem Rosetta Learn History
Java was designed as “C++ minus the confusing bits.” Three decades in, every major languages choice Gosling made has been tested: garbage collection won, single inheritance held, checked exceptions lost, generics-with-erasure is contested, and WORA has been absorbed by every cloud runtime.

Where Java Sits on the Escalation Ladder

From the PRL and CPP research projects, the sysadmin’s escalation ladder places languages by proximity to the metal. Java occupies a distinctive rung: above C++ in abstraction, above Python in performance, and with a runtime (the JVM) that is itself a world-class engineering artifact.

  1. Bash — shell glue
  2. Perl / Python — dynamic languages (PRL, SNK)
  3. Java / Kotlin / Scala / Clojure — JVM languages. Managed runtime, garbage collected, platform-portable via bytecode. Enterprise, big data, Android.
  4. C++ — where objects meet the metal (CPP)
  5. C — the substrate (CEE)
  6. Assembly · Machine Code · Mathematics · Philosophy
  7. Ask Your Sysadmin — “Hey Foxy? Can you fix this? ^.^”

Same Program, Different Languages — A Class with Polymorphism

Java

import java.util.List;

sealed interface Shape permits Circle, Square {}
record Circle(double r) implements Shape {}
record Square(double side) implements Shape {}

double area(Shape s) {
    return switch (s) {
        case Circle c -> Math.PI * c.r() * c.r();
        case Square sq -> sq.side() * sq.side();
    };
}

void main() {
    var shapes = List.<Shape>of(new Circle(2.0), new Square(3.0));
    shapes.forEach(s -> System.out.println(area(s)));
}

Kotlin (the JVM’s pragmatic successor)

sealed interface Shape
data class Circle(val r: Double) : Shape
data class Square(val side: Double) : Shape

fun area(s: Shape) = when (s) {
    is Circle -> Math.PI * s.r * s.r
    is Square -> s.side * s.side
}

fun main() {
    val shapes = listOf<Shape>(Circle(2.0), Square(3.0))
    shapes.forEach { println(area(it)) }
}

Scala (the JVM’s academic successor)

enum Shape:
  case Circle(r: Double)
  case Square(side: Double)

def area(s: Shape): Double = s match
  case Shape.Circle(r)  => math.Pi * r * r
  case Shape.Square(sd) => sd * sd

@main def run() =
  val shapes = List[Shape](Shape.Circle(2.0), Shape.Square(3.0))
  shapes.foreach(s => println(area(s)))

C++ (the ancestor)

#include <memory>
#include <vector>
#include <iostream>
#include <numbers>

struct Shape { virtual ~Shape() = default; virtual double area() const = 0; };
struct Circle : Shape {
    double r; explicit Circle(double r_) : r(r_) {}
    double area() const override { return std::numbers::pi * r * r; }
};
struct Square : Shape {
    double s; explicit Square(double s_) : s(s_) {}
    double area() const override { return s * s; }
};

int main() {
    std::vector<std::unique_ptr<Shape>> shapes;
    shapes.push_back(std::make_unique<Circle>(2.0));
    shapes.push_back(std::make_unique<Square>(3.0));
    for (const auto& s : shapes) std::cout << s->area() << '\n';
}

Java vs C++ — The Deliberate Simplification

Gosling started from C++ and removed what he considered dangerous or unnecessary. The removals define Java:

C++ featureJava’s answer
Manual new/deleteGarbage collection
Pointers + pointer arithmeticObject references only; no arithmetic
Multiple inheritanceSingle inheritance + multiple interface implementation
Operator overloadingNot allowed (except + on String)
Templates (compile-time monomorphized)Generics with type erasure (Java 5)
Header files, preprocessor, #includePackages, import, no preprocessor
Stack vs heap distinctionAll objects on heap (until escape analysis and Project Valhalla’s value types)
RAII via destructorstry-with-resources (Java 7) + AutoCloseable
Undefined behaviorJVM defines most edge cases; no UB in pure Java

Java vs C# — The Microsoft Mirror

C# (Anders Hejlsberg, Microsoft, 2000) is Java’s closest living relative. Microsoft watched Sun’s lawsuit settlement, gave up J++, and built .NET as a parallel universe with the same ideas plus some lessons learned:

  • Reified generics (C# keeps type info at runtime; Java erases)
  • Properties as a first-class language feature (vs. JavaBeans getters/setters)
  • Value types (struct) from day one; Java has chased this via Project Valhalla
  • Async/await before Java (Java has CompletableFuture + virtual threads)
  • LINQ preceded Java’s streams by years
  • Pattern matching matured earlier in C#

Java eventually catches up on most of these. The JVM’s cross-platform maturity remains Java’s advantage.

The JVM as a Polyglot Runtime

One of Java’s most consequential — and least-planned — gifts to the world is the JVM as a multi-language platform. The invokedynamic bytecode (Java 7) was added specifically to make dynamic languages fast on the JVM, and the resulting polyglot ecosystem is enormous:

Java vs Python — The Two-Language Mirror

From the SNK research project: Python has a “two-language problem” where Python glue calls into C/C++ libraries for the heavy lifting. Java’s answer is different: the JVM is already fast enough that you can write the heavy lifting in Java itself. This is why:

  • Hadoop, Spark, Kafka, Flink, Cassandra, Elasticsearch are all Java/Scala — no Python underneath, no C++ underneath
  • Apache Lucene (the search library) is pure Java and powers most enterprise search
  • High-frequency trading uses Java (with Azul Zing or ZGC) where Python would be unthinkable

Python won data science and ML because of NumPy, Pandas, and PyTorch. Java won the data infrastructure that feeds those tools.

Android — Java’s Second Life

Android (Google, 2005 acquisition; Android 1.0 September 2008) chose Java as its primary language and a Java bytecode dialect (Dalvik, later ART) as its runtime. For a decade, Java was the dominant mobile language by sheer volume of installs.

The Google v. Oracle lawsuit (2010–2021) asked whether Google’s reimplementation of the Java standard library APIs was copyright infringement. The US Supreme Court ruled on April 5, 2021 that Google’s use was fair use, settling the question for the industry.

Since 2017, Kotlin has been Google’s recommended Android language. But the runtime is still fundamentally Java bytecode, and billions of Android apps still include Java code.

When to Reach for Java

Yes, use Java

Enterprise web services. Microservices. Big data pipelines. Android apps (or Kotlin). High-performance server apps. Anywhere a mature ecosystem, operational tooling, and a 30-year library corpus matter.

Maybe use Java

Command-line tools (startup time is improving with AOT/Leyden). Scientific computing (ND4J, Apache Commons Math exist, but Python’s ecosystem is richer). Desktop apps (JavaFX exists, Swing works, but native toolkits are smoother).

Consider the JVM (Kotlin/Scala)

Android (Kotlin is official). Data engineering (Scala for Spark). Functional programming on the JVM (Scala, Clojure). When Java’s verbosity becomes friction but the JVM and libraries are still what you want.

Probably use something else

Systems programming (Rust, C++). ML research (Python). Shell glue (bash). Embedded (C). WebAssembly targets (Rust, AssemblyScript).
