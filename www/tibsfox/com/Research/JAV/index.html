WORAWrite Once, Run Anywhere

Why Java? Java is the language that proved a managed runtime could deliver production-grade performance, portability, and reach. Born from a failed set-top box project at Sun, it became the lingua franca of enterprise software, the foundation of Android, the runtime for most of the world’s big-data infrastructure (Hadoop, Spark, Kafka, Flink, Cassandra, Elasticsearch), and the host for an entire family of polyglot languages on the JVM (Scala, Kotlin, Clojure, Groovy). Thirty years in, Java is reinventing itself again with virtual threads, records, sealed types, and pattern matching — the Loom, Amber, Panama, and Leyden OpenJDK projects.

Research method: Primary sources (oracle.com/java, openjdk.org, jcp.org, JSR documents, JEPs, dev.java), canonical book corpus (Gosling, Bloch, Goetz, Lea, Bracha), and cross-reference against the 280-project PNW Research Series corpus

Mission period: April 2026 · Artemis II branch · Research Co-Processor v2

The WORA Promise

Write Once, Run Anywhere — the slogan that defined Java’s first decade. Sun’s insight in 1995 was that if every machine ran the same virtual machine, the programmer could ship a single compiled artifact and have it execute on Solaris, Windows, Mac, Linux, IBM mainframes, set-top boxes, smart cards, and later, a billion Android phones. Compile once to bytecode; the JVM handles the rest.

The tradeoffs were real: slower startup, higher memory, and decades of “Java is slow” folklore. The payoffs were real too: portable binaries, automatic memory management, runtime optimization via HotSpot, and an ecosystem where an application from 2004 still runs on Java 25 in 2026 without recompilation.

The Lineage: From the Green Project to Java 25

1991 Green Project — Gosling, Sheridan, Naughton start a stealth project at Sun. Target: smart consumer electronics. Language named Oak after a tree outside Gosling’s office.
1992 Star7 (*7) prototype — a handheld with animated Duke. Time Warner set-top box bid lost.
1995 Renamed Java (trademark conflict with Oak Technology). May 23, 1995 public announcement at SunWorld; Netscape endorses same day.
Jan 1996 JDK 1.0 — 212 classes, java.awt, java.io, java.net, java.applet. The applets era begins.
1997 JDK 1.1 — inner classes, JDBC, RMI, JavaBeans, reflection.
1998 J2SE 1.2 “Java 2” — Collections Framework (Doug Lea), Swing, JIT compiler, strictfp.
1999 J2EE 1.2 — EJB, JSP, Servlets. Enterprise era begins.
2004 Java 5 Tiger — generics, annotations, enhanced for, autoboxing, varargs, enums, static imports, java.util.concurrent.
2006 OpenJDK — Sun GPLs the JDK. Android 1.0 ships 2008 using a Java dialect.
2010 Oracle acquires Sun (January 27). Google v. Oracle lawsuit begins (August).
2011 Java 7 — Project Coin: try-with-resources, diamond, multi-catch, invokedynamic, NIO.2.
Mar 2014 Java 8 — lambdas, streams, Optional, java.time (JSR 310), default methods. The single most-used Java for years.
Sep 2017 Java 9 — modules (Project Jigsaw), JShell. 6-month cadence announced by Mark Reinhold.
Sep 2018 Java 11 LTS — first LTS under the new cadence. Oracle JDK becomes paid; OpenJDK remains free.
Sep 2021 Java 17 LTS — sealed classes, records (standard), pattern matching for instanceof. Oracle reverses course: Oracle JDK 17+ free under NFTC.
Apr 2021 Google v. Oracle decided by the US Supreme Court — Google’s use of Java APIs ruled fair use.
Sep 2023 Java 21 LTS — virtual threads (Loom), pattern matching for switch, record patterns, sequenced collections.
Mar 2024 Java 22 — Foreign Function & Memory API (Panama) standard, unnamed variables, stream gatherers (preview).
Sep 2025 Java 25 LTS target — continued Amber/Leyden/Valhalla delivery.

The Four Pillars, Java Style

Java is the language that made object-oriented programming mandatory. Every piece of Java code lives inside a class (until Java 21 preview, anyway). The four pillars are implemented with a deliberate narrowing of what C++ offered:

Encapsulation

private fields, public getters/setters. The JavaBeans convention. Access modifiers enforced at the JVM level, not just compile time.

Inheritance

Single inheritance only via extends. java.lang.Object is the root of every class hierarchy. Multiple interface inheritance via implements. Sealed classes (Java 17) for closed hierarchies.

Polymorphism

Every non-final method is virtual by default — the opposite of C++. Dynamic dispatch via the JVM’s invokevirtual/invokeinterface. Pattern matching for switch (Java 21) adds structural polymorphism.

Abstraction

Abstract classes + interfaces + generics (since Java 5). Interfaces gained default methods in Java 8 to enable library evolution. Records (Java 16) for data-oriented abstractions.

What Java Deliberately Removed from C++

Gosling started from C++ and simplified ruthlessly. The removals define Java as much as the additions:
