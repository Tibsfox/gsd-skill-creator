Safety Constraints

All proposed changes in this improvement pack are subject to three non-negotiable constraints:

  1. Backward compatibility: All changes must maintain compatibility with existing skills. The 34 active skills and 57 GSD commands must continue to function without modification.
  2. Learning loop preservation: Security hardening must not break the skill-creator’s core learning loop. The observe-suggest-validate cycle is the system’s primary value proposition.
  3. Muse council review: Changes to trust boundaries require muse council review before implementation. Cedar filters, the ledger records, and the grove map governs.
