PHP — The Web Sibling

Born from Perl, Powers 77% of the Web — WordPress, Laravel, Composer, Modern PHP 8.x
Overview Perl Core Text & NLP CPAN Sysadmin PHP Rosetta Learn
In 1994, Rasmus Lerdorf wrote a set of CGI scripts in Perl to track visits to his online resume. He rewrote them in C to eliminate the overhead of forking Perl on every request — but the Perl DNA was already embedded in the language’s design. PHP’s $ sigils, string interpolation, regex, and associative arrays are all inherited directly from Perl. Thirty years later, PHP powers 76.5% of the server-side web.

Highlights: Modern PHP Features

Enums (8.1)

enum Suit: string {
    case Hearts = 'H';
    case Diamonds = 'D';
    case Clubs = 'C';
    case Spades = 'S';

    public function color(): string {
        return match($this) {
            Suit::Hearts, Suit::Diamonds => 'red',
            Suit::Clubs, Suit::Spades => 'black',
        };
    }
}

Pipe Operator (8.5)

// Before: inside-out nesting
$result = array_map('strtoupper', array_filter($list, 'is_string'));

// After: left-to-right flow (Unix pipe philosophy)
$result = $list
    |> array_filter($$, 'is_string')
    |> array_map('strtoupper', $$);

Readonly Classes (8.2)

readonly class Point {
    public function __construct(
        public float $x,
        public float $y,
    ) {}
}

Property Hooks (8.4)

class User {
    public string $fullName {
        get => "$this->firstName $this->lastName";
        set(string $value) {
            [$this->firstName, $this->lastName] = explode(' ', $value, 2);
        }
    }
}

Why PHP Won the Web

The LAMP stack + shared hosting: Linux, Apache, MySQL, PHP became the default offering of virtually every shared hosting provider from the late 1990s onward. When “getting a website” meant “getting a cPanel account,” PHP was already there, configured and ready. No compilation, no deployment pipeline, no server configuration. Upload a .php file and it works.

The Facebook / Hack Story

One of the most remarkable journeys in software history:

YearEvent
2004Zuckerberg writes Facebook in PHP
2008Scaling walls hit. PHP too slow for billions of requests.
2010HipHop for PHP (HPHPc) — transpiles PHP to C++. 50% CPU reduction.
2013HPHPc replaced by HHVM — JIT-compiling virtual machine for PHP.
2014Hack released — gradually-typed PHP derivative with generics, async/await.
2019HHVM 4.0 drops PHP support. Facebook/Meta is now a Hack shop.

The PHP community’s response was PHP 7 (2015), which doubled performance and proved mainstream PHP could close the gap without forking. PHP 8’s JIT narrowed it further. The irony: Facebook’s pressure to perform made PHP better for everyone else.

Hand in Hand — The Dual-Language Architecture

The typical LAMP-era shop (circa 2000–2008):

[Browser] --> [Apache + mod_php] --> PHP handles web pages
                                     |
                                     v
                              [MySQL database]
                                     ^
                                     |
              [Perl cron jobs] --> ETL, log processing, reports
              [Perl scripts]  --> data import, file transformation
              [Perl daemons]  --> queue workers, monitoring

PHP served the web-facing layer; Perl handled everything behind the curtain. The mod_php vs mod_perl divergence tells the story: mod_php won adoption through simplicity (drop a .php file, it works), while mod_perl was more powerful but required understanding Apache internals. This foreshadowed the languages’ different trajectories: Perl toward power users and system complexity, PHP toward ease-of-use and web accessibility.
