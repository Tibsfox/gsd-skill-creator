Perl was not born from computer science. It was born from linguistics. Larry Wall studied linguistics at UC Berkeley and Seattle Pacific University before working at JPL and Unisys. He was a linguist who happened to write code, and this distinction is not cosmetic — it is the defining fact of Perl’s existence. Where most language designers came from mathematics and formal logic, Wall came from the study of how humans actually communicate: messily, contextually, with redundancy, with multiple valid ways to say the same thing.

Larry Wall — The Linguist Who Built a Language

TMTOWTDI — “There’s More Than One Way To Do It” (pronounced “tim-toady”). Natural languages do not enforce a single correct way to express an idea. Neither would Perl.

The Three Virtues of a Programmer:

  1. Laziness — Makes you write labor-saving programs and document what you wrote so you don’t have to answer so many questions about it.
  2. Impatience — The anger you feel when the computer is being lazy. Makes you write programs that anticipate your needs.
  3. Hubris — Excessive pride that makes you write programs other people won’t want to say bad things about.

Wall described Perl as a “postmodern programming language” (Linux World, 1999): where modernist languages sought one true paradigm, Perl borrowed freely and without apology. Regex from sed/awk, file handling from shell, data structures from C, report formatting from COBOL, OO from wherever it felt like. A bricolage united by practical utility, not theoretical purity.

The O’Reilly Animal Books

The Camel Book (Programming Perl) became Perl’s bible. The Llama Book (Learning Perl) became the curriculum. The camel became the mascot. This was part of O’Reilly’s tradition of animal engravings on technical covers — so recognizable that “the camel book” needed no further explanation. You could walk into any sysadmin’s office and see a shelf of animals: the camel, the llama, the owl (sed & awk), the horse (Unix in a Nutshell). That bookshelf was the curriculum of a generation.

The Perl 6 / Raku saga: At the Perl Conference in 2000, Larry Wall announced Perl 6. It became a legendary example of second-system effect. Grammars as first-class citizens, a type system, multiple dispatch, lazy evaluation, junctions. The running joke: Perl 6 would be released “by Christmas” — without specifying which Christmas. The 2019 rename to Raku was liberation in both directions.

Perl 5 is not dead. It is not even dying. It is the cockroach of programming languages — unkillable, everywhere, quietly doing its job in the walls of civilization.

The CGI Era — Perl Built the Dynamic Web

Before PHP. Before Python on the web. Before Rails, Node.js, or any of it — there was Perl and CGI.

The Common Gateway Interface let a web server execute an external program and return its output as HTTP. The program could be any language, but in practice it was almost always Perl: already on every Unix system, superb at text processing, and HTTP is a text protocol.

#!/usr/bin/perl
print "Content-type: text/html\n\n";
print "<html><body><h1>Hello, World!</h1></body></html>\n";

Perl Culture — Where Code Meets Art

JAPH — “Just Another Perl Hacker”

A tradition of writing programs whose output is “Just another Perl hacker,” using the most creative (often obfuscated) code possible. Started by Randal Schwartz. A signature, a calling card, and a flex.

Obfuscated Perl Contest

Inspired by the Obfuscated C Contest. Entries are working programs compressed into unreadable, beautiful monstrosities. Perl’s syntax flexibility makes it uniquely suited to this art form.

Perl Poetry

Programs that are simultaneously valid Perl and valid poetry. Larry Wall’s linguistic heritage at its purest. The /x modifier and flexible syntax make Perl the only language where this is genuinely possible.

Perl Golf

Solve a problem in the fewest characters possible. The opposite of readable code, and a celebration of Perl’s expressive density.

The Perl Advent Calendar

24 days of Perl articles in December, running since 2000. A community tradition of sharing knowledge and celebrating the language.

Perl Mongers

230+ local user groups worldwide. pm.org coordinates them. From London.pm to Tokyo.pm, these groups are where Perl programmers meet, teach, and maintain the community that keeps the language alive.
