Learning Resources

Books, Tutorials, Community, Mentors — From Beginner to CPAN Author
Overview Perl Core Text & NLP CPAN Sysadmin PHP Rosetta Learn
Both Perl and PHP have exceptional documentation and learning resources built up over decades. The Camel Book taught a generation of sysadmins. PHP.net set the standard for language reference sites. Laracasts showed what video-based programming education could be. This page is the curated path from “I’ve never seen a sigil” to “I just uploaded my first CPAN module.”

ResourceWhat

Learning Strategies

The CPAN Cookbook Method (for Perl)

When you have a concrete problem:

  1. Search MetaCPAN for modules solving similar problems
  2. Read the synopsis (first code example in the POD) — shows intended usage
  3. Read the source code on MetaCPAN’s “Source” tab
  4. Read the tests in the t/ directory — reveals edge cases and idioms
  5. Install with cpanm and try in a throwaway script

This is how Perl programmers learn: by reading other people’s code. CPAN is a library in the literal sense — a place to read and study.

The Packagist Browse Method (for PHP)

Similar approach for PHP:

  1. Search Packagist.org for keywords
  2. Check download counts and stars (popularity heuristic)
  3. Read the README and docs linked on the package page
  4. Clone the repo or browse on GitHub — read the source
  5. composer require vendor/package and try it

The “Write, Then Improve” Method

Both communities emphasize: write bad code first, then improve it. Larry Wall’s three virtues include Impatience — anger at slow computers drives you to write real solutions. PHP’s culture rewards shipping — mod_php won because people could upload a file and see results. Perfectionism is the enemy of learning.
