The Same Grammar, Seven Ways

A minimal arithmetic expression grammar, expressed in the most commonly used notations. All seven describe exactly the same language.

ALGOL 60 BNF (1960) — the original

<expr>   ::= <term> | <expr> "+" <term> | <expr> "-" <term>
<term>   ::= <factor> | <term> "*" <factor> | <term> "/" <factor>
<factor> ::= <number> | "(" <expr> ")"

Wirth EBNF (1977)

expr   = term { ( "+" | "-" ) term } .
term   = factor { ( "*" | "/" ) factor } .
factor = number | "(" expr ")" .

IETF ABNF (RFC 5234)

expr   = term *( ("+" / "-") term )
term   = factor *( ("*" / "/") factor )
factor = number / "(" expr ")"

Yacc / Bison (1975)

%token NUMBER
%left '+' '-'
%left '*' '/'

expr:   expr '+' term
    |   expr '-' term
    |   term
    ;

term:   term '*' factor
    |   term '/' factor
    |   factor
    ;

factor: NUMBER
    |   '(' expr ')'
    ;

ANTLR4

grammar Arith;

expr   : expr ('*'|'/') expr
       | expr ('+'|'-') expr
       | NUMBER
       | '(' expr ')'
       ;

NUMBER : [0-9]+ ;
WS     : [ \t\r\n]+ -> skip ;

PEG / Ford (2004)

Expr    <- Term (('+' / '-') Term)*
Term    <- Factor (('*' / '/') Factor)*
Factor  <- Number / '(' Expr ')'
Number  <- [0-9]+

Tree-sitter (2018)

module.exports = grammar({
  name: 'arith',

  rules: {
    expr: $ => choice(
      $.number,
      prec.left(1, seq($.expr, choice('+', '-'), $.expr)),
      prec.left(2, seq($.expr, choice('*', '/'), $.expr)),
      seq('(', $.expr, ')')
    ),
    number: $ => /[0-9]+/
  }
});

The shape of the grammar is the same in every notation. A terminal here is a literal; a non-terminal there is a rule. What changes is the metasyntax and the tooling. The concept — production rules that describe an infinite language — is identical to what Backus wrote on a blackboard in 1959.

The Through-Line from Post to Tree-sitter

1943 Emil Post — production systems

“Formal Reductions of the General Combinatorial Decision Problem.” Turing-complete string rewriting systems as a foundation for undecidability proofs.

1956 Chomsky — hierarchy of formal languages

Context-free grammars are Type 2, restricted form of Post production systems.

1959 Backus — BNF at ICIP Paris

A restricted form of Post productions, designed to describe programming language syntax precisely. Cites Post explicitly in a footnote.

1960 Naur — ALGOL 60 Report

First programming language fully specified in BNF. 20 pages that changed the world.

1964 Knuth — “Backus-Naur Form”

Two-paragraph letter to CACM renames the notation.

1965 Knuth — LR(k) paper

“On the Translation of Languages from Left to Right.” Theoretical foundation of bottom-up parsing.

1969 DeRemer — LALR(1) thesis

Practical LR parsing with compressed tables.

1975 Johnson — Yacc at Bell Labs

LALR(1) parser generator for Unix. Paired with Lex.

1977 Wirth — EBNF

“What Can We Do About the Unnecessary Diversity of Notation for Syntactic Definitions.” The modern EBNF we all use.

1985 Bison — GNU yacc

Robert Corbett, Richard Stallman. GPL.

1990s Parr — ANTLR (PCCTS)

LL(*) top-down generator with readable error messages.

2001 Leijen — Parsec

Monadic parser combinators in Haskell. The canonical combinator library.

2004 Ford — PEG

Parsing Expression Grammars with ordered choice. No ambiguity possible.

2013 ANTLR4

Adaptive LL(*). Much better error recovery.

2018 Brunsfeld — Tree-sitter

Incremental GLR parser for editors. Runs on every keystroke. Handles broken input gracefully.

2020 Python → PEG

PEP 617. Python 3.9 retires pgen.

2026 BNF still dominant

67 years after Backus’s ICIP paper, every serious language reference manual still has a grammar in BNF or some variant.

The Corpus Connections

PNW Series ProjectALGOL Connection
LSP — Lisp & FunctionalJohn McCarthy was on the ALGOL 60 committee. Lisp (1958) and ALGOL (1960) are the two roots of modern programming languages. McCarthy contributed the if-expression and mandatory recursion to ALGOL 60.
PCH — Punch CardsALGOL was designed to be read on paper, printed in journals, checked by hand — the purest expression of the publication-language ideal, and the constraint thinking that punch-card era programmers practiced.
FOR — FortranALGOL’s sibling rival. Fortran (1957) won scientific computing; ALGOL (1960) won the theory of programming languages. Backus led both efforts.
CEE — C ProgrammingC descends from ALGOL 60 via CPL (1963) → BCPL (1967) → B (1969) → C (1972). Every C block is an ALGOL block.
CPP — C++C++ inherits C’s ALGOL inheritance plus Simula 67’s object-oriented extensions to ALGOL. Stroustrup explicitly credits Simula as the source of classes, inheritance, and virtual dispatch.
JAV — JavaJava is Pascal cleaned up for the Internet (Gosling read Wirth). The JVM inherits from Wirth’s Pascal P-machine (1975).
RST — RustRust’s block structure, lexical scope, and type system are ALGOL-family through and through. The ML-family type system is grafted on top.
GOL — GoGo is deliberately Pascal-flavored. Robert Griesemer, Go co-creator, was Wirth’s doctoral student at ETH Zürich. Go is the Wirth tradition continuing.
ASM — AssemblyThe Burroughs B5000 had an ALGOL-oriented instruction set — the only mainstream machine designed around a specific high-level language.
CLI — Art of the CLIyacc, bison, ANTLR, and Tree-sitter are all CLI tools descended directly from BNF.
RXX — Rexx/ARexxRexx was specified with BNF, as were most modern scripting languages.
MLC — ML/CompilersCompiler theory is the theoretical descendant of BNF + Chomsky hierarchy + LR parsing. Every compilers textbook starts with ALGOL 60.
SST — States, Symbols & TapeThe automata theory underlying the Chomsky hierarchy: finite automata for regular; pushdown automata for context-free; linear-bounded automata for context-sensitive; Turing machines for recursively enumerable.
PRL — Perl/PHPPerl’s block structure, lexical scope (via my), and block-style control flow are Algol-family.
SNK — PythonPython inherits block structure (via indentation), lexical scope, and type system ideas from Algol. Its modules come from Modula-3 per van Rossum.

The One-Line Summary

§ The Six-Decade Shelf Life

John Backus introduced BNF at a UNESCO conference in Paris in June 1959, citing Emil Post’s 1943 production systems as the formal substrate. Peter Naur edited it into the Report on the Algorithmic Language ALGOL 60, published in Communications of the ACM in May 1960. Donald Knuth renamed it in a two-paragraph 1964 letter. Niklaus Wirth extended it in 1977. Stephen Johnson built Yacc from it in 1975. Frank DeRemer made LR parsing practical with LALR(1) in 1969. Terence Parr built ANTLR in the 1990s. Bryan Ford reformulated it as PEG in 2004. Max Brunsfeld built Tree-sitter from it in 2018. Python switched to a PEG-based parser in 2020.

Every step along this 67-year arc kept the BNF inheritance visible. Every step added capability without replacing the formalism. Every programming language reference manual in the world in 2026 still has a grammar section written in a notation descended from what Backus wrote in Paris and Naur cleaned up in the Report.

Sixty-seven years. No other notation in computing has had a longer shelf life. Not one.
