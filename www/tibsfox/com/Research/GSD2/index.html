GSD-2 Architecture

Workflow Systems — Planning · Execution · Agent Coordination · Phase Management
Overview Mission Pack
GSD-2: The next-generation Get Shit Done workflow system. Iron rule: every task fits in one context window. Disk-driven state machine. Milestone-Slice-Task hierarchy. Pi SDK integration. skill-creator as the adaptive learning layer.

Research Modules

01. Architecture & State Machine

Core Design · Execution Engine

Iron rule (task = 1 context window), 3-tier hierarchy (Milestone/Slice/Task), 5-phase execution loop, crash recovery via lock files and forensics, stuck detection, timeout supervision (20/10/30 min).

02. Context Engineering

Token Budget · Pre-loading Strategy

10 managed artifacts, pre-loading strategy, dispatch construction algorithm with token budget, 5-minute cache TTL, must-haves system (truths/artifacts/key links), phase-specific context budgets.

03. Extensions & Subagents

9 Bundled Extensions · 3 Subagents

9 bundled extensions (GSD Core, Background Shell, Git Conventions, Verification Ladder, DECISIONS Protocol, UAT Generation, Stuck Detection, Crash Recovery, Skill Discovery), 3 subagents (Scout/Researcher/Worker).

04. Git Strategy

Branch-per-Slice · Verification Ladder

Branch naming (gsd/M001/S01), atomic task commits, squash merge with UAT summary, 4-level verification ladder (static/artifact/behavioral/human), must-haves as mechanical verification.

05. Pi SDK Integration

badlogic/pi-mono · Multi-Provider

7 Pi SDK packages, two-file loader pattern (loader.ts/cli.ts), InteractiveMode session lifecycle events, 20+ AI provider support, per-phase model allocation, cost tracking and token ledger.

06. Skill-Creator Integration

Full Integration Thesis · 6 Modules

6 integration modules: Pi SDK event bridge, skill discovery path, GSDArtifactReader, PhaseContext type, GSDFeedbackTap, agent composition via preferences.md. 10 success criteria defined.

07. Verification Matrix

Quality Assurance · Coverage Audit
