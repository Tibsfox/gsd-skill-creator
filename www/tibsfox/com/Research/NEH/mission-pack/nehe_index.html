How to Use This Package

1
Open nehe_mission.pdf and read Stage 1 (Vision) to confirm the scope matches your intent. The 12 success criteria define the acceptance boundary.
2
Hand the PDF to the GSD orchestrator (Claude Code, dev branch): gsd execute nehe_mission.pdf — the orchestrator reads the wave plan and begins Wave 0.
3
Confirm CAPCOM Gate 1 after Wave 1 completes: review the visual spot-checks for L01–L12 and the test results for L01–L20. Go/no-go before Wave 2.
4
Confirm CAPCOM Gate 2 after Wave 2: review the DACP bundle schemas, the deprecation map, and the documentation index accuracy (30-query spot-check). Go/no-go before Wave 3.
5
After Wave 3: drop the output directory into the skill-creator dev branch and run python -m pytest tests/ in a fresh venv. All 4 safety-critical tests must pass before merge.
