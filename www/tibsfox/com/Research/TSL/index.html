Overview Platform Quality PR #28 Actions OSS Recursion Verification Mission Pack
On March 26, 2026, a Tessl employee submitted PR #28 to the gsd-skill-creator repository, improving five skills and installing a GitHub Actions workflow for automated skill review. The improvements averaged +60 percentage points per skill — not by changing what the skills knew, but by changing how they presented what they knew. The content was already there. The structure was missing. A skill without a workflow is a wish. A skill without a trigger is invisible. A skill without error handling is dangerous. And a skill creator whose own skills score 0% on a skill review has a credibility problem — one worth fixing.
