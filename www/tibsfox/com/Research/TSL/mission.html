Mission Pack

The original research brief and mission package that initiated the Tessl Skill Review research project. This document provides the foundational research from which all seven modules were developed, covering the Tessl platform, the Agent Skills Specification, PR #28’s five skill improvements, the GitHub Actions workflow, open-source contribution patterns, and the recursive quality loop of a skill creator’s skills being reviewed by an external quality tool.

Source Documents

The mission pack source material comprises a comprehensive 950-line TeX mission document (covering vision, research reference, mission specification, wave execution plan, test plan, and source bibliography) and a 510-line deep research document covering the Tessl platform, skill quality measurement, PR #28 analysis, GitHub Actions integration, open-source contribution patterns, and the recursive quality feedback loop.

Research Scope

  • Tessl Platform: Founded by Guy Podjarny (Snyk founder, ex-Akamai CTO), $125M+ raised, $500–750M valuation, registry of 3,000+ skills, evaluation framework with three scoring components (Validation, Implementation, Activation), CLI, GitHub Actions
  • Skill Quality Problem: Common failure modes (missing frontmatter, weak descriptions, no workflows, no error handling), the parallel to code linting evolution (manual → automated → CI/CD integrated), progressive disclosure model
  • PR #28 Improvements: Five skills improved averaging +60 percentage points — dacp-interpreter (0% → 85%), mfe-synthesis (11% → 63%), mfe-unification (11% → 75%), mfe-change (22% → 68%), mfe-emergence (22% → 75%)
  • GitHub Actions Workflow: tesslio/skill-review action, non-blocking by default, zero external accounts, incremental improvement model
  • Open Source Contribution: Honest disclosure pattern, small reviewable scope, non-invasive tooling, self-service optimization path
  • Recursive Quality: The skill creator’s skills reviewed by an external tool, the adaptive learning layer receiving feedback, the closed loop from creation to evaluation
  • 19 Sources: Tessl docs, agentskills.io specification, TechCrunch, Product Hunt, GitHub repositories, PR #28

Key Findings

The content was already good. The structure was invisible. Adding frontmatter, workflows, and triggers didn’t change what the skills knew — it changed whether anyone could find and use them. The same pattern as linting for code: automated quality gates that catch structural problems before humans have to. The recursive loop — a skill creator whose skills get reviewed by an external quality tool — is the adaptive learning layer working as designed. External feedback improving internal quality. The verification matrix verifies a project about verification. The recursion is the point.
