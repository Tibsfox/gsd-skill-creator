GC tuning wisdom is unusually concentrated. A few books, a few blogs, a few people. If you read every page by Shipilev, Goetz, Pangin, Lea, Kennke, and Oaks, you have read most of the corpus.

The Canonical Book Corpus

Essential

BookAuthorsYearNotes
The Garbage Collection HandbookRichard Jones, Antony Hosking, Eliot Moss2nd ed. 2023The academic canon. Not Java-specific — covers every serious GC algorithm in the literature. If you’re going to read one GC book, read this one.
Java Performance: The Definitive GuideScott Oaks2nd ed. 2020The practical companion. HotSpot internals, GC tuning, JIT, profiling, JMH.
Optimizing JavaBen Evans, James Gough, Chris Newland2018Practical JVM performance engineering. Good chapter on understanding JIT compilation output.
Java Concurrency in PracticeBrian Goetz, Tim Peierls, Joshua Bloch, Joseph Bowbeer, David Holmes, Doug Lea2006The book on java.util.concurrent and the JMM. Indirect relevance to GC: concurrent algorithms use memory differently.
Effective JavaJoshua Bloch3rd ed. 2018Items on allocation discipline, immutability, equals/hashCode, weak and soft references.

Advanced / niche

  • Java Performance Companion — Charlie Hunt, Monica Beckwith, Poonam Parhar, Bengt Rötzlin (2016) — G1 internals, CDS
  • Java Performance — Charlie Hunt & Binu John (2012) — the pre-modern era, still useful for HotSpot architecture
  • Systems Performance (Brendan Gregg, 2nd ed. 2020) — Linux perf tools, flame graphs, not Java-specific but applies
  • BPF Performance Tools (Brendan Gregg, 2019) — eBPF-based profiling including JVM USDT probes

A Reading Order That Works

  1. Start with GC logs. Enable unified logging on a real workload. Open the file in GCViewer. Stare at it until the shapes make sense.
  2. Read Shipilev’s “JVM Anatomy Quarks” series on shipilev.net. Short, dense, essential.
  3. Watch Gil Tene’s “Understanding Java Garbage Collection” talk. 60 minutes, changes how you think about it.
  4. Java Performance: The Definitive Guide by Scott Oaks — work through it, try the examples
  5. Get your hands on JFR. Record a real app. Load it into JMC. Explore every tab.
  6. Eclipse MAT with a real heap dump. Dominator tree. Retained size. Do the tutorial.
  7. The Garbage Collection Handbook — slow, dense, worth it. The theory that underlies everything.
  8. The OpenJDK JEP index. Read every GC-related JEP from the last 5 years.
  9. Subscribe to hotspot-gc-dev. Lurk for a month; you’ll learn more than most books can teach.
  10. Tune a real workload. Measure before, measure after, write it down. This is the only way to become fast at it.
