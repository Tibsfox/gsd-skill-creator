The sysadmin who manually configures servers is doing archaeology. The modern sysadmin writes code that configures servers — code that is version-controlled, tested, reviewed, and reproducible. Infrastructure as Code is not a trend; it is the professional standard.

Configuration Management

Configuration management tools enforce desired state across server fleets. The four major tools represent different philosophies:

Ansible dominates due to its agentless architecture and low barrier to entry. A playbook is readable by anyone who knows YAML. Ansible collections provide reusable modules for cloud providers, network devices, and applications.

# Ansible playbook: ensure Nginx is installed and running
- hosts: webservers
  become: yes
  tasks:
    - name: Install Nginx
      apt:
        name: nginx
        state: present
    - name: Start and enable Nginx
      systemd:
        name: nginx
        state: started
        enabled: yes

Terraform remains the industry standard for infrastructure provisioning. The plan/apply workflow provides a safety net: terraform plan shows exactly what will change before terraform apply executes it. OpenTofu emerged after HashiCorp changed Terraform’s license from MPL to BSL in August 2023, maintained by the Linux Foundation.

GitOps

GitOps uses Git as the single source of truth for infrastructure and application state. Changes are made via pull requests, and a reconciliation controller continuously ensures the live environment matches the Git repository.

Principles:

  • Declarative: Desired state described in Git, not imperative scripts
  • Versioned: Every change has a commit, enabling audit and rollback
  • Automated: Reconciliation controllers detect and correct drift
  • Observable: Divergence between desired and actual state is visible

ArgoCD (Kubernetes-native, web UI, multi-cluster) and Flux (lightweight, CNCF graduated, toolkit-based) are the two leading GitOps controllers. Both watch Git repositories and reconcile Kubernetes manifests automatically.

CI/CD Integration

Infrastructure changes must go through the same CI/CD pipeline as application code: lint (tflint, ansible-lint), validate (terraform validate), security scan (tfsec, checkov, Snyk IaC), plan/diff, approval gate, apply. GitHub Actions, GitLab CI, and Jenkins all support IaC pipelines. The key principle: no manual changes to production infrastructure.

Immutable Infrastructure

Instead of updating servers in place (mutable), build new machine images with every change and replace old instances. Packer builds AMIs/images from templates. Golden images contain the base OS, security hardening, and monitoring agents — application teams layer on top. This eliminates configuration drift and makes rollback trivial: deploy the previous image.

Drift Detection

Infrastructure drift occurs when live state diverges from declared state — caused by manual changes, failed applies, or external modifications. Detection approaches:

  • Terraform plan: Detects drift by comparing state file to live resources
  • NSync (Netflix): Continuous reconciliation between desired state and AWS resources, automated paper on drift patterns
  • InfraFix: Technology-agnostic IaC repair — detects misconfigured infrastructure and generates fixes
  • IaCGen: Deployability-centric code generation evaluated across 153 real-world scenarios
  • Driftctl: Open-source drift detection for Terraform-managed cloud resources

The NSync paper from Netflix and the InfraFix research represent the frontier of automated drift remediation — moving from detection to autonomous correction.
