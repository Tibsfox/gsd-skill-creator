Networking is the connective tissue of all distributed systems. A sysadmin who cannot read a packet capture, configure a firewall rule, or troubleshoot DNS resolution is operating blind. Every outage investigation eventually arrives at the network layer.

DNS

DNS is arguably the most critical infrastructure service. When DNS fails, everything fails. Sysadmins must understand recursive resolution, authoritative servers, zone files, record types (A, AAAA, CNAME, MX, NS, TXT, SRV, PTR), TTL tuning, and DNSSEC.

Server implementations: BIND (the reference implementation, complex), Unbound (recursive resolver, lightweight), PowerDNS (database-backed authoritative), CoreDNS (Kubernetes default, plugin-based). DNS-over-HTTPS (DoH) and DNS-over-TLS (DoT) add encryption to prevent eavesdropping.

DHCP

Dynamic Host Configuration Protocol automates IP address assignment. Key concepts: lease lifecycle (DORA: Discover, Offer, Request, Acknowledge), reservation by MAC address, scope configuration, relay agents for multi-subnet environments. ISC DHCP is being replaced by Kea (ISC’s modern replacement with REST API management).

HTTP & TLS Certificate Management

Every sysadmin manages TLS certificates. Let’s Encrypt (2015) transformed certificate management from an annual paid process into automated 90-day renewals via ACME protocol. Tools: certbot, acme.sh, Caddy (automatic HTTPS). Understanding certificate chains, key types (RSA vs ECDSA), OCSP stapling, and HSTS is essential.

HTTP/2 multiplexing and HTTP/3 (QUIC) change connection behavior in ways that affect load balancing and firewall rules.

SSH Key Management

SSH is the sysadmin’s primary tool for remote access. Best practices:

  • Key-based authentication only — disable password auth in sshd_config
  • Ed25519 keys — smaller, faster, more secure than RSA
  • SSH agent forwarding — use with caution (prefer ProxyJump)
  • ProxyJump (-J) — multi-hop SSH without agent forwarding risk
  • SSH certificates — replace authorized_keys at scale (HashiCorp Vault, Smallstep)
  • Port knocking / fail2ban — additional brute-force protection layers

Firewalls

Linux firewall evolution: ipchains (2.2) → iptables (2.4, Netfilter) → nftables (3.13+, unified framework). UFW (Uncomplicated Firewall) provides a simplified frontend for iptables on Ubuntu. firewalld provides zone-based management on RHEL/Fedora.

Key concepts: chains (INPUT, OUTPUT, FORWARD), tables (filter, nat, mangle), connection tracking (conntrack), stateful vs stateless rules, DNAT/SNAT for port forwarding and masquerading.

Load Balancing

HAProxy (Layer 4/7, high-performance, production standard) and Nginx (web server + reverse proxy + L7 load balancer) dominate. Algorithms: round-robin, least connections, IP hash, weighted. Health checks ensure traffic routes only to healthy backends. keepalived provides VRRP-based failover for high availability.

BGP, OSPF, VLANs & Subnetting

BGP (Border Gateway Protocol) is the routing protocol of the Internet — every ISP and cloud provider uses it. OSPF (Open Shortest Path First) handles internal routing within autonomous systems. Sysadmins managing on-premise infrastructure need to understand VLANs (802.1Q tagging for logical network segmentation), subnetting (CIDR notation, calculating broadcast/network addresses), and inter-VLAN routing.

CIDR: 10.0.0.0/24 = 256 addresses (254 usable) | /16 = 65,536 | /8 = 16,777,216
