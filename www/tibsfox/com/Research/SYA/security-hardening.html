Security is not a feature you add; it is a property of the entire system. Every decision — from which ports to open to how secrets are stored — either strengthens or weakens the overall security posture. The sysadmin is the last line of defense between the attacker and the data.

CIS Benchmarks

The Center for Internet Security (CIS) publishes hardening benchmarks for every major operating system, cloud platform, and application. CIS benchmarks are the industry standard for baseline security configuration. They provide two profiles:

Level 1: Basic security settings that can be applied broadly without significant performance impact or functionality loss

Level 2: Defense-in-depth settings for high-security environments, may restrict some functionality

Automated scanning tools like OpenSCAP, Lynis, and CIS-CAT verify compliance against benchmarks and generate remediation reports.

Patch Management

Unpatched systems are the most exploited attack vector. A mature patch management process includes: vulnerability scanning (Nessus, OpenVAS, Qualys), risk-based prioritization (CVSS scores, CISA KEV catalog), staged rollout (dev → staging → production), rollback plans, and compliance reporting. Unattended-upgrades (Debian) and dnf-automatic (RHEL) automate security patches on individual hosts. Spacewalk, Foreman, and WSUS manage patching at fleet scale.

SIEM (Security Information and Event Management)

SIEM platforms aggregate logs from all systems, apply correlation rules, and generate security alerts. They are the central nervous system of security operations. Key platforms: Elastic SIEM (open source, built on ELK), Splunk (commercial leader), Wazuh (open source, OSSEC-based), QRadar (IBM). Effective SIEM requires tuning: out-of-box rules generate excessive false positives. Iterative refinement based on environment-specific baselines is essential.

SSH Hardening

SSH is the most critical service to harden because it provides direct system access:

# /etc/ssh/sshd_config — hardened settings
PermitRootLogin no
PasswordAuthentication no
PubkeyAuthentication yes
AuthorizedKeysFile .ssh/authorized_keys
MaxAuthTries 3
LoginGraceTime 30
AllowUsers deploy monitor
X11Forwarding no
AllowTcpForwarding no
ClientAliveInterval 300
ClientAliveCountMax 2

Additional layers: fail2ban (ban IPs after failed attempts), port knocking (hide SSH port), SSH certificates (replace static authorized_keys with time-limited certificates issued by a CA).

Mandatory Access Control

SELinux and AppArmor implement mandatory access control at the kernel level, confining processes to only the resources their policy allows — even if the process runs as root.

FeatureSELinuxAppArmor
Policy modelLabel-based (type enforcement)Path-based (profile per binary)
Default distrosRHEL, Fedora, CentOS, RockyUbuntu, Debian, SUSE
ComplexityHigh (steeper learning curve)Medium (more approachable)
ModesEnforcing, Permissive, DisabledEnforce, Complain
Troubleshootingausearch, audit2allow, sealertaa-status, aa-logprof, aa-complain

Never disable SELinux. If something breaks, switch to permissive mode, diagnose with audit2allow, and create a custom policy module. Disabling MAC is a security regression that most compliance frameworks will flag.

PKI & TLS Certificate Management

Public Key Infrastructure underpins all encrypted communication. Sysadmins manage certificate lifecycles: generation (CSR), signing (CA or Let’s Encrypt), deployment, renewal, and revocation. cert-manager automates certificate lifecycle in Kubernetes. Smallstep and HashiCorp Vault PKI provide private CA infrastructure for internal services.

Secrets Management

HashiCorp Vault is the standard for secrets management: dynamic secrets (database credentials generated on demand), secret rotation, audit logging, and integration with Kubernetes, Terraform, and CI/CD pipelines. Alternatives: AWS Secrets Manager, Azure Key Vault, SOPS (encrypted files in git), age (simple file encryption).

Rule: Secrets never belong in source code, environment variables in Dockerfiles, or configuration files committed to version control.
