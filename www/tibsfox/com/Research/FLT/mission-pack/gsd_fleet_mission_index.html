How to Execute

1
Deposit to planning-bridge inbox

Drop gsd_fleet_feature_refinement_mission.pdf into .planning/staging/inbox/ or paste the Stage 3 mission spec directly to Claude Code on the dev branch.

2
Validate chipset before Wave 0

Run gsd:validate-chipset on the fleet chipset configuration. Exit code 0 = proceed. Exit code 2 = BLOCK. Do not skip this gate.

3
Wave 0 → Wave 1 (TOPO registers all 4 teams)

TOPO registers Alpha, Beta, Gamma, Delta in the cluster state store. All four teams launch simultaneously in Wave 1. Monitor via SURGEON health telemetry.

4
Three CAPCOM gates — all human-required

Wave 1 exit · Wave 4 exit · Wave 5 exit. Each gate is structural. No wave begins until CAPCOM approves the previous gate. The gates are not advisory.

5
v1.50-spec tag on clean Wave 6 completion

LOG agent commits all 10 deliverables and tags v1.50-spec on the skills-evolution branch. This tag is the handoff point for the implementation team.
