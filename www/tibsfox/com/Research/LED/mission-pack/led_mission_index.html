8 Research Modules

MODULE 1

LED Electronics Fundamentals

Forward voltage, I-V curves, Ohm's Law, current-limiting resistors, series/parallel wiring, constant-current drivers, strip power, thermal management.

MODULE 2

Microcontroller Programming

Arduino, PIC MPLAB XC8, Raspberry Pi (pigpio), ESP32, RP2040 PIO. PWM generation, timer config, interrupt handling. Cross-platform comparison guide.

MODULE 3

Addressable LED Protocols

WS2812B NZR 800 kHz protocol, APA102 SPI 24 MHz, SK6812 RGBW, WS2813/WS2815. NeoPixel library, FastLED, WLED firmware. Power injection guide.

MODULE 4

Smart Lighting & Ambient Sensing

Philips Hue CLIP API v1/v2, diyHue emulator, TCS34725 RGB sensing, circadian CCT adaptation, Home Assistant. Ambient → Hue automation pipeline.

MODULE 5

DIY Household LED Controllers

MOSFET PWM dimmers, TRIAC phase-cutting overview, WLED firmware (full BOM), IR/RF remote systems, mains safety, 12V/24V strip controllers.

MODULE 6

High-Speed LED & Persistence of Vision

POV physics, APA102 vs WS2812B for POV, RP2040 dual-core PIO architecture, polar image mapping, rotation sensing, wireless power for rotating assemblies.

MODULE 7

Oscilloscopes & Scientific Measurement

Scope types, bandwidth, Nyquist theorem, trigger modes, probe compensation, measuring PWM/WS2812B, DIY scopes (DSO138, ESP32, Pico). Multimeter diode test mode.

MODULE 8

PLC & Industrial LED Control

IEC 61131-3 ladder logic, relay vs transistor output modules, MOSFET driver boards, Modbus TCP integration, hybrid PLC + ESP32 architectures.

How to Execute This Mission

  1. Open the PDF and read Stage 1 (Vision) to understand the domain architecture and success criteria.
  2. Open Claude Code and paste the Stage 3 Wave 0 component specs (F0-a, F0-b, F0-c) to generate HTML templates, the site map schema, and source index.
  3. Once Wave 0 is complete (CAPCOM human approval checkpoint), launch three parallel Claude Code sessions — one per Wave 1 track (A, B, C). Feed each the Stage 3 component specs for their track plus the Wave 0 outputs.
  4. Collect all Wave 1 HTML drafts. Feed them to a single Opus 4.6 session for Wave 2 synthesis: cross-link injection, glossary generation, and code verification.
  5. CAPCOM checkpoint: review the linked site structure before Wave 3.
  6. Run Wave 3 publication tasks: HTML W3C validation, external link checking, safety warning audit, and final site.zip packaging.

"An LED is not a light source. It is a boundary condition — a place where electrons, coached into higher energy states by a voltage differential, fall across a gap and surrender their borrowed energy as photons. The oscilloscope, the multimeter, the color sensor, the PIC's CCP module, the RP2040's PIO state machine: these are all different ways of standing at that boundary and asking: how bright? how fast? what color? when?"
