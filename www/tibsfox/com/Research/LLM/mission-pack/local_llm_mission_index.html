How to Use This Package

  1. Read Stage 1 Orient on the mission scope, chipset configuration, and success criteria before any execution.
  2. Load into GSD Skill Creator Hand Stage 3 to the GSD orchestrator (Claude Code session). It contains everything needed to begin Wave 0 without returning to this document.
  3. Execute Wave 0 first Paula (Haiku) generates shared schemas, VRAM calculator script, and dataset template. These shared artifacts reduce Wave 1 token spend.
  4. Run Wave 1 parallel tracks Track A (Inference/GPU) and Track B (Training/Dataset) can run simultaneously. Bring both outputs to the CAPCOM gate.
  5. HITL gate: review dataset schema Before Opus begins synthesis, review the GSD-dialect dataset design. This is the highest-judgment artifact — it shapes the fine-tuned model's vocabulary.
  6. Wire Claude Code to local model Set ANTHROPIC_BASE_URL, confirm 25K+ context size, run a test task. Only then route real work to the local model.

Quick Start: Claude Code → Ollama

# 1. Install Ollama and pull a coding model
curl -fsSL https://ollama.com/install.sh | sh
ollama pull qwen2.5-coder:32b      # or your fine-tuned adapter

# 2. Point Claude Code to your local model
export ANTHROPIC_BASE_URL="http://localhost:11434"
export ANTHROPIC_API_KEY="ollama"
echo 'export ANTHROPIC_BASE_URL="http://localhost:11434"' >> ~/.bashrc

# 3. Launch Claude Code (it routes to local model)
claude

# 4. Verify: the model name in Claude Code should show
# your local model, not claude-sonnet or claude-opus
