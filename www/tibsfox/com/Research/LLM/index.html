Research Modules

M1: Inference Runtime Landscape

Runtime Track A · Serving layer

Three major runtimes compared: Ollama (developer-first CLI), vLLM (production throughput), LM Studio (GUI + API). OpenAI/Anthropic-compatible API endpoints. PagedAttention for memory efficiency. Speculative decoding for latency reduction. Model format support: GGUF, SafeTensors, AWQ, GPTQ.

M2: GPU Architecture & Quantization

GPU Track A · Hardware layer

VRAM math for consumer GPUs: RTX 3090 (24GB), RTX 4090 (24GB), RTX 5090 (32GB), RTX 4060 Ti (8/16GB). Quantization landscape: GGUF Q4_K_M, Q5_K_M, Q8_0, FP16. Memory formula: parameters x bytes-per-weight + KV cache + overhead. Throughput benchmarks per model size and quantization level.

M3: LoRA & Fine-Tuning Frameworks

LoRA Track B · Training layer

LoRA mechanics: low-rank decomposition of weight update matrices. Framework comparison: Unsloth (2-5x faster, 80% less VRAM), Axolotl, Hugging Face PEFT. Dataset preparation for GSD-dialect training: DACP bundles, YAML schemas, mission pack structures. GGUF export pipeline from fine-tuned checkpoint to servable model.

M4: Model-to-Production Pipeline

Pipeline Synthesis · Operations layer

End-to-end pipeline: model selection, quantization, serving, monitoring. Eval harness for GSD-specific tasks: DACP bundle generation, YAML validation, mission pack authoring. A/B testing between base model and LoRA adapter. Continuous improvement loop with regression detection. Deployment patterns for always-on inference.

M5: Claude Code Integration

Claude Synthesis · Routing layer

ANTHROPIC_BASE_URL redirection to local inference endpoint. Task routing strategy: scaffolding and YAML validation to local model, synthesis and judgment to Anthropic API. LM Studio 0.4.1+ and Ollama Anthropic Messages API support. GSD-dialect fine-tuned model as lightweight subagent in wave execution. Cost analysis: local vs. API per task category.
