$ Shell scripting is the connective tissue of Unix. It grew from operators who needed to automate their work, from the same impulse that produces jigs in a woodshop or macros on a factory floor.

1. Execution Model

The Shebang

#!/bin/bash              # absolute path
#!/usr/bin/env bash      # searches $PATH -- more portable

The #! is processed by the kernel’s binfmt handler, not the shell. env accommodates systems where bash lives in /usr/local/bin (FreeBSD, older macOS).

fork/exec

Every external command: fork() → child calls execve() → parent calls waitpid(). Built-ins (cd, echo, read, export, test) skip this — they run inside the shell process. cd must be a builtin because it modifies the shell’s own state.

Subshells — The #1 Gotcha

x=hello
(x=world; echo "$x")   # prints "world"
echo "$x"               # prints "hello" -- parent unchanged

# Pipelines create subshells too:
count=0
cat file.txt | while read -r line; do ((count++)); done
echo "$count"   # prints 0 in bash < 4.2!

# Fix: redirect instead of pipe
while read -r line; do ((count++)); done < file.txt

source vs execution

source script.sh (or . script.sh) runs in the current shell — no fork, variables persist. This is how .bashrc and virtualenv activate work. Executing (./script.sh) creates a new process; only exported env vars cross the boundary.

2. Variables & Arrays

# Shell vs environment variables
MY_VAR="local only"          # shell variable
export PATH_VAR="/usr/bin"   # environment variable (inherited by children)
readonly CONST="immutable"   # cannot be reassigned

# Indexed arrays (bash, not POSIX)
fruits=(apple banana cherry)
echo "${fruits[0]}"          # apple
echo "${#fruits[@]}"         # 3 (count)
fruits+=(date)               # append

# Associative arrays (bash 4+, must declare)
declare -A colors
colors[sky]=blue
echo "${!colors[@]}"         # keys

Expansion Order (9 Steps)

# The shell processes each word through these stages in order:
1. Brace expansion       {a,b,c} → a b c
2. Tilde expansion        ~/dir → /home/foxy/dir
3. Parameter expansion    ${var}, ${var:-default}, ${var#prefix}
4. Command substitution   $(command)
5. Arithmetic expansion   $(( 2 + 3 ))
6. Process substitution   <(command)
7. Word splitting         on unquoted results of 3-5
8. Pathname expansion     *.txt → file1.txt file2.txt
9. Quote removal          strips outer quotes

Parameter Expansion — The Power Tools

${var:-default}          # use default if var is unset/empty
${var:=default}          # assign default if var is unset/empty
${var:?error message}    # exit with error if var is unset/empty
${var:+alternate}        # use alternate if var IS set

${var#pattern}           # strip shortest prefix match
${var##pattern}          # strip longest prefix match
${var%pattern}           # strip shortest suffix match
${var%%pattern}          # strip longest suffix match
${var/old/new}           # replace first match
${var//old/new}          # replace all matches

${var,,}                 # lowercase (bash 4+)
${var^^}                 # uppercase (bash 4+)
${#var}                  # string length
${var:offset:length}     # substring

4. Redirections & Pipes

# Standard redirections
cmd > file               # stdout to file (overwrite)
cmd >> file              # stdout to file (append)
cmd 2> file              # stderr to file
cmd &> file              # both stdout and stderr to file (bash)
cmd > /dev/null 2>&1     # discard everything (POSIX)

# File descriptors (order matters!)
cmd 2>&1 > file          # WRONG: stderr goes to terminal
cmd > file 2>&1          # RIGHT: both to file

# Here-document and here-string
cat << 'EOF'             # quoted delimiter: no expansion
  $HOME is literal here
EOF

grep "pattern" <<< "search this string"  # here-string

# Process substitution
diff <(sort file1) <(sort file2)
tee >(gzip > compressed.gz) < input.txt

# Named pipes (FIFOs)
mkfifo /tmp/mypipe
producer > /tmp/mypipe &
consumer < /tmp/mypipe

Pipes: How They Actually Work

Each | creates a kernel pipe (64K buffer). Stages run concurrently — not sequentially. The kernel blocks the writer when the buffer is full and the reader when it’s empty. set -o pipefail makes the pipeline return the exit code of the first non-zero stage, not just the last. Check all stages with ${PIPESTATUS[@]}.

5. Control Flow

# test / [ / [[ comparison
[ -f file ]          # POSIX: file exists and is regular
[[ $var == pattern ]]  # Bash: pattern matching, no word splitting
[[ $var =~ regex ]]    # Bash: regex matching
(( x > 5 ))            # Bash: arithmetic comparison

# Case (pattern matching)
case "$response" in
  [yY]|[yY][eE][sS]) echo "yes" ;;
  [nN]|[nN][oO])     echo "no"  ;;
  *)                  echo "what?" ;;
esac

# Select (interactive menu)
select choice in "Option A" "Option B" "Quit"; do
  case $choice in
    "Quit") break ;;
    *) echo "Selected: $choice" ;;
  esac
done

# C-style for loop
for (( i=0; i<10; i++ )); do
  echo "$i"
done

6. Functions & Signal Handling

# Function declaration (POSIX form preferred)
cleanup() {
  rm -f "$tmpfile"
  echo "Cleaned up"
}

# Trap: run cleanup on exit, error, or signal
trap cleanup EXIT            # always runs on exit
trap 'echo "Interrupted"' INT  # Ctrl-C
trap '' TERM                   # ignore SIGTERM

# The EXIT cleanup pattern (essential for robust scripts)
tmpfile=$(mktemp)
trap 'rm -f "$tmpfile"' EXIT
# ... script body ... tmpfile is guaranteed cleaned up

7. Job Control & Process Management

# Background and foreground
long_command &               # run in background
jobs                         # list background jobs
fg %1                        # bring job 1 to foreground
bg %1                        # resume stopped job in background
wait                         # wait for all background jobs
wait $!                      # wait for last background PID

# Detach from terminal
nohup long_command &         # immune to hangup
disown %1                    # remove from job table
setsid command               # new session (full detach)

# File locking
(
  flock -n 9 || { echo "Already running"; exit 1; }
  # ... critical section ...
) 9>/var/lock/myscript.lock

# Parallel execution with wait
for file in *.csv; do
  process_file "$file" &
done
wait                         # wait for all to finish

8. Defensive Scripting

#!/usr/bin/env bash
set -euo pipefail            # THE defensive trio

# set -e: exit on any command failure
# set -u: treat unset variables as errors
# set -o pipefail: pipeline fails if ANY stage fails

The local gotcha: local var=$(command) always succeeds — local masks the exit code! Split the declaration: local var; var=$(command).

# Error trap with line reporting
trap 'echo "Error at line $LINENO" >&2' ERR

# The production template
#!/usr/bin/env bash
set -euo pipefail
trap 'echo "FAILED at line $LINENO" >&2; exit 1' ERR

readonly SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
readonly TMPDIR="$(mktemp -d)"
trap 'rm -rf "$TMPDIR"' EXIT

main() {
  # ... script body ...
  :
}

main "$@"

11. The One-Liner Tradition

# Unique lines without sorting (awk magic)
awk '!seen[$0]++' file.txt
# How it works: seen is an associative array. For each line ($0),
# if seen[$0] is 0 (not yet seen), ! makes it truthy, so the line prints.
# Then ++ increments it to 1, so duplicates are falsy.

# Sum a column
awk '{sum += $3} END {print sum}' data.txt

# Inline Perl for complex text processing
perl -pe 's/(\d{4})-(\d{2})-(\d{2})/$3\/$2\/$1/g' dates.txt

# Sed: number non-empty lines
sed '/./=' file.txt | sed '/./N; s/\n/\t/'

# Process substitution for multi-file comparison
paste <(cut -d, -f1 old.csv) <(cut -d, -f1 new.csv) | awk '$1 != $2'

When to stop: If your one-liner exceeds ~120 characters, or needs a loop, or needs error handling — write a script. If your script exceeds ~100 lines, or needs data structures — write a Python/Perl program. The art is knowing when to escalate.
