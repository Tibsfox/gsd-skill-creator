Pipeline Patterns — The Shell’s Composition Model

Log Analysis Pipeline

# Bash: top 10 404 errors in the last hour
grep "$(date -d '1 hour ago' '+%d/%b/%Y:%H')" access.log \
  | awk '$9 == 404 {print $7}' \
  | sort | uniq -c | sort -rn | head -10

# Perl equivalent (single process, faster for large files):
perl -ane '
  next unless /404/;
  $count{$F[6]}++;
  END { print "$count{$_} $_\n" for sort {$count{$b}<=>$count{$a}} keys %count }
' access.log | head -10

# Python equivalent:
python3 -c "
from collections import Counter
import re
with open('access.log') as f:
    c = Counter(m.group(1) for l in f if '404' in l
                for m in [re.search(r'\"\\w+ (\\S+)', l)] if m)
for url, n in c.most_common(10): print(f'{n:6d} {url}')
"

Data Extraction Pipeline

# Extract, transform, load — the CLI way
cat employees.csv \
  | tail -n +2                       \  # skip header
  | cut -d, -f1,3,5                  \  # name, department, salary
  | sort -t, -k2,2                   \  # sort by department
  | awk -F, '{sum[$2]+=$3; n[$2]++}  \  # average salary per dept
    END{for(d in sum) printf "%s: $%.2f\n", d, sum[d]/n[d]}' \
  | sort -t: -k2 -rn                    # highest paying first

When to Escalate

The hardest skill in the sysadmin’s toolkit: knowing when to stop writing shell and reach for a real language.

SignalStay in ShellEscalate to Perl/Python
Data structuresArrays of stringsNested hashes, objects, trees
Error handlingset -e is enoughNeed try/catch, structured errors
Text processinggrep/sed/awk pipelineMulti-line parsing, state machines
File formatLine-oriented textJSON, XML, YAML, binary
Line count< 100 lines> 100 lines (consider it)
PortabilityPOSIX sh, runs everywhereNeed specific libraries, APIs
PerformancePipe parallelism is enoughCPU-bound, need real threading
TestingManual verificationNeed unit tests, CI integration
MaintenanceWrite-once, document in commentsLong-lived, team-maintained
