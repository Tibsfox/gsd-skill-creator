The POSIX Shell Standard

The POSIX shell (IEEE Std 1003.2, later 1003.1-2017) defines the minimum feature set that any compliant /bin/sh must provide. It is essentially “ksh88 minus the ksh-specific parts” — a codification of the Bourne/Korn shell tradition.

Why POSIX Matters

/bin/sh is not always bash. Debian/Ubuntu use dash (a minimal POSIX shell, ~10x faster than bash for script startup). Alpine uses busybox ash. FreeBSD uses an ash derivative. macOS ships bash 3.2 (frozen at GPLv2). Portable scripts must target POSIX sh — no arrays, no [[ ]], no process substitution, no brace expansion.

Bash — The Universal Shell

Brian Fox (1989), maintained by Chet Ramey since 1994. POSIX.2 superset. GPL v3+.

Strengths

  • Ubiquity. Every Linux distro ships it. Every Docker container that has a shell has bash. Every CI/CD pipeline assumes bash.
  • Readline. Emacs/vi editing, Ctrl-R history search, customizable via ~/.inputrc.
  • Programmable completion. The bash-completion package provides context-sensitive tab completion for hundreds of commands.
  • 30+ years of community knowledge. Every bash question has been answered on Stack Overflow.

Weaknesses

  • Quoting nightmares. Word splitting and globbing interact in surprising ways.
  • Error handling. set -euo pipefail helps but has gotchas (local masks exit codes).
  • No structured data. Everything is a string. Nested data requires external tools (jq).
  • No syntax highlighting without plugins.

Zsh — The Maximalist Shell

Paul Falstad (1990). MIT-like license. macOS default since Catalina.

Killer Features

# Extended globbing with qualifiers
ls **/*.txt          # recursive (also bash 4.0+ with globstar)
ls *.txt(.)          # regular files only
ls *.txt(om)         # order by modification time
ls *(Lk+100)         # files larger than 100 KB
ls *(mw-1)           # modified within last week

# Spell correction
$ gerp foo bar.txt
zsh: correct 'gerp' to 'grep' [nyae]?

oh-my-zsh Ecosystem

Robby Russell (2009). 300+ plugins, 150+ themes, 2,200+ contributors. Made zsh accessible beyond power users. Powerlevel10k (Roman Perepelitsa, 2019) achieves sub-millisecond prompt latency with “instant prompt” caching.

Fish — The Friendly Shell

Axel Liljencrantz (2005). Deliberately not POSIX. Rewritten in Rust for v4.0 (2024).

Out-of-Box Experience

  • Syntax highlighting as you type — valid commands green, invalid red, before Enter.
  • Autosuggestions from history — right arrow to accept.
  • Web-based config — fish_config opens a browser GUI.
  • No configuration needed. Works beautifully with defaults.

Syntax Differences

# Fish syntax (not POSIX)
set -x VAR value              # instead of export VAR=value
if test $count -gt 10         # instead of if [ $count -gt 10 ]
    echo "Too many"
end                            # instead of fi

for file in *.txt
    echo $file
end

function greet
    echo "Hello, $argv[1]"    # $argv instead of $1
end

The POSIX trade-off: Fish scripts won’t run in bash. Bash scripts won’t run in fish. If your workflow involves copying shell snippets from Stack Overflow, fish adds friction. If you want a shell that works perfectly out of the box for interactive use, fish is unmatched.

PowerShell — The Object Pipeline

Jeffrey Snover (2006). .NET objects instead of text. Cross-platform since 2016. MIT license.

# Objects, not text
Get-Process | Where-Object CPU -gt 10 | Sort-Object CPU -Descending | Select-Object -First 5

# Properties and methods, not parsing
(Get-ChildItem *.log).Length | Measure-Object -Sum

# Discoverable
Get-Command *Service*          # find commands by pattern
Get-Help Get-Process -Examples # built-in examples
Get-Process | Get-Member       # inspect object properties

Choosing a Shell

Use CaseRecommendationWhy
Scripts for productionBash (or POSIX sh)Ubiquitous, portable, battle-tested
Interactive power userZsh + oh-my-zshGlobbing, plugins, themes, completion
Interactive ease-of-useFishBest out-of-box experience, zero config
Windows administrationPowerShell.NET objects, Windows integration
Data explorationNushellStructured data pipelines, built-in table viewer
Portable scripts (Docker, CI)POSIX sh (dash)Fastest startup, guaranteed available

The pragmatic answer: Use whatever shell you enjoy interactively. Write scripts in bash (or POSIX sh for maximum portability). Your interactive shell and your scripting shell don’t have to be the same — and for most experienced users, they aren’t.
