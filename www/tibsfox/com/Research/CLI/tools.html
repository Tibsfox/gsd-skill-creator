$ This is the Unix philosophy: Write programs that do one thing and do it well. Write programs to work together. Write programs to handle text streams, because that is a universal interface.
— Doug McIlroy, inventor of Unix pipes

The small tools model: Each classic Unix tool is small enough to fit in your head — cat concatenates, head shows first N lines, tail shows last N, wc counts, tee duplicates a stream. None are impressive alone. Their power comes from composition. A pipeline of five trivial tools can accomplish what would take hundreds of lines in a general-purpose language. And the pipeline is readable, testable, and modifiable without a compiler.

Ken Thompson added a pragmatic edge: “When in doubt, use brute force.” A pipeline that processes a 2 GB log file line by line in 30 seconds is better than an elegant solution that takes a week to write and runs in 3 seconds. The tools are fast enough. The human’s time is the bottleneck.

Composition: Pipes, Filters, and Streams

The | character is the most important operator in the shell. It connects stdout of one process to stdin of the next:

# Count unique IP addresses in an access log
cat access.log | awk '{print $1}' | sort | uniq -c | sort -rn | head -20

# Six tools. One line. No temporary files.
# 1. cat reads the file
# 2. awk extracts the first field (the IP address)
# 3. sort orders the IPs alphabetically (required for uniq)
# 4. uniq -c collapses duplicates and prefixes with a count
# 5. sort -rn sorts numerically in reverse (highest first)
# 6. head -20 shows only the top 20

A filter is any program that reads from stdin and writes to stdout. tee duplicates a stream — writes input to both stdout and a file, letting you inspect mid-pipeline without breaking it:

cat server.log | grep "ERROR" | tee errors.txt | wc -l

# Process substitution: compare two commands as files
diff <(sort file1.txt) <(sort file2.txt)

grep — Pattern Matching

grep (Global Regular Expression Print) is likely the most frequently used Unix tool after cd and ls.

# Basic search with context
grep -n "error" /var/log/syslog          # show line numbers
grep -C 3 "error" /var/log/syslog        # 3 lines context
grep -i "error" /var/log/syslog          # case-insensitive

# Recursive search
grep -r "TODO" src/                       # all files under src/
grep -r --include="*.py" "TODO" src/      # only Python files
grep -rl "deprecated" lib/                # filenames only

# Inversion and counting
grep -v "DEBUG" application.log           # lines NOT matching
grep -c "ERROR" application.log           # count matching lines

# Regular expressions (ERE with -E, PCRE with -P)
grep -E "error|fatal" server.log          # alternation
grep -P "error(?!.*handled)" app.log      # negative lookahead
grep -F "user.name[0]" config.json        # fixed string (no regex)

Modern: ripgrep (rg)

rg "TODO" src/                            # respects .gitignore, colored
rg -t ts "interface" src/                 # only TypeScript files
rg -P "(?<=fn )\w+" src/                  # PCRE2 lookbehind

sed — The Stream Editor

sed processes text line by line, applying transformations without opening an editor. The workhorse for automated text manipulation.

# Substitution (the most common use)
sed 's/old/new/' file.txt                # first occurrence per line
sed 's/old/new/g' file.txt               # all occurrences (global)
sed -i 's/old/new/g' file.txt            # in-place edit

# Address ranges
sed '5s/old/new/' file.txt               # only line 5
sed '10,20s/old/new/g' file.txt          # lines 10-20
sed '/^#/d' file.txt                     # delete comment lines
sed '/START/,/END/d' file.txt            # delete between patterns

# Multiple commands
sed -e 's/foo/bar/g' -e 's/baz/qux/g' file.txt
sed '/^$/d; s/  */ /g' file.txt          # delete blank lines + collapse spaces

# Insert and append
sed '1i\#!/bin/bash' script.sh           # insert at line 1
sed '/pattern/a\new line after' file.txt # append after pattern

# Classic one-liners
sed -n '10,20p' file.txt                 # print lines 10-20 (like head+tail)
sed 's/^[ \t]*//' file.txt               # strip leading whitespace
sed '/^$/d' file.txt                     # remove blank lines

Modern: sd

sd 'old' 'new' file.txt                  # simpler syntax, no escaping
sd -F 'literal.string' 'new' file.txt    # fixed string mode

awk — Pattern-Action Processing

awk is a pattern-action language. For every line of input, if the pattern matches, the action executes. It is a complete programming language disguised as a text tool.

# Field extraction (default separator: whitespace)
awk '{print $1}' file.txt                # first field
awk '{print $NF}' file.txt               # last field
awk -F: '{print $1, $3}' /etc/passwd     # custom delimiter

# Pattern matching
awk '/error/ {print}' server.log         # lines containing "error"
awk '$3 > 100 {print $1, $3}' data.txt   # field 3 greater than 100
awk 'NR >= 10 && NR <= 20' file.txt      # lines 10-20

# Built-in variables
# $0 = entire line, $1..$NF = fields, NR = record number
# NF = number of fields, FS = field separator, OFS = output separator

# Aggregation
awk '{sum += $3} END {print "Total:", sum}' data.txt
awk '{sum += $3; n++} END {print "Avg:", sum/n}' data.txt

# Associative arrays (awk's killer feature)
awk '{count[$1]++} END {for (k in count) print count[k], k}' access.log | sort -rn

# BEGIN/END blocks
awk 'BEGIN {FS=","; OFS="\t"} {print $1, $3}' data.csv

# Printf for formatted output
awk '{printf "%-20s %8.2f\n", $1, $3}' report.txt

find — Filesystem Traversal

# By name and type
find . -name "*.log" -type f              # files matching *.log
find . -name "*.tmp" -type f -delete      # find and delete (careful!)
find /etc -name "*.conf" -type f          # config files under /etc

# By time
find . -mtime -1 -type f                  # modified in last 24 hours
find . -newer reference.txt               # newer than a reference file

# By size and permissions
find . -size +100M                        # files over 100 MB
find . -perm -u+x -type f                # executable files

# Executing commands (-exec vs xargs)
find . -name "*.py" -exec grep -l "TODO" {} \;     # one grep per file
find . -name "*.py" -exec grep -l "TODO" {} +      # batch (faster)
find . -name "*.py" -print0 | xargs -0 grep -l "TODO"  # null-delimited

# Pruning (skip directories)
find . -path ./node_modules -prune -o -name "*.js" -print

Modern: fd

fd "\.log$"                               # regex by default, colored
fd -e py                                  # by extension
fd -e py -x grep -l "TODO"               # execute per result
fd --type f --changed-within 1d           # recently modified

Text Field Tools

# sort: order lines
sort file.txt                             # alphabetical
sort -n file.txt                          # numeric
sort -rn file.txt                         # reverse numeric
sort -t: -k3 -n /etc/passwd              # sort by field 3 (numeric)
sort -u file.txt                          # unique (sort + dedup)

# uniq: collapse duplicates (input must be sorted!)
sort data.txt | uniq                      # remove duplicates
sort data.txt | uniq -c                   # count occurrences
sort data.txt | uniq -d                   # only duplicated lines

# cut: extract columns
cut -d: -f1,3 /etc/passwd                # fields 1 and 3
cut -c1-10 file.txt                      # characters 1-10

# tr: translate/delete characters
echo "Hello World" | tr 'A-Z' 'a-z'     # lowercase
echo "Hello World" | tr -d ' '          # delete spaces
echo "Hello   World" | tr -s ' '        # squeeze repeated spaces
cat file.txt | tr '\n' ' '              # join lines with spaces

# paste: merge files side by side
paste file1.txt file2.txt                # tab-separated columns
paste -d, file1.txt file2.txt            # comma-separated
seq 6 | paste - - -                      # three columns from one stream

# join: SQL-like join on sorted files
join -t, users_sorted.csv orders_sorted.csv

xargs — Building Commands from stdin

# Basic: pass lines as arguments
echo "file1.txt file2.txt" | xargs rm

# Null-delimited (safe for filenames with spaces)
find . -name "*.bak" -print0 | xargs -0 rm

# Placeholder (-I) for complex commands
find . -name "*.jpg" -print0 | xargs -0 -I{} convert {} -resize 800x600 {}

# Batch size
echo {1..100} | xargs -n 10 echo         # 10 args per invocation

# Parallel execution
find . -name "*.png" -print0 | xargs -0 -P4 -I{} optipng {}

# Dry run with echo
find . -name "*.tmp" | xargs echo rm      # shows what would be deleted

System Tools

Process Management

# ps: snapshot of processes
ps aux                                    # all processes, BSD syntax
ps -ef                                    # all processes, POSIX syntax
ps aux --sort=-%mem | head -10            # top 10 by memory
ps -eo pid,user,%cpu,%mem,cmd --sort=-%cpu | head -20

# top/htop: real-time monitoring
top -bn1 | head -20                       # batch mode, one snapshot
htop                                      # interactive, colored

# kill: signal processes
kill -15 PID                              # SIGTERM (graceful)
kill -9 PID                               # SIGKILL (force)
killall -u username                       # kill all by user
pkill -f "pattern"                        # kill by command pattern

Debugging and Introspection

# lsof: list open files
lsof -i :8080                             # what's using port 8080?
lsof -p PID                               # files open by a process
lsof +D /var/log                          # files open in a directory

# strace: system call tracer
strace -e trace=open,read,write cmd       # trace file operations
strace -c cmd                             # system call statistics
strace -p PID -e trace=network            # attach to running process

Network Diagnostics

# ss: socket statistics (replacement for netstat)
ss -tuln                                  # listening TCP/UDP ports
ss -tn state established                  # established connections
ss -tn state established | awk '{print $5}' | cut -d: -f1 | sort | uniq -c | sort -rn

# dig: DNS queries
dig example.com                           # A record
dig +short example.com MX                 # MX records, short output
dig @8.8.8.8 example.com                  # query specific DNS server

diff & patch — Change Management

# Unified diff (the standard for patches)
diff -u old.txt new.txt                   # unified format
diff -u old.txt new.txt > changes.patch   # save as patch file
patch < changes.patch                     # apply the patch

# Recursive directory diff
diff -rq dir1/ dir2/                      # quick: show only which files differ
diff -ru dir1/ dir2/ > directory.patch    # full recursive patch

# Modern: delta (beautiful diffs)
diff -u old.txt new.txt | delta           # colored, syntax-highlighted
git diff | delta                          # or configure git to use delta

Archives and Compression

# tar: tape archive
tar czf archive.tar.gz dir/              # create gzip-compressed
tar cjf archive.tar.bz2 dir/             # create bzip2-compressed
tar cJf archive.tar.xz dir/              # create xz-compressed
tar xf archive.tar.gz                    # extract (auto-detects compression)
tar tf archive.tar.gz                    # list contents

curl & wget — HTTP from the Terminal

# curl: transfer data (supports 25+ protocols)
curl -s https://api.example.com/data | jq .       # silent + JSON
curl -o file.zip https://example.com/file.zip      # save to file
curl -H "Authorization: Bearer $TOKEN" https://api.example.com/protected
curl -X POST -d '{"key":"value"}' -H "Content-Type: application/json" https://api.example.com/data

# wget: download files
wget https://example.com/file.zip                   # download
wget -r -np https://example.com/docs/               # recursive mirror
wget -c https://example.com/large.iso                # resume interrupted download

Scheduling: cron, at, systemd timers

# cron: recurring schedule
crontab -e                                # edit user's crontab
crontab -l                                # list entries

# Cron format: minute hour day-of-month month day-of-week command
# m  h  dom mon dow  command
  0  2  *   *   *    /usr/local/bin/backup.sh       # daily at 2:00 AM
  */15 * *  *   *    /usr/local/bin/health-check.sh  # every 15 minutes
  0  0  1   *   *    /usr/local/bin/monthly-report.sh # first of month

# at: one-time scheduled execution
echo "backup.sh" | at 2:00 AM tomorrow
at -l                                     # list pending jobs

# systemd timers (modern alternative to cron)
# /etc/systemd/system/backup.timer
# [Timer]
# OnCalendar=*-*-* 02:00:00
# Persistent=true
systemctl list-timers                     # list active timers

The Rust Rewrite Wave

A generation of Rust developers has rewritten the classic Unix tools with modern defaults: color output, .gitignore awareness, better ergonomics, and often significantly better performance.

# Sensible aliases for interactive use (scripts stay POSIX)
# In ~/.bashrc or ~/.zshrc:
alias ls='eza'
alias cat='bat --paging=never'
alias grep='rg'
alias find='fd'
alias du='dust'
alias diff='delta'
alias top='btm'
alias ps='procs'

The pragmatic approach: Modern tools are additive, not replacements. Write scripts with grep, sed, awk, find for portability. Use rg, fd, bat, eza interactively for speed and readability. Alias the modern tools in your shell profile. Best of both worlds.

jq & yq — Structured Data on the CLI

jq fills the gap that Unix text tools cannot: nested, structured data.

# Pretty-print JSON
curl -s https://api.example.com/data | jq .

# Extract and transform
echo '{"name": "Alice", "age": 30}' | jq -r '.name'
echo '[{"name":"A","score":85},{"name":"B","score":92}]' | jq '.[] | select(.score > 90)'

# Object construction
echo '{"first": "Alice", "last": "Smith"}' | jq '{full: (.first + " " + .last)}'

# API work
curl -s "https://api.github.com/users/octocat/repos" | jq -r '.[].name'
kubectl get pods -o json | jq -r '.items[] | [.metadata.name, .status.phase] | @tsv'

# yq: YAML processing (Go version by Mike Farah)
yq '.metadata.name' deployment.yaml
yq -i '.spec.replicas = 3' deployment.yaml
yq -o=json '.' config.yaml               # YAML to JSON

fzf transforms any list into an interactive fuzzy-search menu. Three keybindings justify the install:

BindingActionReplaces

Ctrl-TFuzzy-find file, insert pathTab completion for paths

# Fuzzy-find and open in vim
vim $(fzf --preview 'bat --color=always {}')

# Git: checkout branch interactively
git branch | fzf | xargs git checkout

# Git: interactive log browser
git log --oneline | fzf --preview 'git show --color=always {1}'

# Kill a process interactively

# SSH to a host from config
grep "^Host " ~/.ssh/config | awk '{print $2}' | fzf | xargs ssh

Real-World Pipeline Examples

Log Analysis: Top Error Sources in the Last Hour

journalctl --since "1 hour ago" --priority=err --no-pager |
  sed 's/^.*]: //' |                       # strip timestamp and service
  sort | uniq -c | sort -rn | head -20     # top 20 error messages

Data Extraction: CSV Statistics

tail -n +2 responses.csv |                # skip header
  cut -d, -f3 |                            # extract response_time
  sort -n |                                # sort numerically
  awk '{a[NR]=$1; sum+=$1}
    END {
      n=NR
      printf "Count: %d\nMin: %s\nMax: %s\nAvg: %.2f\nP95: %s\n",
        n, a[1], a[n], sum/n, a[int(n*0.95)]
    }'

Disk Space Alert

df -h | awk 'NR>1 {
  gsub(/%/, "", $5)
  if ($5+0 > 85) printf "ALERT: %s is %s%% full (%s)\n", $6, $5, $1
}'

Find and Replace Across a Codebase

find src/ -name "*.py" -print0 |
  xargs -0 grep -l "api.old.example.com" |
  xargs -I{} sed -i 's|api.old.example.com|api.new.example.com|g' {}

Security: Recently Modified Files

find / -mtime -1 -type f \
  -not -path "/proc/*" -not -path "/sys/*" \
  -not -path "/tmp/*" -not -path "/var/log/*" \
  2>/dev/null | xargs -r ls -la | sort -k6,7

Parallel Compilation with Progress

find src/ -name "*.c" -print0 |
  xargs -0 -P$(nproc) -I{} sh -c '
    obj=$(echo {} | sed "s|src/|build/|;s|\.c$|.o|")
    mkdir -p $(dirname "$obj")
    gcc -O2 -c {} -o "$obj" && echo "OK: {}" || echo "FAIL: {}"
  ' 2>&1 |
  tee build.log |
  awk '/^FAIL/{fail++} /^OK/{ok++} END{print ok, "ok,", fail+0, "failed"}'

When Pipes Break Down

ProblemWhy It FailsSolution
Binary dataText tools assume line-oriented UTF-8Use xxd, hexdump, or format-specific tools
Structured dataJSON/XML/YAML don’t split on newlinesUse jq, yq, xmlstarlet
Error handlingPipe hides exit codes of middle stagesset -o pipefail, check $PIPESTATUS
BufferingStdout is block-buffered when piped (4K-64K)stdbuf -oL for line-buffered, unbuffer for raw
Filenames with spacesWord splitting breaks pathsfind -print0 | xargs -0, always quote
State across linesEach pipeline stage is stateless per lineUse awk variables, or escalate to Perl/Python

The escalation rule: If your pipeline needs state, nested data structures, or error recovery — stop. Write a Perl or Python script. The shell is glue. When the glue needs to think, reach for a real language. The art is knowing when to reach.
