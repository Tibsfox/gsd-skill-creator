Overview Wire Protocol Record Types Resolution Zone Management DNSSEC Extensions & Privacy Mission Pack
Every time a human types a URL, every time a microservice calls another microservice, every time an email routes across the planet — the Domain Name System is doing something that looks simple but is, at its bones, a distributed, hierarchical, cryptographically-extensible database operating at planetary scale. DNS was designed in 1983 by Paul Mockapetris to replace the brittle, centrally-maintained HOSTS.TXT file with something that could survive the exponential growth of the ARPANET. It did not merely survive. It became the connective tissue of the internet itself.

Research Modules

M1: Wire Protocol & Message Format

Wire Track A · Binary specification

The 12-byte header with all flag bits (QR, OPCODE, AA, TC, RD, RA, AD, CD, RCODE). Label encoding and compression pointers. Five message sections. UDP/TCP transport rules including EDNS0 expansion. The complete binary specification of every DNS message.

M2: Resource Record Taxonomy

Records Track A · Type catalog

Every IANA-assigned RR type: type code, RDATA wire format, zone file syntax, and operational semantics. From the original RFC 1035 types (A, NS, CNAME, SOA, MX, TXT) through DNSSEC types (DNSKEY, RRSIG, NSEC3) and modern service binding (SVCB, HTTPS).

M3: Resolution Architecture

Architecture Track B · Resolver hierarchy

The full resolution stack: stub resolver through recursive resolver through the root-to-authoritative chain. Referral mechanics, glue records and the bailiwick rule, negative caching, the 13 root server logical addresses and their anycast deployment.

M4: Zone Management

Zones Track B · Zone operations

Zone files ($ORIGIN, $TTL, $INCLUDE, $GENERATE directives). SOA record fields. Zone transfer protocols (AXFR, IXFR). NOTIFY mechanism. Dynamic updates (RFC 2136). TSIG transaction signatures for authenticated zone transfers.

M5: DNSSEC

Security Track C · Cryptographic chain of trust

Zone signing procedure, DNSKEY (KSK vs ZSK), RRSIG structure, DS delegation signer, NSEC chain vs NSEC3 hashed denial. The chain of trust from root trust anchor through TLD to authoritative zone. Validation algorithm and key rollover procedures.

M6: Modern Extensions & Privacy Stack

Extensions Track C · Privacy and transport

EDNS0 (RFC 6891) OPT pseudo-RR and extension mechanism. DNS over TLS (DoT), DNS over HTTPS (DoH), DNS over QUIC (DoQ). QNAME minimization. Oblivious DoH. Client Subnet (ECS). The privacy trade-off matrix across all approaches.
