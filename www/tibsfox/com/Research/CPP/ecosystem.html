Ecosystem & Tooling

The C++ toolchain story has historically been a nightmare. Other languages have one canonical build tool (Cargo, npm, pip). C++ does not. The reasons are partly historical, partly technical (binary ABI compatibility is genuinely hard), and partly cultural. The situation has improved dramatically in the past few years.

Compilers

C++ has multiple production-quality compilers, each with its own personality. Standards conformance has improved enormously over the past decade.

Compiler Explorer — godbolt.org — is Matt Godbolt’s online sandbox that lets you paste C++ code and see what dozens of compilers produce, side by side, with optimization flags of your choice. It is one of the best teaching tools the language has ever had, and one of the best debugging tools when you are chasing “did the optimizer actually inline this?” Most C++ talks at conferences now use it as a live demo platform.

A modern CMake target

cmake_minimum_required(VERSION 3.20)
project(starfield CXX)

set(CMAKE_CXX_STANDARD 20)
set(CMAKE_CXX_STANDARD_REQUIRED ON)
set(CMAKE_CXX_EXTENSIONS OFF)

add_library(starfield_core
    src/orbit.cpp
    src/star.cpp
)
target_include_directories(starfield_core PUBLIC include)
target_compile_features(starfield_core PUBLIC cxx_std_20)

find_package(fmt REQUIRED)
target_link_libraries(starfield_core PUBLIC fmt::fmt)

add_executable(starfield app/main.cpp)
target_link_libraries(starfield PRIVATE starfield_core)

The key idea: every artifact is a target, and properties like include directories and linked libraries are attached to targets, not set globally. Downstream consumers see exactly what your target advertises.

Why dependency management has been hard

C++ deals with three structural problems that other languages mostly avoid:

  1. No standard binary interface. The ABI of a C++ library depends on the compiler, the standard library implementation, its version, the language standard, optimization flags, and sometimes header definitions.
  2. Header-heavy templates. Most modern C++ code lives in headers because templates need to be visible at instantiation. Modules (C++20) are intended to fix this but rollout is gradual.
  3. One Definition Rule (ODR) violations. It is shockingly easy to link two object files that disagree on the layout of a class, and the linker will silently pick one.

For now, the answer is: pick CMake plus Conan or vcpkg, and be consistent.

Sanitizers and Static Analysis

Modern C++ is much easier to write safely than the language used to be, and tooling is a big part of why:

ToolWhat it catchesCost
AddressSanitizer (ASan) -fsanitize=addressHeap/stack overflow, use-after-free, double-free~2× slowdown
UndefinedBehaviorSanitizer (UBSan)Signed overflow, null deref, misaligned access, invalid castsLow
ThreadSanitizer (TSan)Data races5–15× slowdown
MemorySanitizer (MSan)Reads of uninitialized memoryHigher; needs full instrumentation
LeakSanitizer (LSan)Memory leaks at process exitIncluded with ASan
clang-tidyHundreds of checks: Core Guidelines, modernization, bug-prone patternsBuild-time
clang-formatConsistent formatting (Google, LLVM, Mozilla, Chromium, WebKit styles)Free
cppcheckCommon mistakes, editor-friendlyStandalone
ValgrindMemory debugging, very thorough10–30× slowdown
PVS-Studio / CoverityCommercial static analyzersLicensed

The recommendation for any new C++ project: turn on -Wall -Wextra -Wpedantic, run clang-tidy in CI, format with clang-format, and run your test suite under ASan + UBSan. That single configuration eliminates a substantial fraction of the bugs the language has historically been infamous for.

Testing Frameworks

  • Google Test (gtest) & Google Mock (gmock) — Most widely used. Mature, integrates with CMake via gtest_discover_tests.
  • Catch2 — Header-only with BDD-style syntax (SECTION, REQUIRE). v3 split into a static library to fix compile-time bloat.
  • doctest — Lighter-weight Catch2 alternative. Designed for fast compilation.
  • Boost.Test — Older, full-featured, integrated with Boost.
  • CppUTest — Embedded-friendly, small footprint, explicit leak detection.

Game Engines — C++’s Heartland

Almost every commercial 3D engine and almost every AAA title is written in C++ at the core, with scripting languages on top for gameplay:

  • Unreal Engine (Epic) — Core in C++, gameplay in C++ and Blueprints. Powers Fortnite, Gears of War, the new Tomb Raiders, FF VII remake.
  • Unity — Core in C++, scripting in C# via Mono/IL2CPP (which transpiles C# to C++).
  • id Tech 1–7 (id Software) — Doom → Quake → RAGE → Doom Eternal. id Tech 4 was Carmack’s first C++ engine.
  • CryEngine (Crytek) — Far Cry, Crysis lineage.
  • Godot — Open-source, C++ core, GDScript for scripting, optional C#.
  • O3DE (Amazon open-source Lumberyard) — CryEngine descendant.
  • Source Engine (Valve) — Half-Life 2, Portal, Counter-Strike, Dota 2.
  • Frostbite (EA/DICE) — Battlefield, FIFA, Need for Speed.

Middleware: Bullet, NVIDIA PhysX, Havok (physics); OpenAL, FMOD, Wwise (audio); Vulkan, DirectX 12, Metal, OpenGL (graphics APIs) — all bound from C++.

Other Domains

HFT

Single-digit-microsecond latency floors. Hudson River Trading, Citadel Securities, Jump, Tower, Optiver, DRW. Lock-free queues, false sharing, busy-wait spinlocks, kernel bypass networking. Jane Street is the OCaml outlier.

Databases

MySQL, MariaDB, MongoDB, ClickHouse, DuckDB, RocksDB (Meta’s LevelDB fork). The fast-analytics wave is overwhelmingly C++.

Browsers

Chromium/Chrome/Edge/Brave, Firefox/Gecko (with growing Rust: Servo, WebRender, URL parser), WebKit/Safari. There is no major browser engine that is not primarily C++.

ML / Scientific

TensorFlow, PyTorch (LibTorch), ONNX Runtime, TensorRT, OpenCV — all C++ cores with Python wrappers. Eigen, Armadillo, CERN ROOT for numerical and scientific computing.

  • Embedded — C still dominates bare-metal, but modern C++ is increasingly used. AUTOSAR C++14 and MISRA C++ for safety-critical. Arduino is a C++ dialect.
  • Financial engineering — QuantLib, the prominent open-source quantitative finance library.
  • CAD / 3D — Autodesk Maya, 3ds Max, AutoCAD, FreeCAD, OpenCASCADE. Blender mostly C with substantial C++.
  • Telecom — Base stations, switches, routers, packet processors. Carrier-grade software is overwhelmingly C++.
  • Desktop applications — Adobe Photoshop, Illustrator, InDesign, Microsoft Office, Visual Studio. Electron itself is Chromium + Node.js — both C++ underneath.
