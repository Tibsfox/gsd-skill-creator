“Do not try to learn the whole language. Learn modern C++ — the subset that the Core Guidelines recommend — and treat the rest as legacy that you’ll meet when you need to.”

The Good News

C++ has a deserved reputation for being hard to learn well. The good news is that the modern syllabus is much shorter and saner than the historical one. You don’t need to learn everything; you need to learn the modern core well, and pick up the legacy parts when you encounter them. Books written in 2018 or later assume modern style from the start, and resources like cppreference.com and the Core Guidelines give you a canonical source of truth for every idiom.

A Recommended Progression

1 Start: A Tour of C++ (Stroustrup, 3rd edition 2022, ~240 pages) — a fast on-ramp for working programmers. Covers modern style and everything you need to read modern code.
2 Depth: Programming: Principles and Practice Using C++ (Stroustrup, 3rd ed. 2024) for beginners, or Accelerated C++ (Koenig & Moo, 2000) — older but still one of the cleanest intros, teaches modern style by starting with the standard library.
3 Idiom: Effective Modern C++ (Scott Meyers, 2014) — 42 specific items on C++11/14 idioms. The single most useful book for someone who already knows the basics.
4 Reference: cppreference.com — the canonical day-to-day lookup. Better than the standard itself for practical use. Always the first place to check what a function actually does.
5 Free online: learncpp.com — a free online tutorial maintained for years. Excellent for self-study from zero.
6 Videos: CppCon on YouTube — the annual C++ conference. Talks by Stroustrup, Sutter, Lakos, Carruth, Parent, and dozens of others. The “Back to Basics” track is particularly good for learners.
7 Short-form: C++ Weekly by Jason Turner — a long-running YouTube series, one technique per video.
8 Canon: C++ Core Guidelines — Stroustrup and Sutter’s living document at isocpp.github.io/CppCoreGuidelines/. Opinionated and very useful as a reference for “is this idiom recommended?”

Beginner-friendly textbooks

  • Accelerated C++ (Andrew Koenig & Barbara Moo, 2000) — still considered one of the cleanest on-ramps. Teaches by starting with the standard library.
  • C++ Primer (Lippman, Lajoie, Moo, 5th ed. 2012) — 976 pages, thorough, covers C++11.

A Reading Order That Works

  1. A Tour of C++ (Stroustrup, 3rd ed.) — 1 week, serious read
  2. learncpp.com sections 1–10 — hands-on, write code alongside
  3. Effective Modern C++ (Meyers) — 2–3 weeks, work through every item
  4. cppreference.com — reference as you go
  5. C++ Core Guidelines — skim once, return to often
  6. A real project in modern style — CMake, Conan/vcpkg, Catch2 or gtest, ASan in CI
  7. CppCon “Back to Basics” track on YouTube — fill gaps
  8. C++ Templates: The Complete Guide or C++ Concurrency in Action — go deep on the area that matters for your work

This progression takes several months of part-time study and produces a programmer who can read and write modern C++ confidently. It is much less than “learn all of C++.”
