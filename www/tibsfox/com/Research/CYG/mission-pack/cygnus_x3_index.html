How to Use This Package

  1. Review Stage 1 — Confirm scope boundaries, success criteria, and module decomposition match your research intent.
  2. Hand Stage 3 to GSD Orchestrator — The Milestone Specification and Wave Execution Plan are designed to be executed via gsd-skill-creator (dev branch) without additional context.
  3. CAPCOM gate — The mission includes one human-in-the-loop approval gate between Wave 1 and Wave 2 (scope review before synthesis). Plan for this checkpoint.
  4. Model allocation — Follow the 30/60/10 Opus/Sonnet/Haiku split defined in the chipset YAML. The physics modules (MOD-03, MOD-04) require Opus-class judgment.
  5. Verify before archival — Run all 4 safety-critical tests (SC-SRC, SC-NUM, SC-ADV, SC-UNC) before committing final output to the College of Knowledge.
