The Problem

Different tasks require different levels of model capability. A simple code formatting request does not need the same model that solves novel mathematical proofs. The economic logic is clear: route easy tasks to cheap, fast models and hard tasks to expensive, powerful ones. The technical challenge is the routing decision itself.

How does the router know what is hard? The difficulty of a task is not apparent from its surface features. A short prompt can hide extreme complexity. A long prompt can describe a trivial task. The router must estimate difficulty before the task is solved -- if you already knew how hard it was, you would already be partway to solving it.

Current approaches use a small classifier model to estimate difficulty from prompt features. But these classifiers are trained on historical data and fail on novel task types. Step-level routing (re-evaluating difficulty at each reasoning step) improves accuracy but adds latency. The fundamental tension is between routing accuracy and routing cost -- the more compute you spend on routing, the less benefit you get from routing to a cheaper model.

Latest Research

Confidence-Aware Model Routing

arXiv:2601.04861

Proposes routing based on the small model's confidence in its own answer. If the small model produces a high-confidence response, use it. If confidence is low, escalate to the large model. The key insight is that confidence calibration is easier than difficulty estimation -- the model does not need to judge how hard the problem is, only how confident it is in its answer. Achieves 85% of the large model's accuracy at 40% of the cost on coding benchmarks.

CogRouter: Step-Level Routing

arXiv:2602.12662 (CogRouter)

Extends routing from prompt-level to step-level. At each reasoning step, the system evaluates whether the current model can handle the next step or whether escalation is needed. This captures tasks that start easy but encounter hard sub-problems partway through. Uses ACT-R cognitive activation levels to predict step difficulty. More accurate than prompt-level routing but adds ~15% latency overhead from per-step evaluation.

LLM M5 Routing Strategy

LLM Research Track

Internal research milestone examining routing strategies for the LLM project. Compares prompt-level classifiers, confidence-based escalation, and ensemble voting. Finds that ensemble voting (run both models, compare answers) provides the highest accuracy but at 2x cost, negating the economic benefit. Confidence-based routing offers the best cost-accuracy tradeoff for production deployment, but requires well-calibrated confidence estimates.

Our Connection

We already implement manual confidence routing through the GSD set-profile command. The user selects quality, balanced, or budget mode based on their judgment of task difficulty. The open research question is whether this judgment can be automated reliably.

  • GSD set-profile Command: Manual routing between quality (expensive, powerful), balanced (default), and budget (fast, cheap) model profiles. This is confidence routing with a human in the routing loop.
  • Sling-Dispatch Formula Routing: The dispatch pipeline routes work items to agents based on agent capabilities and work item requirements. This is a form of routing, though currently based on capability matching rather than difficulty estimation.
  • Mayor-Coordinator Task Allocation: The mayor estimates task complexity when assigning work to polecats. Making this estimation principled and measurable is the research frontier.

Session 6 Research Connections

New connections discovered during the Artemis II Session 6 research sweep:

  • 3-Lane Hybrid Search (vector + FTS + concept): Our hybrid search implementation provides a concrete confidence signal through lane agreement. When all three lanes (vector similarity, full-text search, and concept graph) return the same result, confidence is high. When only one lane finds a match, confidence is low. The Reciprocal Rank Fusion score (k=60) quantifies this agreement, giving a calibrated confidence estimate without a separate classifier model. This is routing by consensus rather than routing by prediction.
  • Intent Classification for Confidence Routing: The intent classification system built for dataset retrieval routes queries by detected confidence in the query's intent category. Ambiguous queries (low classification confidence) get routed to the more expensive multi-lane search. Clear queries (high classification confidence) get routed to the fast single-lane path. The classifier's confidence in its own routing decision is the routing signal — matching the arXiv:2601.04861 insight that confidence calibration is easier than difficulty estimation.

What Would Constitute a Breakthrough

A routing system that achieves within 2% of oracle routing accuracy (where the oracle knows the optimal model for each task) using less than 5% of total inference budget on routing overhead. The router must generalize to novel task types without retraining, handling distribution shift gracefully.

For our system, the breakthrough is automatic profile selection that replaces human judgment: given a work item description, automatically choose quality/balanced/budget mode with accuracy that matches or exceeds the user's manual selection over a representative workload.
