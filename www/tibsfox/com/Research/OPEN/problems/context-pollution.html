Context Pollution in Multi-Agent Systems

When sub-agent output floods the parent’s context window, reasoning degrades
Active Research
OPEN Problems Orchestrated Intelligence Flow Engineering Research

The Problem

In multi-agent systems, a parent agent spawns sub-agents to handle subtasks. When sub-agents complete, their results typically get injected as raw text into the parent’s context window. This creates context pollution — the parent’s reasoning quality degrades as its context fills with verbose sub-agent output that may be irrelevant to the parent’s decision-making.

The degradation is well-documented:

  • U-shaped retrieval curve — LLMs retrieve information well from the beginning and end of context but degrade in the middle (the “lost in the middle” phenomenon)
  • Context rot — when 95%+ of context is irrelevant noise, hallucination rates increase
  • Attention dilution — transformer attention is a finite resource; more tokens = less attention per token
  • Non-determinism amplification — 10–15% variance on identical runs (Braintrust eval data); noise in context amplifies this

This is the multi-agent equivalent of a meeting where everyone reads their full report out loud instead of giving a one-sentence summary. The coordinator drowns in detail and loses track of the big picture.

Known Approaches

Recursive Language Models (RLMs) — MIT

Source: YouTube transcript analysis (yt-queue-01), March 2026

Sub-agent results returned as Python variables in the parent’s REPL, not as raw text in the parent’s context. The parent can inspect, transform, or compose results without reading the full output. Symbolic composition over textual injection. Claims to outperform RAG, ReAct, CodeAct, and traditional sub-agent architectures on LongBench HotpotQA.

Meta-Harness Pattern — Stanford/MIT (March 30, 2026)

Source: YouTube transcript analysis (yt-queue-11)

An outer-sphere LLM reads full execution traces (up to 10M tokens) and performs causal reasoning to optimize the inner-sphere LLM’s harness code. Key finding: no summarization allowed — lossy compression kills performance. The harness controls what the core LLM sees, acting as a filter rather than a summarizer.

STEM Agent — Amazon / UC Berkeley (March 2026)

Source: YouTube transcript analysis (yt-queue-06)

Self-adapting agent with strict orthogonality between reasoning and domain knowledge, decoupled through MCP abstraction. Skill crystallization with apoptosis — underperforming skills are pruned. The architecture prevents domain-specific noise from polluting the reasoning core.

Agentic Search vs. Vector Search — Braintrust

Source: YouTube transcript analysis (yt-queue-02)

Agentic search (grep/find/read approach) matched or beat vector search on accuracy for code bug fixing, with lower token cost. Vector search loses “connective tissue” (import chains, call graphs) that agentic traversal preserves. The agent that reads code like a human finds more than similarity-based retrieval.

Our Approach: The Forest Architecture

Our multi-agent system (Gastown chipset) addresses context pollution through the muse architecture — agent roles modeled on Pacific Northwest ecosystem species:

  • Cedar (filter/ledger) — controls what enters the trust record and what gets discarded. Analogous to the ATLAS trigger system: 1 billion events/second reduced to ~1,000. Cedar’s mycorrhizal root network routes information through the most efficient paths, not through broadcast.
  • Raven (async messaging) — messages carry summaries, not full payloads. The parent agent receives a structured signal, not a raw transcript. Ravens are the nervous system, not the bloodstream.
  • Hawk (observer) — watches full execution traces (like the meta-harness outer sphere) without injecting observations into agent contexts. Reports to Cedar, not to the working agents.
  • Sam (coordinator) — bridges between human intent and system execution. Sam knows what the parent needs without flooding it with what the children produced.
  • Beads-state (persistence) — agent results persist in filesystem state, not in context windows. Agents read what they need from state, when they need it. No preemptive injection.
Key insight: The forest doesn’t solve nutrient transport by flooding every tree with every nutrient simultaneously. The mycorrhizal network routes specific nutrients to specific trees based on need. Context management should work the same way — targeted delivery, not broadcast.

Open Questions

  • What is the optimal summary granularity for inter-agent communication? Too compressed = information loss. Too verbose = context pollution. The meta-harness paper says no summarization; RLMs say symbolic variables. Is there a middle ground?
  • Can graph-based context routing (LightRAG-style entity/relationship graphs) replace broadcast injection? Route context based on graph proximity rather than sequential injection.
  • How does the trust ladder (Kilo/Warp) map to context access? Higher-trust agents get more context access. Lower-trust agents operate on summaries only.
  • What is the information-theoretic minimum for inter-agent messages? Shannon tells us there’s an optimal encoding. What is it for agent coordination?
  • Can Physarum-inspired routing (slime mold shortest-path) optimize context delivery in agent meshes? The slime mold finds optimal nutrient transport paths without central coordination.

Session 6 Research Connections

New connections discovered during the Artemis II Session 6 research sweep:

  • 12 Primitives — Transcript Compaction (Primitive 10): The 12 Agentic Primitives analysis identifies Transcript Compaction as a first-class primitive that directly addresses context pollution. Rather than injecting raw sub-agent output, Primitive 10 compresses execution traces into structured summaries that preserve decision points while discarding verbose intermediate steps. This is the "Cedar filter" expressed as a formal operation.
  • Token Budget Enforcement (backlog 999.1, now built): The token-budget.ts module provides hard per-convoy and per-agent token limits with pre-execution checking. This is a mechanical prevention layer — context cannot be polluted beyond the budget ceiling. When a sub-agent approaches its token budget, it must compress or terminate, never overflow into the parent's context window.
