The Problem

Not every problem requires the same amount of thinking. Humans naturally spend more cognitive effort on hard problems and breeze through easy ones. Current language models apply roughly uniform compute to every token, regardless of problem difficulty. This is wasteful on easy problems and insufficient on hard ones.

The adaptive thinking problem asks: can a model dynamically assess the difficulty of a prompt and allocate compute proportionally? Simple factual lookups should use minimal reasoning tokens. Complex multi-step proofs should use many. The model should be its own difficulty estimator, adjusting its reasoning depth in real time.

Research in early 2026 shows this is achievable with up to 50% compute reduction on easy tasks while maintaining quality. But the difficulty estimation itself is an open problem -- the model must judge how hard something is before knowing the answer, which is a chicken-and-egg challenge. Current approaches use cognitive architecture theories (ACT-R) and learned routing policies, but neither is robust across all task types.

Latest Research

CogRouter: ACT-R Cognitive Theory for LLM Routing

arXiv:2602.12662

Applies the ACT-R (Adaptive Control of Thought -- Rational) cognitive architecture from psychology to LLM compute routing. Models prompt difficulty using activation levels from cognitive theory: prompts with low "activation" (unfamiliar patterns, complex dependencies) get routed to deeper reasoning. Achieves near-oracle routing performance on mathematical reasoning benchmarks while using only prompt features, no answer lookahead.

Reasoning on a Budget

arXiv:2507.02076

Introduces token budget constraints as a first-class training signal. Models are trained with a "cost" for each reasoning token, learning to achieve correct answers with minimal reasoning overhead. The resulting models produce dense, efficient reasoning chains -- fewer tokens but higher information per token. Cognitive density (useful information per reasoning token) becomes a measurable metric.

Input-Adaptive Allocation (ICLR 2025)

ICLR 2025

Demonstrates that adaptive compute allocation based on input features can reduce total inference compute by 50% with no measurable quality loss on standard benchmarks. Uses a lightweight classifier trained on prompt features to predict the optimal number of reasoning steps. The classifier adds negligible overhead (~0.1% of total compute) and generalizes across task types after training on a diverse mixture.

Our Connection

This maps directly to infrastructure we already have. Our GSD model profiles (quality/balanced/budget) are manual adaptive thinking -- the user or the system decides how much compute to allocate based on task importance. The frontier research is automating what we do by hand.

  • GSD Model Profiles: quality/balanced/budget are coarse-grained adaptive thinking. Research points toward per-prompt granularity, which is much finer than our current per-session granularity.
  • GUPP Propulsion: The interrupt controller already modulates agent execution urgency. Adaptive thinking extends this concept to the reasoning layer itself.
  • Mayor-Coordinator: Task allocation in the Gastown chipset implicitly estimates task difficulty when choosing which agent handles which work item. Making this estimation explicit and principled is the research frontier.

What Would Constitute a Breakthrough

A self-assessing model that reliably estimates its own uncertainty and difficulty judgment for every prompt, allocating compute proportionally with less than 5% accuracy loss versus maximum-compute baselines. The difficulty estimator must work across all task types without per-domain tuning.

For our system specifically, a breakthrough would be automatic profile selection: the model reads the prompt and selects quality/balanced/budget mode without human input, with routing accuracy matching or exceeding human judgment on a representative task distribution.
