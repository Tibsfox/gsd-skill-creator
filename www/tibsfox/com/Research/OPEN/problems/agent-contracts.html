The Problem

An autonomous AI agent given unbounded resources will eventually cause harm -- not necessarily through malice, but through unbounded exploration, runaway cost, or unintended side effects. The solution is contracts: formal specifications that bound what an agent can do, how much it can spend, and what it must guarantee.

An agent contract specifies three things: preconditions (what the agent can assume about its environment), postconditions (what the agent must deliver), and resource bounds (maximum tokens, tool calls, wall-clock time, and cost). If the agent violates its contract, the orchestrator can safely terminate it without data loss.

The research challenge is making contracts tight enough to prevent harm but loose enough to allow the agent to solve the problem. Too-tight contracts prevent useful work. Too-loose contracts fail to prevent harm. The contract must encode the right level of specificity for each constraint, which requires understanding the relationship between resource budgets and task completion probability.

Latest Research

Agent Contracts (COINE 2026)

arXiv:2601.08815

Introduces a formal framework for agent contracts with three key contributions: (1) a contract specification language that encodes resource bounds, safety constraints, and output guarantees; (2) a runtime enforcement mechanism that monitors contract compliance and terminates violating agents; (3) conservation laws that prove total resource consumption across a multi-agent system is bounded if each agent respects its contract. Demonstrates 90% token reduction through contract-guided early termination of agents that are clearly not making progress toward their postconditions.

Behavioral Contracts and Drift Bounds

arXiv:2602.22302

Extends agent contracts with the Drift Bounds Theorem: given a contract and a model's behavioral distribution, the theorem bounds the probability that the agent will violate its contract over a given number of steps. Validated on 1,980 sessions across multiple models and task types. The drift bound provides a principled way to set contract resource limits -- choose the limit where the probability of contract violation drops below a target threshold. Key finding: behavioral drift is predictable from early execution, enabling proactive contract renegotiation before violation.

Our GUPP is an operational implementation of agent contracts. Get Up and Push Protocol specifies resource bounds (token budget ceiling), safety constraints (tool access lists), and termination conditions (watchdog stall detection). We built this as engineering pragmatism; the research community is now formalizing the theory behind what we do operationally.

Our Connection

The agent contracts research formalizes patterns we have been using in production. Our system enforces resource bounds, monitors agent health, and terminates stalled agents. The research provides theoretical grounding for these operational decisions.

  • GUPP (Get Up and Push Protocol): Our interrupt controller enforces execution contracts. Agents have token budgets, tool access lists, and progress expectations. Watchdog monitors for stalls and escalates. This is an agent contract by another name.
  • Hook-Persistence: Pull-based work assignment with single-active-work-item constraints. The hook is a contract: one agent, one task, atomic state transitions, no concurrent mutations.
  • Watchdog: Our stall detection system monitors agent progress and terminates unresponsive agents. This is the contract enforcement mechanism described in the research.
  • Token Budget Ceiling: Hard limit on per-agent token consumption. The research formalizes why this limit is necessary (conservation laws) and how to set it (drift bounds).
  • ACE M3: Agent Compute Engine milestone where resource management is a core requirement. Agent contracts provide the formal foundation.

What Would Constitute a Breakthrough

A compositional contract system where individual agent contracts compose into system-level guarantees. If agent A has contract C_A and agent B has contract C_B, and they interact through a defined protocol, derive the composite contract C_{AB} that bounds the behavior of the combined system. This is the multi-agent analog of compositional verification in formal methods.

A practical breakthrough would be automatic contract inference: given a task description and a target cost budget, automatically generate a contract that is tight enough to prevent waste but loose enough to allow task completion with high probability. The drift bounds work is a step toward this, but current methods require historical execution data for calibration.
