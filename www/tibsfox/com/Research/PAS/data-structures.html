Complex Data Types & Recursive Data Structures

Pointers, Linked Lists, Binary Trees, Variant Records — and Why Pascal Made Data Structures Visible
Overview Blaise Pascal Language Data Structures Implementations Teaching Rosetta Learn

This is the flagship page of the Pascal project. Pascal was the first language to combine record types, typed pointers, recursive type definitions, recursive procedures, strong typing, and dynamic allocation in a clean package. The result was that data structure code in Pascal looked almost exactly like the diagrams in a textbook. The code was the diagram, made precise. This is why Wirth wrote Algorithms + Data Structures = Programs in Pascal.

Why Pascal Excels at Teaching Data Structures

Consider the landscape of 1970. A computer science instructor who wanted to teach data structures had several language options, none satisfactory:

LanguageProblem
FORTRANNo pointer types, no records, no recursion. Linked lists required simulating pointers with array indices — obscuring the concepts being taught.
COBOLRecords but no pointers and no recursion. Dynamic data structures were impossible.
LISPDynamic structures built-in (cons cells) but untyped. Structure was implicit — the type system would not tell you a list was a binary tree.
Algol 60Recursion and block structure but no records and no pointers to dynamic data.
PL/IRecords, pointers, and recursion, but an enormous language with weak typing. Silent automatic conversions; unparameterized pointers.
CNot yet available (1969–1973). Would eventually offer structs and pointers but with weak typing: arbitrary casts, pointer arithmetic, no bounds checking.

Pascal was the first language to combine all six ingredients in a strongly typed package: record types for node structure, pointer types for linking, recursive type definitions through the forward-reference exception, recursive procedures for traversal, strong typing ensuring compile-time safety, and new/dispose for dynamic allocation.

Wirth’s 1976 textbook Algorithms + Data Structures = Programs made the philosophy explicit: data structure design and algorithm design are inseparable. The book used Pascal throughout — not merely because Pascal was Wirth’s language, but because it was the first language designed to express this relationship in a way students could read, write, and reason about.

The Pointer Type and Forward References

A pointer in Pascal is declared using the caret (^) prefix. The type ^T means “pointer to a value of type T.” Pointers are created with new and deallocated with dispose:

var
  p: ^integer;
begin
  new(p);        { allocate an integer on the heap }
  p^ := 42;     { dereference and assign }
  writeln(p^);   { read the value: 42 }
  dispose(p);    { free the memory }
end;

Pascal’s Deliberate Constraints

Unlike C, Pascal prohibits pointer arithmetic entirely. You cannot add an integer to a pointer, subtract pointers, or cast between pointer types and integers. Wirth’s reasoning: pointers exist for building dynamic data structures, not for array manipulation. Arrays already have safe, bounds-checked indexing. Pointer arithmetic is the source of buffer overflows and the majority of memory safety bugs in C.

This design decision echoes throughout language history: Java (1995), C# (2000), Go (2009), and Rust (2015) all adopted Pascal’s philosophy of prohibiting pointer arithmetic in safe code — not C’s.

The Forward Reference Exception

Pascal’s requirement that every identifier be declared before use poses a problem for recursive data structures. A linked list node contains a pointer to another node of the same type — a circular dependency. Pascal solved this with a single, elegant exception:

type
  PNode = ^TNode;    { PNode declared BEFORE TNode exists }
  TNode = record
    data: integer;
    next: PNode;
  end;

A pointer type could reference a type not yet declared, provided that type was declared later in the same type block. This narrowly-scoped exception handled exactly the case that needed it without complicating the single-pass compilation model. It was necessary and sufficient for defining all recursive data structures.

Linked Lists

The linked list is the simplest dynamic data structure and the first one most students encounter. In Pascal, a singly linked list is defined by two type declarations:

type
  PNode = ^TNode;
  TNode = record
    data: integer;
    next: PNode;
  end;

Creating a List

var
  head, newNode: PNode;
begin
  head := nil;  { empty list }

  { Insert at the front }
  new(newNode);
  newNode^.data := 10;
  newNode^.next := head;
  head := newNode;

  new(newNode);
  newNode^.data := 20;
  newNode^.next := head;
  head := newNode;

  { List is now: 20 -> 10 -> nil }
end;

Traversal

procedure PrintList(head: PNode);
var
  current: PNode;
begin
  current := head;
  while current <> nil do
  begin
    write(current^.data, ' ');
    current := current^.next;
  end;
  writeln;
end;

Insertion and Deletion

procedure InsertAfter(node: PNode; value: integer);
var
  newNode: PNode;
begin
  if node <> nil then
  begin
    new(newNode);
    newNode^.data := value;
    newNode^.next := node^.next;
    node^.next := newNode;
  end;
end;

procedure DeleteFirst(var head: PNode);
var
  temp: PNode;
begin
  if head <> nil then
  begin
    temp := head;
    head := head^.next;
    dispose(temp);
  end;
end;

Sorted Insertion

procedure SortedInsert(var head: PNode; value: integer);
var
  newNode, current, prev: PNode;
begin
  new(newNode);
  newNode^.data := value;
  newNode^.next := nil;

  if (head = nil) or (value < head^.data) then
  begin
    newNode^.next := head;
    head := newNode;
  end
  else
  begin
    prev := head;
    current := head^.next;
    while (current <> nil) and (current^.data < value) do
    begin
      prev := current;
      current := current^.next;
    end;
    newNode^.next := current;
    prev^.next := newNode;
  end;
end;

This procedure illustrates edge case handling (empty list, insertion at head), the two-pointer traversal technique, and pointer update mechanics. Students who understand this have grasped the essential mechanics of linked list manipulation.

Disposing an Entire List

procedure DisposeList(var head: PNode);
var
  current, next: PNode;
begin
  current := head;
  while current <> nil do
  begin
    next := current^.next;
    dispose(current);
    current := next;
  end;
  head := nil;
end;

This explicit memory management is itself pedagogically valuable. Students learn that memory is a finite resource, that every allocation must be matched by a deallocation, and that failing to do so causes memory leaks. These lessons are invisible in garbage-collected languages but essential for understanding how computer systems actually work.

Binary Trees

Binary trees are perhaps the data structure where Pascal’s expressiveness is most apparent. The type definition mirrors the mathematical definition almost exactly:

type
  PTree = ^TTree;
  TTree = record
    value: integer;
    left: PTree;
    right: PTree;
  end;

A tree is either empty (nil) or a node containing a value and two subtrees. This recursive definition, expressed through Pascal’s forward-reference mechanism, is as close to a formal mathematical definition as an imperative language can get.

Tree Traversals

The three classical traversals differ only in the placement of the output statement relative to the recursive calls:

{ Inorder: left, root, right -- produces sorted output for BSTs }
procedure InOrder(root: PTree);
begin
  if root <> nil then
  begin
    InOrder(root^.left);
    write(root^.value, ' ');
    InOrder(root^.right);
  end;
end;

{ Preorder: root, left, right -- useful for copying/serializing }
procedure PreOrder(root: PTree);
begin
  if root <> nil then
  begin
    write(root^.value, ' ');
    PreOrder(root^.left);
    PreOrder(root^.right);
  end;
end;

{ Postorder: left, right, root -- useful for deletion and expression evaluation }
procedure PostOrder(root: PTree);
begin
  if root <> nil then
  begin
    PostOrder(root^.left);
    PostOrder(root^.right);
    write(root^.value, ' ');
  end;
end;

The correspondence between code and definition is immediate and complete. A student who understands “visit left subtree, then root, then right subtree” can read the inorder code directly.

Binary Search Trees

A binary search tree (BST) maintains the invariant: for every node, all values in the left subtree are less than the node’s value, and all values in the right subtree are greater. This enables O(log n) search, insertion, and deletion on average.

Insertion

procedure Insert(var root: PTree; val: integer);
begin
  if root = nil then
  begin
    new(root);
    root^.value := val;
    root^.left := nil;
    root^.right := nil;
  end
  else if val < root^.value then
    Insert(root^.left, val)
  else if val > root^.value then
    Insert(root^.right, val);
  { if val = root^.value, duplicate; do nothing }
end;

The recursive structure of the procedure mirrors the recursive structure of the tree. The var parameter means the pointer is passed by reference, allowing the procedure to modify it directly — this is how a new node is attached without returning a value.

Search

function Search(root: PTree; val: integer): boolean;
begin
  if root = nil then
    Search := false
  else if val = root^.value then
    Search := true
  else if val < root^.value then
    Search := Search(root^.left, val)
  else
    Search := Search(root^.right, val);
end;

Finding Minimum and Maximum

function FindMin(root: PTree): integer;
begin
  if root^.left = nil then
    FindMin := root^.value
  else
    FindMin := FindMin(root^.left);
end;

function FindMax(root: PTree): integer;
begin
  if root^.right = nil then
    FindMax := root^.value
  else
    FindMax := FindMax(root^.right);
end;

Deletion

BST deletion has three cases: leaf node (simply remove), one child (replace with the child), and two children (replace with the inorder successor, then delete the successor):

function DeleteNode(root: PTree; val: integer): PTree;
var
  temp, successor: PTree;
begin
  if root = nil then
  begin
    DeleteNode := nil;
    exit;
  end;

  if val < root^.value then
    root^.left := DeleteNode(root^.left, val)
  else if val > root^.value then
    root^.right := DeleteNode(root^.right, val)
  else
  begin
    { Found the node to delete }
    if root^.left = nil then
    begin
      temp := root^.right;
      dispose(root);
      DeleteNode := temp;
      exit;
    end;
    if root^.right = nil then
    begin
      temp := root^.left;
      dispose(root);
      DeleteNode := temp;
      exit;
    end;
    { Two children: find inorder successor }
    successor := root^.right;
    while successor^.left <> nil do
      successor := successor^.left;
    root^.value := successor^.value;
    root^.right := DeleteNode(root^.right, successor^.value);
  end;
  DeleteNode := root;
end;

Stacks and Queues

Pointer-Based Stack

type
  PStackNode = ^TStackNode;
  TStackNode = record
    data: integer;
    next: PStackNode;
  end;

procedure Push(var top: PStackNode; value: integer);
var
  newNode: PStackNode;
begin
  new(newNode);
  newNode^.data := value;
  newNode^.next := top;
  top := newNode;
end;

function Pop(var top: PStackNode): integer;
var
  temp: PStackNode;
begin
  if top <> nil then
  begin
    Pop := top^.data;
    temp := top;
    top := top^.next;
    dispose(temp);
  end;
end;

Array-Based Stack

const
  MaxStack = 100;
type
  TStack = record
    items: array[1..MaxStack] of integer;
    top: 0..MaxStack;
  end;

procedure StackInit(var s: TStack);
begin
  s.top := 0;
end;

procedure Push(var s: TStack; value: integer);
begin
  s.top := s.top + 1;
  s.items[s.top] := value;
end;

function Pop(var s: TStack): integer;
begin
  Pop := s.items[s.top];
  s.top := s.top - 1;
end;

Pointer-Based Queue

type
  PQueueNode = ^TQueueNode;
  TQueueNode = record
    data: integer;
    next: PQueueNode;
  end;

  TQueue = record
    front, rear: PQueueNode;
  end;

procedure Enqueue(var q: TQueue; value: integer);
var
  newNode: PQueueNode;
begin
  new(newNode);
  newNode^.data := value;
  newNode^.next := nil;
  if q.rear = nil then
  begin
    q.front := newNode;
    q.rear := newNode;
  end
  else
  begin
    q.rear^.next := newNode;
    q.rear := newNode;
  end;
end;

function Dequeue(var q: TQueue): integer;
var
  temp: PQueueNode;
begin
  Dequeue := q.front^.data;
  temp := q.front;
  q.front := q.front^.next;
  if q.front = nil then
    q.rear := nil;
  dispose(temp);
end;

Variant Records as Algebraic Data Types

Variant records are Pascal’s mechanism for tagged unions — records where the fields depend on a discriminant (tag) value. They are the most powerful construct in Pascal’s type system and the closest thing to algebraic data types in any mainstream imperative language of the 1970s.

Basic Variant Records

type
  ShapeKind = (skCircle, skRectangle, skTriangle);

  TShape = record
    x, y: real;    { position -- common to all shapes }
    case kind: ShapeKind of
      skCircle:    (radius: real);
      skRectangle: (width, height: real);
      skTriangle:  (base, altitude: real);
  end;

function Area(const s: TShape): real;
begin
  case s.kind of
    skCircle:    Area := 3.14159265 * s.radius * s.radius;
    skRectangle: Area := s.width * s.height;
    skTriangle:  Area := 0.5 * s.base * s.altitude;
  end;
end;

The case kind: ShapeKind of introduces the variant part. Each variant specifies a different set of fields, and the tag field (kind) records which variant is active. The common fields (x, y) are shared by all variants.

The Expression Tree: Where Variant Records Shine

Recursive variant records are structurally identical to algebraic data types in ML or Haskell. An expression tree demonstrates this beautifully:

type
  ExprKind = (ekNumber, ekAdd, ekMul, ekNeg, ekVar);

  PExpr = ^TExpr;
  TExpr = record
    case kind: ExprKind of
      ekNumber: (value: real);
      ekAdd:    (addLeft, addRight: PExpr);
      ekMul:    (mulLeft, mulRight: PExpr);
      ekNeg:    (operand: PExpr);
      ekVar:    (name: char);
  end;

Expression Construction and Evaluation

function MakeNumber(v: real): PExpr;
var e: PExpr;
begin
  new(e);
  e^.kind := ekNumber;
  e^.value := v;
  MakeNumber := e;
end;

function MakeAdd(l, r: PExpr): PExpr;
var e: PExpr;
begin
  new(e);
  e^.kind := ekAdd;
  e^.addLeft := l;
  e^.addRight := r;
  MakeAdd := e;
end;

function Evaluate(e: PExpr; xVal: real): real;
begin
  case e^.kind of
    ekNumber: Evaluate := e^.value;
    ekAdd:    Evaluate := Evaluate(e^.addLeft, xVal) +
                          Evaluate(e^.addRight, xVal);
    ekMul:    Evaluate := Evaluate(e^.mulLeft, xVal) *
                          Evaluate(e^.mulRight, xVal);
    ekNeg:    Evaluate := -Evaluate(e^.operand, xVal);
    ekVar:    Evaluate := xVal;
  end;
end;

Comparison with Modern Algebraic Types

Pascal’s variant records established the pattern that Rust’s enum and Haskell’s algebraic data types would later perfect:

FeaturePascalRustHaskell
Tag fieldExplicit (case kind:)Implicit (compiler-managed)Implicit (constructor)
Type safetyWeak (can read wrong variant)Strong (exhaustive match)Strong (exhaustive match)
RecursionVia pointer indirectionVia Box<T>Lazy evaluation / indirection
Pattern matchingcase on tagmatchcase / pattern equations
Exhaustiveness checkNoYes (compiler enforced)Yes (with warnings)

The key insight is the same across all three: pointer indirection enables recursive types in a statically typed language. Pascal expressed this with ^TExpr; Rust expresses it with Box<Expr>. The concept is identical; the syntax and safety guarantees evolved.

Recursive Type Definitions: The Key Insight

The central mechanism enabling all of Pascal’s dynamic data structures is the interaction between pointers and the forward-reference exception. A type cannot contain itself (that would require infinite storage), but a type can contain a pointer to itself (a pointer has fixed, finite size regardless of what it points to). This is the insight that makes linked lists, trees, and graphs possible in a statically typed language:

{ A linked list node contains a pointer to another node }
PNode = ^TNode;
TNode = record
  data: integer;
  next: PNode;     { recursive: TNode references itself via PNode }
end;

{ A binary tree node contains pointers to two subtrees }
PTree = ^TTree;
TTree = record
  value: integer;
  left, right: PTree;   { doubly recursive }
end;

{ A graph node contains a list of edges to other nodes }
PGraphNode = ^TGraphNode;
PEdge = ^TEdge;
TEdge = record
  target: PGraphNode;
  weight: integer;
  next: PEdge;
end;
TGraphNode = record
  id: integer;
  edges: PEdge;
  next: PGraphNode;
end;

Every data structure in every introductory textbook — linked lists, doubly linked lists, circular lists, binary trees, BSTs, AVL trees, B-trees, heaps, graphs, hash tables with chaining — is built from this one mechanism: records containing typed pointers.

“Algorithms + Data Structures = Programs”

Wirth’s 1976 textbook of this title is the canonical statement of a philosophy that Pascal was designed to embody: the design of a data structure and the design of the algorithm that operates on it are not separate activities but two aspects of a single design problem.

In Pascal, this unity is expressed at the language level. The type declaration defines the data structure; the procedure declaration defines the algorithm; the two are written in the same notation, in the same program, subject to the same type discipline. A binary search tree is not an abstract concept illustrated by pseudocode — it is a Pascal type declaration and a set of Pascal procedures that the compiler can verify, the student can run, and the instructor can grade.

This philosophy was Pascal’s deepest contribution to computer science education. It shaped a generation of textbooks (Aho, Hopcroft, and Ullman’s Data Structures and Algorithms used Pascal; Tenenbaum and Augenstein’s Data Structures Using Pascal was the standard in many departments). It influenced the design of later languages: Ada’s type system, Modula-2’s modules, Oberon’s type extension, and Rust’s ownership model all build on the principle that data structure design and algorithm design are inseparable.

The pedagogical value: Compare implementing a linked list in Pascal versus using java.util.LinkedList in Java. The Java student learns the API — add(), remove(), get() — but not the mechanics. The Pascal student learns how nodes are allocated, how pointers are threaded, how insertion requires updating exactly two pointers, how deletion requires saving a reference before calling dispose. The Pascal student understands why insertion at the head is O(1) while insertion at position k is O(k), because they have written the traversal loop themselves.
