Wirth’s textbooks, Free Pascal setup, the Pensées reading editions, and a path through both the person and the language

The purpose of a programming language is to help you think about problems. The purpose of reading Pascal’s Pensées is to help you think about thinking. This project covers both, because both reward patient attention.

A Staged Learning Path

  1. Install Free Pascal and Lazarus. Free Pascal (freepascal.org) is the modern open-source Pascal compiler supporting ISO 7185, ISO 10206, Turbo Pascal, Delphi, and Mac Pascal dialects. Lazarus (lazarus-ide.org) is the companion IDE — a Delphi-like visual development environment. Both are cross-platform (Linux, macOS, Windows, FreeBSD). Install both to get a complete development environment.
  2. Read Wirth’s original Revised Report (1973). “The Programming Language Pascal (Revised Report)” is freely available online. It is only 35 pages long. Reading the actual specification that defined the language gives you a precision that no tutorial can match. Wirth wrote it in English so clean that it reads like prose.
  3. Work through a Pascal tutorial. The Free Pascal documentation includes a complete language reference and a programmer’s guide. Alternatively, learn-pascal.org provides a browser-based interactive tutorial. Start with program, begin/end, variables, if/then/else, while/for/repeat, procedures, and functions.
  4. Read Wirth’s Algorithms + Data Structures = Programs (1976). The most important computer science textbook written in Pascal. Every data structure — arrays, records, linked lists, trees, hash tables, graphs — is presented with complete Pascal code and rigorous analysis. This is the book that taught a generation of computer scientists to think about programs as compositions of data structures and algorithms.
  5. Implement linked lists, trees, and sorting algorithms in Pascal. Write them yourself. Use pointer types (^Node), new and dispose, recursive traversals, and variant records. This is the core of Pascal’s pedagogy: you learn how data structures work by building them in a language that forces you to be explicit about every pointer and every allocation.
  6. Read the Pensées. A. J. Krailsheimer’s translation (Penguin Classics, 1966, revised 1995) is the standard English edition. Follow the Lafuma numbering. The Pensées are fragments — Pascal died before he could organize them into a book — so read them slowly, a few at a time. The “Wager” fragment (Lafuma 418) is the most famous; the fragments on diversion, custom, and the two infinities are the most profound.
  7. Read the Provincial Letters. Krailsheimer’s translation (Penguin Classics). Pascal’s polemic against Jesuit casuistry, published pseudonymously as “Louis de Montalte.” Eighteen letters, devastatingly funny, a masterpiece of French prose style. Voltaire said they were the first book of genius in French prose.
  8. Read Kernighan’s “Why Pascal is Not My Favorite Programming Language” (1981). Available freely as a Bell Labs technical report. Kernighan’s critique is precise, fair, and devastating. Every complaint is real. Understanding the critique is as important as understanding the language, because it explains why C won and Pascal lost in systems programming.
  9. Compare Pascal to C, Ada, and Modula-2. Write the same small program (linked list, binary tree, file I/O) in all four languages. See how each handles type definitions, pointer arithmetic, separate compilation, and error handling. This is where the design space becomes visible — the trade-offs are not abstract when you have working code in front of you.

Your First Session

Install Free Pascal and create a simple program:

$ fpc hello.pas
$ ./hello

Hello, world!

The source file hello.pas:

program Hello;
begin
  writeln('Hello, world!');
end.

Now try a linked list — save this as linkedlist.pas:

program LinkedListDemo;

type
  PNode = ^Node;
  Node = record
    data: integer;
    next: PNode;
  end;

var
  head, current, temp: PNode;
  i: integer;

begin
  { Build a list of 1..5 }
  head := nil;
  for i := 5 downto 1 do
  begin
    new(temp);
    temp^.data := i;
    temp^.next := head;
    head := temp;
  end;

  { Print the list }
  current := head;
  while current <> nil do
  begin
    writeln(current^.data);
    current := current^.next;
  end;

  { Free the list }
  current := head;
  while current <> nil do
  begin
    temp := current;
    current := current^.next;
    dispose(temp);
  end;
end.

Compile and run: fpc linkedlist.pas && ./linkedlist. You have now used pointer types, new/dispose, records, and pointer dereferencing (^). This is the core of Pascal’s data structure story — every tree, graph, and hash table builds on these primitives.

Reading the Pensées

The Pensées are fragments, not a book. Pascal planned an “Apology for the Christian Religion” but died before writing it. What survives is a collection of notes, some polished, some rough, organized in bundles (liasses) with string. Three major orderings exist:

  • Brunschvicg (1897) — thematic grouping, widely cited, numbers familiar to older scholarship
  • Lafuma (1951) — follows the original manuscript bundles as closely as possible; Krailsheimer’s English translation uses this numbering
  • Sellier (1976) — follows the first copy made by Pascal’s family; increasingly preferred by French scholars

Start with Krailsheimer’s Penguin Classics translation (Lafuma numbering). Read the thematic sections on diversion, wretchedness, the two infinities, and the Wager first. Then browse. The fragments reward rereading — their brevity makes each one a small provocation.
